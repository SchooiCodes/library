Linux Firewall with nftables

Tables, chains, rules, sets, NAT, stateful filtering, and migrating from iptables.

nftables vs iptables

nftables is the modern Linux firewall framework that replaces iptables, ip6tables, arptables, and ebtables with a single unified interface. It uses a simple, structured syntax -- no more complex raw table rules.

Key advantages over iptables:

  • Simpler, more readable syntax
  • Native support for sets, maps, and dictionaries
  • Atomic rule replacement (entire chains replaced atomically)
  • Better performance with nft (generic netlink) vs nfnetlink
  • Single tool for IPv4, IPv6, ARP, and bridge

Install nftables:

sudo apt install nftables
sudo systemctl enable --now nftables

Tables, Chains & Rules

nftables organizes rules into tables and chains. Tables contain chains; chains contain rules:

#!/usr/sbin/nft -f

# Flush existing rules
flush ruleset

# Create table (with ip, ip6, inet, arp, or bridge family)
table inet filter {

    # Base chains (hook into kernel netfilter)
    chain input {
        type filter hook input priority 0; policy drop;

        # Allow established connections
        ct state established,related accept

        # Allow loopback
        iif lo accept

        # SSH (rate-limited)
        tcp dport 22 accept
        
        # ICMP (ping)
        icmp type echo-request accept
    }

    chain forward {
        type filter hook forward priority 0; policy drop;
    }

    chain output {
        type filter hook output priority 0; policy accept;
    }
}

# Apply
sudo nft -f /etc/nftables.conf

Sets & Dictionaries

Sets and dictionaries reduce rule duplication and improve performance:

# Anonymous set (inline)
tcp dport { 80, 443, 8080 } accept

# Named set
set allowed_ports {
    type inet_service
    flags interval
    elements = { 22, 80, 443, 3000-3999, 8080, 8443 }
}

tcp dport @allowed_ports accept

# Dictionary (map) -- different actions per value
chain input {
    tcp dport vmap {
        22: accept,
        80: accept,
        443: accept,
        3306: drop,
    }
}

# Update sets at runtime
nft add element inet filter allowed_ports { 9090 }

Sets support intervals (3000-3999), concatenations (ip saddr . tcp dport), and timeouts for dynamic banning.

NAT with nftables

Configure NAT (masquerading, port forwarding) in nftables:

# NAT table
table inet nat {

    chain prerouting {
        type nat hook prerouting priority -100;
        
        # Port forwarding: WAN:8080 -> 192.168.1.10:80
        tcp dport 8080 dnat to 192.168.1.10:80
    }

    chain postrouting {
        type nat hook postrouting priority 100;
        
        # Masquerade (WAN interface)
        oif eth0 masquerade
    }
}

# IP masquerading (share internet connection)
sudo nft add table inet nat
sudo nft add chain inet nat postrouting '{ type nat hook postrouting priority 100; }'
sudo nft add rule inet nat postrouting oif eth0 masquerade

Logging, Monitoring & Atomic Updates

Debug and monitor nftables traffic:

# Monitor all nftables events (real-time)
sudo nft monitor

# List all rules with packet/byte counters
sudo nft list ruleset

# Add logging
table inet filter {
    chain input {
        # Log dropped packets
        log prefix "nft-drop: " flags all counter drop
    }
}

# Atomic reload
nft -f /etc/nftables.conf  # Whole ruleset replaced atomically

# Zero counters
nft reset counters

# Save current ruleset
sudo nft list ruleset > /etc/nftables.conf
