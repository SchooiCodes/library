📂 Linux Filesystem Hierarchy

Understand the Linux filesystem — /, /etc, /var, /proc, /sys, /dev, mounts, and where everything lives.

Table of Contents

The Root (/)

Everything in Linux starts at the root directory /. Unlike Windows with drive letters, Linux uses a single unified tree. Every file, device, and partition is mounted somewhere under /. The ls / command reveals the top-level directories defined by the Filesystem Hierarchy Standard (FHS).

/bin   -> /usr/bin   (essential user binaries, symlink)
/sbin   -> /usr/sbin  (system binaries, symlink)
/lib   -> /usr/lib   (libraries, symlink)
/usr   (user system resources)
/etc   (configuration files)
/var   (variable data — logs, spools, caches)
/tmp   (temporary files)
/dev   (device files)
/proc   (process virtual filesystem)
/sys   (kernel/device virtual filesystem)
/home   (user home directories)
/root   (root user's home)
/opt   (optional third-party software)
/mnt   (temporary mount points)

/etc — Configuration

/etc holds system-wide configuration files. Most are plain text. This is where you tweak system behavior.

/etc/passwd    user accounts
/etc/shadow    hashed passwords
/etc/group    user groups
/etc/fstab    filesystem mount table
/etc/hostname   system hostname
/etc/hosts    static hostname resolution
/etc/ssh/sshd_config  SSH server config
/etc/nginx/    Nginx web server config
/etc/apt/     APT package manager config

Always back up config files before editing: cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak.

/var — Variable Data

/var contains data that changes while the system runs — logs, databases, spool files, and caches. It's where the system writes runtime information.

/var/log/syslog   main system log
/var/log/auth.log  authentication log
/var/log/nginx/   web server access/error logs
/var/spool/cron/   cron job spools
/var/cache/apt/   APT package cache
/var/lib/mysql/   MySQL/MariaDB data
/var/tmp/      persistent temp files (survives reboot)
Tip: Use du -sh /var/log to check log sizes. Rotate logs with logrotate to avoid filling the disk.

/proc & /sys — Virtual Filesystems

/proc and /sys are virtual filesystems that expose kernel data structures as files. /proc contains process info and system stats; /sys exposes device and driver information.

cat /proc/cpuinfo     CPU details
cat /proc/meminfo     memory usage
cat /proc/uptime     system uptime
cat /proc/1/cmdline    command line of PID 1 (init)
ls /proc/[PID]/fd/    open file descriptors

# /sys — kernel parameters
cat /sys/class/thermal/thermal_zone*/temp
cat /sys/block/sda/queue/scheduler

/dev — Devices

/dev contains device files representing hardware. Block devices (sda, nvme0n1) handle data in blocks; character devices (tty, random) stream data byte-by-byte.

/dev/sda      first SATA/SCSI disk
/dev/sda1      first partition on sda
/dev/nvme0n1    first NVMe SSD
/dev/tty1      first virtual terminal
/dev/null      discards all data
/dev/zero      outputs null bytes
/dev/random     random number generator
/dev/urandom    faster random generator

/tmp

/tmp is world-writable storage for temporary files. Contents are typically cleared on reboot. Many systems mount /tmp as tmpfs (RAM-backed) for speed.

# /tmp is often a tmpfs mount
df -h /tmp
mount | grep /tmp

/usr & /opt

/usr contains user utilities and applications — binaries, libraries, documentation, and source code. Modern Linux distros symlink /bin, /sbin, /lib into /usr. /opt is for third-party or manually installed software (e.g., Google Chrome, JetBrains IDEs).

/usr/bin/      user commands
/usr/lib/      shared libraries
/usr/share/     architecture-independent data
/usr/local/     locally compiled software
/opt/google/chrome Chrome installation

Mounts & fstab

/etc/fstab defines filesystem mounts that are automatically mounted at boot. Each line specifies: device, mount point, filesystem type, options, dump, and pass.

# /etc/fstab example
UUID=abc123 /        ext4 defaults,noatime 0 1
UUID=def456 /home      ext4 defaults     0 2
UUID=ghi789 swap      swap sw        0 0

# Mount a USB drive manually
sudo mount /dev/sdb1 /mnt/usb
# Unmount
sudo umount /mnt/usb
Tip: Always use mount -a after editing /etc/fstab to test the entries before rebooting.
