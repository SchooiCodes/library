Incident Response

Security Breach Response

A comprehensive step-by-step guide for identifying, containing, and recovering from a security breach — from immediate triage to full system restoration.

Read this guide now, before a breach happens. Once you're compromised, every second counts. Print a copy and keep it in your emergency kit. When you're panicking during a breach, you won't want to search for instructions — you'll want a piece of paper telling you exactly what to do.

Overview

A security breach can take many forms: ransomware encrypting your files, a phishing attack compromising your credentials, malware exfiltrating data, or an attacker gaining remote access to your system. Regardless of the vector, the response follows a predictable pattern: disconnect, assess, contain, eradicate, recover.

This guide covers the full incident response lifecycle for individual users and small businesses. If you are part of a larger organization, follow your formal incident response plan (NIST SP 800-61, SANS PICERL) and involve your security team immediately.

First rule: Do not panic. A calm, methodical response is far more effective than a frantic one. Follow the steps in order. Skipping steps (especially the containment phase) can make the situation worse.

Immediate Response Steps

The first five minutes of a breach determine the outcome. Everything you do in this phase is about containment — stopping the attacker from doing more damage and preserving evidence.

Step 1: Disconnect Immediately

Pull the ethernet cable and turn off Wi-Fi immediately. If on a phone, enable airplane mode. The attacker cannot exfiltrate data or issue commands if the connection is severed.

# Physical disconnect is best
1. Pull ethernet cable from the computer
2. Turn off Wi-Fi (hardware switch or settings)
3. If using a laptop, disable all network adapters:
   Device Manager → Network Adapters → Disable
Do not shut down yet. Shutting down may destroy volatile evidence (running processes, network connections, memory-resident malware). Disconnect the network, then proceed with live analysis.

Step 2: Assess the Situation

Before you act, understand what you're dealing with:

  • Ransomware: Files have been renamed/encrypted, ransom note displayed
  • Malware infection: Unusual popups, browser redirects, slow performance, unknown processes
  • Account compromise: Password changed without your action, emails sent from your account, suspicious login alerts
  • Remote access: Mouse moves on its own, programs open unexpectedly, unknown remote desktop sessions
  • Data breach: Company notification, Have I Been Pwned alert, suspicious activity on financial accounts

Step 3: Boot to Safe Mode or Rescue Environment

Safe Mode loads only essential drivers and services, preventing most malware from running. This gives you a clean environment to scan and remove threats.

# Windows Safe Mode
Shift + Restart → Troubleshoot → Advanced Options → Startup Settings → Restart → Press 4/5/F4/F5

# Or interrupt boot 3 times
Power on → hold power button during Windows logo → repeat 3x → Automatic Repair → Advanced Options

If the system is too compromised to boot even into Safe Mode, use a rescue USB (Hiren's Boot CD PE, SystemRescue, or a Linux live USB) to scan the drive externally.

Step 4: Scan for Malware

Run multiple scanning tools. No single tool catches everything.

# Recommended scanning order (Windows)
1. Malwarebytes (full scan)
2. AdwCleaner (adware/PUP focused)
3. Microsoft Safety Scanner (offline, boot-time scan)
4. ClamAV (open-source, command-line, thorough)

# From a Linux rescue USB
sudo clamscan -r --bell -i /mnt/windows/

Step 5: Investigate Running Processes and Connections

Use Sysinternals tools to identify malicious processes:

# Process Explorer (replacement for Task Manager)
→ Launch procexp.exe as Administrator
→ Look for unsigned processes, suspicious names (misspellings), high CPU/memory
→ Right-click → Check VirusTotal → Submit to VirusTotal for unknown entries

# TCPView (network connections)
→ Launch tcpview.exe
→ Look for connections to unknown IP addresses
→ Close suspicious connections immediately
→ Note remote IPs for firewall blocking

# Autoruns (startup persistence)
→ Launch autoruns.exe
→ Hide Microsoft entries (Options → Hide Microsoft Entries)
→ Review remaining entries for suspicious items
→ Disable (do not delete) suspicious entries

Step 6: Contain and Block

Once you've identified the threat, prevent it from spreading:

  • Block identified C2 (command & control) IP addresses at the firewall
  • Remove malicious startup entries using Autoruns
  • Kill malicious processes with Process Explorer
  • Disable compromised user accounts
  • If ransomware: remove the infected drive and work from a backup on a clean system
# Block IPs at Windows Firewall (PowerShell Admin)
New-NetFirewallRule -DisplayName "Block Malicious C2" -Direction Outbound -RemoteAddress 192.0.2.100 -Action Block

# Or using the GUI
wf.msc → Outbound Rules → New Rule → Custom → Remote IP Address → Add

Step 7: Change Credentials from a Clean Device

Do not change passwords on the infected machine — the attacker may be capturing keystrokes. Use a separate, trusted device (phone, tablet, or another computer).

Step 8: Restore from Backup

Only restore data after the malware is confirmed removed. Scan backup files before restoring. If the backup is on a drive that was connected during the infection, it may also be compromised — use a backup that predates the infection.

Pro tip: If you have a clean system image backup (e.g., Clonezilla or Macrium Reflect), restoring it is the fastest and most reliable recovery method. This is why you take backups before disaster strikes.

8-Step Incident Response Checklist

Quick-reference checklist for when every second counts. Follow these steps in order.

  1. Disconnect — Pull the ethernet cable and turn off Wi-Fi immediately. If on a phone, enable airplane mode.
  2. Boot to safe mode — Restart holding Shift (Windows) or boot from a rescue USB. Do not let the malware load.
  3. Scan offline — Run Malwarebytes, AdwCleaner, and ClamAV from safe mode or a rescue environment.
  4. Check processes — Use Process Explorer and Autoruns to find and disable suspicious entries.
  5. Check connections — Use TCPView to see if malware is phoning home. Block IPs at the firewall.
  6. Change credentials — Use a clean device to change passwords. Enable 2FA everywhere.
  7. Restore from backup — Only after the malware is confirmed removed. Scan backup files before restoring.
  8. Report — File a report with local cybercrime authorities (IC3, local police) and notify your bank if financial data is involved.
Print this checklist. Tape it to your monitor or keep it in your go-bag. When adrenaline is pumping, you will not think clearly — you will follow the paper.

Password Emergency Flow

If you suspect any account has been compromised, act immediately. Use a clean, trusted device — do not change passwords on a potentially infected machine. The attacker may be monitoring keystrokes or session tokens.

Order matters. Attackers often pivot from one compromised account to another. If an attacker controls your email, they can reset every other password. Always start with email.

10-Step Password Emergency Protocol

  1. Email accounts first — If an attacker controls your email, they can reset every other password. Change email passwords immediately. Start with your primary email, then any secondary/recovery emails.
  2. Password manager — Change your master password and check for any new, unknown entries. Revoke all active sessions. Check if the password manager itself reported any unusual activity.
  3. Financial accounts — Banking, PayPal, credit cards, crypto exchanges, investment accounts, and any payment processors. Check recent transactions for unauthorized activity.
  4. Social media — Facebook, Twitter/X, Instagram, LinkedIn, Discord, Telegram, Signal, WhatsApp. Check active sessions and logged-in devices. Revoke all unknown sessions.
  5. Cloud storage — Google Drive, iCloud, OneDrive, Dropbox, Nextcloud. Check for file changes you didn't make. Check sharing settings for unauthorized shares. Review recent activity logs.
  6. Domain & DNS — If you own domains, check that registrar and DNS settings haven't been changed. Attackers often redirect domain traffic to phishing sites. Verify WHOIS contact info.
  7. Device logouts — Use "sign out of all devices" on every service. Revoke all session tokens and API keys. Regenerate OAuth tokens for connected apps.
  8. Enable 2FA — Add two-factor authentication to every service that supports it. Use an authenticator app (Aegis, 2FAS, Google Authenticator), not SMS if possible. Print backup codes and store them offline.
  9. Check forwarding rules — Attackers often set up email forwarding to silently read your mail. Check all forwarding rules, filters, and auto-reply settings on every email account. Look for rules that delete or archive incoming mail.
  10. Monitor — Watch your accounts for suspicious activity for the next 30 days. Set up login alerts where available. Check Have I Been Pwned regularly. Review credit reports for unauthorized accounts.

Using chntpw to Reset Windows Passwords

If you are locked out of a local Windows account and need emergency access, chntpw can reset the password from a Linux live USB.

# Boot from a Linux live USB (Ubuntu, SystemRescue, etc.)
# Open a terminal

# Identify the Windows partition
sudo fdisk -l
# Look for the NTFS partition (e.g., /dev/sda2)

# Mount the Windows partition
sudo mkdir -p /mnt/windows
sudo mount /dev/sda2 /mnt/windows

# Navigate to the SAM directory
cd /mnt/windows/Windows/System32/config

# List users
sudo chntpw -l SAM

# Reset a specific user's password
sudo chntpw -u username SAM

# Interactive menu:
# 1 - Clear (blank) user password
# 2 - Change password
# q - Quit
# Select option 1 and confirm with 'y'
chntpw does not work with Microsoft accounts. It only resets local Windows account passwords. For Microsoft accounts, use the online password reset process from a trusted device.

Bitwarden Emergency Access

Bitwarden Emergency Access allows designated emergency contacts to request access to your vault after a configurable waiting period. This is critical for ensuring family members or trusted colleagues can reach your accounts if you are incapacitated.

# Setup process:
1. Log in to your Bitwarden web vault
2. Go to Settings → Emergency Access
3. Click "Add Emergency Contact"
4. Enter the email of your trusted contact
5. Choose their access level:
   - View: Read-only access to vault items
   - Takeover: Full access, can remove you
6. Set the waiting period (1-7 days):
   - 1 day: Fast access, minimal delay
   - 7 days: Maximum security but slow
7. Your contact will receive an email to accept
8. Once accepted, they can initiate access at any time
Pro tip: Set up Emergency Access now, even if you don't think you need it. Also store a physical copy of your 2FA recovery codes in a safe or safety deposit box. When a breach happens, you may lose access to your authenticator app.

Emergency Boot Decision Tree

When your computer won't start or you suspect tampering, follow this decision tree to identify the right recovery path.

1. Power on → no POST beep / blank screen?
  → Remove all peripherals (USB drives, printers)
  → Reseat RAM modules
  → Clear CMOS (remove battery or jumper)
  → Test with minimal hardware (one stick of RAM, onboard GPU)
  → If still dead: likely motherboard or PSU failure

2. POST but no boot ("Operating System not found")?
  → Enter BIOS (F2/Del/F12) → check boot order
  → Verify drive is detected in BIOS
  → Boot from rescue USB → check partition table with TestDisk
  → Check if Boot Configuration Data is corrupt:
  →   bcdedit /store C:\Boot\BCD /enum
  →   bcdedit /rebuild (from Windows RE Command Prompt)

3. Windows loads but crashes to bluescreen?
  → Boot Safe Mode (F8 or Shift+Restart)
  → Uninstall recent updates (Settings → Windows Update → Update History)
  → Run System Restore to a point before the issue
  → chkdsk /f C: (check filesystem integrity)
  → sfc /scannow (check system file integrity)
  → dism /online /cleanup-image /restorehealth

4. Bluescreen / Kernel panic with error code?
  → Note the error code (e.g., CRITICAL_PROCESS_DIED, IRQL_NOT_LESS_OR_EQUAL)
  → Boot from rescue media
  → Check drive health: CrystalDiskInfo (Windows) or smartctl (Linux)
  →   sudo smartctl -a /dev/sda
  → Test RAM: Memtest86 (boot from USB, let it run at least 1 pass)
  → Restore from backup or reinstall if hardware is failing

5. Ransomware / extortion screen?
  → IMMEDIATELY disconnect power (hold power button 10 seconds)
  → Do not boot into the infected OS
  → Boot from rescue USB (Hiren's, SystemRescue, Ubuntu Live)
  → Do not mount or touch the infected drive unnecessarily
  → Research decryption: check nomoreransom.org for free decryption tools
  → If no decryption available: restore from clean backup
  → As last resort: send one encrypted file to a recovery service
  → Never pay the ransom — it funds criminal operations and does not guarantee data recovery

6. Suspected rootkit / bootkit?
  → Boot from clean media (do not use the installed OS)
  → Use a  boot-time antivirus scanner (Windows Defender Offline, Kaspersky Rescue Disk)
  → Check MBR/GPT integrity: mbrfix /drive 0 fixmbr
  → Rebuild boot sector if corrupted
  → If confirmed: wipe and reinstall. Rootkits are nearly impossible to remove completely.

Boot Environment Comparison

Safe Mode (Windows)

Best for: Malware removal, driver issues, system restore. Loads minimal drivers and services. Most malware does not load here. Can't access the internet unless you choose Safe Mode with Networking.

Live USB (Linux)

Best for: Password reset (chntpw), data recovery, disk imaging. Boots a full OS from USB without touching the internal drive. Can mount and repair Windows partitions. No risk of malware execution from the host OS.

Windows RE

Best for: Startup repair, system restore, command prompt recovery. Built-in recovery environment accessible via Advanced Startup. Contains restore points, safe mode options, and a command prompt for manual repairs.

Hiren's Boot CD PE

Best for: All-in-one rescue. Windows PE environment with pre-loaded tools: partition managers, password reset, data recovery, malware removal, registry editor, disk diagnostics, and browser.

Tool Usage Flow

Different breach scenarios require different tools. Use this flow chart to pick the right tool for your situation.

Scenario A: Suspicious Popups / Adware / Browser Redirects

1. Boot into Safe Mode with Networking
2. Run AdwCleaner (quick scan)
3. Reset browser settings (Chrome: Settings → Reset → Restore settings to defaults)
4. Check for unwanted extensions in all browsers
5. Run Malwarebytes (threat scan)
6. Run Check for unknown scheduled tasks (taskschd.msc)
7. Run Autoruns → disable suspicious startup entries
8. Reboot normally → verify clean

Scenario B: Ransomware / File Encryption

1. DISCONNECT POWER immediately (hold power button)
2. Remove the drive and connect to a clean system via USB adapter
3. Identify the ransomware family:
   → Upload a ransom note to nomoreransom.org
   → Check the encrypted file extension against known ransomware DBs
4. Check for decryption tools:
   → nomoreransom.org
   → Emsisoft Decrypter (emsisoft.com/ransomware-decryption-tools)
   → Avast/AVG Ransomware Decryption Tools
5. If decryption available → use the tool
6. If NO decryption available → restore from backup
7. If NO backup → data recovery is unlikely, consult a specialist
8. Wipe and reinstall the operating system

Scenario C: Malware / Trojan / Backdoor

1. Disconnect from network
2. Capture volatile evidence (Process Explorer, TCPView logs)
3. Boot into Safe Mode
4. Run offline scanners:
   → Malwarebytes (full scan)
   → Windows Defender Offline (boot-time scan)
   → ClamAV (command-line, thorough)
5. Use Process Explorer with VirusTotal integration to identify unknown processes
6. Use Autoruns to find persistence mechanisms:
   → Run keys, Scheduled Tasks, Services, Drivers
7. Use TCPView to identify C2 connections, block IPs at firewall
8. Remove threats in Safe Mode
9. Change all passwords from a clean device
10. Monitor for 30 days for re-infection

Scenario D: Phishing / Credential Theft

1. Do NOT click any links or open attachments in suspicious emails
2. Check Have I Been Pwned for your email addresses
3. Change password for the affected account IMMEDIATELY (from a clean device)
4. Enable 2FA on the affected account
5. Check for unauthorized forwarding rules (especially email)
6. Review account activity logs for unauthorized access
7. If financial account: contact your bank immediately
8. Check other accounts that share the same password
9. Report the phishing email to your email provider (Gmail: Report Phishing)
10. Forward phishing URLs to reportphishing@apwg.org (APWG)

Scenario E: Physical Access / Device Theft

1. Locate device:
   → Windows: account.microsoft.com/devices
   → Android: google.com/android/find
   → Apple: icloud.com/find
2. Lock the device remotely
3. Display a message with contact info on the lock screen
4. If recovery seems unlikely → remote wipe
5. Change passwords for all accounts that were logged in on the device
6. Revoke session tokens for all services
7. If the device had unencrypted sensitive data → notify affected parties
8. File a police report (needed for insurance claims)
9. Update device inventory and review physical security procedures

Prevention & Preparation

The best breach response is the one you never need. Take these steps now to prepare for the worst.

Backup Strategy

Follow the 3-2-1 rule: 3 copies, 2 different media, 1 off-site. Use the Backup Strategies guide for implementation.

Emergency USB

Create a Ventoy multi-boot drive with Hiren's Boot CD PE, a Linux live ISO, Windows install media, and Memtest86. Update ISOs every 6 months.

Password Hygiene

Use a password manager. Enable 2FA everywhere. Store backup codes offline. Set up Bitwarden Emergency Access. Use unique passwords for every account.

Incident Plan

Print this guide and the 8-step checklist. Store them with your emergency USB. Review the procedure with family members or trusted colleagues who may need to act in your absence.

You are now prepared. Bookmark this page, download the portable tools, and build your emergency USB today. When (not if) a breach happens, you will know exactly what to do.
