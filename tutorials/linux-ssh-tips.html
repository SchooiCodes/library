🔑 SSH Tips & Tricks

Advanced SSH usage — config files, key management, tunnels, jump hosts, agent forwarding, and security hardening.

SSH Config File

Save time by defining hosts in ~/.ssh/config. No more typing long connection strings.

# ~/.ssh/config

Host myserver
  HostName 192.168.1.100
  User pi
  Port 2222
  IdentityFile ~/.ssh/mykey

Host github
  HostName github.com
  User git
  IdentityFile ~/.ssh/github-key

# Then connect with:
ssh myserver
ssh github

You can also use wildcards to apply settings to groups of hosts:

Host *.local
  User admin
  Port 22
  Compression yes

Key Management

Use Ed25519 keys — faster and more secure than RSA:

# Generate Ed25519 key
ssh-keygen -t ed25519 -a 100 -f ~/.ssh/id_ed25519

# Copy key to server
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server

# Or manually
cat ~/.ssh/id_ed25519.pub | ssh user@server "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
Tip: Use ssh-keygen -t ed25519 -a 100 for better KDF rounds (default is 16). Higher -a values make cracking the key harder.

Port Forwarding (Tunnels)

Forward ports through SSH for secure remote access.

# Local forwarding: access remote service locally
# Remote service on server:5432 -> localhost:5432
ssh -L 5432:localhost:5432 user@server

# Remote forwarding: expose local service to remote
# Local service on localhost:3000 -> server:8080
ssh -R 8080:localhost:3000 user@server

# Dynamic forwarding (SOCKS proxy)
ssh -D 1080 user@server
# Then configure browser to use SOCKS5 proxy at localhost:1080

Jump Hosts

Reach a target server through a bastion/jump host without manual chaining:

# Direct jump host
ssh -J user@jump-host user@target-server

# With config file
Host jump
  HostName jumphost.example.com
  User bastion

Host internal
  HostName 10.0.1.50
  User appuser
  ProxyJump jump

# Then simply:
ssh internal

Agent Forwarding

Forward your local SSH agent to a remote server so you can SSH from there using your local keys (e.g., to pull from GitHub).

# Start the agent
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# Connect with agent forwarding
ssh -A user@server

# In config file:
Host server
  ForwardAgent yes

# Security: use -a instead of -A to explicitly enable only when needed

Hardening SSH

Edit /etc/ssh/sshd_config on the server for better security:

# Disable password authentication
PasswordAuthentication no

# Disable root login
PermitRootLogin prohibit-password

# Use only key-based auth
PubkeyAuthentication yes

# Use a non-standard port (optional, reduces log noise)
Port 2222

# Allow only specific users
AllowUsers alice bob

# Restart SSH
sudo systemctl restart sshd

Additional tips: use fail2ban to block brute force attempts, keep SSH updated, and audit logs with journalctl -u ssh.
