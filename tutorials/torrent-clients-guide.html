🟢 Transmission Setup

Best for: Headless servers, minimal resource usage, macOS users.

Installation

# Ubuntu/Debian
sudo apt install transmission-daemon transmission-remote-cli
sudo systemctl stop transmission-daemon
# Edit config before starting
sudo nano /etc/transmission-daemon/settings.json
sudo systemctl start transmission-daemon

# macOS (GUI)
brew install --cask transmission

# Docker
docker run -d --name=transmission \
  -p 9091:9091 -p 51413:51413 \
  -v /path/to/downloads:/downloads \
  ghcr.io/linuxserver/transmission

Quick Config Settings

# Edit /etc/transmission-daemon/settings.json (stop daemon first)
{
  "rpc-authentication-required": true,
  "rpc-username": "admin",
  "rpc-password": "yourpassword",
  "rpc-whitelist-enabled": false,
  "download-dir": "/path/to/downloads",
  "incomplete-dir": "/path/to/incomplete",
  "peer-port": 51413,
  "port-forwarding-enabled": true,
  "encryption": 1
}

🟣 Deluge Setup

Best for: Plugin ecosystem, lightweight daemon + rich GUI.

Installation

# Ubuntu/Debian
sudo apt install deluged deluge-web deluge-console
sudo systemctl enable --now deluged

# Start the web interface
deluge-web --fork

# Access at http://localhost:8112 (password: deluge)

# Docker
docker run -d --name=deluge \
  -p 8112:8112 -p 58846:58846 -p 58946:58946 \
  -v /path/to/config:/config \
  -v /path/to/downloads:/downloads \
  ghcr.io/linuxserver/deluge

🔒 VPN Binding (Kill Switch)

VPN binding ensures your real IP is never exposed — even if the VPN drops, the torrent client stops immediately.

qBittorrent

  1. Go to Settings → Advanced → Network Interface
  2. Select your VPN interface (e.g., tun0, wg0, utun4)
  3. Only traffic through that interface will be allowed

Transmission

# Add to settings.json:
"bind-address-ipv4": "10.8.0.2"  # Your VPN IP

Deluge

  1. Go to Preferences → Network → Network Interface
  2. Select your VPN interface from the dropdown
Verify your IP: After binding, check your torrent IP at check.torrentprivacy.com or by downloading a test torrent and checking the "peers" list.

🔌 Port Forwarding

Port forwarding improves connection speed by allowing incoming peer connections. Without it, you can only connect to peers that DO have open ports (slower speeds).

  1. Choose a port (49152-65535 range recommended)
  2. Set it in your client's connection settings
  3. Forward the port in your router: Settings → NAT → Port Forwarding
  4. If using a VPN with port forwarding, use that port instead (e.g., Mullvad, ProtonVPN, AirVPN)
  5. Verify at canyouseeme.org

💻 Headless & Web UI

All three clients offer web interfaces for remote management:

  • qBittorrent: http://your-server:8080 (enable in Settings → Web UI)
  • Transmission: http://your-server:9091
  • Deluge: http://your-server:8112 (password: deluge)

Combine with a VPN for privacy. See the 📖 VPN Setup Guide for configuration.
