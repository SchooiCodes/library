✍️ Neovim/Vim Editor Guide

From beginner to power user — navigation, modes, Lua config, plugins, and more.

Why Neovim?

Neovim is a modern fork of Vim, designed to extend the classic modal editor with better extensibility, built-in LSP support, a Lua runtime for configuration, and a more maintainable codebase. It is fully compatible with Vim but adds powerful features that make it an excellent choice for developers who want a fast, keyboard-driven editor.

Basic Navigation

The most important skill in Neovim is moving efficiently without touching the mouse:

KeyAction
h j k lLeft / Down / Up / Right
wJump forward to start of next word
bJump backward to start of word
eJump to end of word
0Jump to beginning of line
$Jump to end of line
ggJump to first line of file
GJump to last line of file
:42Jump to line 42
Ctrl+d / Ctrl+uPage down / Page up
%Jump to matching bracket
zzCenter cursor on screen

Modes

Neovim is a modal editor. Every key behaves differently depending on the current mode:

ModeHow to EnterPurpose
NormalEsc (default)Navigation, manipulation, commands. This is your resting mode.
Inserti, a, oTyping text. i inserts before cursor, a after, o opens new line.
VisualvSelecting text. V for line-wise, Ctrl+v for block-wise.
Command-line:Running commands (:w, :q, :e, :%s/foo/bar/g)

Pro tip: Never stay in Insert mode. Normal mode is your home — use it for navigating and manipulating, only switch to Insert mode when you need to type new text.

Configuration with Lua (init.lua)

Neovim's configuration file is ~/.config/nvim/init.lua. Here is a minimal starter:

-- ~/.config/nvim/init.lua

-- Basic options
vim.opt.number = true          -- Show line numbers
vim.opt.relativenumber = true  -- Relative line numbers
vim.opt.tabstop = 2
vim.opt.shiftwidth = 2
vim.opt.expandtab = true       -- Spaces instead of tabs
vim.opt.mouse = 'a'            -- Enable mouse in all modes
vim.opt.termguicolors = true   -- True color support
vim.opt.clipboard = 'unnamedplus' -- System clipboard

-- Leader key
vim.g.mapleader = ' '

-- Basic keymaps
vim.keymap.set('n', '<leader>w', ':w<CR>', { desc = 'Save file' })
vim.keymap.set('n', '<leader>q', ':q<CR>', { desc = 'Quit' })
vim.keymap.set('n', '<leader>e', ':NvimTreeToggle<CR>', { desc = 'Toggle file tree' })

-- Bootstrap lazy.nvim plugin manager
local lazypath = vim.fn.stdpath('data') .. '/lazy/lazy.nvim'
if not vim.loop.fs_stat(lazypath) then
  vim.fn.system({
    'git', 'clone',
    'https://github.com/folke/lazy.nvim.git',
    '--branch=stable',
    lazypath,
  })
end
vim.opt.rtp:prepend(lazypath)

Plugin Manager: lazy.nvim

lazy.nvim is the most popular plugin manager for Neovim. It provides lazy-loading, automatic setup, and a clean declarative syntax:

-- Load plugins with lazy.nvim
require('lazy').setup({
  { 'catppuccin/nvim', name = 'catppuccin', priority = 1000 },
  { 'nvim-telescope/telescope.nvim', dependencies = { 'nvim-lua/plenary.nvim' } },
  { 'nvim-treesitter/nvim-treesitter', build = ':TSUpdate' },
  { 'VonHeikemen/lsp-zero.nvim', dependencies = {
    'neovim/nvim-lspconfig',
    'hrsh7th/nvim-cmp',
    'hrsh7th/cmp-nvim-lsp',
  }},
  { 'folke/which-key.nvim', config = true },
})

Essential Plugins

  • telescope.nvim — Fuzzy finder for files, buffers, help tags, grep results, and more. The single most impactful plugin for productivity.
  • nvim-treesitter — Syntax highlighting, code navigation, and structural editing powered by tree-sitter. Dramatically better than regex-based highlighting.
  • lsp-zero.nvim — A beginner-friendly way to set up LSP (Language Server Protocol) for autocompletion, go-to-definition, hover docs, diagnostics, and more.
  • which-key.nvim — Popup that shows available keybindings as you type the leader key. Essential for learning your config.
  • nvim-tree.lua — File explorer sidebar with git status icons.
  • alpha-nvim — A customizable dashboard (like the vim-startify plugin).

Learning Resources

  • vimtutor — Built-in interactive tutorial. Run nvim +Tutor or vimtutor in your terminal.
  • Vim Adventures — A fun game that teaches Vim navigation.
  • Neovim Official Docs — :help is your best friend. :help user-manual is an excellent starting point.
  • awesome-neovim — Curated list of plugins, colorschemes, and configurations.
  • YouTube channels: ThePrimeagen, TJ DeVries (Neovim core maintainer), Josean Martinez, chris@machine.
Remember: Learning Vim motions takes time. Expect to be slow for the first week. Stick with it — after a month, you'll likely be faster than you ever were with mouse + menu.
