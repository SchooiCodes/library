Planning Your Homelab

Define your goals first: learning Kubernetes, self-hosting media services, running VMs, or a mix. Start small and scale up. A homelab can be a single Raspberry Pi or a rack of enterprise servers.

  • Learning lab: 1-2 old PCs or NUCs, Proxmox, Docker
  • Media server: NAS + mini PC running Jellyfin / Plex
  • Network lab: Managed switch, router/firewall (OPNsense), VLANs

Hardware Options

HardwareProsCons
Old desktop / laptopFree, powerful enough to startPower hungry, noisy, bulky
Intel NUC / mini PCLow power, quiet, smallLimited expansion, pricier
Raspberry Pi 4/5Very low power, silentARM limitations, less RAM
Used enterprise serverHigh core/ram count, cheapLoud, hot, high power draw
Tip: Start with a used mini PC (e.g., HP EliteDesk 800 G3/G4, ~$80-120 on eBay). They offer great performance per watt for a first homelab.

Hypervisor — Proxmox

Proxmox VE is a Debian-based hypervisor that supports both KVM VMs and LXC containers. Install on bare metal, then manage via web UI.

# Download Proxmox VE ISO, write to USB:
dd if=proxmox-ve_*.iso of=/dev/sdX bs=4M status=progress

# After install, access web UI:
https://your-server-ip:8006

# Basic post-install:
apt update && apt dist-upgrade -y
pveversion -v

Docker & Services

Run services in Docker containers for isolation and easy management. Install Docker on a VM or LXC:

curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER

# docker-compose.yml example:
version: "3.8"
services:
 nginx:
  image: nginx:alpine
  ports:
   - "80:80"
  volumes:
   - ./html:/usr/share/nginx/html

Popular self-hosted services: Jellyfin (media), Vaultwarden (passwords), Nextcloud (cloud), AdGuard Home (DNS), Portainer (Docker management).

Security

  • Use strong passwords and SSH keys (disable password auth)
  • Isolate services with VLANs and firewall rules
  • Keep systems updated — enable auto-updates for security patches
  • Use reverse proxy (Nginx/Caddy) with SSL (Let's Encrypt) for web services
  • Monitor logs with Grafana + Loki or Splunk free

Maintenance

Automate updates, backup configs, and monitor disk health:

# Smartmontools for disk health
sudo apt install smartmontools
sudo smartctl -H /dev/sda

# Regularly check:
# - Disk space: df -h
# - Memory: free -h
# - Uptime & load: uptime
# - Docker logs: docker logs -f container_name
