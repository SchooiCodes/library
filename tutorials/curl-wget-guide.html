🌐 curl & wget Guide

Master HTTP requests from the command line — curl for APIs, wget for downloads, headers, authentication, and scripting.

Table of Contents

Getting Started with curl

curl is a swiss-army knife for HTTP. Every developer should have it memorised for API work, debugging, and automation.

Basic GET Request

curl https://api.github.com/users/SchooiCodes

GET with Headers

curl -H "Authorization: Bearer ghp_yourtoken" \
   -H "Accept: application/vnd.github.v3+json" \
   https://api.github.com/user

POST with JSON Body

curl -X POST https://jsonplaceholder.typicode.com/posts \
   -H "Content-Type: application/json" \
   -d '{"title":"foo","body":"bar","userId":1}'
Use curl -i to include response headers in the output — invaluable for debugging status codes and rate limits.

Follow Redirects & Save Output

curl -L -o output.html https://httpbin.org/redirect/3

Authentication & Cookies

curl -u username:password https://httpbin.org/basic-auth/username/password
curl -c cookies.txt -b cookies.txt https://httpbin.org/cookies

Mastering wget

wget excels at recursive downloads, mirroring sites, and resuming interrupted transfers.

Simple Download with Resume

wget -c https://releases.ubuntu.com/22.04/ubuntu-22.04.3-desktop-amd64.iso

Recursive Site Mirror

wget --mirror --convert-links --adjust-extension \
   --page-requisites --no-parent \
   https://example.com/docs/

Download with Custom Filename & Rate Limit

wget -O archive.zip --limit-rate=500k https://example.com/large.zip
The -c (continue) flag in wget lets you resume a partial download — a lifesaver on flaky connections.

API Testing with curl

Combine curl with jq for powerful API pipelines:

curl -s https://api.github.com/repos/curl/curl/releases/latest | jq '.tag_name'

Measure Response Time

curl -w "\nTotal: %{time_total}s\nHTTP: %{http_code}\n" \
   -o /dev/null -s https://example.com

Scripting curl

#!/bin/bash
for id in 1 2 3 4 5; do
 curl -s "https://jsonplaceholder.typicode.com/posts/$id" | jq '.title'
done

Quick Reference

CommandDescription
curl -sSSilent but show errors
curl -vVerbose (full request/response)
curl -kSkip SSL verification (dev only)
wget -i urls.txtDownload from a list of URLs
wget -q --show-progressQuiet with progress bar
Pro tip: alias curldbg to curl -sS -w "\nHTTP %{http_code} | %{time_total}s\n" -o /dev/null for quick endpoint health checks.
