What is WSL?

WSL (Windows Subsystem for Linux) lets you run a full Linux distribution directly inside Windows — no virtual machine needed. WSL2 uses a real Linux kernel for full system call compatibility. It provides native file system access, seamless integration with Windows tools, and is perfect for developers who need Linux tools on Windows.

Why WSL? Run bash, Python, Node.js, Docker, git, and thousands of Linux packages without dual-booting or setting up a VM.

Installation

  1. Open PowerShell as Administrator
  2. Run: wsl --install

This installs WSL2 and Ubuntu by default. Restart when prompted, then set up your Linux username and password on first launch.

Older Windows versions: Enable "Windows Subsystem for Linux" in Windows Features first, then run wsl --set-default-version 2.

Basic WSL Commands

Run these from PowerShell or CMD:

CommandDescription
wsl --list --verbose or wsl -l -vList installed distros with their WSL version
wsl --set-version <name> 2Convert a distro to WSL2
wsl --set-default-version 2Set WSL2 as the default for new installs
wsl --set-default <name>Set the default WSL distro
wsl --export <name> <file.tar>Export a distro to a tar archive
wsl --import <name> <folder> <file.tar>Import a distro from a tar archive
wsl --terminateShut down a specific WSL instance
wsl --shutdownShut down all WSL instances
wsl --install -d <distro>Install a specific distro (Ubuntu, Debian, Kali, Alpine, etc.)

Accessing Files

FromAccessPath
Windows → WSL filesFile Explorer\\wsl$\Ubuntu\home\yourname\
WSL → Windows filesTerminal/mnt/c/Users/YourName/
Best practice: Keep project files inside the WSL filesystem (not /mnt/c/...) for better I/O performance. VS Code integration: run code . in your WSL terminal — it automatically opens with the Remote - WSL extension.

Package Management

# Update package lists
sudo apt update && sudo apt upgrade -y

# Install a package
sudo apt install <package>

# Remove a package
sudo apt remove <package>

# Clean up unused packages
sudo apt autoremove

# Search for packages
sudo apt search <term>

Setting Up a Dev Environment in WSL

Node.js

curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash - && sudo apt install -y nodejs

Python

Usually pre-installed. Install pip:

sudo apt install python3 python3-pip

Docker

Install Docker Desktop for Windows with WSL integration enabled.

Git

sudo apt install git

VS Code

Install VS Code on Windows, then run code . in your WSL terminal. Requires the Remote - WSL extension.

Useful Tools to Install in WSL

sudo apt install build-essential curl wget git unzip neofetch htop
  • fish shell — Install with sudo apt install fish, set as default with chsh -s /usr/bin/fish
  • fzf — sudo apt install fzf — fuzzy finder for command-line
  • ripgrep — sudo apt install ripgrep — blazingly fast grep replacement

Networking

  • WSL2 uses NAT — it has a different IP from Windows by default
  • Access WSL from Windows — use localhost (ports are forwarded automatically)
  • Access Windows from WSL — use the Windows IP (check with cat /etc/resolv.conf)
  • Port forwarding — WSL2 ports are automatically forwarded to Windows; no extra config needed

Troubleshooting

IssueSolution
General WSL issueswsl --shutdown then restart — fixes most problems
Corrupted distrowsl --unregister <name>, then reinstall
Low disk spacesudo apt autoremove and sudo journalctl --vacuum-size=100M
WSL not startingCheck that virtualization is enabled in BIOS
DNS issuesCreate /etc/wsl.conf with [network] generateResolvConf = false and set a manual DNS server
