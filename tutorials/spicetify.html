Spotify Modding

Spicetify

Customize the Spotify desktop client with themes, extensions, and custom CSS using the Spicetify CLI modding tool.

Overview

Spicetify is a command-line tool that modifies the Spotify desktop client (Windows, Linux, macOS). It allows you to install themes, extensions, and custom CSS without patching the actual Spotify binary — instead, it modifies the client's web-based UI assets.

Spicetify works by injecting modified CSS and JavaScript into Spotify's interface during launch. It supports both the current Spicetify Creator (v2) and the legacy Spicetify v1 (which works on older Spotify versions).

Safety Warning. Download Spicetify only from the official GitHub repository: https://github.com/spicetify/cli. Do not use pre-compiled binaries from untrusted sources. Always verify checksums if available.
Spotify Premium Required. Spicetify only works with the official Spotify desktop client, which requires a Premium account (or you can use it in offline mode with a free account, but functionality is limited).

Installation

Windows

# Install via PowerShell (Run as Administrator):
iwr -useb https://raw.githubusercontent.com/spicetify/cli/main/install.ps1 | iex

# Or install via winget:
winget install spicetify.cli

# After installation, apply the mod:
spicetify
spicetify backup apply

Linux

# Install via the install script:
curl -fsSL https://raw.githubusercontent.com/spicetify/cli/main/install.sh | sh

# Or via package managers:
# Arch (AUR):
yay -S spicetify-cli

# Homebrew (Linuxbrew):
brew install spicetify/cli/spicetify

# Nix:
nix-shell -p spicetify-cli

# After installation, set Spotify path and apply:
spicetify config spotify_path /usr/share/spotify  # Adjust for your distro
spicetify backup apply

On Linux, you may need to set the Spotify path manually. Common locations:

# Snap install:
/snap/spotify/current/usr/share/spotify

# Flatpak install:
/var/lib/flatpak/app/com.spotify.Client/current/active/files/extra/share/spotify

# Native install (Arch):
/opt/spotify

macOS

# Install via Homebrew:
brew install spicetify/cli/spicetify

# Or use the install script:
sh -c "$(curl -fsSL https://raw.githubusercontent.com/spicetify/cli/main/install.sh)"

# Apply:
spicetify backup apply

Post-Installation

After running spicetify backup apply, Spotify will open with Spicetify injected. You should see a small Spicetify icon in the top-left corner of the Spotify window.

Spicetify Marketplace

The Marketplace is an in-app browser for themes and extensions. You can install it directly from Spicetify:

# Install the Marketplace extension:
spicetify config extensions marketplace.js
spicetify apply

# Restart Spotify. You will see a "Marketplace" icon in the sidebar.

The Marketplace lets you browse, install, and update themes and extensions without touching the terminal. It is community-driven, so vet extensions before installing them.

Themes

Themes completely overhaul the Spotify UI. They are written in CSS and JavaScript.

Installing a Theme Manually

# List available themes:
spicetify config current_theme

# Install a theme from the community repository:
# Clone or download a theme to ~/.config/spicetify/Themes/
git clone https://github.com/spicetify/spicetify-themes.git
cp -r spicetify-themes/NameOfTheme ~/.config/spicetify/Themes/

# Apply the theme:
spicetify config current_theme NameOfTheme
spicetify apply

Popular Themes

  • Dribbblish — A colorful, modern theme with rounded album art.
  • Text — Minimal text-only UI — no album art or images, just typography.
  • Sleek — Glassmorphism design with blur effects and dark/light variants.
  • Bloom — Beautiful gradient-based theme with dynamic colors from album art.
  • Turntable — Vinyl record player visualizer for the "Now Playing" view.
  • Onepunch — Compact layout inspired by OnePunch, good for small screens.
  • Dark Night — A true dark theme with AMOLED black backgrounds.

Color Schemes

Many themes support color schemes. You can switch between them:

# Switch to a different color scheme for the current theme:
spicetify config color_scheme SchemeName
spicetify apply

# See available schemes in the theme's directory:
ls ~/.config/spicetify/Themes/YourTheme/color.ini

Extensions

Extensions add functionality to Spotify. They are JavaScript files that hook into Spotify's internal API.

ExtensionFunction
skip-or-go-while-listeningRemoves the "Listening on Spotify" banner when skipping or seeking.
fullscreenAdds a native fullscreen button for the Now Playing view.
shuffle+Better shuffle algorithm — avoids playing the same songs repeatedly.
volume-profileSet different volume levels per device.
bookmarksSave positions within long tracks or podcasts.
trashbinRecently removed songs from playlists — undo removal.
old-layoutReverts Spotify's UI to the classic layout.
Ad Blocking. Spicetify does not block audio ads. Extensions that claim to block ads will likely stop working after a Spotify update. Use a Pi-hole or a modified client at your own risk. This guide does not endorse ad circumvention on free-tier accounts.

Custom CSS

You can add your own CSS rules to fine-tune Spotify's appearance without a full theme:

# Create or edit the custom CSS file:
nano ~/.config/spicetify/CustomCSS/custom.css

# Example custom CSS:
/* Hide the "Upgrade to Premium" button */
button[data-testid="upgrade-button"] { display: none !important; }

/* Round the album art corners */
.main-coverSlotExpanded-cover img { border-radius: 16px !important; }

/* Make the queue sidebar wider */
.main-queue-tabBar { min-width: 400px !important; }

/* Hide podcasts from the homepage */
.main-home-homeHeader + * { display: none !important; }

After editing, apply with spicetify apply.

Backup & Restore

Before making changes, always back up your original Spotify files. Spicetify does this automatically, but you can also do it manually:

# Backup current config and theme selections:
spicetify backup

# Restore Spotify to its original state:
spicetify restore

# Back up your entire Spicetify configuration (themes, extensions, settings):
cp -r ~/.config/spicetify ~/.config/spicetify.backup.$(date +%Y%m%d)

# Restore from backup:
cp -r ~/.config/spicetify.backup.* ~/.config/spicetify
spicetify apply

The spicetify restore command is useful after a Spotify update that breaks the mod — run restore, wait for Spotify to update, then apply again.

Troubleshooting

Spicetify Says "Spotify is not detected"

You need to set the Spotify path manually:

spicetify config spotify_path /path/to/spotify

# Find the Spotify binary location:
# Linux:
which spotify
# macOS:
mdfind kMDItemKind==Application | grep Spotify
# Windows (usually):
C:\Users\%USERNAME%\AppData\Roaming\Spotify

Spicetify Not Loading After Spotify Update

Spotify updates overwrite the files Spicetify patches. Run spicetify apply again after an update.

Theme or Extension Not Working

Check the Spicetify logs:

spicetify debug

# Or view the Spotify developer console:
# Open Spotify, right-click, select "Inspect" (if enabled).
# Look for red error messages in the Console tab.

Spicetify Fails to Apply with "Incompatible Version"

Your Spicetify version may be too old for the latest Spotify client. Update Spicetify:

# Windows:
spicetify upgrade

# Linux/macOS (Homebrew):
brew upgrade spicetify-cli

# Manual:
git -C ~/.spicetify pull

Marketplace Shows "Not Installed"

Re-install the Marketplace extension:

spicetify config extensions marketplace.js
spicetify apply

Tips & Best Practices

  • Backup before updates. Always run spicetify backup before updating Spotify or Spicetify.
  • Use the Marketplace. It simplifies theme and extension management. You can browse without leaving Spotify.
  • Keep themes minimal. Too many heavy CSS animations can slow down the client.
  • Check compatibility. Some themes/extensions conflict. Install one at a time and test.
  • Join the community. The Spicetify Discord and Reddit (/r/spicetify) are great for finding new themes and getting help.
  • Automate re-apply on login. On Linux, add spicetify apply to your startup scripts if Spotify auto-updates break Spicetify frequently.
  • Do not use ad-block extensions. They are fragile, easily detected, and may violate Spotify's ToS more seriously than cosmetic mods.
  • Use color schemes. Many themes have preset color schemes. Experiment with them before writing custom CSS.
  • Read logs. If something does not work, spicetify debug usually tells you exactly what is wrong.
