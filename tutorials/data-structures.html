Data structures — arrays, linked lists, trees, hash tables, and advanced structures.

Linear & Tree Structures

Arrays: contiguous memory, O(1) random access, O(n) insert/delete. Dynamic arrays: ArrayList, vector, List — amortized O(1) append. Linked lists: singly (next pointer), doubly (prev + next), circular. Operations: O(n) search, O(1) insert/delete at head. Stacks: LIFO — push/pop O(1), used for undo, expression evaluation, DFS. Queues: FIFO — enqueue/dequeue O(1), used for BFS, scheduling. Deque: double-ended queue. Trees: Binary Tree (each node 0-2 children), Binary Search Tree (left < node < right — O(log n) average), AVL Tree (self-balancing — rotations), Red-Black Tree (approximate balance — used in TreeMap, std::map), B-Tree (multi-way — database indexes), Trie (prefix tree — autocomplete), Segment Tree (range queries), Fenwick Tree (prefix sums).

Hash Tables & Graphs

Hash tables: key-value pairs with O(1) average lookup. Hash function: maps key to bucket index. Collision resolution: chaining (linked list per bucket), open addressing (linear probing, quadratic probing, double hashing). Load factor: resize when table is X% full. HashMap (Java), unordered_map (C++), dict (Python), Map (JS). Graphs: directed/undirected, weighted/unweighted. Representations: adjacency matrix (Vx2, dense, O(1) edge check), adjacency list (list per vertex, sparse, O(V+E) traversal). DFS: stack/recursion, O(V+E). BFS: queue, O(V+E). Dijkstra: shortest path (non-negative weights) — O(E log V). A*: heuristic-guided pathfinding. Topological sort: DAG ordering — Kahn's algorithm. Union-Find: disjoint sets — Kruskal's MST.
