Modern Shell Scripting (Zsh & Fish)

Syntax, functions, autocompletion, theming, and practical shell scripting in Zsh and Fish.

Zsh vs Fish vs Bash

Modern shells offer significant improvements over traditional Bash:

  • Zsh -- backwards-compatible with Bash, with enhanced completion, globbing, and theming (Oh My Zsh)
  • Fish -- 'friendly interactive shell' with syntax highlighting, autosuggestions, and a web-based configuration
  • Bash -- the POSIX standard, available everywhere but lacking modern ergonomics

Check your current shell and change default:

echo $SHELL
which zsh
chsh -s $(which zsh)  # Change default shell

# Install fish
sudo apt install fish
chsh -s $(which fish)

Zsh Scripting & Configuration

Zsh scripts resemble Bash but with powerful enhancements:

#!/usr/bin/env zsh

# Arrays are 1-indexed
fruits=("apple" "banana" "cherry")
echo $fruits[2]  # banana

# Glob qualifiers -- filter by type, age, etc.
echo *.txt(.Om)  # Regular files, sorted by modification time

# Extended globbing
setopt extended_glob
ls ^*.txt     # Everything except .txt files
ls **/*.py    # Recursive glob (already works)

# Named directory expansion
hash -d docs=~/Documents/projects
echo ~docs  # Expands to ~/Documents/projects

Customize Zsh with ~/.zshrc. Use Oh My Zsh for plugin management and Powerlevel10k for theming.

Fish Scripting & Configuration

Fish has its own syntax -- not compatible with POSIX sh, but cleaner:

#!/usr/bin/env fish

# Variables -- no $ for assignment
set name "Alice"
echo $name

# Lists (1-indexed)
set fruits apple banana cherry
echo $fruits[2]  # banana

# Functions
function hello
    set name $argv[1]
    echo "Hello, $name!"
end

hello "World"

# Command substitution uses () not $()
echo (date)

# Conditional
grep -q "error" /var/log/syslog; and echo "Found errors"

# Autoload functions from ~/.config/fish/functions/

Fish configuration is in ~/.config/fish/config.fish. Use Fisher or Oh My Fish for plugin management.

Autocompletion & Prompt Customization

Modern shells excel at providing rich completions:

# Zsh -- write completions
# ~/.zsh/completion/_mycommand
#compdef mycommand

_mycommand() {
  local -a subcommands
  subcommands=(
    'start:Start the service'
    'stop:Stop the service'
    'status:Show status'
  )
  _describe 'command' subcommands
}

# Prompt customization (Powerlevel10k)
# In ~/.zshrc:
source ~/powerlevel10k/powerlevel10k.zsh-theme

# Fish prompt customization
function fish_prompt
    set_color green
    echo -n (prompt_pwd)
    set_color normal
    echo -n ' $ '
end

Install popular prompt themes: Starship (cross-shell), Powerlevel10k (Zsh), Pure (Zsh/Fish).

Error Handling & Debugging

Modern shell scripts should include robust error handling:

#!/usr/bin/env zsh

set -e          # Exit on error
set -o pipefail # Fail pipeline if any command fails
set -u          # Error on undefined variables

# Trap errors for cleanup
trap 'echo "Error on line $LINENO"; cleanup' ERR

cleanup() {
    rm -f /tmp/build_*
    echo "Cleaned up"
}

# Validate arguments
if [[ $# -lt 1 ]]; then
    echo "Usage: $0 <directory>" >&2
    exit 1
fi

# Debugging
set -x  # Print commands before executing
# ... debug code ...
set +x  # Disable debug
