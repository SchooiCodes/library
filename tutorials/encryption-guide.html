🔐 Encryption Guide

GPG for email/files, VeraCrypt for disk encryption, LUKS for full-disk, gocryptfs for per-directory encryption, and TLS basics.

GPG — Keys, Encrypt & Sign

GNU Privacy Guard implements OpenPGP. Use it to encrypt files, emails, and verify signatures.

# Generate a key pair
gpg --full-generate-key

# List keys
gpg --list-keys
gpg --list-secret-keys

# Export public key
gpg --export --armor my.email@example.com > public.asc

# Import a key
gpg --import friend.asc

# Encrypt a file (to recipient)
gpg --encrypt --recipient friend@example.com secret.txt

# Decrypt
gpg --decrypt secret.txt.gpg

# Sign a file
gpg --sign document.txt

# Verify signature
gpg --verify document.txt.gpg

# Encrypt and sign
gpg --encrypt --sign --recipient friend@example.com doc.txt
Tip: Always set an expiration date on your keys. Use a hardware token (YubiKey) to store private keys for better security.

LUKS Full-Disk Encryption

Linux Unified Key Setup — standard for Linux disk encryption. Encrypts entire block devices.

# Encrypt a disk partition
sudo cryptsetup luksFormat /dev/sdX1

# Open the encrypted volume
sudo cryptsetup open /dev/sdX1 cryptroot

# Format and mount
sudo mkfs.ext4 /dev/mapper/cryptroot
sudo mount /dev/mapper/cryptroot /mnt

# Add a passphrase
sudo cryptsetup luksAddKey /dev/sdX1

# Dump header info
sudo cryptsetup luksDump /dev/sdX1

# Backup LUKS header
sudo cryptsetup luksHeaderBackup /dev/sdX1 --header-backup-file luks-header-backup.bin

For full-disk encryption during install, most distros (Ubuntu, Fedora, Arch) offer LUKS as an option at setup time.

VeraCrypt Volumes

Cross-platform (Windows, Linux, macOS). Supports hidden volumes and plausible deniability.

# Create a volume (CLI)
veracrypt --create /path/to/volume.hc
# Follow prompts — size, password, filesystem

# Mount
veracrypt /path/to/volume.hc /mnt/veracrypt1

# Dismount
veracrypt --dismount /mnt/veracrypt1

# Create a hidden volume (nested inside outer volume)
# Use GUI for easier hidden volume creation

VeraCrypt also supports encrypting whole partitions or system drives (Windows only for system encryption).

gocryptfs — Per-Directory Encryption

Encrypted overlay filesystem in userspace (uses FUSE). Encrypts individual files, not the whole disk.

# Initialize a gocryptfs directory
mkdir ~/crypt ~/crypt.mount
gocryptfs -init ~/crypt
# This creates gocryptfs.conf and gocryptfs.diriv

# Mount
gocryptfs ~/crypt ~/crypt.mount

# Unmount
fusermount -u ~/crypt.mount

# Uses AES-256-GCM, per-file IVs, and file names are encrypted

Excellent for cloud sync folders (Nextcloud, Dropbox) — only encrypted files leave your device.

SSL/TLS Overview

TLS (Transport Layer Security) encrypts data in transit — HTTPS, SMTPS, etc.

  • Certificate Authority (CA) issues trusted certificates (Let's Encrypt is free).
  • Self-signed certs work for internal use but browsers warn.
  • mTLS — both client and server authenticate with certificates.
# Generate a self-signed cert (OpenSSL)
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 365 -nodes

# Get a Let's Encrypt cert (Certbot)
sudo certbot --nginx -d example.com

# Check cert details
openssl x509 -in cert.pem -text -noout

Password Managers

Encryption is useless if passwords are weak. Use a password manager:

  • Bitwarden — Open source, cloud-sync, self-host option.
  • KeePassXC — Local-only, offline database, cross-platform.
  • Vaultwarden — Self-hosted Bitwarden-compatible server.
