Digital Forensics & Incident Response

DFIR methodology — evidence collection, disk forensics, memory analysis, and containment.

Incident Response Process

The SANS PICERL model: Preparation (playbooks, tools, training), Identification (detect anomaly, classify severity), Containment (isolate affected systems, short-term + long-term), Eradication (remove threat, patch vulnerability), Recovery (restore systems, monitor for re-infection), Lessons Learned (post-mortem, improve processes).

Key metrics: MTTR (mean time to respond), dwell time (attacker presence inside network before detection — average 200+ days). Automation with SOAR platforms (Splunk Phantom, Palo Alto XSOAR) reduces response time.

Disk & File System Forensics

Disk forensics examines storage media for evidence. Process: acquire (create forensic image — dd, FTK Imager, Guymager; verify with hash — SHA-256), analyze (Autopsy/Sleuth Kit — recover deleted files, analyze file system metadata, timeline analysis), report.

Key artifacts: $MFT (NTFS master file table — all files, timestamps), $UsnJrnl (update sequence number — file change log), $LogFile (NTFS journal), Recycle Bin ($Recycle.Bin), prefetch files (execution history), LNK files (shortcuts — file access evidence), and browser history (Chrome, Firefox, Edge).

Memory Forensics

Memory forensics analyzes RAM dumps for evidence that doesn't touch disk. Capture: LiME (Linux Memory Extractor), WinPmem, FTK Imager (live memory). Analysis: Volatility 3 (open-source, Python — pslist, netscan, cmdscan, mimikatz, malfind, procdump).

Evidence in memory: running processes (including hidden — DKOM), network connections, command history, injected code, encryption keys, clipboard contents, plaintext passwords. Memory analysis captures the state of the system at the moment of acquisition — crucial for detecting fileless malware and rootkits.

Network Forensics

Network forensics captures and analyzes network traffic. Tools: Wireshark/TShark (packet analysis — follow TCP streams, export objects, filter by protocol), tcpdump (CLI capture), Zeek (formerly Bro — network analysis framework, protocol detection), and Snort/Suricata (IDS/IPS with logging).

Key evidence: connection logs (source/destination IP, port, protocol, timestamps), DNS queries (exfiltration via DNS tunneling), HTTP requests (C2 callbacks, data exfiltration), TLS handshake parameters (JA3 fingerprint for malware identification), and file transfers (exportable for malware analysis).

DFIR Tools Ecosystem

Enterprise: Splunk (SIEM, log analysis), Elastic Security (ELK stack for security), CrowdStrike Falcon (EDR), Microsoft Defender for Endpoint, SentinelOne. Open-source: Wazuh (SIEM + XDR), TheHive (incident management), MISP (threat intelligence sharing), Velociraptor (endpoint visibility, collection, and response).

Essential free tools: KAPE (Kroll Artifact Parser Extractor — collect and parse Windows artifacts), Eric Zimmerman's tools (MFT Explorer, ShellBags Explorer, Timeline Explorer), Hayabusa (Windows event log fast forensics), and YARA (pattern matching for malware identification).
