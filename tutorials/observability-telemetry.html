Observability & Telemetry

Observability — logs, metrics, traces, OpenTelemetry, and alerting.

Table of Contents (2 sections)

  • Logs & Metrics
  • Tracing & OpenTelemetry

Logs & Metrics

Observability: ability to infer system state from external outputs. Three pillars: logs (discrete events), metrics (aggregated time-series), traces (request flow). Logging: structured logs (JSON — timestamp, level, service, message, trace_id). Log levels: DEBUG, INFO, WARN, ERROR, FATAL. Log aggregation: ELK (Elasticsearch, Logstash, Kibana), Loki (Grafana, Promtail — logQL, cheap), Splunk (enterprise). Metrics: Prometheus (pull model, time-series database, PromQL queries, Alertmanager). Metric types: counter (requests total, always increases), gauge (CPU, memory, current value), histogram (latency buckets, quantiles), summary (pre-computed quantiles). RED method: Rate (requests/sec), Errors (failed requests/sec), Duration (latency). USE method: Utilization, Saturation, Errors (for resources — CPU, memory, disk, network). Grafana: dashboards, panels (time series, stat, table, bar gauge), alerts, annotations, variables. Grafana Cloud or self-hosted. Loki: log aggregation like Prometheus but for logs, label-based, cost-effective. Promtail: agent to ship logs. LogQL: query logs with filters, parsers, metrics from logs.

Tracing & OpenTelemetry

Distributed tracing: trace a request across services. Trace: tree of spans. Span: operation with name, start time, duration, attributes, status. Trace context: trace ID + span ID propagated via W3C Trace Context headers (traceparent, tracestate). OpenTelemetry (OTel): vendor-neutral standard — APIs, SDKs, Collector, semantic conventions. Instrumentation: automatic (auto-instrumentation agents for Java, Python, Node.js, .NET, Go) and manual (create spans, add attributes/events). OTel Collector: receiver (OTLP, Jaeger, Prometheus), processor (batch, filter, attributes, sampling), exporter (to backend: Jaeger, Zipkin, Prometheus, Loki, Tempo, Datadog, Honeycomb). Sampling: head-based (decision at start of trace), tail-based (decision after full trace, based on content). Jaeger: open-source tracing backend (UI, storage backend — Elasticsearch, Cassandra, Kafka). Grafana Tempo: tracing backend integrated with Grafana, object storage (S3/GCS). Service maps: visualize dependencies. Alerting: based on metrics (high error rate, latency SLO), logs (error patterns), traces (slow traces). SLOs: service level objectives, error budgets. On-call: PagerDuty, OpsGenie, Grafana OnCall. Observability patterns: health check endpoints, /metrics endpoint, /health endpoint. Logs vs Metrics vs Traces: logs for detailed debugging, metrics for trends/alerts, traces for request flow.
