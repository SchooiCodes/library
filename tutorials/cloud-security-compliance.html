Cloud Security & Compliance

Secure cloud infrastructure — IAM, encryption, compliance frameworks, and audits.

Shared Responsibility Model

Security in the cloud is a shared responsibility between provider and customer. AWS/GCP/Azure secure the cloud (physical security, hypervisor, network infrastructure). Customers secure what's IN the cloud (data, applications, IAM configuration, network controls).

Common customer mistakes: overly permissive IAM policies, unencrypted S3 buckets, default VPC settings, open security groups (0.0.0.0/0 SSH), and lack of encryption. The customer is always responsible for data classification, access management, and compliance.

IAM Best Practices

Cloud IAM best practices: grant least privilege (start with deny-all, add minimum permissions), use roles/groups instead of individual permissions, use temporary credentials (STS, IAM Roles Anywhere) instead of long-term keys, and rotate access keys every 90 days.

Implement permission boundaries (maximum permissions a role can have), service control policies (SCPs in AWS Organizations — guardrails for accounts), and attribute-based access control (ABAC — tags-based permissions for dynamic scaling). Use Access Analyzer or Cloud IAM Recommender for unused permissions.

Data Encryption

Encrypt data at rest: server-side encryption (SSE-S3/AES-256, SSE-KMS/customer-managed keys, SSE-C/customer-provided keys) and client-side encryption (encrypt before upload). Encrypt in transit: TLS 1.2+ for all API endpoints, enforce HTTPS-only bucket policies, and use VPN/Direct Connect for hybrid networks.

KMS (AWS) / Cloud KMS (GCP) / Azure Key Vault manage encryption keys. Important: rotate CMKs annually, use automatic key rotation when available, enable key deletion protection, and audit key usage with CloudTrail or equivalent.

Compliance Frameworks

Major compliance frameworks: SOC 2 (service organization controls, Type I/II), HIPAA (US healthcare privacy), PCI DSS (payment card industry), GDPR (EU data protection), FedRAMP (US government), ISO 27001 (information security management).

Cloud providers publish compliance reports and maintain authorized regions/zones for each framework. Customers often need to configure services to meet compliance — e.g., HIPAA requires BAA (Business Associate Agreement), encryption, access logging, and data residency.

Cloud Security Posture Management

CSPM tools continuously monitor cloud environments for misconfigurations and compliance violations. They detect: open security groups, unencrypted data, overly permissive IAM, exposed S3 buckets, and deviations from compliance benchmarks (CIS, NIST, SOC 2).

Tools: AWS Security Hub (centralized security findings), GCP Security Command Center (threat detection, vulnerability scanning), Azure Defender (CSPM + workload protection). Open-source: Prowler (AWS CIS benchmark tool), ScoutSuite (multi-cloud auditing). Integrate CSPM into CI/CD to catch misconfigs before deployment.
