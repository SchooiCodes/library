HTML Document Structure

Every HTML document follows the same basic skeleton. The <!DOCTYPE html> declaration tells the browser this is HTML5, and all content lives between <html> tags.

<!DOCTYPE html>
<html lang="en">
<head>
 <meta charset="UTF-8">
 <meta name="viewport" content="width=device-width, initial-scale=1.0">
 <title>My First Page</title>
</head>
<body>
 <h1>Hello, World!</h1>
 <p>This is my first webpage.</p>
</body>
</html>

HTML tags define the structure and meaning of your content. Here are the most frequently used ones:

CSS Selectors

CSS selectors target HTML elements to apply styles. You can select by element type, class, ID, or more complex patterns.

/* Element selector — targets all <p> */
p {
 color: #333;
 font-size: 16px;
}

/* Class selector — targets class="card" */
.card {
 background: white;
 border-radius: 8px;
 padding: 20px;
}

/* ID selector — targets id="header" */
#header {
 background: #1a1a2e;
 color: white;
}

/* Descendant — <a> inside <nav> */
nav a {
 text-decoration: none;
}

/* Pseudo-class — hover state */
button:hover {
 background: #0056b3;
}
Naming Convention: Use kebab-case for CSS class names (e.g., .nav-menu, .section-title). Keep class names semantic — describe what the element is, not what it looks like.

The Box Model

Every element in CSS is a rectangular box. The box model controls how much space an element takes up:

/* Box model breakdown */
.element {
 width: 300px;     /* Content area */
 padding: 20px;     /* Space inside the border */
 border: 2px solid #333;/* The visible edge */
 margin: 15px;     /* Space outside the border */
}

/* Use border-box to include padding/border in width */
* {
 box-sizing: border-box;
}

Flexbox & Grid

Modern CSS layout is done with Flexbox (one-dimensional) and Grid (two-dimensional). Both make responsive design much simpler than older float-based methods.

/* Flexbox — perfect for nav bars, centering, rows */
.flex-container {
 display: flex;
 justify-content: space-between;
 align-items: center;
 gap: 16px;
 flex-wrap: wrap;
}

/* Grid — perfect for page layouts, card grids */
.grid-container {
 display: grid;
 grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
 gap: 24px;
}

/* Centering with Flexbox */
.centered {
 display: flex;
 justify-content: center;
 align-items: center;
 min-height: 100vh;
}

Responsive Design

Responsive design ensures your site looks good on any screen size. The key ingredients are relative units, flexible layouts, and media queries.

/* Use relative units instead of fixed px */
.container {
 max-width: 960px;
 width: 90%;
 margin: 0 auto;
}

/* Media queries — change layout at breakpoints */
@media (max-width: 768px) {
 .nav-menu {
  display: none;    /* Hide desktop nav */
 }

 .hamburger {
  display: block;   /* Show hamburger menu */
 }

 .grid-container {
  grid-template-columns: 1fr; /* Single column */
 }
}

/* Viewport units for full-screen sections */
.hero {
 height: 100vh;     /* Full viewport height */
 padding: 5vw;     /* 5% of viewport width */
}

Publishing Your Site

Once your site is ready, you need a way to put it online. Here are the easiest free options:

PlatformHow It Works
GitHub PagesPush your code to a username.github.io repo — it's live instantly
NetlifyDrag & drop your folder onto netlify.com
VercelConnect your Git repo and it auto-deploys on every push
Cloudflare PagesFree hosting with global CDN, connect via Git
 
# Deploy to GitHub Pages (quick start)
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/your-username/your-repo.git
git push -u origin main

# Then enable Pages in repo Settings → Pages → Deploy from main branch
