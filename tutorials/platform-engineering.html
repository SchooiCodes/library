Platform Engineering

Platform engineering — internal developer platforms, Backstage, and golden paths.

Table of Contents (2 sections)

  • IDPs & Golden Paths
  • Platform Components

IDPs & Golden Paths

Platform engineering: build internal developer platform (IDP) to improve developer experience (DevEx) and reduce cognitive load. Golden paths: paved roads for common tasks (deploy service, add database, set up CI/CD). Developer portals: Backstage (Spotify's open-source — software catalog, scaffolder (templates), TechDocs, plugins), Port, Humanitec (internal developer platform), Cortex, Firestartr. Software catalog: register components (services, libraries, websites, APIs) with metadata (owner, lifecycle, system, dependsOn). Backstage scaffolder: YAML templates (steps: fetch template, create PR, register in catalog). Backstage plugins: CI/CD (Jenkins, GitHub Actions), monitoring (Datadog, Grafana), security (Snyk), cost (CloudCost). Scorecards: check ownership, docs, CI/CD coverage. IDP architecture: control plane (orchestration) + data plane (compute). Provisioning: Terraform, Pulumi, Crossplane for infrastructure. Self-service: developers request resources via portal. Environment management: ephemeral environments (review apps). Platform team: product mindset — treat developers as customers, OKRs for adoption. Scorecard metrics: service ownership %, documentation %, CI/CD %.

Platform Components

CI/CD: unified pipeline platform — builds, tests, deploys. Feature: pipeline templates, promotion gates. Kubernetes: namespace per service, RBAC, resource quotas, network policies. Argo CD: GitOps (app of apps pattern). Crossplane: control plane — managed resources (RDS, S3, IAM). Terraform: modules for infrastructure. Service mesh: Istio (mTLS, traffic routing, observability). Gateways: Kong (API gateway for north-south traffic). Secret management: external-secrets-operator (sync secrets from Vault/AWS Secrets Manager/Google Secret Manager). Observability stack: Prometheus + Grafana + Loki + Tempo (or Datadog/Honeycomb). Cost showback: track infrastructure cost per team/service. Backstage TechDocs: docs-as-code (MkDocs, material theme). Error tracking: Sentry. Backstage integrations: GitHub, GitLab, Jenkins, Argo CD, SonarQube, Snyk, Datadog, PagerDuty. Score: portal to measure DORA metrics (deployment frequency, lead time, MTTR, change failure rate). Platform maturity: level 0 (no platform), level 1 (scripts/manuals), level 2 (automated CI/CD), level 3 (self-service portal), level 4 (developer platform with guardrails). Internal market: platform team = provider, dev teams = consumers. Focus on: reducing time from commit to production. Measure: developer satisfaction (surveys), onboarding time (hours to first deploy).
