Windows Recovery Environment (WinRE)

The Windows Recovery Environment (WinRE) is a minimal recovery OS built into Windows. It provides access to troubleshooting tools when Windows fails to boot. Access WinRE by:

  • Interrupting the boot process 3 times (power off during Windows logo).
  • Holding Shift while clicking Restart from the login screen or Start menu.
  • Booting from a Windows installation USB → Repair your computer (bottom-left).
  • Running shutdown /r /o /f /t 0 from an admin command prompt.

Once inside WinRE, navigate: Troubleshoot → Advanced options to access Command Prompt, System Restore, Startup Repair, and more.

# Boot directly into WinRE from Windows
shutdown /r /o /f /t 0

# Check if WinRE is enabled on your system
reagentc /info

# Enable or disable WinRE
reagentc /enable
reagentc /disable

# Set a custom WinRE image path
reagentc /setreimage /path C:\Recovery\WindowsRE

DISM — Repair the Component Store

Deployment Image Servicing and Management (DISM) repairs the Windows component store — the source files that SFC uses to restore corrupt system files. If SFC fails, run DISM first:

# Check the health of the component store
DISM /Online /Cleanup-Image /CheckHealth

# Scan for corruption (more thorough than CheckHealth)
DISM /Online /Cleanup-Image /ScanHealth

# Repair corruption using Windows Update as the source
DISM /Online /Cleanup-Image /RestoreHealth

# Repair using a mounted ISO/WIM as the source (offline repair)
DISM /Online /Cleanup-Image /RestoreHealth /Source:C:\mount\windows\install.wim /LimitAccess

# Repair a Windows image offline (e.g., from WinRE)
DISM /Image:C:\ /Cleanup-Image /RestoreHealth /Source:D:\Sources\install.wim
Tip: If DISM cannot contact Windows Update for replacement files, use the /Source parameter pointing to a Windows ISO or a install.wim file. You can also specify /LimitAccess to prevent DISM from using Windows Update at all.

SFC — System File Checker

System File Checker (SFC) scans protected system files and replaces corrupted ones with cached copies from the component store:

# Scan and repair system files
sfc /scannow

# Scan once without repairing
sfc /verifyonly

# Scan a specific file
sfc /VERIFYFILE=C:\Windows\System32\kernel32.dll

# Repair a specific file
sfc /SCANFILE=C:\Windows\System32\kernel32.dll

# Run SFC at boot (useful when Windows won't start)
# Boot to WinRE → Command Prompt
sfc /scannow /offbootdir=C:\ /offwindir=C:\Windows

SFC results are logged at %WinDir%\Logs\CBS\CBS.log. Look for lines with [SR] to see which files were repaired.

bootrec — BCD Repair

The Boot Configuration Data (BCD) stores boot parameters. If you see Bootmgr is missing, 0xc000000f, or a blank screen after POST, the BCD is likely corrupt:

# All bootrec commands run from WinRE Command Prompt

# Rebuild the boot sector
bootrec /fixboot
bootrec /fixmbr

# Rebuild the BCD store (scans for Windows installations)
bootrec /rebuildbcd

# If /rebuildbcd finds no installations, try:
bcdedit /export C:\bcdbackup
attrib C:\boot\bcd -h -r -s
ren C:\boot\bcd bcd.old
bootrec /rebuildbcd

# For UEFI systems, the BCD is on the EFI System Partition:
bootrec /fixboot
bootrec /rebuildbcd
Danger Zone: Running bootrec /fixboot on a system with a corrupted boot sector can sometimes make things worse if the wrong partition is targeted. Always back up the BCD (bcdedit /export C:\bcd.bak) before modifying it. On UEFI systems, ensure you are writing to the correct EFI partition.

System Restore

System Restore reverts system files, registry keys, and installed programs to a previous state without affecting personal files. Restore points are created automatically before major changes or on a schedule:

# Create a restore point manually from the command line
powershell -Command "Checkpoint-Computer -Description 'Before Driver Update' -RestorePointType MODIFY_SETTINGS"

# List all restore points
powershell -Command "Get-ComputerRestorePoint"

# Launch System Restore from GUI
rstrui.exe

# System Restore from WinRE
# Troubleshoot → Advanced Options → System Restore

# Restore from command line (not directly supported, use rstrui)

System Restore only monitors system drives. Enable it manually on data drives via System Protection tab (sysdm.cpl → System Protection → Configure). Restore points consume disk space — allocate 5-10% of the drive for them.

Reset This PC

When repair tools fail, Reset This PC reinstall Windows while offering choices about keeping your files:

  • Keep my files — Reinstalls Windows, preserves personal files in C:\Users, removes apps and settings. Best first step before a full wipe.
  • Remove everything — Wipes all personal files, apps, and settings. Use this before selling a PC or as a last resort.
  • Cloud download — Downloads a fresh Windows image from Microsoft (recommended — more reliable than the local image).
  • Local reinstall — Uses the local Windows image on your system (faster, but may include existing corruption).
# Reset This PC from Settings
# Settings → System → Recovery → Reset PC

# Reset from command line
systemreset --factoryreset

# Cloud download reset from command line
systemreset --factoryreset --cloud

# Reset from WinRE: Troubleshoot → Reset This PC

Recovery Drive

A USB Recovery Drive boots Windows RE even if the hard drive is dead or the WinRE partition is corrupt. Create one before disaster strikes:

# Create a Recovery Drive via the GUI
# Search "Create a recovery drive" → Check "Back up system files to the recovery drive" → Select USB (8GB+) → Create

# Create from command line
# Requires the RecoveryDrive wizard — no native CLI, but can be scripted:
Start-Process "recdisc.exe"

# Alternative: create a bootable WinPE USB manually
# Download the Windows ADK, then:
copype amd64 C:\WinPE
MakeWinPEMedia /ISO C:\WinPE C:\WinPE.iso
# Use Rufus or Ventoy to write the ISO to USB
Tip: Always test your recovery USB by booting from it at least once. A recovery drive that fails to boot is worse than none at all. Also create a separate backup of your BitLocker recovery key — without it, encrypted drives are unrecoverable even with WinRE.

Third-party Rescue Tools

When built-in tools are not enough, these third-party tools can save your system:

  • Hiren's Boot CD PE — A full Windows PE environment with disk tools, password recovery, data rescue, and registry editing. Boots from USB.
  • Medicat USB — All-in-one USB toolkit combining Hiren's, Linux live ISOs, antivirus scanners, and hardware diagnostics.
  • Dell OS Recovery Tool — Downloads and creates recovery media for Dell systems (also works for non-Dell in many cases).
  • Macrium Reflect — Free disk imaging and cloning. Create a full system image before making major changes — restore in under 30 minutes.
  • TestDisk & PhotoRec — Retrieve deleted partitions (TestDisk) and recover lost files (PhotoRec) even after formatting.
