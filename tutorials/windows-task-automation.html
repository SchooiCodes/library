⏰ Windows Task Automation

Automate Windows with Task Scheduler, scheduled tasks via PowerShell, startup scripts, and event-triggered automation.

Task Scheduler GUI

Task Scheduler (taskschd.msc) is the graphical interface for creating and managing scheduled tasks. It is organised into Task Scheduler Library (all existing tasks) and Actions/Conditions/Settings panes per task.

Key concepts when building a task:

  • Triggers — Time-based (daily, weekly, on a schedule), event-based (on log, on startup, on idle), or at system boot/logon.
  • Actions — Start a program, send an email, or display a message. Most automation runs a .bat, .ps1, or .exe.
  • Conditions — Only run on AC power, only if the network is available, or only if the computer is idle.
  • Settings — Run missed tasks as soon as possible, restart on failure, stop after X duration.
Tip: When running PowerShell scripts from Task Scheduler, set Action → Program/script to powershell.exe and Add arguments to -ExecutionPolicy Bypass -File "C:\Scripts\my-task.ps1". Always test the task by right-clicking → Run.

schtasks.exe CLI

The schtasks.exe command-line tool lets you create, query, modify, and delete tasks without touching the GUI. Essential for scripting and remote management:

# Create a daily task that runs a PowerShell script at 6 AM
schtasks /create /tn "Daily Backup" /tr "powershell.exe -ExecutionPolicy Bypass -File C:\Scripts\backup.ps1" /sc daily /st 06:00 /ru SYSTEM

# Create a task that runs at every user logon
schtasks /create /tn "User Logon Script" /tr "C:\Scripts\logon-notify.bat" /sc onlogon /ru %USERNAME%

# Run a task on demand
schtasks /run /tn "Daily Backup"

# Query all scheduled tasks (remote machine supported)
schtasks /query /fo LIST /v

# Export a task as XML (backup or deploy elsewhere)
schtasks /query /tn "Daily Backup" /xml > backup-task.xml

# Import a task from XML
schtasks /create /xml backup-task.xml /tn "Daily Backup"

# Delete a task
schtasks /delete /tn "Daily Backup" /f

PowerShell ScheduledJobs

PowerShell's ScheduledJob module wraps Task Scheduler functionality in native PowerShell cmdlets. Jobs remain manageable via Get-Job and Receive-Job:

# Create a trigger (daily at 2 AM)
$trigger = New-JobTrigger -Daily -At "02:00AM"

# Define job options (run even if the computer is on battery)
$options = New-ScheduledJobOption -RunElevated -RequireNetwork

# Register the scheduled job
Register-ScheduledJob -Name "Nightly Cleanup" `
  -ScriptBlock {
    $logs = Get-ChildItem "C:\Logs" -Recurse -Filter "*.log"
    $cutoff = (Get-Date).AddDays(-14)
    $logs | Where-Object { $_.LastWriteTime -lt $cutoff } | Remove-Item -Force
  } `
  -Trigger $trigger `
  -ScheduledJobOption $options

# List all scheduled jobs
Get-ScheduledJob

# Run immediately and check output
Start-Job -DefinitionName "Nightly Cleanup"
Get-Job -Name "Nightly Cleanup" | Receive-Job -Keep

# Add a second trigger (e.g., at system startup)
$trigger2 = New-JobTrigger -AtStartup
Set-ScheduledJob -Name "Nightly Cleanup" -Trigger @($trigger, $trigger2)

# Unregister a job
Unregister-ScheduledJob -Name "Nightly Cleanup"

Event Triggers

Tasks can fire when specific events appear in the Windows Event Log. Create an XML event filter that targets a specific Event ID and source, then bind it to a scheduled task:

# Create an event-triggered task via PowerShell
$action = New-ScheduledTaskAction -Execute "powershell.exe" -Argument "-ExecutionPolicy Bypass -File C:\Scripts\on-critical-error.ps1"

# Trigger on Event ID 41 (Kernel-Power — unexpected shutdown)
$trigger = New-ScheduledTaskTrigger -Custom -RepetitionInterval (New-TimeSpan -Minutes 5) `
  -Xml ''

# Register as a scheduled task
Register-ScheduledTask -TaskName "On Critical Error Alert" -Action $action -Trigger $trigger -RunLevel Highest -User "SYSTEM"

# Alternative: use schtasks with an event-query XML
# Save this as event-query.xml:
# <QueryList><Query Id="0"><Select Path="Application">*[System[Provider[@Name='.NET Runtime'] and (EventID=1026)]]</Select></Query></QueryList>
# schtasks /create /tn ".NET Crash Alert" /tr "powershell.exe -File alert.ps1" /sc onevent /ec "event-query.xml" /ru SYSTEM
Danger Zone: Event-triggered tasks can fire hundreds of times if not carefully scoped. Always use a specific Event ID + Source filter, and configure Settings → "If the task is already running, then the following rule applies: Do not start a new instance" to prevent runaway processes.

Startup Scripts

There are several ways to run scripts at startup. Each serves a different purpose:

  • Startup folder (shell:startup) — runs at user logon, visible to the user, easy to manage. Place shortcuts to scripts here.
  • Group Policy Startup Script (gpedit.msc → Computer Configuration → Windows Settings → Scripts → Startup) — runs as SYSTEM, before the user logs in.
  • Run/RunOnce registry keys — HKLM\Software\Microsoft\Windows\CurrentVersion\Run (all users) and HKCU\... (current user).
# Add a script to the Startup folder via PowerShell
$shortcut = (New-Object -ComObject WScript.Shell).CreateShortcut("$env:APPDATA\Microsoft\Windows\Start Menu\Programs\Startup\DiskCheck.lnk")
$shortcut.TargetPath = "powershell.exe"
$shortcut.Arguments = "-ExecutionPolicy Bypass -File C:\Scripts\disk-check.ps1"
$shortcut.Save()

# Add a startup program via the registry
New-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Run" `
  -Name "DiskCheck" -Value "powershell.exe -ExecutionPolicy Bypass -File C:\Scripts\disk-check.ps1" -PropertyType String -Force

# Group Policy startup script path
# Copy the script to C:\Windows\System32\GroupPolicy\Machine\Scripts\Startup\

Practical Examples

📀 Automatic Disk Cleanup

Run cleanmgr /sageset first to configure cleanup categories, then schedule a weekly cleanup:

schtasks /create /tn "Weekly Disk Cleanup" /tr "cleanmgr /sagerun:1" /sc weekly /d SUN /st 03:00 /ru SYSTEM

📮 Email Alert on Low Disk Space

Create a scheduled job that checks disk space and sends an email if below 10%:

$trigger = New-JobTrigger -Daily -At "08:00"
$options = New-ScheduledJobOption -RunElevated
Register-ScheduledJob -Name "LowDiskAlert" -ScriptBlock {
  $drives = Get-CimInstance Win32_LogicalDisk -Filter "DriveType=3"
  foreach ($d in $drives) {
    $pct = [math]::Round(($d.FreeSpace / $d.Size) * 100, 1)
    if ($pct -lt 10) {
      Send-MailMessage -To "admin@example.com" -From "monitor@example.com" `
        -Subject "LOW DISK: $($d.DeviceID)" -Body "$($d.DeviceID) has $pct% free." `
        -SmtpServer "smtp.example.com"
    }
  }
} -Trigger $trigger -ScheduledJobOption $options

🔄 Rotate System Restore Points

# Create a task that removes restore points older than 30 days
schtasks /create /tn "Rotate Restore Points" /tr "powershell.exe -Command Get-ComputerRestorePoint | Where-Object { \$_.CreationTime -lt (Get-Date).AddDays(-30) } | ForEach-Object { Checkpoint-Computer -Description 'Pre-Cleanup' -RestorePointType MODIFY_SETTINGS }" /sc daily /st 04:00 /ru SYSTEM

🖥️ Remote Task Management

# Create a task on a remote machine
schtasks /create /s PC02 /tn "Remote Backup" /tr "C:\Scripts\backup.bat" /sc daily /st 22:00 /ru SYSTEM

# Query remote task status
schtasks /query /s PC02 /tn "Remote Backup" /v

# Run a remote task immediately
schtasks /run /s PC02 /tn "Remote Backup"
