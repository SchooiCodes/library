What is the Registry?

The Windows Registry is a hierarchical database that stores low-level settings for the operating system and applications. It contains configuration data for hardware, software, user preferences, and system policies. Think of it as the Windows brain — nearly every setting is stored here.

The registry is organised into keys (folders), subkeys, and values (name/data pairs). Values come in types including REG_SZ (string), REG_DWORD (32-bit integer), REG_QWORD (64-bit integer), REG_BINARY, and REG_MULTI_SZ (multi-string).

HKEY Root Keys

There are five root keys, each serving a distinct purpose:

  • HKEY_CLASSES_ROOT (HKCR) — File associations, COM classes, and shell extensions.
  • HKEY_CURRENT_USER (HKCU) — Settings for the currently logged-in user.
  • HKEY_LOCAL_MACHINE (HKLM) — System-wide hardware and software configuration.
  • HKEY_USERS (HKU) — Contains a subkey for every loaded user profile.
  • HKEY_CURRENT_CONFIG (HKCC) — Current hardware profile (mostly legacy).

regedit Navigation

regedit is the built-in GUI editor. Launch it from Start or with Win+R → regedit. The left pane shows the key hierarchy; the right pane shows values. You can right-click to create new keys/values, rename, modify, export, and search (Ctrl+F).

# Open regedit from the command line
regedit

# Export a specific key to a .reg file
regedit /e "C:\backup\mykey.reg" "HKEY_CURRENT_USER\Control Panel\Desktop"

# Import a .reg file silently
regedit /s "C:\tweaks\custom.reg"
Tip: Press Ctrl+F in regedit to search key names, value names, and data. Use View → Filter to narrow results. Searching the entire registry can be slow — start from a specific root key when possible.

reg.exe CLI

The reg.exe command-line tool lets you query, add, delete, and compare registry entries without opening regedit. Essential for scripting:

# Query a key and all its values
reg query "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion" /s

# Add a new DWORD value
reg add "HKCU\Control Panel\Desktop" /v WallpaperStyle /t REG_DWORD /d 2 /f

# Delete a value
reg delete "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v BadProgram /f

# Compare two registry keys
reg compare "HKLM\Software\MyApp" "HKLM\Software\MyApp_Backup"

# Copy a key to another location
reg copy "HKLM\Software\MyApp" "HKLM\Software\MyApp_Backup" /s

.reg Files

A .reg file is a plain-text script that adds, modifies, or deletes registry keys and values. Double-clicking one merges it into the registry. They follow a specific format:

Windows Registry Editor Version 5.00

; Add a value under HKCU
[HKEY_CURRENT_USER\Control Panel\Desktop]
"WallpaperStyle"=dword:00000002
"TileWallpaper"=dword:00000000

; Delete a value (note the minus sign)
[HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run]
"OldProgram"=-

; Delete an entire key
[-HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\Installer]

Common Tweaks

📂 Context Menu Items

Add or remove items from the right-click context menu in File Explorer:

# Remove "Cast to Device" from context menu
[-HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Shell Extensions\Blocked]
"{7AD84985-87B4-4a16-BE58-8B72A5B390F7}"=-

# Add "Open with Notepad" for all files
[HKEY_CLASSES_ROOT\*\shell\Open with Notepad]
@="Open with Notepad"
"Icon"="notepad.exe"
[HKEY_CLASSES_ROOT\*\shell\Open with Notepad\command]
@="notepad.exe \"%1\""

🚀 Startup Programs

Programs that run at user login are registered here:

# List startup programs
reg query "HKCU\Software\Microsoft\Windows\CurrentVersion\Run"
reg query "HKLM\Software\Microsoft\Windows\CurrentVersion\Run"

# Add a program to user startup
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v MyApp /t REG_SZ /d "C:\tools\myapp.exe" /f

🖥️ Visual & Performance Tweaks

# Disable transparency effects (slight performance gain)
reg add "HKCU\Software\Microsoft\Windows\DWM" /v EnableTransparency /t REG_DWORD /d 0 /f

# Remove the "3D Objects" folder from This PC
reg delete "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\MyComputer\NameSpace\{0DB7E03F-FC29-4DC6-9020-FF41B59E513A}" /f

Backup & Restore

Always back up a key before editing it. Mistakes in the registry can break applications or prevent Windows from booting:

# Export a single key to a .reg backup file
regedit /e "C:\RegistryBackups\HKLM-System.reg" "HKEY_LOCAL_MACHINE\SYSTEM"

# Backup the entire registry (huge file — not recommended daily)
regedit /e "C:\RegistryBackups\full-backup.reg"

# Restore by double-clicking the .reg file, or:
regedit /s "C:\RegistryBackups\HKLM-System.reg"

# Create a System Restore Point before major edits (works from CLI)
powershell -Command "Checkpoint-Computer -Description 'Before Registry Edit' -RestorePointType MODIFY_SETTINGS"

Safety & Best Practices

Danger Zone: Editing the wrong key can render Windows unbootable. Never delete or modify keys in HKLM\SYSTEM, HKLM\SAM, or HKLM\SECURITY unless you know exactly what you are doing. Always create a System Restore Point or full registry export before making changes.
  • Test on a VM first — validate registry tweaks in a safe environment.
  • One change at a time — this makes it easy to isolate the cause if something breaks.
  • Use .reg files for rollback — create an "undo" .reg file before applying any change.
  • Administrator rights — many HKLM edits require an elevated regedit or reg.exe.
  • 64-bit vs 32-bit — on 64-bit Windows, 32-bit apps see a redirected view under HKLM\SOFTWARE\WOW6432Node.
Tip: Use Registrar Registry Manager or RegCool as more powerful alternatives to regedit — they offer tabbed browsing, colour-coded value types, and advanced search-and-replace across the entire hive.
