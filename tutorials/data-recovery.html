Data Recovery & Forensics

Recover deleted files, repair corrupted drives, and salvage data from failing storage.

Table of Contents (3 sections)

  • Imaging Failing Drives
  • File Recovery Tools
  • Data Rescue Scenarios

Imaging Failing Drives

First rule of data recovery: work on a copy. Tools: ddrescue (Linux) reads failing drives, skips bad sectors on first pass, retries later. Creates an image file you can safely work on. ddrescue -d -r3 /dev/sda image.img mapfile.log. Clonezilla: bit-for-bit disk cloning for healthy drives. Never attempt recovery on the original drive — further damage is almost certain. Bad sectors multiply as the drive is used.

File Recovery Tools

Recuva (Windows, free): recover files from recently deleted, formatted drives. Deep scan mode finds older files. Good for NTFS, FAT, exFAT. PhotoRec (cross-platform, free): recovers 480+ file types, ignores filesystem (works on formatted/corrupted drives). TestDisk: repair partition tables, recover deleted partitions. R-Studio (paid, powerful): RAID reconstruction, network recovery, advanced algorithms. For SSDs: less chance of recovery due to TRIM — stop using the drive immediately.

Data Rescue Scenarios

Deleted files (not overwritten yet) → run Recuva/PhotoRec immediately. Formatted drive → TestDisk to recover partition, then recover files. Corrupted filesystem → fsck (Linux), chkdsk (Windows) — may make things worse, image first. Physical damage → send to professional recovery service ($500-$3000+). Water damage → remove power immediately, dry for 72+ hours in silica gel. Ransomware → backup is the only reliable recovery — decryption tools exist for some variants (nomoreransom.org).
