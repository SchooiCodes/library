Combinatorics & Discrete Optimization

Counting principles, graph coloring, integer programming, and approximation algorithms.

Counting Principles

Fundamental counting rules: product rule, sum rule, inclusion-exclusion. Permutations and combinations with and without repetition.

from math import comb, perm
n, k = 10, 3
print(f'P({n},{k}) = {perm(n,k)}')  # 720
print(f'C({n},{k}) = {comb(n,k)}')  # 120
# Stars and bars: C(n+k-1, k-1) ways to put n items into k bins
print(f'Stars & bars: {comb(10+4-1, 4-1)}')  # 286

Pigeonhole principle, Catalan numbers, Stirling numbers, and generating functions for complex counting.

Graph Coloring & Ramsey Theory

Graph coloring assigns colors to vertices so adjacent vertices differ. Chromatic number χ(G) is the minimum colors needed.

import networkx as nx
G = nx.petersen_graph()
coloring = nx.greedy_color(G, strategy='largest_first')
print(f'Colors used: {len(set(coloring.values()))}')
# The Petersen graph has chromatic number 3

Four color theorem: every planar graph is 4-colorable. Ramsey numbers guarantee monochromatic substructures in edge-colored graphs.

Integer Programming

Integer linear programming optimizes linear objective with integer variables. Branch and bound, cutting planes, and heuristics.

from scipy.optimize import milp, Bounds, LinearConstraint
c = [-1, -2]  # Maximize x + 2y (negated for minimization)
A = [[1, 1], [2, 1]]
b = [5, 8]
constraints = LinearConstraint(A, -np.inf, b)
bounds = Bounds(0, [np.inf, np.inf])
# Integer constraints
integrality = [1, 1]  # Both variables integer
result = milp(c=c, constraints=constraints, bounds=bounds, integrality=integrality)
print(f'Optimal: x={result.x[0]}, y={result.x[1]}, obj={-result.fun}')

Knapsack, traveling salesman, facility location problems. NP-hardness and the importance of approximation algorithms.

Network Flow & Matching

Maximum bipartite matching, assignment problem, and min-cost flow are fundamental combinatorial optimization problems.

from scipy.optimize import linear_sum_assignment
cost = np.array([[9, 2, 7], [6, 4, 3], [5, 8, 1]])
row_ind, col_ind = linear_sum_assignment(cost)
print(f'Assignment: workers {row_ind} → tasks {col_ind}')
print(f'Total cost: {cost[row_ind, col_ind].sum()}')

Hungarian algorithm for assignment (O(n³)). Edmonds-Karp for max flow. Min-cost max flow for logistics and scheduling.

Approximation Algorithms

For NP-hard problems, approximation algorithms guarantee solutions within a factor of optimal. Examples: greedy set cover, vertex cover 2-approximation.

# Greedy set cover algorithm
def set_cover(universe, subsets):
    elements = set(universe)
    cover = []
    while elements:
        best = max(subsets, key=lambda s: len(elements & s))
        elements -= best
        cover.append(best)
    return cover

universe = set(range(10))
subsets = [set([0,1,2]), set([2,3,4]), set([4,5,6]), set([6,7,8,9]), set([1,3,5])]
cover = set_cover(universe, subsets)
print(f'Cover size: {len(cover)} (optimal would be ≤ {len(cover)} · ln({len(universe)}))')

PTAS (polynomial-time approximation scheme), FPTAS, and hardness of approximation results (inapproximability).
