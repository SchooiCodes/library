Windows Group Policy Editor

Local Group Policy, administrative templates, security policies, and computer configuration management.

What is Group Policy?

Group Policy is a Microsoft Windows infrastructure that allows administrators to implement specific configurations for users and computers. It controls the operating system, applications, and user settings in an Active Directory environment. Local Group Policy Editor (gpedit.msc) provides the same functionality on standalone Windows machines.

Group Policy settings are organized into two main categories:

  • Computer Configuration -- applied at system startup, affects all users
  • User Configuration -- applied at user logon, affects specific users

Each category contains Administrative Templates (registry-based policies), Windows Settings (security, scripts, folder redirection), and software installation settings.

Opening & Navigating Local Group Policy Editor

Launch and navigate the Group Policy Editor:

# Open Local Group Policy Editor (Windows Pro/Enterprise only)
gpedit.msc

# Open Group Policy Management (for domain environments)
gpmc.msc

# Resultant Set of Policy (what policies are applied)
rsop.msc

# Command-line policy update
gpupdate /force
gpresult /r  # View effective policy
gpresult /h policy-report.html  # HTML report

Navigate the tree: Computer Configuration > Windows Settings > Security Settings > Local Policies > Security Options is the most common path for security hardening.

Key Security Policies

Essential security policies to configure:

# Password Policy (Computer > Windows Settings > Security > Account Policies)
# - Enforce password history: 24 passwords remembered
# - Maximum password age: 60 days
# - Minimum password length: 14 characters
# - Password must meet complexity requirements: Enabled

# Account Lockout Policy
# - Account lockout threshold: 5 invalid attempts
# - Account lockout duration: 30 minutes
# - Reset account lockout counter after: 30 minutes

# User Rights Assignment (Security Settings > Local Policies)
# - Deny log on through Remote Desktop Services: add Guests, Local Accounts

# Security Options
# - Network access: Do not allow anonymous enumeration of SAM accounts
# - Microsoft network server: Digitally sign communications (always)

Administrative Templates & Registry-Based Policies

Administrative Templates control hundreds of Windows features via registry. Common examples:

# Computer > Admin Templates > Windows Components > Windows Update
# - Configure Automatic Updates: Enabled (4 - Auto download and install)
# - Specify intranet Microsoft update service location (for WSUS)

# User > Admin Templates > Start Menu and Taskbar
# - Remove Run menu from Start Menu: Disabled (keep Run)
# - Remove notification and action center: Disabled

# Computer > Admin Templates > System > Device Installation
# - Prevent installation of removable devices: Enabled (block USB drives)

# Custom ADMX templates can be added:
# Copy .admx files to C:\Windows\PolicyDefinitions\
# Copy .adml files to C:\Windows\PolicyDefinitions\en-US\

Managing Policies via PowerShell

PowerShell provides programmatic access to Group Policy:

# Get installed Administrative Templates
Get-ADMXTemplate -Name | Format-Table

# Backup all Local Group Policy settings
$backupPath = "C:\Backups\GPO_$(Get-Date -Format yyyyMMdd)"
Backup-GPO -Name "Local Policy" -Path $backupPath

# Reset Local Group Policy to defaults
Remove-Item -Recurse -Force $env:windir\System32\GroupPolicy\*
gpupdate /force

# Check specific policy setting
# Use LGPO.exe tool from Microsoft for advanced management
# Download: https://www.microsoft.com/en-us/download/details.aspx?id=55319
