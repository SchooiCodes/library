AWS Fundamentals

Amazon Web Services — EC2, S3, Lambda, RDS, and core cloud services.

EC2 Virtual Machines

EC2 (Elastic Compute Cloud) provides scalable virtual servers. Choose instance families: general purpose (t3, m7g), compute optimized (c7g), memory optimized (r7g, x2iedn), GPU (p4d, g5), and storage optimized (i3, d2).

Key features: AMIs (Amazon Machine Images — pre-configured OS templates), security groups (virtual firewalls), EBS volumes (persistent block storage), elastic IPs (static public IPs), and auto-scaling groups (automatically adjust capacity). Pricing: on-demand, reserved (1-3 year commitment, up to 72% off), spot (up to 90% off, interruptible).

S3 Object Storage

S3 (Simple Storage Service) stores objects (files) in buckets. Objects have a key (path), value (data), metadata, and version ID. S3 is infinitely scalable, durable (99.999999999%), and accessible via HTTP/REST, SDKs, or CLI.

Storage classes: Standard (frequent access), Intelligent-Tiering (auto-cost optimization), Standard-IA (infrequent access), One Zone-IA (cheaper, single AZ), Glacier (archive, minutes-hours retrieval), and Deep Archive (long-term, 12hr retrieval, cheapest). Lifecycle policies automate transitions between tiers.

Lambda Serverless

AWS Lambda runs code without provisioning servers. Supported languages: Python, Node.js, Java, Go, Ruby, .NET, Custom Runtime (Rust, C++). Triggers: S3 events, API Gateway, SQS, DynamoDB Streams, CloudWatch Events, and more.

Limits: 15 min max execution, 128MB-10GB memory, 512MB-10GB ephemeral storage, 1MB payload. Cold starts add latency — mitigate with Provisioned Concurrency (keep functions warm) or SnapStart (Lambda snapshots for Java). Lambda Power Tuning tool finds optimal memory/performance balance.

RDS & Databases

RDS (Relational Database Service) manages MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, and Amazon Aurora. Features: automated backups, multi-AZ (high availability), read replicas (read scaling), and automatic storage scaling.

DynamoDB is AWS's NoSQL key-value/document database. Single-digit millisecond latency at any scale, auto-scaling, DAX (DynamoDB Accelerator — in-memory cache). Aurora is AWS's MySQL/PostgreSQL-compatible database with 5x faster performance, auto-scaling storage up to 128TB.

IAM Security

AWS Identity and Access Management (IAM) controls access to AWS resources. Users (individuals), Groups (collections of users), Roles (assumed by services/users), and Policies (JSON documents defining permissions) form the access model.

Best practices: least privilege (only grant needed permissions), use roles instead of long-term keys, enable MFA for root user, use IAM Access Analyzer (identifies external access), and rotate access keys regularly. Policy conditions restrict access by IP, time, VPC, and MFA status.
