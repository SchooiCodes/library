Media Server Automation Stack

Automate your entire media library — from searching and downloading to organizing and streaming. This guide covers the "*arr" stack with qBittorrent, Prowlarr, Bazarr, and Stremio integration.

Disclaimer: This guide is for educational purposes. Ensure you have permission to download any content and use a VPN for privacy.

Sonarr / Radarr / Lidarr

These tools automate TV shows, movies, and music downloading. They monitor your wantlist and automatically search, download, sort, and rename files.

Sonarr (TV Shows)

# Docker run example
docker run -d \
  --name sonarr \
  -p 8989:8989 \
  -v /path/to/data:/config \
  -v /path/to/tv:/tv \
  -v /path/to/downloads:/downloads \
  --restart unless-stopped \
  linuxserver/sonarr

Radarr (Movies)

docker run -d \
  --name radarr \
  -p 7878:7878 \
  -v /path/to/data:/config \
  -v /path/to/movies:/movies \
  -v /path/to/downloads:/downloads \
  --restart unless-stopped \
  linuxserver/radarr

Lidarr (Music)

docker run -d \
  --name lidarr \
  -p 8686:8686 \
  -v /path/to/data:/config \
  -v /path/to/music:/music \
  -v /path/to/downloads:/downloads \
  --restart unless-stopped \
  linuxserver/lidarr

Initial Setup

  1. Access each web UI at its respective port (8989, 7878, 8686).
  2. Add your download client (qBittorrent) under Settings → Download Client.
  3. Add your indexers in Settings → Indexers (or connect Prowlarr).
  4. Add shows/movies/artists to monitor.
  5. Configure quality profiles and root folders.

Bazarr for Subtitles

Bazarr automatically downloads subtitles for your Sonarr and Radarr libraries.

docker run -d \
  --name bazarr \
  -p 6767:6767 \
  -v /path/to/config:/config \
  -v /path/to/movies:/movies \
  -v /path/to/tv:/tv \
  --restart unless-stopped \
  linuxserver/bazarr

Configuration

  1. Access Bazarr at port 6767.
  2. Go to Settings → Sonarr/Radarr and add your server URLs and API keys.
  3. Under Subtitles → Languages, select your preferred languages.
  4. Enable "Upgrade" to automatically find better subtitles.
  5. Bazarr will scan existing media and download missing subtitles.

Prowlarr — Indexer Manager

Prowlarr acts as a central indexer manager for Sonarr, Radarr, Lidarr, and Bazarr — one indexer configuration shared across all apps.

docker run -d \
  --name prowlarr \
  -p 9696:9696 \
  -v /path/to/config:/config \
  --restart unless-stopped \
  linuxserver/prowlarr

Setup

  1. Access Prowlarr at port 9696.
  2. Go to Settings → Apps and add Sonarr, Radarr, Lidarr, and Bazarr.
  3. Go to Indexers and add your preferred indexers (public or private).
  4. Prowlarr syncs all indexers to your connected apps automatically.
Tip: Prowlarr supports Jackett indexers too. You can migrate from Jackett by importing your existing indexers in Prowlarr.

qBittorrent with VPN Binding

qBittorrent is a free, open-source torrent client. Binding it to your VPN interface prevents IP leaks if the VPN drops.

docker run -d \
  --name qbittorrent \
  -p 8080:8080 \
  -p 6881:6881 \
  -p 6881:6881/udp \
  -v /path/to/config:/config \
  -v /path/to/downloads:/downloads \
  --restart unless-stopped \
  linuxserver/qbittorrent

VPN Binding Setup

  1. Access qBittorrent web UI at port 8080 (default: admin/adminadmin).
  2. Go to Settings → Advanced → Network Interface.
  3. Select your VPN interface (e.g., tun0, wg0, or the VPN adapter name).
  4. Go to Settings → Connection → Listening Port and set a port (6881).
  5. Enable "Use UPnP / NAT-PMP" only if needed.
Critical: VPN binding ensures that if your VPN disconnects, qBittorrent stops all network activity — no IP leaks.

Stremio + Debrid Setup

Stremio provides a Netflix-like interface for streaming torrents. Combined with Real-Debrid, you get instant 4K streams from cached torrents.

Setup Steps

  1. Install Stremio on your device.
  2. Create an account and log in.
  3. Sign up for Real-Debrid (paid, ~€3/month).
  4. Install the Torrentio addon (configure with your Real-Debrid API key).
  5. Browse and play — Torrentio fetches cached torrents via Real-Debrid.

Docker Compose for the Full Stack

Deploy everything with a single docker-compose.yml file.

version: "3.8"
services:
  sonarr:
    image: linuxserver/sonarr
    container_name: sonarr
    ports:
      - "8989:8989"
    volumes:
      - ./config/sonarr:/config
      - /data/tv:/tv
      - /data/downloads:/downloads
    restart: unless-stopped

  radarr:
    image: linuxserver/radarr
    container_name: radarr
    ports:
      - "7878:7878"
    volumes:
      - ./config/radarr:/config
      - /data/movies:/movies
      - /data/downloads:/downloads
    restart: unless-stopped

  lidarr:
    image: linuxserver/lidarr
    container_name: lidarr
    ports:
      - "8686:8686"
    volumes:
      - ./config/lidarr:/config
      - /data/music:/music
      - /data/downloads:/downloads
    restart: unless-stopped

  bazarr:
    image: linuxserver/bazarr
    container_name: bazarr
    ports:
      - "6767:6767"
    volumes:
      - ./config/bazarr:/config
      - /data/movies:/movies
      - /data/tv:/tv
    restart: unless-stopped

  prowlarr:
    image: linuxserver/prowlarr
    container_name: prowlarr
    ports:
      - "9696:9696"
    volumes:
      - ./config/prowlarr:/config
    restart: unless-stopped

  qbittorrent:
    image: linuxserver/qbittorrent
    container_name: qbittorrent
    ports:
      - "8080:8080"
      - "6881:6881"
      - "6881:6881/udp"
    volumes:
      - ./config/qbittorrent:/config
      - /data/downloads:/downloads
    restart: unless-stopped
# Start the entire stack
docker-compose up -d

# Check logs for all services
docker-compose logs -f

# Stop everything
docker-compose down
Tip: Run the stack on an always-on device (NAS, Raspberry Pi, or VPS). Configure port forwarding or a reverse proxy (Nginx) for remote access. Use Tailscale for secure remote access without opening ports.
