🛡️ DNS Privacy & Filtering Guide

Block ads, trackers, and malware at the DNS level with Pi-hole, AdGuard Home, and NextDNS.

What is DNS Filtering?

DNS filtering intercepts DNS queries and blocks requests to known malicious, ad-serving, or tracking domains before they reach your device. Unlike browser ad blockers, it works network-wide — every device on your network gets protected automatically, including IoT gadgets, smart TVs, and phones that can't run extensions.

Three main approaches: Self-hosted (Pi-hole, AdGuard Home), Cloud-based (NextDNS, ControlD), and Router-level (FreshTomato, OpenWrt with adblock). Each has trade-offs in control, privacy, and ease of setup.

Pi-hole Setup

Pi-hole is a DNS sinkhole that runs on a Raspberry Pi or any Linux server. It blocks ads and trackers at the network level, improving speed and privacy for every device.

# One-line install (Debian/Ubuntu/Raspbian)
curl -sSL https://install.pi-hole.net | bash

# During setup:
# - Select a network interface (eth0 or wlan0)
# - Choose an upstream DNS provider (Cloudflare, Quad9, etc.)
# - Enable the web admin interface
# - Note the generated admin password

# After install, set Pi-hole as your DHCP DNS server
# Or configure your router to hand out Pi-hole's IP as DNS

Access the admin panel at http://pi.hole/admin or http://YOUR_PI_IP/admin. The dashboard shows total queries, blocked percentage, top blocked domains, and top clients.

Key features: Adlists (curated blocklists from Firebog, OISD, etc.), Regex blacklist/whitelist, DHCP server, Local DNS records, Conditional forwarding for local domain resolution, API for integration with Home Assistant and Grafana.

Pro tip: Add the oisd blocklist for comprehensive coverage: Settings → Blocklists → Add: https://big.oisd.nl

AdGuard Home Setup

AdGuard Home is a modern alternative to Pi-hole with a cleaner UI, built-in DHCP, encryption (DoH/DoT), and parental controls. It runs on Linux, Windows, macOS, or as a Docker container.

# Docker install (recommended)
mkdir -p ~/adguard && cd ~/adguard
cat > docker-compose.yml << 'EOF'
version: "3"
services:
  adguard:
    image: adguard/adguardhome
    container_name: adguard
    ports:
      - 53:53/tcp
      - 53:53/udp
      - 80:80/tcp
      - 3000:3000/tcp
    volumes:
      - ./work:/opt/adguardhome/work
      - ./conf:/opt/adguardhome/conf
    restart: unless-stopped
EOF
docker-compose up -d

Access the setup wizard at http://YOUR_IP:3000. After initial setup, the admin panel runs on port 80. Features include: DNS query log with search/filter, Client-specific blocking, Safe Search enforcement, Blocked service management, and DNS-over-HTTPS/TLS server.

NextDNS Setup

NextDNS is a cloud-based DNS filtering service with a generous free tier (300,000 queries/month). No hardware required — configure it on your router, device, or browser.

# Router setup (most routers)
# Set DNS servers to:
# 45.90.28.0
# 45.90.30.0

# Or use DNS-over-HTTPS:
# https://dns.nextdns.io/YOUR_CONFIG_ID

# Linux setup with stubby:
sudo apt install stubby
sudo nano /etc/stubby/stubby.yml
# Add your NextDNS config under upstream_recursive_servers

Configuration options: Blocklists (OISD, Hagezi, 1Hosts), Allowlist/Denylist, Parental controls, NAT64 for IPv6-only networks, Query logs with analytics, and multiple profiles for different networks.

Install the CLI client for advanced features: curl -fsSL https://nextdns.io/install | sh. This enables encrypted DNS on Linux without configuring systemd-resolved.

DNS-over-HTTPS & DNS-over-TLS

DNS encryption prevents your ISP from seeing which websites you visit. Standard DNS is unencrypted and can be intercepted, modified, or logged by anyone on the network path.

# Systemd-resolved (most modern Linux)
sudo nano /etc/systemd/resolved.conf
# Add:
[Resolve]
DNS=1.1.1.1 1.0.0.1
DNSOverTLS=yes
DNSSEC=yes
Cache=yes

# Then restart:
sudo systemctl restart systemd-resolved

Browser-level DoH: Firefox: Settings → Network Settings → Enable DNS over HTTPS → Choose provider (Cloudflare, NextDNS). Chrome: Uses system DNS by default, enable via chrome://flags/#dns-over-https.

Public DoH/DoT providers: Cloudflare (1.1.1.1), Quad9 (9.9.9.9), NextDNS (custom), AdGuard DNS (94.140.14.14), Mullvad DNS. For maximum privacy, run your own recursive resolver with Unbound alongside Pi-hole/AdGuard Home.

Troubleshooting & Maintenance

Common issues and fixes for DNS filtering setups:

  • No queries showing up — Other devices might be using hardcoded DNS (Google Nest, Chromecast). Block port 53 on router except for your DNS server.
  • Some sites broken — Whitelist the domain in your blocklist manager. Check query logs to find which blocklist caused it.
  • Slow DNS resolution — Your upstream DNS provider might be slow. Use a caching resolver like Unbound for better performance.
  • VPN interference — VPNs bypass local DNS. Configure your VPN to use your DNS filter's IP, or use NextDNS which works alongside VPNs.
  • Docker networking — Make sure your DNS server container's port 53 is accessible on the host network. Use network_mode: host if needed.
