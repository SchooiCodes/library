Hardware Security Module

HSM for cryptographic key management — PKCS#11, AWS CloudHSM, and key ceremonies.

Table of Contents (2 sections)

  • HSM Concepts
  • Cloud HSM & Key Ceremonies

HSM Concepts

HSM (Hardware Security Module): tamper-resistant hardware for cryptographic operations. Keys generated inside HSM — never leave in plaintext. Standards: FIPS 140-2/140-3 Level 2-4 (security certification), PKCS#11 (Cryptoki API — open standard for HSM access). Operations: key generation (RSA, ECC, AES), signing (code signing, document signing), encryption/decryption, key wrapping, true random number generation (TRNG).

Cloud HSM & Key Ceremonies

Cloud HSM: AWS CloudHSM (dedicated HSM, VPC), AWS KMS (managed, automatic key rotation), Azure Key Vault (HSM-backed, BYOK), Google Cloud HSM (Cloud KMS with FIPS 140-2 Level 3). Key ceremonies: process for creating master keys — multi-person authorization (m of n), crypto officer + crypto user roles, tamper-evident seals, video recording, paper audit trail. PKCS#11 CLI: pkcs11-tool --module libcloudhsm.so --login --list-objects.
