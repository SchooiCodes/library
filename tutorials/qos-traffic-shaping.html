QoS & Traffic Shaping

Prioritization, shaping, policing, DiffServ, CoS, and practical QoS configuration on routers and Linux.

What is QoS?

Quality of Service (QoS) is a set of techniques to manage network resources by prioritizing certain types of traffic over others. QoS is essential for real-time applications like VoIP, video conferencing, and online gaming, where latency and jitter must be minimized.

Three fundamental QoS actions:

  • Classification -- identifying traffic by type (DSCP, CoS, ACLs)
  • Marking -- setting priority bits in packet headers
  • Policing/Shaping -- enforcing bandwidth limits

DiffServ: DSCP & Trust Boundaries

DiffServ (Differentiated Services) is the primary QoS architecture for modern networks. DSCP (Differentiated Services Code Point) marks packets with 6-bit priority values:

# Common DSCP values:
# EF (46)   -- Expedited Forwarding: VoIP RTP (lowest latency)
# AF41 (34) -- Assured Forwarding 4.1: Video conferencing
# AF31 (26) -- Assured Forwarding 3.1: Streaming video
# AF21 (18) -- Assured Forwarding 2.1: Business-critical data
# AF11 (10) -- Assured Forwarding 1.1: Standard data
# CS0 (0)   -- Best effort
# CS6 (48)  -- Network control (routing protocols)
# CS7 (56)  -- Network control (reserved)

# Cisco trust boundary configuration
interface GigabitEthernet0/1
 description Access port
 ml qos trust device cisco-phone
 ml qos trust cos

# Rewrite DSCP on untrusted ports
policy-map MARK-ACCESS
 class VOIP
  set dscp ef
 class VIDEO
  set dscp af41
 class BUSINESS
  set dscp af21
 class DEFAULT
  set dscp 0

Shaping vs Policing

Traffic shaping buffers excess packets; policing drops them:

# Policing (dropping excess packets)
policer LIMIT-10M {
    if-exceeding {
        bandwidth-percent 10
        burst-packet-count 1500
    }
    then discard;
}

# Shaping (buffering excess packets)
interface GigabitEthernet0/0
 traffic-shape rate 10000000 1000000 1000000

# Cisco MQC (Modular QoS CLI) shaping
policy-map SHAPE-50M
 class class-default
  shape average 50000000

# Linux tc shaping (HTB)
tc qdisc add dev eth0 root handle 1: htb default 30
tc class add dev eth0 parent 1: classid 1:1 htb rate 100mbit
tc class add dev eth0 parent 1:1 classid 1:10 htb rate 10mbit ceil 20mbit

QoS Configuration Examples

Real-world QoS configurations for different network roles:

# ISP/Edge Router -- limited bandwidth WAN link
class-map match-any VOIP
 match ip dscp ef
class-map match-any VIDEO
 match ip dscp af41 af42 af43

policy-map WAN-EDGE
 class VOIP
  priority percent 10    # Strict priority queue
 class VIDEO
  bandwidth percent 30   # Guaranteed bandwidth
 class BUSINESS
  bandwidth percent 30
 class class-default
  fair-queue
  random-detect          # WRED for TCP

interface Serial0/0/0
 bandwidth 50000        # 50 Mbps
 service-policy output WAN-EDGE

# Linux tc with fq_codel (modern, no manual config needed)
tc qdisc replace dev eth0 root fq_codel
# Fair Queuing with Controlled Delay -- adaptive AQM

Monitoring QoS & Troubleshooting

Verify QoS is working correctly:

# Cisco
show policy-map interface Serial0/0/0
show policy-map interface Serial0/0/0 class VOIP
# Look for: "Queue depth 0", "Packets matched 0" (if no traffic)

# Linux tc
# -s = statistics
tc -s qdisc show dev eth0
tc -s class show dev eth0

# Check for drops (overlimit) in shaping classes
# If drops > 0, consider increasing shapred rate or reducing offered load

# End-to-end QoS verification
iperf3 -u -b 10M -l 1400 -t 30 -c 192.168.1.100
# Check jitter and packet loss at receiver
