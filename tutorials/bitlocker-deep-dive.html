BitLocker Overview & Architecture

BitLocker Drive Encryption is a full volume encryption feature in Windows Professional and Enterprise editions. It encrypts the entire operating system volume, protecting data at rest. BitLocker uses AES encryption (128-bit or 256-bit) in XTS or CBC mode with an Elephant diffuser for disk sector encryption.

Key components:

  • TPM (Trusted Platform Module) -- stores encryption keys securely; validates boot integrity
  • BitLocker System Boot -- requires a separate, unencrypted system partition for boot files
  • Recovery Key -- 48-digit numeric password used when TPM fails or boot chain changes
  • Secure Boot -- ensures only trusted firmware and OS components load

Enabling BitLocker

Enable BitLocker via Control Panel or PowerShell:

# Check TPM status
tpm.msc
Get-Tpm

# Enable BitLocker (Control Panel)
# Control Panel > System and Security > BitLocker Drive Encryption > Turn on BitLocker

# Enable via PowerShell (Admin)
Enable-BitLocker -MountPoint "C:" -EncryptionMethod XtsAes256 -UsedSpaceOnly \
    -RecoveryPasswordProtector -SkipHardwareTest

# Add recovery key protector
Add-BitLockerKeyProtector -MountPoint "C:" -RecoveryPasswordProtector

# Backup recovery key to Microsoft Account
Backup-BitLockerKeyProtector -MountPoint "C:" -KeyProtectorId "{GUID}"

# Check encryption status
Get-BitLockerVolume -MountPoint "C:"

TPM & Key Protectors

BitLocker uses multiple key protectors to secure the Volume Master Key (VMK):

# TPM protector (default) -- key released only if boot integrity check passes
# TPM+PIN -- requires user PIN at boot
# TPM+Startup Key -- requires USB key at boot
# Recovery Password -- 48-digit key for rescue scenarios
# Password (Data Drives) -- password-based protection for fixed/removable drives

# Configure TPM+PIN for enhanced security
# Group Policy: Computer > Admin Templates > Windows Components > BitLocker
# "Configure TPM startup PIN" > Enabled
# Set PIN: 4-20 digits

# Add TPM with PIN protector
Add-BitLockerKeyProtector -MountPoint "C:" -TpmAndPinProtector

# View protectors
Get-BitLockerVolume -MountPoint "C:" | Select-Object KeyProtector

Recovery & Data Recovery

When BitLocker prompts for recovery, use these methods:

# Common triggers for recovery mode:
# - BIOS/UEFI update
# - Boot configuration changes
# - TPM firmware update
# - Disk controller changes (AHCI vs RAID)

# Suspend BitLocker (before hardware changes)
Suspend-BitLocker -MountPoint "C:" -RebootCount 2

# Resume protection
Resume-BitLocker -MountPoint "C:"

# Find recovery key:
# 1. Microsoft Account (https://account.microsoft.com/devices/recoverykey)
# 2. Active Directory (if domain-joined)
# 3. Print or USB backup

# Recovery key file
# C:\Users\%USERNAME%\AppData\Roaming\Microsoft\RecoveryKey\*.txt

# Manual unlock (if recovery key known)
Unlock-BitLocker -MountPoint "D:" -RecoveryPassword "48-digit-key"

BitLocker Management & Enterprise Deployment

Manage BitLocker at scale with MBAM or PowerShell:

# MBAM (Microsoft BitLocker Administration and Monitoring)
# - Part of Microsoft Desktop Optimization Pack (MDOP)
# - Centralized key escrow, compliance reporting, self-service recovery

# Manage-BDE command-line tool
manage-bde -status C:
manage-bde -protectors -get C:
manage-bde -off C:   # Decrypt
manage-bde -on C:     # Encrypt with defaults

# Enterprise deployment via GPO
# Computer > Admin Templates > Windows Components > BitLocker Drive Encryption
# - Choose drive encryption method and cipher strength: XTS-AES 256-bit
# - Require additional authentication at startup: Enabled (TPM+PIN)
# - Deny write access to removable drives not protected by BitLocker: Enabled
