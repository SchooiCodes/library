Web Performance Optimization

Core Web Vitals, lazy loading, caching, CDN, and performance budgets.

Table of Contents (2 sections)

  • Core Web Vitals
  • Loading & Caching

Core Web Vitals

LCP (<2.5s): optimize hero image (preload, fetchpriority=high), compress, CDN. CLS (<0.1): set image dimensions, space for embeds, font-display:optional. FID/INP (<100ms/200ms): minimize main thread blocking, code-split, lazy JS. TTFB: fast server response, CDN, preconnect. Measuring: Lighthouse, PageSpeed Insights, web-vitals library, CrUX. Performance budget: 300KB JS, 100KB CSS, 1MB total.

Loading & Caching

Code splitting: dynamic import(), React.lazy, route-based. Images: WebP/AVIF, srcset, loading=lazy, blur placeholders. Fonts: subset, variable, preload, font-display:optional. Preload: critical resources. Preconnect: early origin connections. Service worker: Workbox (precache, runtime strategies). Cache: Cache-Control immutable for hashed assets. Brotli: better than gzip. HTTP/2 multiplexing. HTTP/3 QUIC. Minimize: HTML, CSS, JS (terser, cssnano). Critical CSS: inline above-fold.
