The 3-2-1 Backup Rule

The golden rule of backups: 3 copies of your data, on 2 different media types, with 1 copy off-site.

  • 3 copies: 1 primary + 2 backups
  • 2 media: e.g., internal HDD + external drive, or NAS + cloud
  • 1 off-site: different physical location (cloud, friend's house, safety deposit box)
Tip: RAID is not a backup. RAID protects against disk failure but not against accidental deletion, corruption, ransomware, or disasters. Always have a separate backup system.

rsync

Fast, flexible file syncing. Copies only changed parts of files. Works over SSH for remote backups.

# Local backup
rsync -avh --delete /home/user/ /mnt/backup/

# Remote backup over SSH
rsync -avz -e ssh /home/user/ user@remote:/backup/

# Useful flags:
# -a : archive mode (preserves permissions, timestamps, etc.)
# -v : verbose
# -z : compress during transfer
# --delete : remove files in destination that no longer exist in source

Borg Backup

Deduplicating, compressed, encrypted backup tool. Ideal for large data sets with many repeated files.

# Install Borg
sudo apt install borgbackup

# Initialize a repository
borg init --encryption=repokey /mnt/backup/borg-repo

# Create a backup
borg create --stats --progress \
 /mnt/backup/borg-repo::{hostname}-{now:%Y-%m-%d} \
 /home/user /etc

# List archives
borg list /mnt/backup/borg-repo

# Restore
borg extract /mnt/backup/borg-repo::archive-name

Borg deduplicates automatically — subsequent backups are very fast and space-efficient.

restic

Cross-platform backup tool with multiple storage backends (local, S3, Backblaze B2, SFTP, etc.).

# Initialize
restic init --repo /mnt/backup/restic-repo

# Backup
restic --repo /mnt/backup/restic-repo backup /home/user

# Restore
restic --repo /mnt/backup/restic-repo restore latest --target /restore

# With remote backend (S3):
restic -r s3:s3.amazonaws.com/bucket-name init

Duplicati

GUI-based backup tool with encryption, compression, and cloud storage support. Runs as a web service.

# Install Duplicati
sudo apt install duplicati
sudo systemctl enable duplicati
sudo systemctl start duplicati

# Access web UI at http://localhost:8200
# Configure backup destinations: local, Google Drive, OneDrive, S3, etc.
# Schedule backups via the UI

Cloud Backups

Popular cloud storage targets for backups:

  • Backblaze B2 — $0.006/GB/month, S3-compatible
  • Wasabi — $0.0069/GB/month, no egress fees
  • AWS S3 / Glacier — many tiers, cheap cold storage
  • rsync.net — ZFS-based, supports Borg/restic directly

Automation

Use cron (Linux) or Task Scheduler (Windows) to automate backups:

# Cron: daily backup at 2 AM
0 2 * * * /usr/local/bin/backup-script.sh

# Example backup script:
#!/bin/bash
borg create --stats /mnt/backup::{hostname}-{now:%Y-%m-%d} /home/user
borg prune --keep-daily 7 --keep-weekly 4 --keep-monthly 6 /mnt/backup

Testing Backups

An untested backup is not a backup. Schedule regular restore tests:

# Monthly restore test
borg extract --dry-run /mnt/backup::latest
# Or restore to a temp directory and verify files

Document your backup strategy, test quarterly, and ensure you can recover from a total loss.
