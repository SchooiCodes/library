🌍 Linux Networking Guide

Configure networking on Linux — ip, nmcli, netplan, bridges, bonding, routing, and diagnostics.

The ip Command

The modern replacement for ifconfig, route, and arp. Part of the iproute2 package.

# Show all network interfaces and addresses
ip addr show

# Show routing table
ip route show

# Show neighbor (ARP) table
ip neigh show

# Bring interface up/down
ip link set eth0 up
ip link set eth0 down

# Add an IP address
ip addr add 192.168.1.100/24 dev eth0

# Add a default gateway
ip route add default via 192.168.1.1

nmcli (NetworkManager)

NetworkManager is the default network service on most desktop Linux distros. nmcli controls it from the command line.

# List connections
nmcli connection show

# List devices
nmcli device status

# Connect to Wi-Fi
nmcli device wifi connect "SSID" password "password"

# Create a static IP connection
nmcli connection add type ethernet con-name "static-eth0" \
 ifname eth0 ip4 192.168.1.100/24 gw4 192.168.1.1

# Modify connection (add DNS)
nmcli connection mod "static-eth0" ipv4.dns "1.1.1.1 8.8.8.8"
nmcli connection down "static-eth0" && nmcli connection up "static-eth0"
Tip: Use nmcli connection show --active to see only active connections, and nmcli device wifi list to scan nearby Wi-Fi networks.

netplan

Netplan is the default network config on modern Ubuntu. It uses YAML files in /etc/netplan/ and generates systemd-networkd or NetworkManager config.

# /etc/netplan/01-netcfg.yaml
network:
 version: 2
 renderer: networkd
 ethernets:
  eth0:
   dhcp4: true
  eth1:
   addresses:
    - 10.0.0.100/24
   routes:
    - to: 10.0.1.0/24
     via: 10.0.0.1

# Apply config
sudo netplan apply

# Debug
sudo netplan --debug apply

Bridges & Bonding

Bridge: Connects multiple interfaces like a virtual switch. Used by VMs and containers.

# Create bridge with netplan
network:
 version: 2
 ethernets:
  eth0:
   dhcp4: false
 bridges:
  br0:
   interfaces: [eth0]
   dhcp4: true

Bonding: Combines multiple physical NICs for redundancy or increased throughput.

# Bond two interfaces (mode 1 = active-backup)
network:
 version: 2
 bonds:
  bond0:
   interfaces: [eth0, eth1]
   parameters:
    mode: active-backup
    mii-monitor-interval: 100
   dhcp4: true

Routing

Manage complex routing with policy-based routing and multiple routing tables:

# Add a static route
sudo ip route add 10.0.0.0/16 via 192.168.1.254

# Delete a route
sudo ip route del 10.0.0.0/16

# Enable IP forwarding (for router/gateway)
sudo sysctl -w net.ipv4.ip_forward=1
# Make permanent in /etc/sysctl.conf:
# net.ipv4.ip_forward = 1

Diagnostics

ToolUsage
ssSocket statistics — replaces netstat. Show listening ports: ss -tlnp
tcpdumpPacket capture. tcpdump -i eth0 port 80
mtrCombines ping + traceroute in real time. mtr google.com
nmapNetwork scanner. nmap -sn 192.168.1.0/24
pingTest reachability and latency
iperf3Network throughput test. iperf3 -c server
# Quick connectivity test
ping -c 4 8.8.8.8

# Check DNS resolution
dig google.com +short

# Trace path to host
mtr -r google.com
