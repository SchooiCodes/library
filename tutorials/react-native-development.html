React Native Cross-Platform

Build mobile apps for iOS and Android with React Native and Expo.

Table of Contents (5 sections)

  • React Native Setup & Basics
  • Expo Managed Workflow
  • State Management
  • Native Modules & APIs
  • Debugging & Publishing

React Native Setup & Basics

React Native uses JavaScript/TypeScript to build native mobile apps. Core components: View (div), Text (span), ScrollView, FlatList, TextInput, TouchableOpacity, Pressable. Styles use JavaScript objects (StyleSheet.create) with flexbox layout.

React Navigation handles routing: Stack (screen stack), Tab (bottom/top tabs), Drawer (side menu). React Native CLI (more control, native modules) vs Expo (managed workflow, OTA updates, no Xcode/Android Studio needed for development).

Expo Managed Workflow

Expo simplifies React Native development. It provides pre-built native modules (camera, maps, notifications, biometrics, location, haptics) with no native code configuration. Expo Go app lets you test on real devices instantly via QR code.

Expo Application Services (EAS): EAS Build (cloud builds), EAS Submit (app store uploads), EAS Update (over-the-air updates). Expo SDK 52+ includes expo-router (file-based routing like Next.js). For custom native modules, use Expo Dev Client (eject only what you need).

State Management

React Native state management: useState/useReducer (local), useContext (shared across tree), Redux Toolkit (global, predictable state), Zustand (simple, hook-based), and React Query/TanStack Query (server state, caching, auto-refetch).

AsyncStorage (key-value) for simple persistence. MMKV (fast, synchronous) for performance-critical storage. WatermelonDB for relational data. Realm for real-time sync. For complex apps: Redux + Redux Toolkit Query (RTK Query) integrates API caching with global state.

Native Modules & APIs

When you need platform-specific functionality: Native Modules (Android Java/Kotlin, iOS Swift/Objective-C) and Fabric (the new rendering architecture). Turbo Modules (faster, synchronous native calls) are replacing the old bridge.

libraries: react-native-camera (barcode scanning), react-native-maps (Google/Apple Maps), react-native-push-notifications, react-native-biometrics (Face ID/Touch ID). Use React Native Directory (reactnative.directory) to find quality libraries.

Debugging & Publishing

Debugging: React DevTools (component inspector, profiler), Flipper (network inspector, layout editor, React DevTools plugin), Chrome DevTools (console, breakpoints), React Native Debugger (standalone, Redux integration). Use console.log with reactotron for richer logging.

Publishing: build AAB (Android) and IPA (iOS) via EAS Build or locally. Configure app icons, splash screens (expo-splash-screen), and app.json (name, slug, version, orientation, deep linking). Submit to Play Store and App Store — React Native apps are native apps and follow same review processes.
