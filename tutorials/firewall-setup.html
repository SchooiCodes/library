What is a Firewall?

A firewall monitors and controls incoming/outgoing network traffic based on security rules. It acts as a barrier between trusted internal networks and untrusted external networks (like the internet). Firewalls can be hardware-based, software-based, or both.

Linux typically uses netfilter (iptables/nftables) in the kernel; user-space tools like UFW and firewalld provide simplified management on top.

UFW (Uncomplicated Firewall)

Default on Ubuntu. Simple syntax for common rules.

# Enable UFW
sudo ufw enable

# Default policies
sudo ufw default deny incoming
sudo ufw default allow outgoing

# Allow SSH, HTTP, HTTPS
sudo ufw allow ssh
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# Allow from specific IP
sudo ufw allow from 192.168.1.100 to any port 22

# Deny a port
sudo ufw deny 3306

# Delete a rule
sudo ufw delete allow 80/tcp

# Status with rules
sudo ufw status verbose
Tip: Always enable SSH before enabling UFW if connecting remotely, or you'll lock yourself out!

firewalld (RHEL/Fedora)

Uses zones to manage traffic by trust level. Common zones: public, home, internal, dmz.

# Start and enable
sudo systemctl enable --now firewalld

# Check status and zones
sudo firewall-cmd --state
sudo firewall-cmd --get-active-zones

# Add services to public zone
sudo firewall-cmd --zone=public --add-service=http --permanent
sudo firewall-cmd --zone=public --add-service=https --permanent

# Add a port
sudo firewall-cmd --zone=public --add-port=8080/tcp --permanent

# Rich rules (advanced)
sudo firewall-cmd --zone=public --add-rich-rule='rule family="ipv4" source address="10.0.0.0/24" accept' --permanent

# Reload to apply
sudo firewall-cmd --reload
sudo firewall-cmd --list-all

nftables Basics

The modern replacement for iptables. Uses a cleaner syntax with tables, chains, and rules.

# Create a table and chain
sudo nft add table inet filter
sudo nft add chain inet filter input { type filter hook input priority 0\; policy drop\; }

# Allow loopback, established, SSH
sudo nft add rule inet filter input iif lo accept
sudo nft add rule inet filter input ct state established,related accept
sudo nft add rule inet filter input tcp dport 22 accept

# Allow HTTP/HTTPS
sudo nft add rule inet filter input tcp dport { 80, 443 } accept

# Save rules
sudo nft list ruleset > /etc/nftables.conf

# Restore
sudo nft -f /etc/nftables.conf

Windows Defender Firewall

Built into Windows. Managed via GUI (wf.msc) or PowerShell.

# PowerShell commands
New-NetFirewallRule -DisplayName "Allow SSH" -Direction Inbound -Protocol TCP -LocalPort 22 -Action Allow

New-NetFirewallRule -DisplayName "Block SMB" -Direction Outbound -Protocol TCP -LocalPort 445 -Action Block

# Block all inbound except whitelist
Set-NetFirewallProfile -Profile Domain,Public,Private -DefaultInboundAction Block

# Logging
Set-NetFirewallProfile -Profile Public -LogFileName %SystemRoot%\System32\LogFiles\Firewall\pfirewall.log -LogAllowed True

Testing with nmap

Always verify your firewall rules from an external machine.

# Scan common ports
nmap -sS -p 22,80,443,8080 

# Full port scan (slow)
nmap -p- 

# Detect firewall
nmap -sA 

# Version detection
nmap -sV 

# UDP scan
nmap -sU -p 53,161,500
