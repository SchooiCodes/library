Kubernetes Deep Dive

Kubernetes architecture, deployments, services, ingress, and cluster management.

Table of Contents (4 sections)

Pods, Deployments & StatefulSets

Pods are the smallest deployable units in Kubernetes — one or more containers sharing network/storage. Deployments manage stateless applications with declarative updates, rollbacks, and scaling. ReplicaSet ensures the desired number of pod replicas.

StatefulSets manage stateful applications (databases) with stable network identities (pod-name-N), ordered deployment/scaling, and persistent storage per pod via PersistentVolumeClaims. DaemonSets run a pod on every node (logging, monitoring, CNI).

Services & Networking

Services expose pods to network traffic. Types: ClusterIP (internal only, default), NodePort (static port on each node, 30000-32767), LoadBalancer (cloud LB, external IP), and ExternalName (DNS alias). Services use label selectors to route to matching pods.

Ingress controllers (nginx, Traefik, HAProxy) provide HTTP/HTTPS routing, TLS termination, and virtual hosting. Gateway API is the next-gen replacement for Ingress, supporting TCP/UDP/gRPC routing, HTTP header manipulation, and cross-namespace references.

Helm Charts & Package Management

Helm is the Kubernetes package manager. Charts are packaged K8s resources (templates + values.yaml). Install: helm install my-release ./chart. Charts can be stored in repositories, OCI registries, or Git.

Helmfile adds environment management — declare desired state of releases across clusters and environments. Kustomize (built into kubectl) provides overlay-based customization without templating — it patches YAML rather than rendering templates.

Cluster Monitoring & Observability

Prometheus is the standard K8s monitoring system. It scrapes metrics from pods, nodes, and services. Alertmanager handles alert routing (PagerDuty, Slack, email). Grafana visualizes metrics with pre-built dashboards.

kube-state-metrics exposes cluster-level metrics (deployments, nodes, pods). Node Exporter provides hardware/OS metrics. Loki (Grafana) aggregates logs. Jaeger/Tempo provide distributed tracing. The ELK stack (Elasticsearch, Logstash, Kibana) is an alternative for log aggregation.
