🔒 BitLocker Guide

Full-disk encryption on Windows — enabling, managing, recovery, and alternatives.

What is BitLocker?

BitLocker Drive Encryption is a full-disk encryption feature built into Windows Pro, Enterprise, and Education editions. It encrypts the entire drive using AES with either 128-bit or 256-bit keys, protecting data at rest against unauthorized access if the device is lost or stolen.

BitLocker integrates with a Trusted Platform Module (TPM) to validate system integrity at boot. If the TPM detects tampering (modified boot files, BIOS changes, or hardware swaps), BitLocker enters recovery mode and requires a recovery key before the drive can be accessed.

Note: BitLocker is not available on Windows Home. For those editions, consider Device Encryption (a lighter variant) or third-party tools like VeraCrypt.

Requirements

  • TPM 1.2+ (TPM 2.0 recommended) — for measured boot and key protection
  • Windows edition: Pro, Enterprise, or Education
  • UEFI firmware with Secure Boot enabled
  • At least two partitions on the drive (system/reserved and OS)

Enabling BitLocker

Open Control Panel → BitLocker Drive Encryption → Turn on BitLocker. The wizard will walk you through the following choices:

Unlock Method

BitLocker needs to unlock the drive before the OS boots. Choose one of these protectors:

  • TPM + PIN — Requires a PIN at every boot (strongest protection against physical theft)
  • TPM + USB key — Requires a USB drive with a startup key inserted at boot
  • TPM only — Drive unlocks automatically if the TPM validates integrity (most convenient)
  • Password — A pre-boot password (available without TPM, or as an additional protector)

Save Your Recovery Key

If BitLocker enters recovery mode, you will need this 48-digit key to regain access. Save it in at least two places:

  • Microsoft account — Accessible at aka.ms/myrecoverykey
  • USB flash drive — Store the key as a text file
  • File — Save to a secure location (not on the encrypted drive)
  • Print — Physical backup (keep it safe)
Best practice: Save to your Microsoft account and export a file to offline storage. Losing your recovery key means permanent data loss.

Encryption Mode

Windows offers two cipher modes:

  • XTS-AES (default for new drives) — Stronger, but not compatible with Windows 8.1 or older
  • AES-CBC — Compatible with older Windows versions and some third-party tools like dislocker

Encryption Scope

  • Used disk space only — Faster, encrypts only occupied sectors (ideal for new/empty drives)
  • Full drive encryption — Encrypts the entire drive including free space (takes longer, recommended for drives already in use)

Managing BitLocker with manage-bde

manage-bde is the command-line tool for BitLocker management. Run these from an elevated Command Prompt or PowerShell.

CommandDescription
manage-bde -statusCheck encryption status of all drives
manage-bde -protectors -get C:View all key protectors on drive C:
manage-bde -changepin C:Change the BitLocker PIN for C:
manage-bde -protectors -disable C:Suspend BitLocker (reboot allowed, no recovery needed)
manage-bde -protectors -enable C:Re-enable BitLocker after suspension
manage-bde -off C:Decrypt the drive (turns off BitLocker)

Recovery Scenarios

BitLocker will prompt for the recovery key at boot if the TPM measurement changes. Common triggers:

  • TPM firmware update or replacement
  • Hardware changes (new motherboard, RAM, GPU)
  • Boot configuration changes (modifying boot order, disabling Secure Boot)
  • BIOS/UEFI update
  • Bootkit or tampering detection (malware trying to modify boot components)

Finding Your Recovery Key

  • Microsoft account: Visit aka.ms/myrecoverykey and sign in with the Microsoft account linked to the device
  • USB drive: Insert the USB where you saved the key — BitLocker should detect it automatically
  • Active Directory: If the device is domain-joined, the recovery key is stored in AD. Ask your IT admin or check Active Directory Users and Computers → BitLocker Recovery (requires appropriate permissions)
  • Azure AD / Entra ID: Keys are available under the device's BitLocker key blade at portal.azure.com

BitLocker To Go

BitLocker To Go encrypts removable drives (USB flash drives, external SSDs/HDDs). Once enabled, the drive is password-protected. To access the drive on another computer, the user must enter the password or provide the recovery key. Compatible with Windows 7 and later (a read-only agent is available for older systems).

Decrypting Offline with Dislocker

If you have the recovery key or the user's password, you can decrypt or mount a BitLocker volume from Linux using dislocker:

# Install dislocker
sudo apt install dislocker  # Debian/Ubuntu
sudo pacman -S dislocker   # Arch

# Mount a BitLocker-encrypted partition
sudo dislocker /dev/sda2 -r /path/to/recovery-key -- /mnt/bitlocker

# The decrypted contents appear as a file (dislocker-file)
# which you can then mount as a loop device
sudo mount -o loop /mnt/bitlocker/dislocker-file /mnt/decrypted

Dislocker supports both AES-CBC and XTS-AES modes, so it works with modern BitLocker volumes.

BitLocker vs VeraCrypt

FeatureBitLockerVeraCrypt
LicenseProprietary (built into Windows)Open-source (free)
Platform supportWindows onlyWindows, macOS, Linux
Encryption algorithmsAES-128/256 (XTS or CBC)AES, Serpent, Twofish, cascades
TPM integrationNative (TPM + PIN/USB/password)None (password or keyfile only)
Hidden volumesNot supportedPlausible deniability (hidden OS/volume)
PerformanceHardware-accelerated (AES-NI)Hardware-accelerated (AES-NI)
Ease of useIntegrated into Windows, very simpleRequires installing a driver, more manual
AuditabilityClosed-source (no public audit)Source available, publicly audited

Bottom line: BitLocker is the most convenient option on Windows — it's frictionless, integrates with TPM, and works out of the box. VeraCrypt is the better choice if you need cross-platform support, open-source transparency, or features like hidden volumes for plausible deniability.

Pro tip: On Windows, you can encrypt the OS drive AND a secondary data drive with BitLocker. Remember: encryption only protects data at rest — once the system is unlocked, all users with access can read the files as normal.
