From bare metal to a fully functional Arch Linux system — a complete step-by-step walkthrough.

What is Arch Linux?

Arch Linux is an independently developed, x86-64 general-purpose GNU/Linux distribution that strives to provide the latest stable versions of most software by following a rolling release model. Unlike point-release distributions (Ubuntu, Fedora), Arch is perpetually up-to-date — there is no need to reinstall or upgrade between major versions.

Arch is built around the KISS principle (Keep It Simple, Stupid), which in Arch's context means: avoiding unnecessary additions, modifications, or complications. The system is designed for users who want to understand and control every aspect of their OS.

Key components:

  • pacman — Arch's package manager. Simple, fast, and powerful. Syntax: pacman -Syu (update), pacman -S <package> (install), pacman -R <package> (remove).
  • AUR (Arch User Repository) — A community-driven repository of PKGBUILD scripts that allow you to install packages not in the official repos. Managed with helpers like yay or paru.
  • DIY Philosophy — Arch gives you a bare minimum base system and trusts you to build it up. No bloat, no defaults you have to undo — just what you choose to install.

Download ISO & Create Bootable USB

Download the latest Arch Linux ISO from the official site:

# Direct download
https://archlinux.org/download/

# Or via torrent (faster, more reliable)
https://archlinux.org/releng/releases/

Create a bootable USB drive using Ventoy (recommended) or Rufus:

  • Ventoy (cross-platform): Install Ventoy on a USB stick, then simply copy the ISO file onto the drive. Ventoy can boot multiple ISOs — no need to reformat for each one. sudo dd if=ventoy*.iso of=/dev/sdX or use the GUI installer.
  • Rufus (Windows): Select the USB drive, choose the ISO, keep defaults (GPT, UEFI), and write in DD mode.
  • dd (Linux): sudo dd if=archlinux-*.iso of=/dev/sdX bs=4M status=progress && sync

Installation — Step by Step

1. Verify Boot Mode

After booting from the USB, verify whether you are in UEFI or BIOS mode:

ls /sys/firmware/efi/efivars

If the directory exists, you are booted in UEFI mode. If not, you are in BIOS/Legacy mode. This determines your partition table and bootloader choice later.

2. Connect to the Internet

For Wi-Fi, use iwctl (iwd — iNet Wireless Daemon, included in the live environment):

iwctl

# Inside iwctl:
station list        # List available wireless devices
station wlan0 scan     # Scan for networks
station wlan0 get-networks # Show available networks
station wlan0 connect "YourSSID"
# Enter the Wi-Fi password when prompted
exit

For Ethernet, just plug in — it should work automatically via DHCP. Verify connectivity:

ping -c 3 archlinux.org

3. Partition Disks

Use fdisk or parted to partition your disk. Identify your disk first:

lsblk
fdisk -l

For a UEFI system (the most common setup in 2026):

fdisk /dev/nvme0n1 # or /dev/sda for SATA drives

# Inside fdisk:
g          # Create a new GPT partition table
n          # New partition
 Partition number: 1
 First sector: (default)
 Last sector: +512M  # EFI System Partition
 Type: EFI System (or 1)

n          # New partition
 Partition number: 2
 Last sector: +8G   # Swap (adjust size based on RAM)
 Type: Linux swap (or 19)

n          # New partition
 Partition number: 3
 Last sector: (default — rest of the disk)
 Type: Linux filesystem (or 20)

w          # Write changes and exit

For BIOS/Legacy systems, use an MBR partition table (o) instead of GPT. No ESP is needed — just a root partition and optionally swap.

4. Format Partitions

# Format the ESP (EFI System Partition)
mkfs.fat -F32 /dev/nvme0n1p1

# Format the root partition
mkfs.ext4 /dev/nvme0n1p3

# Set up swap
mkswap /dev/nvme0n1p2
swapon /dev/nvme0n1p2

Adjust device names and partition numbers (nvme0n1p1, sda1, etc.) to match your setup.

5. Mount Partitions

# Mount root to /mnt
mount /dev/nvme0n1p3 /mnt

# Mount ESP to /mnt/boot (for UEFI)
mount --mkdir /dev/nvme0n1p1 /mnt/boot

6. Install Base System

pacstrap /mnt base linux linux-firmware base-devel

This installs the absolute minimum: the base system, the Linux kernel, firmware for hardware support, and base-devel (compilers, make, autotools — needed for building AUR packages later).

You may also want to include additional packages at this stage:

pacstrap /mnt base linux linux-firmware base-devel \
 vim man-db man-pages texinfo networkmanager

7. Generate fstab

genfstab -U /mnt >> /mnt/etc/fstab

Verify the fstab file to ensure all partitions are correctly listed:

cat /mnt/etc/fstab

8. Chroot and Configure

arch-chroot /mnt

# Set timezone
ln -sf /usr/share/zoneinfo/Region/City /etc/localtime
hwclock --systohc

# Set locale
echo "en_US.UTF-8 UTF-8" >> /etc/locale.gen
locale-gen
echo "LANG=en_US.UTF-8" > /etc/locale.conf

# Set hostname
echo "myhostname" > /etc/hostname

# Set root password
passwd

# Add a regular user (with wheel group for sudo)
useradd -m -G wheel -s /bin/bash myusername
passwd myusername

# Grant sudo to wheel group
echo "%wheel ALL=(ALL:ALL) ALL" >> /etc/sudoers.d/wheel

9. Install Bootloader

For UEFI systems — systemd-boot (recommended):

bootctl install

# Create a boot entry
cat > /boot/loader/entries/arch.conf << 'EOF'
title  Arch Linux
linux  /vmlinuz-linux
initrd /initramfs-linux.img
options root=PARTUUID=$(blkid -s PARTUUID -o value /dev/nvme0n1p3) rw
EOF

# Set default loader
echo "default arch.conf" >> /boot/loader/loader.conf

For BIOS/Legacy systems — GRUB:

pacman -S grub
grub-install --target=i386-pc /dev/nvme0n1
grub-mkconfig -o /boot/grub/grub.cfg

10. Network Setup

If you installed networkmanager during pacstrap, enable and start it:

systemctl enable NetworkManager
# systemctl start NetworkManager # (done in chroot or on first boot)

Alternatively, use systemd-networkd (simpler, no dependencies):

systemctl enable systemd-networkd systemd-resolved

# Configure Ethernet (DHCP)
cat > /etc/systemd/network/20-wired.network << 'EOF'
[Match]
Name=enp* ens* eth*

[Network]
DHCP=yes
EOF

Exit chroot, unmount, and reboot:

exit
umount -R /mnt
reboot

Post-Installation Setup

AUR Helper (yay)

The AUR (Arch User Repository) is one of Arch's greatest strengths. Install yay — a popular AUR helper:

sudo pacman -S --needed git base-devel
git clone https://aur.archlinux.org/yay.git
cd yay
makepkg -si

Now you can install packages from both the official repos and the AUR with a single command:

yay -S package-name

Desktop Environment

Choose your preferred DE or WM:

  • GNOME — sudo pacman -S gnome gnome-tweaks && sudo systemctl enable gdm
  • KDE Plasma — sudo pacman -S plasma sddm && sudo systemctl enable sddm
  • Hyprland (Wayland tiling compositor) — sudo pacman -S hyprland waybar wofi (no display manager needed; start from ~/.bash_profile or a login manager like SDDM)

Essential Packages

sudo pacman -S firefox alacritty neovim htop btop \
 git curl wget zip unzip file-roller \
 noto-fonts noto-fonts-cjk noto-fonts-emoji \
 pipewire pipewire-pulse wireplumber \
 rsync ffmpeg imagemagick

Keeping Arch Updated

Arch's rolling release means updates are continuous. The update command is simple:

sudo pacman -Syu

This synchronizes the package databases (-Sy) and upgrades all packages (-Su). Some tips:

  • Never do partial upgrades — pacman -Sy package (without -u) can break your system. Always use -Syu to keep the system consistent.
  • Check Arch News before major updates — Visit archlinux.org/news or use pacman -Syu --needed after checking. The archlinux-contrib package provides arch-audit for checking known vulnerabilities.
  • Use pacman cache — Packages are cached in /var/cache/pacman/pkg/. If an update breaks something, you can downgrade: sudo pacman -U /var/cache/pacman/pkg/old-package.pkg.tar.zst. Clean the cache periodically: sudo pacman -Sc (removes old versions) or paccache -r (from pacman-contrib).

Rolling Release Caveats

  • Always read before updating — Subscribe to the Arch Linux news feed or check it before running a large update. Manual interventions are occasionally required (e.g., when pacman itself has a breaking change, or when a major library like glibc, openssl, or python has a soname bump).
  • Downgrading is possible but not trivial — As mentioned, cached packages can be reinstalled. For AUR packages, you can use yay -S --nodiff --rebuild package or check the AUR page for older PKGBUILDs.
  • Backup your system — Use timeshift or snapper with Btrfs snapshots, or simply back up /etc and your home directory regularly.
  • Know how to fall back — From the bootloader menu, you can select a fallback initramfs (initramfs-linux-fallback.img) if the default one fails. Having an Arch USB handy for emergency chroot is a good habit.
You're rolling! Arch is as stable as you make it. Stick to the official repos, read the news, and don't install random AUR packages without inspecting the PKGBUILD. The Arch Wiki is your bible — bookmark it.
