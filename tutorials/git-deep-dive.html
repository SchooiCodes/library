Version Control (Git Deep Dive)

Git internals — branching, rebasing, hooks, and advanced workflows.

Git Internals & Branching

Git: content-addressable filesystem. Objects: blob (file content), tree (directory — blobs + trees), commit (snapshot + parent + tree + author/message), tag (annotated tag). References: branches (movable refs pointing to commits), HEAD (current branch), tags (immutable refs). Packfiles: delta compression, gc (garbage collection). .git/objects: first 2 chars = directory, rest = filename. Branches: lightweight (just a ref). Merge: fast-forward (no branch divergence), 3-way merge (merge commit). Rebase: replay commits on top of another branch — linear history, rewrite commits (never rebase public branches). Cherry-pick: apply specific commit. Reflog: local history of HEAD movements — emergency recovery. Reset: --soft (move HEAD), --mixed (unstage), --hard (discard changes). Stash: temporarily save working directory.

Advanced Workflows & Hooks

Git hooks: .git/hooks/ — pre-commit (lint, format), commit-msg (conventional commit check), pre-push (run tests), post-receive (deploy). Hooks are local — share via .githooks/ + git config core.hooksPath. Signed commits: git commit -S (GPG key). Submodules: external repo in subdirectory — git submodule update --init --recursive. worktrees: multiple branches checked out simultaneously. bisect: binary search for bug introduction — git bisect start; git bisect bad; git bisect good; automated with git bisect run. Filter-branch / filter-repo: rewrite history. Partial clone: git clone --filter=blob:none (only fetch on demand). Sparse checkout: check out only specific directories. Convention: Conventional Commits (feat:, fix:, chore:, docs:, BREAKING CHANGE:). Git flow: develop + feature branches + release/hotfix. Trunk-based: short-lived branches, main deployed often.
