SDDM Display Manager Customization

Theme, configure, and customize SDDM — the modern Linux display manager for KDE Plasma.

SDDM Overview

SDDM (Simple Desktop Display Manager) is the default login manager for KDE Plasma and many other Linux desktops. It supports QML-based theming, multiple seat support, and Wayland sessions.

Configuration file: /etc/sddm.conf (system-wide) or ~/.config/sddm.conf (user override). Theme directory: /usr/share/sddm/themes/.

SDDM supports autologin, multi-monitor setups, HiDPI scaling, and virtual keyboard on touchscreens.

Installing & Switching Themes

Popular SDDM themes: Sugar Candy, Breeze (default), Catppuccin, Layan, and Chili. Install from your distro's repos or manually from GitHub.

# Install via package manager
sudo pacman -S sddm sddm-kcm  # Arch
sudo apt install sddm sddm-theme-*  # Debian/Ubuntu

# Manual theme installation
git clone https://github.com/MarianArlt/sddm-sugar-candy
sudo cp -r sddm-sugar-candy /usr/share/sddm/themes/

Set the theme in /etc/sddm.conf under [Theme]: Current=sugar-candy.

Custom QML Theme Development

SDDM themes are QML applications. Create a directory with metadata.desktop and Main.qml:

[Desktop Entry]
Name=My Custom Theme
Comment=A personalized SDDM login
Type=QML

— Main.qml —
import QtQuick 2.0
import SddmComponents 2.0

Rectangle {
    width: 1920; height: 1080
    color: "#1a1a2e"
    Text {
        anchors.centerIn: parent
        text: "Welcome"
        color: "#e0e0e0"
        font.pixelSize: 48
    }
}

Components available: TextField, PasswordField, Button, Clock, Keyboard, and UserList.

Multi-Monitor & HiDPI Config

SDDM detects all monitors and can be configured to show the login screen on a specific display:

[General]
# Show login on specific display
DisplayServer=:0

[Theme]
Current=breeze
# Enable HiDPI scaling
Scale=1.5
# Cursor theme
CursorTheme=breeze_cursors

[Users]
# Show all users or only listed
MaximumUid=60000
MinimumUid=1000

Use sddm-greeter --test-mode --theme /path/to/theme to preview a theme without logging out.

Autologin & Session Management

Autologin skips the login screen — useful for single-user machines or kiosks:

[Autologin]
User=yourusername
Session=plasma.desktop

[General]
# Hibernate instead of sleep on lid close
HaltCommand=/usr/bin/systemctl poweroff
RebootCommand=/usr/bin/systemctl reboot

Manage sessions via /usr/share/xsessions/ (X11) and /usr/share/wayland-sessions/ (Wayland). Switch between Plasma, GNOME, i3, Hyprland, etc.
