Natural Language Processing

NLP fundamentals — tokenization, embeddings, transformers, and LLM fine-tuning.

Text Preprocessing

Text preprocessing converts raw text into a clean format suitable for ML models. Steps include lowercasing, removing punctuation, tokenization (splitting into words/subwords), stemming (Porter, Snowball), lemmatization (WordNet, spaCy), and stop word removal.

Libraries: NLTK, spaCy, Hugging Face tokenizers. Subword tokenization (BPE, WordPiece, SentencePiece) handles out-of-vocabulary words by breaking them into known subword units — essential for modern LLMs.

Word Embeddings

Word embeddings map words to dense vector representations where semantic similarity corresponds to vector proximity. Methods: Word2Vec (CBOW, Skip-gram), GloVe (co-occurrence matrix factorization), FastText (subword information), and contextual embeddings (BERT, ELMo).

Contextual embeddings from transformers like BERT produce different vectors for the same word in different contexts (e.g., "bank" in "river bank" vs "savings bank"), solving the polysemy problem that plagued static embeddings.

Transformer Architecture

The Transformer (Vaswani et al., 2017) replaced RNNs/LSTMs with self-attention. Key components: multi-head attention (queries, keys, values), positional encoding, feed-forward networks, layer normalization, and residual connections.

Self-attention computes weighted sums of all positions, enabling parallel computation and capturing long-range dependencies. BERT (encoder-only) excels at understanding tasks; GPT (decoder-only) excels at generation. Attention is all you need!

Fine-Tuning LLMs

Fine-tuning adapts a pre-trained language model to a specific task. Methods: full fine-tuning (updates all weights), LoRA (Low-Rank Adaptation — trains small rank matrices injected into attention layers), and adapters (small bottleneck layers between transformer blocks).

LoRA is the most popular method — it reduces trainable parameters by 10,000x while maintaining quality. QLoRA adds 4-bit quantization to fit large models on consumer GPUs. Tools: Hugging Face PEFT, Axolotl, Unsloth.

NLP Evaluation Metrics

NLP metrics: BLEU (precision-based, machine translation), ROUGE (recall-based, summarization), METEOR (semantic matching), BERTScore (embedding-based), and perplexity (language model quality).

For LLMs: human evaluation, MT-Bench (multi-turn conversation quality), AlpacaEval (following instructions), and bias/toxicity benchmarks (TruthfulQA, RealToxicityPrompts). Always use multiple metrics — each captures different aspects of quality.
