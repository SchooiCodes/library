Cloud Cost Optimization

Reduce cloud bills — reserved instances, autoscaling, spot instances, and FinOps.

Understanding Cloud Pricing Models

Cloud providers offer three pricing models: on-demand (full price, flexibility), reserved/committed (1-3 year, 30-70% discount), and spot/preemptible (interruptible, 60-90% discount). The right mix depends on workload predictability.

Compute pricing varies by instance family, region, and OS licensing. Data transfer costs are often the hidden expense — egress (data leaving the cloud) is usually more expensive than ingress. Storage costs depend on capacity, IOPS, and access frequency.

Reserved & Spot Instances

Reserved instances (RIs) guarantee capacity at a discount: standard (1yr/3yr, 30-72% off), convertible (different instance families, slightly less discount), and savings plans (flexible, per-hour compute commitment). Analyze your baseline usage and reserve that portion.

Spot instances are 60-90% off but can be terminated with 2-minute notice. Use for: batch processing, CI/CD, stateless web servers, ML training that supports checkpointing. Avoid for: databases, stateful services, latency-sensitive production. Diversify across instance types and zones.

Autoscaling Strategies

Autoscaling aligns resource usage with demand, avoiding over-provisioning. Compute autoscaling: add/remove instances based on CPU, memory, request count, or custom metrics. Schedule-based scaling (predictable patterns) and predictive scaling (ML-based) are more proactive.

Kubernetes: HPA (Horizontal Pod Autoscaler — scale pods by CPU/memory/custom metrics), VPA (Vertical Pod Autoscaler — adjust pod resources), and KEDA (event-driven scaling — scale based on queue depth, Kafka lag, etc.). Scale-to-zero for non-production environments.

Storage Tier Optimization

Storage tiering automatically moves data between access tiers based on age and access patterns. AWS S3 Intelligent-Tiering monitors access patterns and moves objects with no tiering fees. Lifecycle policies define rules: transition from Standard to IA after 30 days, to Glacier after 90, delete after 365.

EBS gp3 volumes provide baseline performance independent of volume size — no need to over-provision. Use EBS snapshots for backups (incremental, stored in S3). Delete unattached volumes and old snapshots. Use instance store for temporary data (ephemeral, included in instance cost).

FinOps & Cost Monitoring

FinOps (Financial Operations) combines finance, engineering, and business to manage cloud spending. The lifecycle: inform (visibility and allocation), optimize (right-sizing, discounts), operate (continuous improvement, governance).

Tools: AWS Cost Explorer (visualize spend, usage), Azure Cost Management (budgets, alerts, recommendations), GCP Cost Table (hierarchy-based tracking). Third-party: Vantage (multi-cloud, AI recommendations), CloudHealth (VMware), Infracost (IaC cost estimation in CI). Tag resources with cost center, environment, and project for accurate allocation.
