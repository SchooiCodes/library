Cloud Cost Optimization

Reduce cloud costs -- right-sizing, reserved instances, spot instances, and FinOps.

Table of Contents (2 sections)

  • Compute & Storage
  • Monitoring & FinOps

Compute & Storage

Right-sizing: match instance to workload -- CloudWatch metrics (CPU, memory, network), AWS Compute Optimizer (ML recommendations). Reserved Instances: 1yr/3yr, Standard/Convertible, partial/full/no upfront. Savings Plans: flexible discount (compute or EC2 instance). Spot Instances: up to 90% off, use for fault-tolerant, stateless, flexible workloads. Spot Fleet/EC2 Fleet: diversified across pools. EBS gp3: baseline 3000 IOPS included, only pay extra provisioned. S3 lifecycle: transition to IA/Glacier. S3 Intelligent-Tiering: auto savings. Data transfer: expensive (use CloudFront, VPC endpoints, same-AZ).

Monitoring & FinOps

AWS Cost Explorer: visualize spend, cost trends, usage. AWS Budgets: alerts at threshold (cost, usage, RI coverage). Cost allocation tags: track by project/team/environment. Trusted Advisor: cost optimization checks (idle instances, underutilized). Compute Optimizer: right-sizing, auto-scaling recommendations. S3 Storage Lens: storage usage, optimization. FinOps: culture + practice -- track, optimize, operate. Graviton: ARM-based instances (20-40% cheaper). Serverless: Lambda no cost when idle. Container right-sizing: adjust CPU/memory requests. RDS: use Reserved Instances, Aurora Serverless. NAT Gateway: expensive for high bandwidth (use NAT instance or VPC endpoints). Savings: commitments (RI/SP) 30-60% off, spot (60-90% off), right-sizing (10-20%). Multi-cloud: cost comparison across clouds. Kubernetes: cluster autoscaler, node pools, spot/preemptible nodes. Commitments: most savings come from commitments.
