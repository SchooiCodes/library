Deep Backup Strategies

Comprehensive backup strategies beyond 3-2-1 — automation, verification, and disaster recovery.

Table of Contents (3 sections)

  • Backup Methodologies
  • Tools & Automation
  • Disaster Recovery Testing

Backup Methodologies

3-2-1 rule: 3 copies, 2 different media types, 1 off-site. Enhancements: 3-2-1-1-0 (3 copies, 2 media, 1 off-site, 1 offline/air-gapped, 0 errors after verify). GFS (Grandfather-Father-Son): daily (7 days), weekly (4 weeks), monthly (12 months). Incremental Forever: one full backup, then only changed data forever. Differential: full backup + all changes since last full. Modern approach: Borg/Restic (deduplicated, encrypted, versioned backups).

Tools & Automation

Restic: fast, encrypted, deduplicated — supports local, SFTP, S3, Backblaze B2, REST server. One command: restic -r s3:s3.amazonaws.com/bucket backup /home — uses snapshots, prune old ones. Borg: similar but Python-based, better deduplication, supports append-only mode (protects backups from ransomware). Borg + Vorta: GUI frontend. Duplicati: web UI, supports cloud targets, AES-256 encryption. Duplicacy: commercial alternative to Restic with web UI.

Disaster Recovery Testing

Untested backups are worthless. Test quarterly: restore a random file (check integrity), restore a full server (time how long), verify encrypted backups (test decryption), test off-site access (can you reach your cloud/remote backup?). Practice scenarios: SSD failure (restore from cloud), ransomware (restore from offline backup), house fire (restore from off-site), accidental deletion (restore single file). Log test results and fix failures.
