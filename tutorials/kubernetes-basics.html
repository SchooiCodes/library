What is Kubernetes?

Kubernetes (K8s) is an open-source container orchestration platform that automates deployment, scaling, and management of containerized applications. Originally developed by Google, it's now the industry standard for running containers in production.

Core concepts:

  • Cluster — A set of worker machines (nodes) that run containerized applications
  • Node — A worker machine in the cluster (physical or virtual)
  • Pod — The smallest deployable unit, one or more containers sharing network/storage
  • Deployment — Declarative update for Pods and ReplicaSets
  • Service — Stable network endpoint for a set of Pods
  • Namespace — Virtual cluster for organizing resources

Installing Kubernetes

For learning, use Minikube (local single-node cluster) or kind (Kubernetes in Docker). For production, consider kubeadm, Rancher RKE2, or managed services (EKS, GKE, AKS).

# Minikube (local learning)
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube
minikube start --cpus=4 --memory=8192

# kind (Kubernetes in Docker)
curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.24.0/kind-linux-amd64
chmod +x ./kind && sudo mv ./kind /usr/local/bin/kind
kind create cluster --name dev

# kubectl (the CLI tool)
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
chmod +x kubectl && sudo mv kubectl /usr/local/bin/

Your First Deployment

# Create a deployment
kubectl create deployment nginx --image=nginx:alpine

# Scale it to 3 replicas
kubectl scale deployment nginx --replicas=3

# Expose it as a service
kubectl expose deployment nginx --port=80 --target-port=80 --type=NodePort

# Check status
kubectl get pods
kubectl get deployments
kubectl get services

# Describe for details
kubectl describe pod nginx-XXXXX

# Port-forward to access locally
kubectl port-forward service/nginx 8080:80
# Now open http://localhost:8080

YAML Manifests

Kubernetes resources are defined declaratively in YAML files. This is the idiomatic way to manage your cluster.

# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
 name: my-app
 labels:
  app: my-app
spec:
 replicas: 3
 selector:
  matchLabels:
   app: my-app
 template:
  metadata:
   labels:
    app: my-app
  spec:
   containers:
    - name: app
     image: my-app:latest
     ports:
      - containerPort: 3000
     env:
      - name: DATABASE_URL
       value: "postgres://user:pass@db:5432/myapp"
     resources:
      requests:
       memory: "256Mi"
       cpu: "250m"
      limits:
       memory: "512Mi"
       cpu: "500m"
     livenessProbe:
      httpGet:
       path: /health
       port: 3000
      initialDelaySeconds: 30
      periodSeconds: 10
---
# service.yaml
apiVersion: v1
kind: Service
metadata:
 name: my-app-service
spec:
 selector:
  app: my-app
 ports:
  - port: 80
   targetPort: 3000
 type: ClusterIP
# Apply manifests
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml

# Delete resources
kubectl delete -f deployment.yaml

Networking & Ingress

Services provide internal networking. Ingress exposes HTTP/HTTPS routes from outside the cluster to services.

# ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
 name: app-ingress
 annotations:
  nginx.ingress.kubernetes.io/rewrite-target: /
spec:
 ingressClassName: nginx
 rules:
  - host: app.example.com
   http:
    paths:
     - path: /
      pathType: Prefix
      backend:
       service:
        name: my-app-service
        port:
         number: 80

# Install NGINX Ingress Controller
kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/cloud/deploy.yaml

# Service types:
# ClusterIP — Internal only (default)
# NodePort — Exposes on each node's IP at a static port
# LoadBalancer — External load balancer (cloud providers)
# ExternalName — DNS alias

Storage & ConfigMaps

# PersistentVolumeClaim
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
 name: data-pvc
spec:
 accessModes:
  - ReadWriteOnce
 resources:
  requests:
   storage: 10Gi

# ConfigMap (non-sensitive config)
apiVersion: v1
kind: ConfigMap
metadata:
 name: app-config
data:
 APP_ENV: production
 LOG_LEVEL: info

# Secret (base64-encoded)
apiVersion: v1
kind: Secret
metadata:
 name: app-secrets
type: Opaque
data:
 API_KEY: c29tZS1hcGkta2V5

# Using them in a pod
spec:
 containers:
  - name: app
   envFrom:
    - configMapRef:
      name: app-config
   volumeMounts:
    - name: data
     mountPath: /data
 volumes:
  - name: data
   persistentVolumeClaim:
    claimName: data-pvc
