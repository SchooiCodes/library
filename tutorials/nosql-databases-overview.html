NoSQL Databases Overview

NoSQL — document, key-value, wide-column, graph databases and use cases.

Table of Contents (2 sections)

  • Document & Key-Value
  • Wide-Column & Graph

Document & Key-Value

Document stores: MongoDB (BSON, flexible schema, indexes, aggregation pipeline, replica sets, sharding), Couchbase (N1QL, SQL-like queries), CouchDB (replication-focused, REST API), Firebase Firestore (serverless, real-time). Use: catalogs, content management, user profiles, JSON-heavy data. Key-value stores: Redis (in-memory, persistence, pub/sub, streams, sorted sets, Lua scripting), Memcached (simple, distributed cache), DynamoDB (AWS, managed, auto-scaling, DAX), Etcd (distributed config, Raft consensus). Use: caching, session storage, real-time leaderboards, distributed locks, config storage. Consistency: DynamoDB (eventual or strongly consistent), Redis (master-replica, persistence configurable). CAP theorem: partition tolerance + availability (AP) or consistency (CP). MongoDB: CP (by default). Cassandra: AP.

Wide-Column & Graph

Wide-column (column-family): Apache Cassandra (AP, peer-to-peer, no single point of failure, CQL, tunable consistency, compaction strategies), ScyllaDB (C++ rewrite, faster), HBase (CP, Hadoop ecosystem, strong consistency, region servers), Google Bigtable (managed, spanner-based). Data model: row key -> column families -> columns. Use: time-series data, IoT, recommendation engines, event logging, large-scale analytics. Graph databases: Neo4j (property graph, Cypher query language, ACID, graph algorithms, Bloom visualization), Amazon Neptune (managed, property graph/RDF), ArangoDB (multi-model, native graph), JanusGraph (open-source, distributed). Use cases: social networks, fraud detection, knowledge graphs (Google Knowledge Graph), recommendation engines, network/infra topology, supply chain, identity resolution. RDF vs Property graph: RDF (W3C standard, semantic web, SPARQL), property graph (label+property, Neo4j, Gremlin). Multi-model: ArangoDB (documents + graph + key-value). Time-series: InfluxDB (purpose-built), TimescaleDB (SQL-based), Prometheus (monitoring metrics). Search: Elasticsearch (full-text search, analytics, aggregation). Choosing: data model drives choice.
