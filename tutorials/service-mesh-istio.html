Service Mesh (Istio)

Service mesh with Istio — sidecar proxy, traffic management, security, and observability.

Table of Contents (2 sections)

  • Istio Architecture
  • Traffic & Security

Istio Architecture

Istio: sidecar (Envoy proxy) injected into each pod — intercepts all traffic. Control plane (istiod): Pilot (service discovery, traffic rules), Citadel (certificate management, mTLS), Galley (config validation). Data plane: Envoy proxies — handle load balancing, circuit breakers, retries, timeouts. Installation: istioctl install --set profile=demo. Sidecar injection: namespace label istio-injection=enabled or annotation sidecar.istio.io/inject: "true" on pod.

Traffic & Security

VirtualService: HTTP routing — match URI, rewrite, redirect, retry, fault injection, mirror. DestinationRule: subsets (version labels), load balancer (ROUND_ROBIN, LEAST_CONN, RANDOM), connection pool (max connections, max pending requests), outlier detection (consecutive errors, ejection time). mTLS: PeerAuthentication (STRICT, PERMISSIVE, DISABLE), DestinationRule tls mode. AuthorizationPolicy: action (ALLOW, DENY), rules (source: principals, namespaces, IP blocks; to: methods, paths).
