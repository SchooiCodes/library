🔑 Password Security Guide

Password managers, 2FA/MFA, passkeys, creating strong passwords, and security best practices for accounts.

Password Managers

A password manager generates, stores, and auto-fills strong unique passwords for every account. Never reuse passwords.

Bitwarden

Open source, cloud or self-hosted. Apps for all platforms.

# CLI usage
bw login
bw generate --length 20 --uppercase --lowercase --number --symbol
bw list items --search "example"
bw get password "example.com"

# Self-host with Vaultwarden (Docker)
docker run -d --name vaultwarden \
 -v /opt/vaultwarden:/data \
 -p 8080:80 \
 vaultwarden/server:latest

KeePassXC

Local-only, offline. Database file encrypted with AES-256. Cross-platform.

# CLI
keepassxc-cli db-create --set-password mypasswords.kdbx
keepassxc-cli show mypasswords.kdbx -s "GitHub"
keepassxc-cli add mypasswords.kdbx -u myuser -l https://github.com -t "GitHub"
keepassxc-cli generate --length 30
Tip: Always back up your vault export (encrypted) to offline storage. Losing your master password + 2FA = permanent lockout.

2FA / MFA

Two-factor authentication adds a second layer beyond your password.

  • TOTP — Time-based one-time passwords (30s rotating). Apps: Aegis (Android), Raivo OTP (iOS), Ente Auth (cross-platform), Bitwarden Authenticator.
  • SMS — Avoid if possible. SIM swapping is a common attack.
  • Hardware keys — FIDO2/WebAuthn. YubiKey, Nitrokey, SoloKey. Phishing-resistant.
  • Backup codes — Always save backup codes when enabling 2FA. Store them with your password manager.
# Generate a TOTP secret (for custom implementation)
# Use your authenticator app to scan the QR from the service

# YubiKey CLI example
ykman oath add --uri "otpauth://totp/example:user?secret=JBSWY3DPEHPK3PXP&issuer=example"

Passkeys

Passkeys are the modern replacement for passwords. Based on WebAuthn/FIDO2 — your device generates a key pair, the private key never leaves your device, and you authenticate with biometrics (face, fingerprint) or PIN.

  • Cross-platform: Apple, Google, Microsoft all support passkeys with cloud sync.
  • Phishing-proof: Passkeys are scoped to the domain — a fake site can't use them.
  • Hardware-backed: Private keys are stored in secure enclaves (TPM, Secure Enclave).
  • Passwordless: No password to type or leak — just a biometric prompt.

Start using passkeys on Google, GitHub, Microsoft, PayPal, and other major services.

Creating Strong Passwords

# Good: long, random, unique
# 1s3cr3t -> bad (short, common pattern)
# P@ssw0rd123! -> bad (common base)

# Bitwarden/KeePass generated:
# uG8$kL2#mN5*vQ7^wR1

# Diceware method (memorable passphrase):
# correct-horse-battery-staple
# Use 5-7 words from a 7776-word diceware list

# Check password strength
# zxcvbn (Dropbox's library):
zxcvbn-cli -l 4 "MySecurePassphrase!-2025"

Rules: Minimum 16 characters (ideally 20+). Use all character classes. Never reuse. Change only if compromised (not on a schedule).

Recognizing Phishing

  • Check the URL carefully — g00gle.com is not google.com.
  • Hover over links before clicking. Verify the destination in status bar.
  • Legitimate services never ask for your password via email or SMS.
  • Urgency is a red flag — "Your account will be closed in 24 hours!"
  • Enable email aliases (SimpleLogin, Firefox Relay) to identify which service leaked your email.
Tip: A password manager prevents phishing — it won't auto-fill on a fake domain because the URL doesn't match.

Account Recovery

Plan for lockout before it happens:

  1. Export and backup your password manager vault regularly.
  2. Store recovery codes in a secure offline location (safe, encrypted USB).
  3. Set up a recovery email (use a different provider than your main).
  4. Use a dead man's switch service for critical data inheritance.
  5. Print backup seeds for hardware wallets and passkeys.

Digital Legacy — designate someone you trust with instructions (but not the actual passwords). Many password managers have emergency access features.
