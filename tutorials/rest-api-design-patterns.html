REST API Design Patterns

REST API patterns -- HATEOAS, pagination, filtering, error handling.

Table of Contents (2 sections)

  • Resource Design
  • Error Handling & Docs

Resource Design

REST: representational state transfer. Resources: nouns (/users, /orders/{id}). Naming: plural, lowercase, kebab-case (order-items). Actions: HTTP methods (POST, GET, PUT, PATCH, DELETE). Relationships: /users/{id}/orders (sub-resources). Status: 200 OK, 201 Created, 204 No Content, 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 409 Conflict, 422 Unprocessable, 429 Too Many, 500 Internal. Pagination: cursor-based (next_token, preferred for consistency) vs offset/limit (page, per_page). Filtering: ?status=active&created_after=2024-01-01. Sorting: ?sort=created_at,-name (desc with -). Sparse fields: ?fields=id,name,email (reduce payload). HATEOAS: links in responses -- rel: self, next, prev, related. Content negotiation: Accept header (application/vnd.api+json). Versioning: URL (/v1/users), header (Accept: version=1), or no version (evolve API).

Error Handling & Docs

Error response (RFC 7807 Problem Details): { type, title, status, detail, instance }. Validation: JSON Schema request/response validation. Idempotency: Idempotency-Key header for POST/PATCH (retry safety). Bulk operations: batch endpoints (POST /batch), bulk creation via array. Async operations: 202 Accepted + Location header pointing to status endpoint. Webhooks: event callbacks -- register URL, POST event payload, retry on failure. Rate limiting: headers (X-RateLimit-Limit, Remaining, Reset), 429 response. Cache: ETag, Last-Modified, Cache-Control. Security: HTTPS, CORS, CSP, rate limiting. Documentation: OpenAPI 3.0 (YAML/JSON), Swagger UI, ReDoc, Stoplight. OpenAPI: paths, methods, parameters, request bodies, responses, schemas (JSON Schema). Code generation: OpenAPI Generator, Swagger Codegen. Testing: contract testing (Pact), integration tests, Postman collections. SDKs: auto-generate client SDKs from OpenAPI.
