Flutter Development

Cross-platform mobile with Flutter — widgets, state management, and platform channels.

Table of Contents (2 sections)

  • Widgets & Layout
  • State Management & Native

Widgets & Layout

Flutter: compiled to native ARM code — Dart language. Widgets: StatelessWidget vs StatefulWidget (mutable state, setState). Layout: Container, Row, Column, Stack, Expanded, Flexible, Padding, SizedBox, ListView (scrollable), GridView. Material Design: Scaffold, AppBar, Drawer, BottomNavigationBar, FloatingActionButton, Card, ListTile, Chip, SnackBar. Cupertino (iOS-style): CupertinoPageScaffold, CupertinoNavigationBar, CupertinoTabBar. Text styling: TextStyle(fontSize, fontWeight, color, fontFamily). Theme: ThemeData, Theme.of(context). Platform-aware branching.

State Management & Native

State: setState (simple), Provider, Riverpod, BLoC/Cubit, GetX. Provider: ChangeNotifierProvider(create: (context) => Counter()), context.watch() / context.read(). Riverpod: ref.read(counterProvider.notifier).increment(). Navigation: go_router (declarative routing, deep links). HTTP: http or dio packages. Platform channels: MethodChannel('channel'), invokeMethod('getBatteryLevel') calling native Kotlin/Swift. Packages: pub.dev — camera, location, maps, local_auth (biometrics), sqflite (SQLite), shared_preferences. Flutter Web: experimental web support.
