Shizuku & Hail

Manage system apps, debloat your device, and freeze/unfreeze apps without root using Shizuku and Hail.

Overview

Shizuku is a system-level service that gives apps elevated privileges without root access. It starts a background process that uses the Android shell (ADB) or root permissions to run commands. Apps like Hail, Kanade, and Orion Store connect to Shizuku to perform operations that normally require root — like freezing system apps, disabling components, or silently installing APKs.

Hail is an app freezer that uses Shizuku to disable apps. It can freeze both user and system apps, effectively removing them from your device until you unfreeze them. Kanade is a similar tool focused on debloating and managing system app states.

Safety Warning. Freezing system apps can cause boot loops or soft-brick your device if you freeze critical components. Always research what an app does before freezing it. Keep a list of what you froze so you can restore them if something breaks.

Shizuku Setup

Method 1: ADB (No Root)

This is the most common method and works on any Android device with USB debugging enabled.

  1. Enable Developer options on your device: Settings > About phone > Tap "Build number" 7 times.
  2. Enable USB debugging in Developer options.
  3. Install Shizuku from F-Droid or GitHub: https://github.com/RikkaApps/Shizuku/releases
  4. Connect your phone to your computer via USB.
  5. Run the ADB command to start Shizuku:
# On your computer, with ADB installed:
adb shell sh /data/data/moe.shizuku.privileged.api/files/start.sh

# You should see: "Shizuku started"
# Keep USB connected or use wireless debugging.

Method 2: Wireless Debugging (Android 11+)

No USB cable needed:

  1. In Developer options, enable Wireless debugging.
  2. Tap "Wireless debugging" and select "Pair device with pairing code".
  3. In Shizuku app, tap "Pairing" and enter the code shown on your device.
  4. Once paired, Shizuku can start wirelessly.

Method 3: Root

If your device is rooted, Shizuku can start directly without ADB:

# Open Shizuku app
# Tap "Start via root"
# Grant root permission when prompted
# Shizuku will start automatically on boot.

Verifying Shizuku is Running

Open the Shizuku app. It will show "Shizuku is running" with the process ID and uptime. The "Granted apps" section lists which apps have permission to use Shizuku.

Hail — App Freezer

Hail uses Shizuku (or root) to freeze and unfreeze apps. It provides a clean Material You interface and supports batch operations.

Installation

# From F-Droid:
# Search for "Hail" and install.

# From GitHub Releases:
# https://github.com/aistra0528/Hail/releases

Freezing Apps

  1. Open Hail. It will automatically detect Shizuku.
  2. The main screen shows all installed apps. System apps are marked with a yellow icon.
  3. Tap an app to select it. Select multiple by long-pressing and tapping others.
  4. Tap the Freeze button (snowflake icon) in the bottom bar.
  5. Hail will disable the selected apps using pm disable commands via Shizuku.

UI Features

  • Search bar — Find apps by name or package name.
  • Filters — Show all / user / system / frozen apps.
  • Sort — By name, install date, or package name.
  • Batch actions — Freeze, unfreeze, or uninstall multiple apps at once.
  • Activity launcher — Launch specific activities within an app.
  • Component manager — Enable/disable specific components (services, receivers, activities).
  • App info — Quick link to the system's app info page.

Component Management

Hail lets you disable specific components within an app, not just the whole app. For example, you can disable only the tracking service in an app while leaving the app itself functional. This is an advanced feature — use with care.

Kanade — Alternative Debloater

Kanade is another Shizuku-powered tool focused on debloating. It provides a curated list of safe-to-remove packages for various OEMs (Samsung, Xiaomi, OnePlus, etc.).

Key Differences from Hail

  • Preset lists — Kanade includes debloat presets for popular brands. You can freeze/remove all known bloatware from your device with one tap.
  • Uninstall focus — Kanade prefers uninstalling apps over freezing. It can remove apps entirely (if possible) instead of just disabling them.
  • Package explorer — Detailed view of each package's activities, services, receivers, and permissions.
  • Backup before uninstall — Kanade can back up APKs before removing them, so you can restore if needed.

Installation

# From F-Droid:
# Search for "Kanade" and install.

# From GitHub:
# https://github.com/KanadeApp/Kanade/releases

Using Kanade Presets

  1. Open Kanade and grant Shizuku permission.
  2. Go to the Presets tab.
  3. Select your device manufacturer (e.g., Xiaomi, Samsung).
  4. Review the list of packages that will be affected.
  5. Tap "Apply preset" to freeze/uninstall all listed packages.
  6. Reboot your device after applying presets.

Restoring Frozen Apps

If you freeze an app and something breaks, restoring is straightforward:

  1. Open Hail (or Kanade).
  2. Switch to the Frozen tab or apply the "Frozen" filter.
  3. Select the app(s) you want to restore.
  4. Tap the Unfreeze button (sun icon).
  5. The app will be enabled immediately. No reboot needed.

If you froze a critical app and the device is in a boot loop, boot into Safe Mode. In Safe Mode, all user-disabled apps are re-enabled. Then unfreeze the problematic app and reboot normally.

# Boot into Safe Mode (varies by device):
# 1. Turn off the device.
# 2. Press and hold Power + Volume Down until the logo appears.
# 3. Release Power but keep holding Volume Down until Safe Mode appears.
# 4. In Safe Mode, use Hail to unfreeze the problematic app.

Troubleshooting

Shizuku Won't Start via ADB

  • Ensure USB debugging is enabled.
  • Try revoking USB debugging authorizations and re-authorizing.
  • Update ADB on your computer: adb version should be 1.0.41+.
  • On Windows, you may need the proper USB driver for your device.
  • Try a different USB port or cable.

Shizuku Keeps Stopping

  • Battery optimization may be killing Shizuku. Add it to the "Unrestricted" or "No optimization" list in battery settings.
  • On MIUI/HyperOS, enable "Autostart" for Shizuku in the security app.
  • If using root startup, ensure the Shizuku service is set to start at boot.

Hail Freeze Button Not Working

  • Shizuku is not running. Start it first.
  • Hail is not listed in Shizuku's granted apps. Open Shizuku and grant permission to Hail.
  • Some apps (especially Google Play Services and system launchers) cannot be frozen by Android's security model.

Device Acting Unstable After Freezing

  • Unfreeze all recently frozen apps immediately.
  • If the device is still unstable, boot into Safe Mode and unfreeze everything.
  • If you cannot access Safe Mode, you may need to factory reset via recovery. This is why keeping a list of frozen packages is critical.

Kanade Preset Removes Too Much

  • Kanade presets are aggressive. Review the package list before applying.
  • Use the backup feature in Kanade before running a preset.
  • If something breaks, restore from the backup or unfreeze individual packages in Hail.

Tips & Best Practices

  • Start small. Freeze one or two apps at a time and test your device before freezing more.
  • Never freeze the launcher. If you freeze your home screen app, you will be stuck. Always have a second launcher installed before freezing the default one.
  • Do not freeze Google Play Services. Many apps depend on it. Freezing it will cause widespread instability, even if you use microG.
  • Use Shizuku+ADB on first setup. Once Shizuku is running, it will keep running until reboot. You can set it to start on boot via a quick ADB command or a Tasker profile.
  • Keep Shizuku updated. New Android versions sometimes break Shizuku. Follow the GitHub repo for updates.
  • Combine with Orion Store. Use Orion Store to export APKs before freezing apps you might want to reinstall later.
  • Do not freeze system updater. If you freeze com.google.android.gms or the OEM update service, you may prevent security patches from installing.
  • Export frozen list. Hail does not have a built-in export feature, but you can screenshot the Frozen tab or copy the package names manually into a notes app.
  • Restart Shizuku after reboot. If you used ADB to start Shizuku, you will need to re-run the ADB command after every reboot — unless you set up auto-start via a shell script or Tasker.
  • Use Kanade for one-time debloat, Hail for ongoing management. Kanade is great for removing a batch of bloatware on a new device. Hail is better for daily management (freezing/unfreezing as needed).
