What is Markdown?

Markdown is a lightweight markup language created by John Gruber in 2004 that lets you write formatted text using a plain-text editor. It was designed to be easy to read and write, and can be converted to HTML and other formats. Markdown is now used everywhere — README files, documentation, forum posts, notes, and even books.

The philosophy of Markdown: "A Markdown-formatted document should be publishable as-is, as plain text, without looking like it's been marked up with tags or formatting instructions."

Basic Syntax

Headers

# H1 — Document Title
## H2 — Section
### H3 — Subsection
#### H4 — Sub-subsection
##### H5
###### H6

Text Formatting

**Bold text**
*Italic text*
~~Strikethrough~~
`Inline code`
***Bold and italic***

Lists

1. First ordered item
2. Second ordered item
  - Nested unordered item
  - Another nested item
3. Third ordered item

- Unordered item
- Another unordered item
 - Indented with two spaces

Links & Images

[Link text](https://example.com)
[Link with title](https://example.com "Title text")
![Alt text](https://example.com/image.png)

Code Blocks

```python
def hello():
  print("Hello, World!")
```

Use triple backticks with an optional language identifier for syntax highlighting.

Tables

| Header 1 | Header 2 | Header 3 |
|----------|:--------:|---------:|
| Left   | Center  |  Right |
| Row 2  | Data   |   $42 |

Colons in the separator row control alignment: left (:---), center (:---:), right (---:).

Blockquotes

> This is a blockquote.
> It can span multiple lines.
>
>> Nested blockquotes work too.

Horizontal Rules

---
***
___

Extended Syntax

Many Markdown processors support additional features beyond the original spec:

# Footnotes
Here is a sentence with a footnote.[^1]

[^1]: This is the footnote content.

# Task Lists
- [x] Completed task
- [ ] Unfinished task
- [ ] Another task

# Math (LaTeX)
$$ E = mc^2 $$

Inline math: $E = mc^2$

# Definition Lists
Term
: Definition of the term

# Strikethrough
~~This text is crossed out~~

# Highlight
==Highlighted text==

Markdown Flavors

FlavorBased OnKey Features
CommonMarkStandardized MarkdownStrict spec, unambiguous parsing, used by many tools as a baseline
GFM (GitHub Flavored Markdown)CommonMarkTask lists, tables strikethrough, autolinks, emoji shortcodes (:smile:), URL references to issues/PRs
MultiMarkdownOriginal MarkdownFootnotes, tables, citations, math, metadata, cross-references
PandocExtended MarkdownBidirectional conversion between dozens of formats, citations via BibTeX, raw HTML/LaTeX blocks
ObsidianCommonMark + GFMWiki-style links ([[Page Name]]), embeds (![[note]]), callouts, Dataview queries, graph view

CommonMark is the closest thing to an official standardized spec. GFM extends CommonMark with features useful for software development on GitHub. If you're writing documentation on GitHub, GFM is your default.

Editors

Recommended editors:
  • Typora — Minimalist WYSIWYG Markdown editor. What you see is what you get. Paid (one-time), available on all platforms.
  • VS Code — Excellent Markdown support with built-in preview, extensions (Markdown All in One, Markdown Preview Enhanced, Paste Image).
  • Obsidian — Knowledge base that works on local Markdown files. Graph view, plugins, backlinks. Excellent for personal note-taking and wiki-style documentation.
  • Neovim — With markdown.nvim, glow (terminal Markdown preview), and markdown-preview.nvim for browser preview.
  • StackEdit — Browser-based Markdown editor with sync to Google Drive, Dropbox, and GitHub.
  • Obsidian — The most popular note-taking app built on Markdown. Supports plugins, themes, and a vibrant community.

For terminal users, glow renders Markdown files directly in the terminal with syntax highlighting and tables.

Converting Markdown

Markdown can be converted to many other formats using these tools:

ToolCommandOutput Formats
Pandocpandoc input.md -o output.htmlHTML, PDF, DOCX, LaTeX, EPUB, reveal.js slides, and 40+ more
markednpx marked README.mdHTML (Node.js CLI)
multimarkdownmultimarkdown file.mdHTML, LaTeX, OPML
glowglow README.mdTerminal rendered (not conversion, but viewing)

Pandoc is the Swiss Army knife of document conversion. It can read Markdown, reStructuredText, LaTeX, HTML, EPUB, DOCX, and many other formats, and convert between them losslessly when possible. For PDF output, you'll need a LaTeX engine like pdflatex or xelatex installed.

Markdown Best Practices

  • Be consistent with spacing — Always put a blank line before headers, lists, and code blocks. This ensures correct parsing across all Markdown flavors.
  • Use reference-style links for readability: [text][ref] and [ref]: https://... at the bottom of the document.
  • Avoid trailing whitespace — Two spaces at the end of a line creates a line break (<br>) in original Markdown, which can be surprising.
  • Use fenced code blocks with language tags — Triple backticks with a language identifier (```python) enable syntax highlighting.
  • Escape special characters with a backslash when you need the literal character: \*not italic\*.
  • Use relative links for internal documentation so links work regardless of where the files are hosted.
  • Keep lines short (80-120 characters) for better diff readability in version control.

Following these best practices ensures your Markdown renders correctly everywhere — GitHub, GitLab, VS Code preview, Obsidian, and any other Markdown renderer.
