Linux Server Administration

Administer Linux servers — SSH, firewalls, monitoring, user management, and automation.

Initial Server Setup

First steps on a new Linux server:

# Update system
sudo pacman -Syu   # Arch
sudo apt update && sudo apt upgrade  # Debian/Ubuntu
sudo dnf upgrade   # Fedora

# Create non-root user with sudo
useradd -m admin
passwd admin
usermod -aG wheel admin  # or sudo group on Debian

# Harden SSH
# Edit /etc/ssh/sshd_config:
# PermitRootLogin no
# PasswordAuthentication no (after key setup)
# Port 2222 (non-standard port)
sudo systemctl restart sshd

Configure firewall: ufw (simple) or nftables (modern): ufw allow 2222/tcp && ufw enable.

User & Permission Management

Linux user and group management:

# Users
useradd -m -G wheel,users -s /bin/bash johndoe
passwd johndoe
usermod -aG docker johndoe  # Add to supplementary group
deluser johndoe             # Delete user

# Groups
groupadd developers
usermod -aG developers johndoe

# Sudoers — grant specific commands
# visudo /etc/sudoers.d/johndoe
johndoe ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart nginx

# ACLs — fine-grained permissions
setfacl -m u:johndoe:rx /var/www/html
setfacl -R -m u:johndoe:rwx /var/www/uploads

Monitoring & Logging

Monitor server health:

# Resource monitoring
htop                        # Interactive process viewer
nmon                        # Performance monitoring
iotop                       # I/O usage per process
nethogs                     # Per-process network usage

# Service monitoring — systemd
systemctl status nginx
journalctl -u nginx --since "1 hour ago"
journalctl -f -u nginx     # Follow logs

# Remote monitoring stack
# Prometheus + Node Exporter + Grafana
# Or lightweight: netdata (one-command install)
bash <(curl -Ss https://my-netdata.io/kickstart.sh)

# Logwatch — daily log summaries
sudo pacman -S logwatch
echo "/usr/sbin/logwatch --output mail --mailto admin@example.com" >> /etc/cron.daily/

Backup & Automation

Automate server maintenance:

# Cron syntax
# min hour day month weekday command
0 3 * * * /usr/bin/systemctl restart nginx  # Daily 3am restart
0 2 * * 7 /usr/bin/pacman -Syu --noconfirm  # Weekly update (Arch)

# Ansible — configuration management
# ~/ansible/inventory.ini
[webservers]
server1 ansible_host=192.168.1.10

# ~/ansible/playbook.yml
---
- hosts: webservers
  tasks:
    - name: Install nginx
      package: name=nginx state=latest
    - name: Start nginx
      service: name=nginx state=started enabled=yes

ansible-playbook -i inventory.ini playbook.yml

Security Hardening Checklist

Server hardening essentials:

☐ Disable root SSH login
☐ Use SSH key authentication only
☐ Change SSH default port
☐ Install and configure fail2ban
☐ Enable firewall (nfstables/ufw)
☐ Keep packages updated (automatic security updates)
☐ Set up SELinux or AppArmor
☐ Disable unused services (systemctl list-unit-files | grep enabled)
☐ Use automated vulnerability scanning: Lynis, ClamAV
☐ Regular backups with off-site copy
☐ Monitor failed login attempts: journalctl -u sshd | grep Failed
☐ Kernel hardening: sysctl -w kernel.randomize_va_space=2
