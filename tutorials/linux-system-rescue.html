Linux System Rescue Toolkit

Rescue, recover, and repair Linux systems — live USB tools, chroot, fsck, and data recovery.

Live USB Rescue Distributions

Essential rescue distros to keep on a USB drive:

  • SystemRescue — Arch-based, includes fsck, rsync, ddrescue, testdisk, parted, btrfs tools. sudo pacman -S sysrescue or download ISO.
  • GParted Live — Boot to repair/manage partitions, resize filesystems, recover deleted partitions.
  • Clonezilla — Disk imaging and cloning. Supports disk-to-disk, disk-to-image, over network.
  • Ubuntu Live — Full desktop environment with Try Ubuntu option. Good for driver issues.

Create with Ventoy for multi-boot: just copy ISOs to a Ventoy drive.

Chroot Recovery

Chroot into a broken system to repair packages, reinstall bootloader, or fix configs:

# Boot from live USB, mount partitions
mount /dev/sdXn /mnt
mount /dev/sdXn /mnt/boot  # If separate EFI/boot

# Bind mount system directories
for i in /dev /proc /sys /run; do mount --bind $i /mnt$i; done

# Chroot in
chroot /mnt /bin/bash
source /etc/profile
export PS1="(chroot) \\w $ "

# Now fix things: reinstall GRUB, fix configs, reinstall packages
pacman -S linux  # Reinstall kernel
pacman -S grub   # Reinstall bootloader
grub-install /dev/sdX
grub-mkconfig -o /boot/grub/grub.cfg

Filesystem Repair & Data Recovery

Check and repair filesystems:

# Check filesystem (unmount first!)
sudo umount /dev/sdXn
sudo fsck.ext4 -f /dev/sdXn  # ext4
sudo fsck.f2fs /dev/sdXn      # F2FS
sudo btrfs check /dev/sdXn    # Btrfs (use --repair carefully)

# Recover deleted files
testdisk /dev/sdX   # Partition recovery & file undelete
photorec /dev/sdX   # File carving (recovers by signature, loses filenames)

# Clone failing drive
ddrescue -d -r3 /dev/sdX /dev/sdY rescue.log

For SSDs, TRIM/fstrim after repairs: sudo fstrim -v /.

Bootloader Repair

Common boot failures and fixes:

# GRUB reinstall (from chroot)
grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB
grub-mkconfig -o /boot/grub/grub.cfg

# Fix UEFI boot entry
efibootmgr --create --disk /dev/sdX --part 1 --loader /EFI/GRUB/grubx64.efi --label "Linux"

# Systemd-boot reinstall
bootctl install

# Kernel panic on boot — check the initramfs
dracut --force  # Arch/Fedora
update-initramfs -u  # Debian/Ubuntu

# Boot with fallback initramfs from GRUB advanced menu

Remote Rescue & Log Analysis

Access broken system over network:

# Start SSH on live USB
sudo systemctl start sshd
# Get IP: ip addr
# Connect from another machine: ssh user@live-ip

# Analyze logs
journalctl -p 3 -xb  # Show errors from current boot
journalctl --list-boots
journalctl -b -1 -p 3  # Errors from previous boot

# Check systemd unit failures
systemctl --failed

# Xorg/Wayland issues
cat /var/log/Xorg.0.log | grep EE
journalctl -u display-manager
