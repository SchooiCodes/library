What is PowerShell?

PowerShell is a modern shell and scripting language developed by Microsoft. Built on top of the .NET framework, it's far more powerful than the traditional Command Prompt (CMD). It uses an object-oriented approach where commands produce structured data (objects) rather than plain text, making it incredibly flexible for automation and system administration.

Unlike CMD, which is limited to running batch commands, PowerShell gives you full access to the .NET ecosystem, remote management capabilities, and a consistent verb-noun command structure that makes learning new commands intuitive.

Opening PowerShell

There are several ways to launch PowerShell on Windows:

  • Press Win + X → Select Windows Terminal or PowerShell (Admin)
  • Right-click the Start button → Windows Terminal
  • Press Win + R, type powershell, and press Enter
  • Search for "PowerShell" in the Start menu

For administrative tasks, always run PowerShell as Administrator to avoid permission errors.

Basic Commands (Cmdlets)

PowerShell commands are called cmdlets (pronounced "command-lets"). They follow a consistent Verb-Noun naming convention. This makes it easy to guess command names — if you know the action and the target, you can often form the right command.

Tip: Use Get-Command to list all available cmdlets and Get-Help to learn how any command works. PowerShell even has Get-Help -Online to open the full Microsoft docs.

Navigation

Moving around the filesystem in PowerShell is similar to CMD, but with more power under the hood:

# Get current location (like pwd)
Get-Location

# Change to a directory
Set-Location C:\
Set-Location "C:\Program Files"

# Shortcuts
cd ..    # Go up one directory
cd ~    # Go to home directory
cd -    # Go to previous directory

# List contents
Get-ChildItem     # All items (ls/dir)
Get-ChildItem -Recurse # Recursive listing
Get-ChildItem *.txt  # Filter by extension
Get-ChildItem -Hidden # Show hidden files

Working with Files

PowerShell provides a complete set of file manipulation cmdlets. Most have familiar aliases:

# Creating items
New-Item file.txt -ItemType File    # Create a file
New-Item folder -ItemType Directory   # Create a folder (md)
New-Item "folder\sub" -ItemType Directory -Force # Create nested folders

# Deleting
Remove-Item file.txt          # Delete file (del)
Remove-Item folder -Recurse      # Delete folder recursively (rm -r)

# Copying & Moving
Copy-Item source.txt dest.txt      # Copy file (copy)
Copy-Item C:\source\* D:\dest\ -Recurse # Copy folder
Move-Item source.txt C:\dest\      # Move file (move)
Rename-Item old.txt new.txt       # Rename (ren)

# Reading & Writing Content
Get-Content file.txt          # Read file (type / cat)
Get-Content file.txt -Tail 10      # Last 10 lines (tail)
Set-Content file.txt "Hello World"   # Write text (echo >)
Add-Content file.txt "More text"    # Append text (echo >>)

Pipelines

Pipelines are where PowerShell truly shines. Instead of passing plain text, PowerShell pipes objects between commands. This means you can filter, sort, and transform data without messy string parsing.

# Find processes using more than 10% CPU
Get-Process | Where-Object CPU -gt 10

# Top 5 processes by memory
Get-Process | Sort-Object WorkingSet64 -Descending | Select-Object -First 5

# Show only running services
Get-Service | Where-Object Status -eq 'Running'

# List file names only
Get-ChildItem *.txt | ForEach-Object { $_.Name }

# Chain multiple operations
Get-Process | Where-Object CPU -gt 10 | Sort-Object CPU -Descending | Select-Object Name, CPU

Variables

Variables in PowerShell start with $. They can store any type of .NET object:

# Basic variables
$name = "John Doe"
$number = 42
$pi = 3.14159
$isAdmin = $true

# Arrays
$list = @(1, 2, 3, 4, 5)
$empty = @()
$mixed = @("text", 42, $true)

# Hashtables (dictionaries)
$config = @{
 Path = "C:\Temp"
 Recursive = $true
 Force = $false
}

# String interpolation
$greeting = "Hello, $name!"  # "Hello, John Doe!"
$calc = "2 + 2 = $($2 + 2)" # "2 + 2 = 4"

# List all variables
Get-Variable

# Check if variable exists
Test-Path variable:name

Scripting Basics

PowerShell scripts are saved with a .ps1 extension. They let you automate complex tasks by combining cmdlets, control flow, and logic.

Execution Policy: By default, Windows blocks script execution. Run this command once to allow local scripts: Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
# Comments start with #
# This is a comment, ignored by PowerShell

# Simple script structure
param(
 [string]$Name = "World"  # Parameter with default
)

Write-Host "Hello, $Name!" -ForegroundColor Green

# Conditionals
if ($Name -eq "Admin") {
 Write-Host "You have admin access" -ForegroundColor Cyan
} elseif ($Name -ne "") {
 Write-Host "Welcome, user" -ForegroundColor Yellow
} else {
 Write-Host "No name provided" -ForegroundColor Red
}

# Loops
foreach ($item in (Get-ChildItem *.txt)) {
 Write-Host "Processing: $($item.Name)"
}

for ($i = 0; $i -lt 10; $i++) {
 Write-Host "Count: $i"
}

# Error handling
try {
 Get-Content "nonexistent.txt" -ErrorAction Stop
} catch {
 Write-Host "File not found: $_" -ForegroundColor Red
}

Example Script: File Organizer

This script organizes your Downloads folder by file extension — automatically creates folders and moves files in:

# File Organizer — Sorts Downloads by extension
$path = "C:\Users\$env:USERNAME\Downloads"

# Check if path exists
if (!(Test-Path $path)) {
 Write-Host "Path not found: $path" -ForegroundColor Red
 exit 1
}

# Get all files with extensions
$files = Get-ChildItem $path | Where-Object { $_.Extension -ne "" }

foreach ($file in $files) {
 $ext = $_.Extension.TrimStart('.')
 $folder = Join-Path $path $ext

 # Create folder if it doesn't exist
 if (!(Test-Path $folder)) {
  New-Item $folder -ItemType Directory | Out-Null
  Write-Host "Created folder: $ext" -ForegroundColor Yellow
 }

 # Move file into folder
 Move-Item $_.FullName (Join-Path $folder $_.Name) -Force
 Write-Host "Moved: $($_.Name) → $ext\" -ForegroundColor Green
}

Write-Host "Done! Organized $($files.Count) files." -ForegroundColor Cyan

Example Script: System Info

A quick system information report using WMI/CIM queries:

# System Information Report
Write-Host "=== System Information ===" -ForegroundColor Cyan
Write-Host "Computer: $env:COMPUTERNAME"
Write-Host "User: $env:USERNAME"

# OS Info
$os = Get-CimInstance Win32_OperatingSystem
Write-Host "OS: $($os.Caption)"
Write-Host "Version: $($os.Version)"
Write-Host "Architecture: $($os.OSArchitecture)"
Write-Host "Last Boot: $($os.LastBootUpTime)"

# RAM
$cs = Get-CimInstance Win32_ComputerSystem
$ramGB = [math]::Round($cs.TotalPhysicalMemory / 1GB, 2)
Write-Host "RAM: $ramGB GB"

# CPU
$cpu = Get-CimInstance Win32_Processor
Write-Host "CPU: $($cpu.Name)"
Write-Host "Cores: $($cpu.NumberOfCores)"
Write-Host "Logical Processors: $($cpu.NumberOfLogicalProcessors)"

# Disk
$disk = Get-CimInstance Win32_LogicalDisk -Filter "DriveType=3"
foreach ($d in $disk) {
 $freeGB = [math]::Round($d.FreeSpace / 1GB, 2)
 $sizeGB = [math]::Round($d.Size / 1GB, 2)
 $pctFree = [math]::Round(($d.FreeSpace / $d.Size) * 100, 1)
 Write-Host "Disk $($d.DeviceID): $freeGB GB free of $sizeGB GB ($pctFree%)"
}

# Uptime
$uptime = (Get-Date) - $os.LastBootUpTime
Write-Host "Uptime: $($uptime.Days) days, $($uptime.Hours) hours"
Write-Host "============================" -ForegroundColor Cyan

Useful One-Liners

Handy one-liners for everyday system administration:

# Top 10 processes by RAM usage
Get-Process | Sort-Object WorkingSet64 -Descending | Select-Object -First 10 Name, WorkingSet64

# Remove all .tmp files recursively
Get-ChildItem -Recurse -Filter *.tmp | Remove-Item

# Start all stopped services
Get-Service | Where-Object Status -eq 'Stopped' | Start-Service

# Kill a process by name
Stop-Process -Name "notepad" -Force

# Find files larger than 100MB
Get-ChildItem -Recurse | Where-Object { $_.Length -gt 100MB }

# Get IP configuration (like ipconfig)
Get-NetIPAddress | Select-Object InterfaceAlias, IPAddress, PrefixLength

# Test network connectivity with continuous ping
ping google.com -t

# Export running processes to CSV
Get-Process | Export-Csv processes.csv -NoTypeInformation

# Monitor CPU usage (refreshes every 2 seconds)
while ($true) { Clear-Host; Get-Process | Sort-Object CPU -Descending | Select-Object -First 5 Name, CPU; Start-Sleep 2 }
