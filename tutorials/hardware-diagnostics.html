🔧 Hardware Diagnostics Guide

Why Hardware Diagnostics?

Hardware failures can strike without warning — corrupted files, random crashes, boot loops, or complete data loss. Running diagnostics before a catastrophic failure lets you identify failing components and replace them on your terms. The goal is proactive detection: catch a dying drive or faulty RAM stick while you can still back up your data.

This guide covers the essential tools and procedures for testing every major component in a PC.

Memory (RAM) Testing

Faulty RAM causes crashes, BSODs, file corruption, and unexplainable application errors. Because RAM errors can be intermittent, rigorous testing is essential.

Memtest86

Memtest86 is the gold standard for RAM testing. It runs outside the operating system, giving it full access to all memory without interference.

  • Preparation: Download the ISO from memtest86.com and write it to a USB drive using Rufus (Windows) or dd (Linux).
  • Booting: Reboot and select the USB drive from your boot menu (usually F12, Esc, or Del). Memtest86 loads automatically.
  • Running: The test runs in multiple passes (called "passes") that exercise every memory address with various patterns. Let it run for at least 1 full pass (30–60 minutes typically). For a thorough test, 4 passes or more is recommended — especially if you already suspect instability.
  • Interpreting results: The top-right corner shows pass count and error count. Any errors at all (even 1) indicate a problem. Possible causes:
    • Faulty RAM module — remove or replace it
    • Wrong slot configuration — check your motherboard manual for correct dual-channel population
    • Overclock/voltage too low — try resetting to JEDEC defaults in BIOS
    • Dirty contacts — remove RAM and carefully clean the gold contacts with a soft eraser, then reseat

Storage Testing

Drive failures are the most common cause of permanent data loss. Catching a failing drive early can save your files.

CrystalDiskInfo (Windows)

CrystalDiskInfo reads SMART data from your drives and gives a health status. The most critical SMART attributes to watch:

AttributeWhat It MeansWarning Signs
Reallocated SectorsCount of bad sectors remapped to spare areasAny non-zero value is suspicious; rapidly increasing values = drive is dying
Current Pending SectorSectors that are unstable and may become reallocatedNon-zero = the drive is struggling; back up immediately
Raw Read Error RateFrequency of read errorsHigh or increasing values indicate a failing head or platter surface
Reallocated Event CountNumber of reallocation eventsShould be zero; any increase means the spare pool is shrinking

GSmartControl (Linux)

GSmartControl is a graphical front-end for smartctl that provides the same SMART information as CrystalDiskInfo. Install with sudo apt install gsmartcontrol (Debian/Ubuntu) or your distro's package manager. It shows pass/fail status for each attribute and can run extended self-tests.

HDTune / HDD Low Level Format Tool (Windows)

HDTune performs benchmarks (read/write speed), error scans, and health monitoring. The Error Scan tab checks every sector — red blocks indicate bad sectors. HDD Low Level Format Tool (HDDLLF) zero-fills a drive, wiping all data and forcing the drive to reallocate bad sectors. Warning: this is destructive and should only be used on a drive you intend to completely wipe.

badblocks (Linux)

The badblocks utility scans a block device for defective sectors. A non-destructive read-write test is the most thorough:

# Non-destructive read-write test (preserves data)
sudo badblocks -svn /dev/sdX

# Destructive write test (wipes ALL data)
sudo badblocks -svw /dev/sdX

# Combine with fsck to mark bad sectors
sudo fsck -cc /dev/sdX1

The -v flag enables verbose output; -s shows progress. Be extremely careful with the device path — writing to the wrong disk is irreversible.

CPU / GPU Testing

CPU and GPU issues manifest as crashes under load, graphical artifacts, or thermal throttling. Use these tools to verify stability and performance.

CPU-Z (Windows)

CPU-Z is a lightweight tool that displays detailed information about your CPU: model, codename, core count, cache sizes, stepping, and supported instruction sets. It also shows motherboard model, BIOS version, chipset, and memory timings (DRAM frequency, CAS latency). Use it to verify that your hardware matches what you expect (e.g., after buying used parts).

GPU-Z (Windows)

GPU-Z provides the same level of detail for your graphics card: GPU name, die revision, memory type/size, driver version, BIOS version, and — crucially — the PCIe link speed under load. If your card is running at x8 3.0 when it should be x16 4.0, you may have a contact issue or a bandwidth bottleneck. The Sensors tab shows real-time temperature, core/memory clock, fan speed, and voltage.

HWInfo

HWInfo is the Swiss Army knife of system monitoring. Available for Windows (and Bootable USB via Hiren's), it shows everything: CPU/GPU temperatures, voltages, fan speeds, power draw, and sensor warnings. Run it in Sensors Only mode to see all real-time readings in one window. It can log to CSV for long-term monitoring.

Stress Testing: Prime95 & FurMark

Prime95 pushes your CPU to 100% by computing Mersenne primes. It generates immense heat — monitor temperatures with HWInfo while it runs. If your CPU exceeds 95°C or the system crashes, your cooling is inadequate or your CPU is unstable at its current voltage/frequency. FurMark does the same for GPUs — it renders a fur-covered torus at maximum detail. Watch for:

  • Temperature spikes above 85°C (GPU-dependent, check your model's limits)
  • Artifacts (glitches, colored squares, flickering)
  • Driver crashes or black screens
Warning: Stress tests generate extreme heat. Ensure your cooling system is clean and functional before running them. Stop immediately if temperatures approach dangerous levels. Never walk away from a stress test — monitor it continuously.

s-tui (Linux Terminal)

s-tui is a terminal-based CPU stress monitor that displays real-time graphs of CPU frequency, utilization, and temperature in your terminal. Install with sudo apt install s-tui or pip install s-tui. Run it and press s to start/stress the CPU using the built-in stress backend. It's lightweight, works over SSH, and gives you an instant snapshot of CPU health without a GUI.

Disk Health Thresholds — When to Replace

SMART AttributeHealthyWarningReplace Immediately
Reallocated Sectors01–50 (and stable)>50 or rapidly increasing
Current Pending Sectors01–10>10 or growing
Uncorrectable Sector Count01–5>5
Raw Read Error RateNear zeroElevated (varies by drive)Consistently high or climbing
Power-On Hours (HDD)<30,00030,000–60,000>60,000 (drives wear out)

For SSDs, also monitor Media Wearout Indicator and Percentage Used. When the latter hits 100%, the drive has exhausted its write endurance — replace it even if it still works.

Temperature Monitoring

Overheating causes thermal throttling, crashes, and permanent damage. Monitor your temps regularly.

Linux — sensors Command

The lm-sensors package provides the sensors command-line tool:

# Install
sudo apt install lm-sensors

# Detect sensors (run once)
sudo sensors-detect

# View temperatures, voltages, fan speeds
sensors

Typical output shows CPU package temp, core temps, motherboard temp, GPU temp (if supported), and fan RPMs. For continuous monitoring, use watch sensors (updates every 2 seconds).

Windows — HWInfo

HWInfo provides the most comprehensive temperature monitoring on Windows. Run it in Sensors Only mode and look for:

  • CPU (Tctl/Tdie) — Ryzen; CPU Package — Intel
  • GPU Temperature — most modern GPUs
  • Hot Spot / Memory Junction — important for AMD GPUs and RTX 30-series+
  • VRM temperature — motherboard voltage regulators
  • NVMe/SATA temperature — drives can throttle above 70°C

When your operating system won't boot, you need a bootable environment to run diagnostics from. These are essential tools for any repair kit:

Pro tip: Keep a USB drive with Hiren's Boot CD PE and another with Memtest86. Between these two, you can diagnose 90% of hardware failures without reinstalling an OS.

Quick Diagnostic Flowchart

  1. Computer crashes / freezes? → Run Memtest86 first (RAM errors are the #1 cause of random crashes).
  2. Files corrupting or drive errors? → Check SMART data with CrystalDiskInfo / GSmartControl.
  3. Random shutdowns under load? → Monitor temperatures with HWInfo or sensors. Check fans and thermal paste.
  4. Graphics artifacts or game crashes? → Run FurMark + check GPU-Z sensors for hotspot temps.
  5. System won't boot at all? → Boot from Hiren's Boot CD PE or SystemRescue and test from there.
Rule of thumb: If a component fails any diagnostic test even once, replace it. Intermittent failures become permanent failures — it's always a matter of time. Back up your data before running destructive tests like badblocks -w or HDDLLF.
