SNMP Overview & Versions

Simple Network Management Protocol (SNMP) is the standard protocol for monitoring and managing network devices. It collects data from routers, switches, servers, printers, and other IP-enabled devices. SNMP uses a manager-agent model where agents expose data through MIBs (Management Information Bases).

Three major versions exist:

  • SNMPv1 -- basic read/write, plaintext community strings (obsolete)
  • SNMPv2c -- improved protocol, bulk requests, still plaintext community
  • SNMPv3 -- authenticated and encrypted communication (recommended)

Install SNMP tools on Linux:

sudo apt install snmp snmpd snmp-mibs-downloader

MIBs, OIDs & SNMP Walk

MIBs define the structure of SNMP data. OIDs (Object Identifiers) uniquely identify managed objects:

# Common OIDs (from 1.3.6.1.2.1 - SNMP MIB-2 internet)
# .1.3.6.1.2.1.1      -- System group
# .1.3.6.1.2.1.2      -- Interfaces group
# .1.3.6.1.2.1.4      -- IP group
# .1.3.6.1.2.1.6      -- TCP group
# .1.3.6.1.2.1.25     -- Host Resources

# snmpwalk -- retrieve entire OID tree
snmpwalk -v2c -c public 192.168.1.1 .1.3.6.1.2.1

# Query specific OID
snmpget -v2c -c public 192.168.1.1 .1.3.6.1.2.1.1.5.0  # sysName
snmpget -v2c -c public 192.168.1.1 .1.3.6.1.2.1.2.1.0  # Number of interfaces

# snmpbulkwalk -- faster for large MIBs
snmpbulkwalk -v2c -c public 192.168.1.1 .1.3.6.1.2.1.25.2.3.1.3  # Storage types

Configuring SNMPv3 (Authenticated & Encrypted)

SNMPv3 provides robust security with authentication (SHA) and encryption (AES):

# Linux snmpd.conf (/etc/snmp/snmpd.conf)
# Create SNMPv3 user
createUser monitor SHA "authPassword123" AES "encryptPassword456"
rouser monitor authpriv

# Restrict to specific subnet
agentAddress udp:161,udp6:[::1]:161

# Cisco SNMPv3 configuration
snmp-server group MONITORS v3 priv
snmp-server user monitor MONITORS v3 auth sha authPassword123 priv aes 128 encryptPassword456
snmp-server host 192.168.1.100 version 3 priv monitor

# Query SNMPv3 device
snmpwalk -v3 -u monitor -l authPriv \
    -a SHA -A "authPassword123" \
    -x AES -X "encryptPassword456" \
    192.168.1.1 .1.3.6.1.2.1.1

SNMP Traps & Notifications

SNMP traps are unsolicited messages from agents to managers:

# snmptrapd configuration (/etc/snmp/snmptrapd.conf)
authCommunity log,execute,net public
traphandle default /usr/bin/traptoemail --admin@example.com

# Cisco trap configuration
snmp-server enable traps
snmp-server enable traps snmp authentication linkdown linkup
snmp-server enable traps config
snmp-server host 192.168.1.100 version 2c public udp-port 162
snmp-server host 192.168.1.100 version 3 priv monitor

# Test trap generation
snmptrap -v2c -c public 192.168.1.100:162 '' \
    .1.3.6.1.4.1.99999.1 .1.3.6.1.4.1.99999.1.1 s "Test Trap"

# Automation with trap receivers: Nagios, Zabbix, Prometheus (snmp_exporter)

Monitoring Tools & Best Practices

Integrate SNMP with popular monitoring platforms:

# Prometheus snmp_exporter
# snmp.yml generator:
generate: {\n  - module: if_mib\n    walk: [1.3.6.1.2.1.2, 1.3.6.1.2.1.31.1.1]\n    max_repetitions: 25\n\n# librenms -- full SNMP discovery and monitoring
# docker run -d --name librenms -p 80:80 librenms/librenms:latest

# Zabbix SNMP template
zabbix_get -s 192.168.1.1 -k "snmpwalk[.1.3.6.1.2.1.1.5.0]"

# Best practices:
# - Use SNMPv3 with authPriv security level
# - Limit SNMP access to monitoring servers only
# - Set appropriate community strings (not 'public' in production)
# - Tune polling intervals (30s for critical, 5m for standard)
# - Monitor both interface utilization and errors
