Ruby Philosophy & Setup

Ruby is a dynamic, object-oriented language designed by Yukihiro Matsumoto with a focus on developer happiness and readability. Everything in Ruby is an object, including numbers and classes.

Install Ruby via a version manager:

# Install rbenv
git clone https://github.com/rbenv/rbenv.git ~/.rbenv
~/.rbenv/bin/rbenv init

# Install Ruby
rbenv install 3.3.0
rbenv global 3.3.0

# Or use RVM
curl -sSL https://get.rvm.io | bash
rvm install 3.3.0

ruby --version

Elegant Syntax & Blocks

Ruby's syntax prioritizes natural language readability. Blocks are first-class code chunks:

# Everything is an object
5.times { print "Hi! " }

# Blocks with do/end
[1, 2, 3].map do |n|
  n * 2
end
# => [2, 4, 6]

# Symbol shorthand
users.map(&:name)  # Same as users.map { |u| u.name }

# Implicit returns -- last expression is returned
def square(x)
  x * x  # No return keyword needed
end

Blocks can capture local variables (closures) and are used extensively for iteration, callbacks, and DSLs.

Metaprogramming & Dynamic Features

Ruby allows modifying classes and objects at runtime -- this is called metaprogramming:

# Open classes -- add methods to existing classes
class String
  def shout
    upcase + "!"
  end
end

"hello".shout  # "HELLO!"

# method_missing -- catch undefined method calls
class DynamicProxy
  def method_missing(name, *args, &block)
    puts "Called #{name} with #{args}"
  end
end

# define_method -- create methods dynamically
%w[get post put delete].each do |verb|
  define_method(verb) do |path, &block|
    puts "#{verb.upcase} #{path}"
  end
end

Ruby on Rails -- MVC Framework

Ruby on Rails is the flagship Ruby web framework, embracing Convention over Configuration:

# Install Rails
gem install rails
rails new my_app
cd my_app
bin/rails server

# Generate scaffold (full CRUD)
bin/rails generate scaffold Post title:string body:text
bin/rails db:migrate

# Routes
# config/routes.rb
Rails.application.routes.draw do
  resources :posts
  root 'posts#index'
end

# Model with validations
class Post < ApplicationRecord
  validates :title, presence: true, length: { minimum: 5 }
  has_many :comments, dependent: :destroy
end

Testing with RSpec & Best Practices

RSpec is the dominant testing framework in Ruby:

# Gemfile
group :test do
  gem 'rspec-rails'
  gem 'factory_bot_rails'
end

# spec/models/post_spec.rb
RSpec.describe Post, type: :model do
  it 'is valid with valid attributes' do
    post = build(:post, title: 'My Post')
    expect(post).to be_valid
  end
  
  it 'is invalid without a title' do
    post = build(:post, title: nil)
    expect(post).not_to be_valid
    expect(post.errors[:title]).to include("can't be blank")
  end
end

Other testing tools: Minitest (built-in), Capybara (integration tests), FactoryBot (test data).
