Windows Performance Monitor

Data collector sets, performance counters, reports, and troubleshooting with PerfMon.

Performance Monitor Overview

Performance Monitor (PerfMon) is a Windows tool that collects and displays real-time performance data. It helps identify bottlenecks in CPU, memory, disk, network, and application performance. PerfMon can log data over time, alert on thresholds, and generate HTML reports.

Launch PerfMon via:

perfmon.msc
perfmon /rel   # Reliability Monitor
perfmon /sys   # System Summary

Performance Monitor works with Performance Counters -- numeric metrics published by the OS, drivers, and applications. Thousands of counters exist for processors, memory, disks, network adapters, processes, and .NET runtime.

Working with Performance Counters

Add and analyze performance counters in real-time:

# In PerfMon GUI:
# 1. Expand Monitoring Tools > Performance Monitor
# 2. Click the green + button
# 3. Select counters (use Ctrl+click for multiple)

# Essential counters to monitor:
# \Memory\Pages/sec          -- Memory pressure
# \Processor(_Total)\% Processor Time  -- CPU usage
# \PhysicalDisk(_Total)\% Disk Time     -- Disk activity
# \Network Interface(*)\Bytes Total/sec -- Network throughput

# Logman command-line (create data collector)
logman create counter system_monitor \
    -c "\Processor(_Total)\% Processor Time" \
       "\Memory\Available MBytes" \
       "\PhysicalDisk(_Total)\% Disk Time" \
    -o "C:\PerfLogs\system_monitor.blg" \
    -f bincirc -max 500 -si 00:00:15

Data Collector Sets & Logging

Data Collector Sets (DCS) are groups of counters, configuration, and alerts that run on a schedule:

# Create a DCS via GUI:
# 1. Expand Data Collector Sets > User Defined
# 2. Right-click > New > Data Collector Set
# 3. Choose "Create from a template" or "Create manually"

# Create via PowerShell
$dcs = New-Object -COM Pla.DataCollectorSet
$dcs.DisplayName = "My Custom Monitor"
$dcs.Duration = 86400  # 24 hours
$dcs.SubdirectoryFormat = 1  # Day-monotonic
$dcs.SubdirectoryFormatPattern = "yyyy-MM-dd"
$dcs.Commit("My Custom Monitor", $null, 3)  # 3 = add/modify

# System Collector Sets (built-in):
# - System Diagnostics: 60-second high-resolution snapshot
# - System Performance: Performance analysis with report
# - Active Directory Diagnostics: Domain controller health

Analyzing Performance Reports

PerfMon generates detailed HTML reports for data collector sets:

# Generate report from a .blg file
relog.exe C:\PerfLogs\system_monitor.blg -f CSV -o report.csv

# Use PowerShell to analyze counter data
$counters = Get-Counter -Counter @(
    "\Processor(_Total)\% Processor Time",
    "\Memory\Available MBytes"
) -SampleInterval 5 -MaxSamples 60

$counters.CounterSamples | ForEach-Object {
    [PSCustomObject]@{
        TimeStamp = $_.TimeStamp
        CPU = ($_.CookedValue | Where Path -like "*Processor*").CookedValue
        Memory = ($_.CookedValue | Where Path -like "*Memory*").CookedValue
    }
}

# View built-in report
# PerfMon > Reports > User Defined > select your DCS > right-click > View Report

Alerts & Troubleshooting Scenarios

Set up alerts for proactive monitoring:

# Create alert via GUI:
# Data Collector Sets > User Defined > New > Data Collector Set
# Choose "Performance Counter Alert"

# Configure alert conditions:
# - \Processor(_Total)\% Processor Time > 90% for 10 seconds
# - \LogicalDisk(C:)\% Free Space < 10%
# - \Memory\Available MBytes < 512

# Action: Log an entry in the application event log
# Or run a script:
# powershell.exe -Command "Send-MailMessage -To admin@example.com ..."

# Common troubleshooting with PerfMon:
# High CPU: Identify which process in Process\% Processor Time
# Memory leak: Monitor Process\Private Bytes over time
# Disk bottleneck: Check PhysicalDisk\Avg. Disk Queue Length > 2
