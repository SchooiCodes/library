What is Proxmox VE?

Proxmox Virtual Environment (VE) is an open-source virtualization platform that combines KVM for virtual machines and LXC for containers in a single web interface. It's the go-to choice for homelabs and small-to-medium deployments, offering enterprise features without the licensing costs of VMware or Hyper-V.

Key features: Web-based management UI, ZFS filesystem support, Built-in backups with vzdump, Live migration, High availability clustering, Container and VM templates, Role-based access control, REST API for automation.

Installation

Download the Proxmox VE ISO from proxmox.com/downloads. Write it to USB with Ventoy or dd.

# Write ISO to USB
sudo dd if=proxmox-ve_8.2.iso of=/dev/sdX bs=1M status=progress

# Minimum requirements:
# - 64-bit CPU with Intel VT-x/AMD-V
# - 2GB RAM (4GB+ recommended)
# - 16GB disk (SSD recommended)
# - Static IP address

# During install:
# - Select filesystem: ZFS (RAID1 for production) or ext4/XFS
# - Set root password and email
# - Configure network (static IP, gateway, DNS)
# - Hostname must be fully qualified (e.g., pve1.home.local)

After installation, access the web UI at https://YOUR_IP:8006. Login as root with the password set during installation.

Post-Install Configuration

Essential steps after first boot:

# Remove subscription nag (for homelab use)
sed -i "s/if (data.status !== \'Active\')/if (false)/" /usr/share/javascript/proxmox-widget-toolkit/proxmoxlib.js
systemctl restart pveproxy

# Add the community repository
nano /etc/apt/sources.list.d/ceph.list
# Comment out enterprise repo, add:
deb http://download.proxmox.com/debian/ceph-quincy bookworm no-subscription

# Update system
apt update && apt full-upgrade -y

# Enable IOMMU for PCI passthrough
nano /etc/default/grub
# GRUB_CMDLINE_LINUX_DEFAULT="quiet intel_iommu=on"
update-grub

# Install useful tools
apt install -y htop iotop net-tools bridge-utils

Creating Virtual Machines

Proxmox supports both VMs (full virtualization) and containers (lightweight OS-level virtualization).

# Download an ISO via CLI
cd /var/lib/vz/template/iso
wget https://releases.ubuntu.com/24.04/ubuntu-24.04.1-live-server-amd64.iso

# Create a VM via CLI (qm)
qm create 100 --name "ubuntu-server" --memory 2048 --cores 2
qm set 100 --net0 virtio,bridge=vmbr0
qm set 100 --scsihw virtio-scsi-pci --scsi0 local-lvm:32
qm set 100 --ide2 local:iso/ubuntu-24.04.1-live-server-amd64.iso,media=cdrom
qm set 100 --boot order=ide2
qm start 100

# Create an LXC container (pct)
pveam update
pveam available | grep ubuntu
pveam download local ubuntu-24.04-standard_24.04-2_amd64.tar.zst
pct create 200 local:vztmpl/ubuntu-24.04-standard_24.04-2_amd64.tar.zst \
  --storage local-lvm --memory 1024 --cores 1 \
  --net0 name=eth0,bridge=vmbr0,ip=dhcp \
  --hostname ubuntu-container

Backups & Snapshots

Proxmox has built-in backup capabilities via vzdump and the web UI.

# Manual backup of a VM
vzdump 100 --dumpdir /var/lib/vz/dump --compress zstd

# Snapshot (quick, but no separate file)
qm snapshot 100 pre-upgrade --description "Before kernel update"

# Rollback to snapshot
qm rollback 100 pre-upgrade

# Automated backup schedule
# Datacenter → Backup → Add
# Select: VMs to backup, storage location, schedule, retention

Best practices: Store backups on a separate NAS or NFS share, Use zstd compression (fast + good ratio), Set retention policies (e.g., 7 daily, 4 weekly, 3 monthly), Test restores regularly.

Networking & Bridge Setup

Proxmox uses Linux bridges to connect VMs to the network. The default vmbr0 bridge connects to your physical NIC.

# /etc/network/interfaces - Linux Bridge
auto lo
iface lo inet loopback

auto eno1
iface eno1 inet manual

auto vmbr0
iface vmbr0 inet static
  address 192.168.1.100/24
  gateway 192.168.1.1
  bridge-ports eno1
  bridge-stp off
  bridge-fd 0

# VLAN-aware bridge (for VLAN tagging)
auto vmbr1
iface vmbr1 inet manual
  bridge-ports eno2
  bridge-stp off
  bridge-fd 0
  bridge-vlan-aware yes
  bridge-vids 2-4094
