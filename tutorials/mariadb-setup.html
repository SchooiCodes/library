What is MariaDB?

MariaDB is a community-developed fork of MySQL, created by the original MySQL developers after Oracle's acquisition. It's a drop-in replacement for MySQL with additional storage engines, performance improvements, and better licensing. Most MySQL commands, tools, and connectors work with MariaDB without modification.

Choose MariaDB for: Better performance with more concurrent connections, Extra storage engines (Aria, ColumnStore, Spider), Built-in Galera cluster support, More frequent security updates, Open-source governance.

Installation

MariaDB is available on all major Linux distributions, macOS, and Windows.

# Debian/Ubuntu
sudo apt update && sudo apt install mariadb-server mariadb-client

# Rocky/Alma/RHEL 9
sudo dnf install mariadb-server mariadb

# Arch Linux
sudo pacman -S mariadb
sudo mysql_install_db --user=mysql --basedir=/usr --datadir=/var/lib/mysql

# Docker
sudo docker run -d --name mariadb \
 -e MYSQL_ROOT_PASSWORD=strongpassword \
 -e MYSQL_DATABASE=myapp \
 -e MYSQL_USER=appuser \
 -e MYSQL_PASSWORD=userpass \
 -p 3306:3306 \
 -v mariadb_data:/var/lib/mysql \
 mariadb:latest

Security Hardening

Run the security script immediately after installation to set a root password, remove anonymous users, disable remote root login, and remove test databases.

sudo mysql_secure_installation

# Manual security steps:
sudo mysql -u root

-- Set root password
ALTER USER 'root'@'localhost' IDENTIFIED BY 'StrongPass123!';

-- Remove anonymous users
DELETE FROM mysql.user WHERE User='';

-- Disable remote root login
DELETE FROM mysql.user WHERE User='root' AND Host NOT IN ('localhost', '127.0.0.1', '::1');

-- Remove test database
DROP DATABASE IF EXISTS test;
DELETE FROM mysql.db WHERE Db='test' OR Db='test\_%';

-- Reload privileges
FLUSH PRIVILEGES;
Security note: Never expose port 3306 directly to the internet. Use a VPN, SSH tunnel, or reverse proxy with authentication for remote access.

Creating Databases & Users

-- Connect as root
sudo mysql -u root -p

-- Create a database
CREATE DATABASE myapp CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- Create a user and grant permissions
CREATE USER 'appuser'@'localhost' IDENTIFIED BY 'StrongPass123!';
GRANT ALL PRIVILEGES ON myapp.* TO 'appuser'@'localhost';

-- User for web application (from any host)
CREATE USER 'webapp'@'192.168.1.%' IDENTIFIED BY 'WebAppPass456!';
GRANT SELECT, INSERT, UPDATE, DELETE ON myapp.* TO 'webapp'@'192.168.1.%';

-- Apply changes
FLUSH PRIVILEGES;

Best practices: Use localhost for apps on the same machine, Use CIDR notation for network access, Grant minimum required privileges, Use UTF8MB4 for full Unicode support (emojis, special chars).

Backup & Restore

# Full database backup
mysqldump -u root -p --all-databases > full_backup.sql

# Single database backup
mysqldump -u root -p myapp > myapp_backup.sql

# Backup with compression
mysqldump -u root -p myapp | gzip > myapp_$(date +%F).sql.gz

# Backup specific tables
mysqldump -u root -p myapp users orders > partial_backup.sql

# Restore
mysql -u root -p myapp < myapp_backup.sql

# Restore compressed backup
gunzip < myapp_2026-01-01.sql.gz | mysql -u root -p myapp

Automation script: Use cron to run nightly backups with automatic retention:

#!/bin/bash
BACKUP_DIR="/var/backups/mysql"
mkdir -p $BACKUP_DIR
mysqldump --all-databases --single-transaction --routines --triggers | gzip > $BACKUP_DIR/db_$(date +%Y%m%d_%H%M%S).sql.gz
# Keep only last 7 days
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

Performance Tuning

MariaDB's performance heavily depends on your server's resources and workload. Tune these key settings in /etc/mysql/mariadb.conf.d/50-server.cnf:

[mysqld]
# InnoDB settings
innodb_buffer_pool_size = 1G    # 70% of available RAM for dedicated DB servers
innodb_log_file_size = 256M     # Larger = better write performance
innodb_flush_log_at_trx_commit = 2 # 1=ACID (slow), 2=performance (1s data loss)
innodb_file_per_table = 1      # Each table has its own file

# Connection settings
max_connections = 150        # Increase for busy apps
thread_cache_size = 8        # Reduce thread creation overhead

# Query cache (disable in MariaDB 10.6+)
query_cache_type = 0
query_cache_size = 0

# Temporary tables
tmp_table_size = 64M
max_heap_table_size = 64M

# Slow query log
slow_query_log = 1
slow_query_log_file = /var/log/mysql/slow.log
long_query_time = 2

Replication & High Availability

Master-slave replication provides read scaling and failover capability.

# On Master server
sudo nano /etc/mysql/mariadb.conf.d/50-server.cnf
[mysqld]
server-id = 1
log_bin = /var/log/mysql/mysql-bin.log
binlog_do_db = myapp

# Create replication user
CREATE USER 'replicator'@'%' IDENTIFIED BY 'ReplPass789!';
GRANT REPLICATION SLAVE ON *.* TO 'replicator'@'%';

# On Slave server
sudo nano /etc/mysql/mariadb.conf.d/50-server.cnf
[mysqld]
server-id = 2
relay-log = /var/log/mysql/mysql-relay-bin.log

# Connect slave to master
CHANGE MASTER TO
 MASTER_HOST='192.168.1.100',
 MASTER_USER='replicator',
 MASTER_PASSWORD='ReplPass789!',
 MASTER_LOG_FILE='mysql-bin.000001',
 MASTER_LOG_POS=0;
START SLAVE;
SHOW SLAVE STATUS\G
