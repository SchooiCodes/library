HTTP & HTTPS

HTTP/1.1, HTTP/2, HTTP/3 — methods, headers, caching, TLS, and security.

Table of Contents (2 sections)

  • HTTP Protocol & Versions
  • Caching & TLS

HTTP Protocol & Versions

HTTP: request-response protocol. Methods: GET (read), POST (create), PUT (replace), PATCH (partial update), DELETE, HEAD, OPTIONS, CONNECT. Status codes: 1xx (informational — 101 Switching Protocols), 2xx (success — 200 OK, 201 Created, 204 No Content), 3xx (redirect — 301 Moved Permanently, 302 Found, 304 Not Modified), 4xx (client error — 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 429 Too Many Requests), 5xx (server error — 500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable). HTTP/2: multiplexed streams (one TCP connection), header compression (HPACK), server push, binary framing layer. HTTP/3: QUIC (UDP-based), 0-RTT, connection migration, no head-of-line blocking.

Caching & TLS

Caching headers: Cache-Control (max-age, no-cache, no-store, public, private, immutable, stale-while-revalidate), ETag (entity tag — conditional requests with If-None-Match), Last-Modified (If-Modified-Since), Expires (deprecated in favor of Cache-Control). Cookies: Set-Cookie (name=value; Domain; Path; Secure; HttpOnly; SameSite=Strict/Lax/None). CORS: Access-Control-Allow-Origin, preflight OPTIONS request. TLS: handshake (ClientHello > ServerHello > Certificate > Key Exchange > Finished), cipher suites (TLS_AES_128_GCM_SHA256), TLS 1.3 (1-RTT handshake, 0-RTT resumption, no RC4/3DES/RC5, deprecated RSA key exchange). HSTS: Strict-Transport-Security header. mTLS: mutual TLS — client also presents certificate.
