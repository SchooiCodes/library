Cryptography Fundamentals

Cryptography — symmetric, asymmetric, hashing, digital signatures, and PKI.

Table of Contents (2 sections)

  • Symmetric & Asymmetric
  • Hashing & Signatures

Symmetric & Asymmetric

Symmetric encryption: same key for encrypt/decrypt. AES (Advanced Encryption Standard) — 128/192/256 bit keys, modes (ECB — not safe, CBC, GCM — authenticated encryption, CTR, CCM). GCM: AES-GCM recommended (encryption + authentication tag). ChaCha20-Poly1305: modern stream cipher (mobile-friendly). Key exchange problem: how to share the secret key? Asymmetric (public-key) cryptography: public key (encrypt, verify) + private key (decrypt, sign). RSA: factoring large numbers, key sizes 2048/4096 bits, slow, use for key exchange or signing. ECC (Elliptic Curve Cryptography): smaller keys for same security (256-bit ECC = 3072-bit RSA). ECDH: Elliptic Curve Diffie-Hellman — key agreement. Ed25519: modern EdDSA signature scheme (fast, secure). Hybrid encryption: ECDH key exchange + AES-GCM for data (TLS does this). Key generation: random enough entropy, secure random generator /dev/urandom. Key storage: HSM (hardware security module), TPM, secure enclave, secrets manager. Key rotation: regular rotation, versioned keys (KMS). Post-quantum: algorithms (CRYSTALS-Kyber, Dilithium), NIST standardization. TLS: hybrid key exchange (ECDH + Kyber).

Hashing & Signatures

Cryptographic hash: one-way, collision-resistant, deterministic. SHA-2: SHA-256, SHA-512 (secure). SHA-3: Keccak-based. BLAKE2/BLAKE3: faster than SHA-2. HMAC: keyed-hash message authentication (verify integrity + authenticity). PBKDF2, bcrypt, scrypt, Argon2: key derivation functions (password hashing) — slow, salt, memory-hard (Argon2id recommended). Use: password storage (Argon2id, bcrypt), data integrity checks, file deduplication. Digital signatures: sign with private key, verify with public key. RSA signature: RSASSA-PKCS1-v1_5 (legacy), RSASSA-PSS (probabilistic). ECDSA: Elliptic Curve Digital Signature Algorithm (used by Bitcoin, TLS). EdDSA: Ed25519 (modern, deterministic). Use: code signing, document signing, email (S/MIME), blockchain identities. PKI (Public Key Infrastructure): Certificate Authority (CA), intermediate CAs, end-entity certificates. X.509 certificates: subject, issuer, validity, public key, extensions (SAN, key usage, extended key usage). Certificate chain: root CA -> intermediate CA -> server cert. TLS/SSL: server certificate authentication. Mutual TLS (mTLS): both sides present certificates. ACME protocol (Let's Encrypt): automated certificate issuance. Certificate pinning: constraint-check cert (risky). Certificate transparency: public logs of issued certs. CRL/OCSP: certificate revocation checking. GPG: OpenPGP for email/file encryption and signing. Web of trust: decentralized trust model (GPG).
