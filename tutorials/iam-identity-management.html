IAM & Identity Management

Identity and access management — SSO, SAML, OAuth 2.0, OIDC, and just-in-time access.

Table of Contents (2 sections)

  • IAM Fundamentals
  • Just-in-Time Access

IAM Fundamentals

IAM: who can do what on which resource. Authentication (who you are) vs Authorization (what you can do). Federation: SSO across orgs — SAML 2.0 (XML-based, enterprise) vs OAuth 2.0 + OIDC (token-based, modern apps). SAML flow: IdP (identity provider) sends signed assertion to SP (service provider). OIDC: ID token (JWT) + access token + refresh token. /authorize → /token → /userinfo endpoints.

Just-in-Time Access

JIT: temporary privilege elevation on demand. Okta: JIT group membership, approval workflows. Azure AD PIM: Privileged Identity Management — activation request, timebound, approval, audit. AWS IAM: role-based, Security Token Service (STS) for temporary credentials. Google Cloud: Privileged Access Manager (PAM), Temporal Access for service accounts. Tools: Teleport (SSO + SSH/K8s proxy), Boundary (HashiCorp session management).
