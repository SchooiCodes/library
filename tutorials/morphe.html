File Transfer

Morphe

P2P encrypted file transfer tool for secure, high-speed, peer-to-peer sharing with no central server.

Overview

Morphe is a peer-to-peer (P2P) encrypted file transfer tool that eliminates the need for a central relay server. It uses WebRTC for browser-to-browser transfers or a custom P2P protocol for CLI-to-CLI communication. Files are transferred directly between devices after a lightweight signalling handshake takes place over a public rendezvous server.

Key features include end-to-end encryption (E2EE) using NaCl/libsodium, support for multi-user "rooms," resumable transfers, and NAT traversal via STUN/TURN. Morphe works on LAN, across the internet, and through most firewalls.

Installation

Go Install

go install github.com/morphe-org/morphe@latest

Prebuilt Binary

# Linux x86_64
curl -LO https://github.com/morphe-org/morphe/releases/latest/download/morphe-linux-amd64.tar.gz
tar -xzf morphe-linux-amd64.tar.gz
sudo mv morphe /usr/local/bin/

Build from Source

git clone https://github.com/morphe-org/morphe.git
cd morphe
make build
sudo mv build/morphe /usr/local/bin/

Verify installation:

morphe --version

How P2P Transfer Works

Morphe uses a three-step handshake to establish a direct encrypted channel:

  1. Signalling — The sender connects to a public rendezvous server and creates a "room." The server returns a room code (e.g. purple-axolotl-42).
  2. NAT Traversal — Both peers exchange SDP offers/answers via the rendezvous server. STUN determines public IPs; TURN is used as fallback when symmetric NAT blocks direct connections.
  3. Direct P2P — Once a data channel opens, the file transfer begins. The rendezvous server is no longer involved; data flows directly peer-to-peer.

This architecture means Morphe can transfer files of any size with no bandwidth limits, no file size caps, and no intermediate storage.

Diagram of the Flow

Sender                    Rendezvous Server            Receiver
  |                              |                          |
  |--- create room ------------->|                          |
  |<--- room code (xyz) ---------|                          |
  |                              |                          |
  |                              |<--- join room (xyz) ----|
  |<--- SDP offer ------------ SDP relay --- SDP offer --->|
  |--- SDP answer ----------> SDP relay --- SDP answer --->|
  |                              |                          |
  |========= encrypted P2P channel ========================>|
  |=== file data flows directly, no server involvement ====>|

Sending Files

Basic Send

morphe send ~/Documents/report.pdf

This command creates a room and returns a one-time code. Share the code with the recipient over a side channel (e.g. Signal, Matrix, text message).

Send with Custom Room Name

morphe send --room my-secure-room ~/Documents/report.pdf

Send to Multiple Recipients

morphe send --room team-share ~/Documents/project.zip --peers 5

Up to --peers simultaneous connections are allowed (default 2).

Send Text (stdin)

cat secret.txt | morphe send --stdin

Send Directory

morphe send --recursive ~/Photos/Vacation2026/

Directories are automatically archived as a tar.gz before transfer.

Command Options

Receiving Files

Basic Receive

morphe receive

You will be prompted to enter the room code. Paste the code shared by the sender.

Direct Receive with Code

morphe receive purple-axolotl-42

Receive to Specific Directory

morphe receive --output ~/Downloads/incoming purple-axolotl-42

Receive from Specific Room

morphe receive --room my-secure-room

Wait for Multiple Files

morphe receive --wait

The --wait flag keeps the room open so the sender can send additional files without creating a new room.

Encryption Details

Morphe uses the NaCl cryptographic library (libsodium) for all encryption operations:

  • Key Exchange: X25519 Diffie-Hellman
  • Symmetric Encryption: XSalsa20-Poly1305 (authenticated encryption)
  • Hashing: BLAKE2b for integrity checks
  • Key Derivation: Argon2id (t=2, m=64MB, p=4) for password-based room keys

Each transfer generates a fresh ephemeral key pair. The session key is derived from the room code combined with the ephemeral keys, ensuring forward secrecy. If a room code is leaked, past transfers remain secure.

Note: Because encryption is end-to-end, the rendezvous server never sees file contents, filenames, or metadata. Only hashed room IDs and ICE candidates pass through the signalling server.

Advanced Options

Custom Relay (TURN)

On restrictive networks that block UDP, configure a TURN server:

morphe send --relay turn:turn.example.com:3478 \
  --relay-username user --relay-password pass ~/file.zip

Speed Optimization

Enable compression for text-heavy files:

morphe send --compress ~/Documents/codebase.tar.gz

Increase the UDP buffer size for better throughput on high-latency links:

export MORPHE_UDP_BUFFER=16777216
morphe send ~/bigfile.iso

Run Your Own Rendezvous Server

git clone https://github.com/morphe-org/morphe-rendezvous.git
cd morphe-rendezvous
go run . --port 9090 --domain example.com

# Then point clients to your server:
export MORPHE_RENDEZVOUS=wss://rendezvous.example.com

Troubleshooting

IssueSolution
Connection fails — "no ICE candidates"Your network may block UDP. Use a TURN relay (--relay). Try a different STUN server.
Room not foundRoom codes expire after 5 minutes by default. Re-send with --room for persistent room.
Slow transfer speedEnable --compress for text files. Increase MORPHE_UDP_BUFFER. Check if you're falling back to TURN (add latency).
File corrupted after transferMorphe verifies BLAKE2b hashes automatically. If mismatch, re-transfer. Check disk space on receiver.
Firewall blocking WebSocketEnsure outbound TCP port 443 (WSS) is open. The signalling handshake uses WSS, then data uses UDP.
Multiple receivers not connectingIncrease --peers value on sender. Each receiver must connect within the timeout window (default 2 min).
