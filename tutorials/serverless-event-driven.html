Serverless & Event-Driven

Serverless architecture -- Lambda, event-driven patterns, and best practices.

Table of Contents (2 sections)

  • Lambda & Event Sources
  • Event-Driven Patterns

Lambda & Event Sources

AWS Lambda: function-as-a-service. Triggers: S3 (object created), API Gateway (HTTP/REST), DynamoDB Streams, Kinesis, SQS, SNS, EventBridge, Cognito. Integration: directly invoke (sync), event source mapping (async poll). Event source mapping: batches, partial batch response, filter criteria, destinations (DLQ, success). Lambda@Edge: run at CloudFront edge. Lambda layers: shared dependencies. Lambda SnapStart: faster cold starts (Java). Provisioned concurrency: pre-warmed environments. Reserved concurrency: guaranteed capacity. Runtimes: Node, Python, Java, Go, .NET, Ruby, custom (Rust via provided.al2). Limits: 15 min timeout, 10GB memory, 512MB /tmp, 6MB request/response. Observability: CloudWatch Logs, X-Ray tracing (distributed tracing with segments/subsegments). Powertools: Lambda best practices libraries.

Event-Driven Patterns

EventBridge: event bus -- schemas, rules, targets (Lambda, SQS, SNS, Step Functions). Events: AWS services, custom apps, SaaS partners (Shopify, Datadog). EventBridge Pipes: point-to-point integration (source -> filter/enrich -> target, with optional DLQ and enrichment Lambda). SQS: queues -- standard (at-least-once, high throughput), FIFO (exactly-once, 300 txn/s). Dead-letter queues: failed messages. SNS: pub/sub -- topics, subscriptions (SQS, Lambda, email, SMS, HTTP). Step Functions: state machine -- tasks, choices, parallel, map, wait, error handling. Express vs Standard workflows. API Gateway: REST, HTTP, WebSocket APIs -- stages, usage plans, API keys, caching, request/response transformation. SAM: Serverless Application Model (simplified CloudFormation for serverless). CDK: Infrastructure as Code (TypeScript, Python, Java, C#, Go). Amplify: full-stack app framework. Best practices: idempotency, DLQ for failures, structured logging, least privilege IAM. Cold start mitigation: SnapStart (Java), provisioned concurrency, interpreted languages (Node/Python) start fast.
