Server Setup & SSH Hardening

After installing your Linux distribution of choice (Ubuntu Server, Debian, Rocky Linux, or Alpine are common), the first step is to secure SSH access:

# Edit SSH daemon config
sudo nano /etc/ssh/sshd_config

# Critical hardening settings:
Port 2222                    # Change from default 22 to reduce bot scans
PermitRootLogin no           # Disable root login
PasswordAuthentication no    # Require SSH keys only
PubkeyAuthentication yes
AllowUsers alice bob         # Restrict which users can SSH in
ClientAliveInterval 300      # Disconnect idle sessions after 5 min
MaxAuthTries 3               # Max 3 auth attempts before disconnect

# Restart SSH to apply
sudo systemctl restart sshd
Warning: Never disable PasswordAuthentication until you have verified that key-based login works! Keep a second terminal session open while testing to avoid locking yourself out.

User Management & Permissions

# Create a new user
sudo useradd -m -s /bin/bash alice
sudo passwd alice

# Add user to sudo group
sudo usermod -aG sudo alice

# Delete a user
sudo userdel -r bob       # -r removes home directory

# View user info
id alice
who
last

Linux file permissions use a three-digit octal system (owner, group, others):

CommandMeaningOctal
chmod 755 filerwxr-xr-x (owner full, group/others read+execute)4+2+1=7, 4+1=5, 4+1=5
chmod 644 filerw-r--r-- (owner read+write, others read only)6, 4, 4
chmod 600 filerw------- (owner only, private key files)6, 0, 0
chmod 700 filerwx------ (owner full, e.g. .ssh dir)7, 0, 0
 
# Change ownership
sudo chown alice:www-data /var/www/app
# -R for recursive changes
sudo chown -R alice:alice /home/alice

Systemd Services & journalctl

systemd is the init system used by most modern Linux distributions. Services are defined in .service files:

# /etc/systemd/system/myapp.service
[Unit]
Description=My Application
After=network.target postgresql.service
Requires=postgresql.service

[Service]
Type=simple
User=alice
WorkingDirectory=/opt/myapp
ExecStart=/usr/bin/node server.js
Restart=on-failure
RestartSec=5
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
# systemd commands
sudo systemctl start myapp         # Start a service
sudo systemctl stop myapp          # Stop a service
sudo systemctl enable myapp        # Start on boot
sudo systemctl disable myapp       # Don't start on boot
sudo systemctl status myapp        # Show status and recent logs
sudo systemctl restart myapp       # Restart
sudo systemctl daemon-reload       # Reload after editing .service files

# View logs with journalctl
sudo journalctl -u myapp           # Logs for a specific service
sudo journalctl -u myapp -f        # Follow logs in real time
sudo journalctl -u myapp --since "1 hour ago"
sudo journalctl -xe                # Show last logs with explanations

Firewall (ufw / iptables / nftables)

UFW (Uncomplicated Firewall) is the easiest way to manage iptables rules:

# UFW basics
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 2222/tcp            # SSH on non-default port
sudo ufw allow 80/tcp              # HTTP
sudo ufw allow 443/tcp             # HTTPS
sudo ufw enable
sudo ufw status verbose

nftables is the modern replacement for iptables (default on Debian 12+, Fedora, Arch):

# Simple nftables ruleset
sudo nft add table inet filter
sudo nft add chain inet filter input { type filter hook input priority 0 \; }
sudo nft add rule inet filter input tcp dport {22, 80, 443} accept
sudo nft add rule inet filter input ct state established,related accept
sudo nft add rule inet filter input iif lo accept
sudo nft add rule inet filter input drop

Nginx / Apache Basics

Nginx is the most popular web server and reverse proxy. Here is a basic site config:

# /etc/nginx/sites-available/myapp
server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /static/ {
        alias /var/www/myapp/static;
        expires 30d;
    }
}

Enable the site: sudo ln -s /etc/nginx/sites-available/myapp /etc/nginx/sites-enabled/ and sudo nginx -t && sudo systemctl reload nginx.

For SSL certificates, use Certbot from Let's Encrypt: sudo certbot --nginx -d example.com.

Pro tip: Set up fail2ban to automatically block IPs with multiple failed SSH login attempts. Combined with SSH key-only auth and a non-standard port, your server will be significantly more secure.
