Android App Development

Build Android apps with Kotlin, Jetpack Compose, and Material Design 3.

Table of Contents (5 sections)

Kotlin Fundamentals

Kotlin is the preferred language for Android development (officially by Google). Key features: null safety (types end with ? for nullable), data classes, sealed classes, extension functions, coroutines (structured concurrency), and flow (reactive streams).

Kotlin compiles to JVM bytecode and is interoperable with Java. Learn: collections (listOf, mapOf, filter, map), scope functions (let, apply, run, with, also), delegates (lazy, observable), and sealed interfaces for state representation.

Jetpack Compose UI

Jetpack Compose is Android's modern UI toolkit using declarative Kotlin. Instead of mutating XML views, you define composable functions that describe the UI for a given state. Recomposition automatically updates the UI when state changes.

Key composables: Column, Row, Box (layouts), Text, Button, Image, TextField (elements), LazyColumn/LazyRow (recycler view replacement). Modifiers style composables (padding, size, clickable, background). Material Design 3 theming via MaterialTheme with dynamic color support.

Navigation & State Management

Navigation Compose manages screen transitions. Define a NavHost with routes. Pass arguments via NavArgs. ViewModel holds UI state and survives configuration changes (rotation). StateFlow represents reactive state in ViewModels.

State hoisting: state moves up to the ViewModel, composables receive state as parameters and emit events as callbacks. SavedStateHandle survives process death. For complex state: MVI (Model-View-Intent) with sealed classes for intents and states, or Redux-like Unidirectional Data Flow.

Room Database & Networking

Room is the SQLite ORM for Android. Annotate entities (@Entity), DAOs (@Dao for queries), and the database (@Database). Room generates SQL at compile time. Use Kotlin Flow with Room for reactive queries — UI updates automatically when data changes.

Networking: Retrofit (type-safe HTTP client) with OkHttp (interceptors for logging, auth, caching). Ktor client (Kotlin-native, coroutine-based). Serialization: kotlinx.serialization or Moshi for JSON. Use Hilt (dependency injection) to manage Retrofit and Room instances.

Publishing to Play Store

Prepare your app: set versionCode/versionName, generate signed App Bundle (AAB) via Android Studio, create a Google Play Console account ($25 fee), fill store listing (title, description, screenshots, feature graphic), content rating questionnaire, and set pricing/distribution.

Publishing: upload AAB to Play Console, review track (Internal testing → Closed alpha → Open beta → Production), set release percentage for staged rollout, monitor crash reports (Firebase Crashlytics), user reviews, and ratings. Follow Google's policy guidelines to avoid rejection.
