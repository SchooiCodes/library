GitHub Actions CI/CD

Automate workflows with GitHub Actions — CI/CD, matrix builds, artifacts, and deployment.

Table of Contents (2 sections)

  • Workflow Fundamentals
  • Artifacts & Deployment

Workflow Fundamentals

GitHub Actions: YAML workflows in .github/workflows/. Trigger: push, pull_request, schedule, workflow_dispatch, release. Jobs: run in parallel by default, dependencies via needs:. Steps: uses (actions/checkout@v4), run (shell commands), name, id, with (inputs). Matrix: strategy.matrix: { os: [ubuntu-latest, windows-latest], node: [18, 20] } — generates NxM job combinations. Runners: GitHub-hosted (ubuntu, windows, macos) or self-hosted (scale sets).

Artifacts & Deployment

Artifacts: actions/upload-artifact@v4 (upload build outputs), actions/download-artifact@v4 (download in later jobs). Caching: actions/cache@v4 — cache node_modules, pip, Maven ~/.m2. Environments: deployment environments with protection rules, secrets, and approval gates. OIDC: authenticate to cloud providers without static secrets. Composite actions: reusable step groups (action.yml in subdirectory). GitHub-hosted runner specs: 2-4 vCPU, 7-14GB RAM, 14-70GB SSD.
