Installation & First Launch

  1. Download VS Code from code.visualstudio.com
  2. During installation, check "Add to PATH" and "Register as default editor"
  3. Launch from terminal: code . opens the current folder

The VS Code interface is split into five main areas:

  • Activity Bar — Leftmost sidebar with icons for Explorer, Search, Source Control, Run & Debug, Extensions
  • Side Bar — Contextual panel next to Activity Bar (file tree, search results, etc.)
  • Editor — Central area where you edit files
  • Panel — Bottom area for terminal, problems, output, debug console
  • Status Bar — Bottom bar showing branch, errors, encoding, line/column

Integrated Terminal

  • Open with Ctrl+`
  • Split terminal: Ctrl+Shift+5
  • Default shell: Configure in settings under terminal.integrated.defaultProfile
  • Multiple terminals: Click the + icon in the terminal panel
  • Run tasks, git commands, and npm scripts directly without leaving the editor

Snippets & Emmet

Emmet is built-in for HTML and CSS. Type a CSS-like expression and press Tab to expand:

div.container>ul.list>li.item*5 + Tab

To create custom snippets: Ctrl+Shift+P → "Configure User Snippets". Choose a language and define your snippets:

// Example HTML snippet (html.json):
"HTML Boilerplate": {
 "prefix": "html5",
 "body": [
  "<!DOCTYPE html>",
  "<html lang=\"en\">",
  "<head>",
  " <meta charset=\"UTF-8\">",
  " <title>${1:Document}</title>",
  "</head>",
  "<body>",
  " $0",
  "</body>",
  "</html>"
 ],
 "description": "HTML5 boilerplate"
}
// Example JS snippet (javascript.json):
"Console Log": {
 "prefix": "clog",
 "body": "console.log('${1:message}');",
 "description": "Log to console"
}

Debugging

  • Open the Run and Debug view: Ctrl+Shift+D or click the play button in the Activity Bar
  • Set breakpoints by clicking the gutter next to line numbers
  • Step through code: Step Over (F10), Step Into (F11), Continue (F5)
  • Inspect variables in the Watch, Call Stack, and Breakpoints panels
  • Create a launch configuration in .vscode/launch.json for customized debugging setups
