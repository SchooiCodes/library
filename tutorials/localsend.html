LocalSend

Cross-platform LAN file transfer app that works entirely offline — no internet, no cloud, no server.

Overview

LocalSend is a free, open-source, cross-platform file transfer app that works over local area networks (LAN) with zero internet dependency. It uses mDNS/DNS-SD (Bonjour/Avahi) for device discovery and communicates over HTTP with TLS. No accounts, no cloud relay, no data leaves your network.

Think of it as AirDrop for everyone — Android, iOS, Windows, macOS, and Linux all talk to each other seamlessly. Transfers are encrypted with TLS 1.3 and can reach wire speeds on gigabit LAN. Android and iOS versions include a QR-code scanner for faster pairing.

Sending Files

Using the GUI

  1. Open LocalSend on the sender device.
  2. Tap "Send".
  3. Select files or tap the clipboard icon to send text.
  4. Wait for nearby devices to appear (this takes 1-3 seconds over mDNS).
  5. Tap the receiver's device name.
  6. The receiver gets a popup — they tap "Accept" and the transfer starts.

Send to Multiple Devices

Tap multiple device names in the peer list. Each device will receive the files simultaneously. The transfer speed adapts per peer — a slow device won't slow down the others.

Send via CLI (Linux/macOS)

LocalSend includes a command-line tool localsend-cli for scripting:

# Send a file to the first discovered device
localsend-cli send ~/Documents/report.pdf

# Send to a specific device by name
localsend-cli send --device "Alice-Phone" ~/Photos/vacation.jpg

# Send to all discovered devices
localsend-cli send --all ~/shared/notes.txt

Receiving Files

Default Behavior

When a sender initiates a transfer, your device shows a notification with the sender's name, file name, and size. Tap "Accept" to start downloading. Files land in your system's Downloads folder by default.

Auto-Accept (Trusted Devices)

Pair devices to skip the confirmation prompt:

# On both devices, enable pairing mode
Settings > Security > Pair with device

# A QR code appears on one device; scan it with the other.
# Once paired, future transfers from this device auto-accept.

Save Location

Change the default save directory:

Settings > Receive > Save to folder
# Choose an external SD card, network share, or cloud-synced folder

Pairing Devices

Pairing creates a trust relationship so you don't need to manually accept every transfer. The process uses a QR code containing the device fingerprint (a hash of its TLS certificate):

  1. Go to Settings > Security > Pair with device on Device A.
  2. Select "Generate QR Code".
  3. On Device B, select "Scan QR Code".
  4. Both devices display the same fingerprint string. Verify it matches, then confirm.

Paired devices appear with a heart icon in the peer list. You can revoke pairing at any time from the same settings menu.

Security Model

LocalSend was designed with privacy as a first principle:

  • No Internet Required — All discovery and transfer happens over LAN. If your router is disconnected from the WAN, LocalSend still works perfectly.
  • TLS 1.3 Encryption — Each device generates a self-signed TLS certificate on first launch. All HTTP traffic is encrypted with TLS 1.3, perfect forward secrecy (PFS) via ECDHE.
  • mDNS Only — Device discovery uses mDNS (multicast DNS), which never leaves the local subnet. No DNS queries go to the internet.
  • No Telemetry — LocalSend has zero analytics, no crash reporting, no usage data collection. The app requests no unnecessary permissions.
  • Ephemeral Ports — Each transfer uses a random high-port (50000-60000), reducing the attack surface.
Note: Because TLS certificates are self-signed, the first connection to a new device shows a warning. Verify the fingerprint out-of-band (show the screen to the other person). After pairing, the certificate is pinned.

Troubleshooting

IssueSolution
Devices not discovering each otherEnsure both devices are on the same subnet/VLAN. Check that mDNS traffic (UDP 5353) isn't blocked by firewall. On enterprise Wi-Fi, client isolation may prevent discovery.
Transfer fails midwayCheck disk space on receiver. Ensure both devices stay connected to the same network. Try sending smaller batches.
iOS can't see AndroidiOS restricts mDNS to foreground apps. Keep LocalSend open on both devices. Disable "Private Wi-Fi Address" for your network if issues persist.
Firewall blockingOpen TCP ports 50000-60000 for inbound connections. Also allow UDP 5353 for mDNS. On Windows, allow LocalSend in Defender Firewall.
Slow transfer on gigabit LANCheck if Wi-Fi is the bottleneck (2.4 GHz vs 5 GHz). Try wired Ethernet. Disable any VPN that might be routing traffic off-LAN.
Certificate warning every timePair the devices. The certificate fingerprint is pinned after pairing, suppressing the warning.
