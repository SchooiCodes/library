🔐 Linux File Permissions

Master chmod, chown, umask, special permissions (SUID/SGID/sticky), ACLs, and SELinux basics.

Permission Basics (rwx)

Every file and directory in Linux has three permission triples: owner, group, and others. Each triple consists of read (r), write (w), and execute (x). Use ls -l to view them.

-rwxr-xr-- 1 alice users 1024 May 27 10:00 script.sh
|└┬┘└┬┘└┬┘
| │ │ └── others: r--
| │ └───── group: r-x
| └──────── owner: rwx
└────────── file type (-=file, d=directory, l=symlink)

For directories, r allows listing contents, w allows creating/deleting entries, and x allows entering the directory.

chmod — Symbolic & Numeric

chmod changes permissions. Use symbolic mode (u+x, g-w) or numeric mode (octal 0-7 per triple).

# Symbolic mode
chmod u+x script.sh    # add execute for owner
chmod g-w file.txt    # remove write for group
chmod o+r file.txt    # add read for others
chmod a+x script.sh    # add execute for all

# Numeric mode (r=4, w=2, x=1)
chmod 755 script.sh    # rwxr-xr-x
chmod 644 file.txt    # rw-r--r--
chmod 600 secret.key   # rw-------
chmod 700 private_dir   # rwx------

# Recursive
chmod -R 755 /path/to/dir
Tip: Use chmod 755 for scripts/directories and chmod 644 for regular files. Never use 777 in production.

chown & chgrp

chown changes the owner and/or group of a file. chgrp changes only the group. Only root can change ownership.

# Change owner
sudo chown alice file.txt

# Change owner and group
sudo chown alice:staff file.txt

# Change group only
sudo chgrp staff file.txt

# Recursive
sudo chown -R alice:staff /home/alice

umask

umask sets default permission masks for newly created files and directories. It subtracts from the base permissions (666 for files, 777 for directories).

umask 022  # default on most systems
       # files: 666-022 = 644 (rw-r--r--)
       # dirs: 777-022 = 755 (rwxr-xr-x)

umask 077  # restrictive — only owner has access
       # files: 600, dirs: 700

Set umask in ~/.bashrc or /etc/profile for persistent changes.

SUID / SGID / Sticky Bit

Special permission bits enable elevated behavior. SUID runs a file as its owner; SGID runs it as its group or inherits group on directories; the sticky bit restricts deletion in shared directories.

# SUID (s on owner's x) — e.g., /usr/bin/passwd
chmod u+s program     # rwsr-xr-x
chmod 4755 program

# SGID (s on group's x) — new files inherit group
chmod g+s shared_dir    # rwxrwsr-x
chmod 2775 shared_dir

# Sticky Bit (t on others' x) — e.g., /tmp
chmod +t /shared      # rwxrwxrwt
chmod 1777 /shared

# Remove special bits
chmod u-s program
chmod g-s shared_dir
chmod -t /shared

ACLs with setfacl/getfacl

ACLs (Access Control Lists) provide fine-grained permissions beyond the standard owner/group/others model. They allow granting permissions to specific users or groups.

# View ACLs
getfacl file.txt

# Grant read/write to a specific user
setfacl -m u:bob:rw- file.txt

# Grant read to a specific group
setfacl -m g:developers:r-- file.txt

# Remove a specific entry
setfacl -x u:bob file.txt

# Recursive with default inheritance
setfacl -R -m u:bob:rx dir/
setfacl -d -m u:bob:rx dir/ # default for new files
Tip: Use ls -l to check if a file has ACLs — a + at the end of the permission string (e.g., -rw-r--r--+) indicates ACLs are set.

SELinux Overview

SELinux (Security-Enhanced Linux) adds Mandatory Access Control (MAC). Every process and file has a security context. Policies define exactly what each process can access, beyond standard permissions.

# Check SELinux status
getenforce     # Enforcing / Permissive / Disabled

# View security context
ls -Z file.txt
ps -Z

# Change context
chcon -t httpd_sys_content_t /var/www/html/index.html

# Restore default context
restorecon -v /var/www/html/index.html

If you see "Permission denied" but standard permissions look correct, SELinux may be blocking the operation. Check audit2allow and /var/log/audit/audit.log.
