Linux Boot Process & GRUB

GRUB configuration, initramfs, UEFI, Secure Boot, and boot troubleshooting.

The Linux Boot Sequence

When you press the power button, this sequence runs:

  1. BIOS/UEFI -- initializes hardware, runs POST, reads boot device
  2. Bootloader (GRUB) -- loads from EFI System Partition or MBR, presents menu
  3. Kernel -- decompresses itself, initializes subsystems, mounts initramfs
  4. initramfs -- loads storage drivers, mounts root filesystem
  5. init/systemd -- PID 1, starts services, reaches target

GRUB 2 is the default bootloader for most Linux distributions. It supports both BIOS (legacy) and UEFI boot modes.

GRUB Configuration Files

GRUB's main configuration files:

# Main config -- auto-generated, DO NOT EDIT directly
/boot/grub/grub.cfg

# User-editable settings
/etc/default/grub

# Custom scripts (executed during grub-mkconfig)
/etc/grub.d/
  00_header        # Sets defaults
  10_linux         # Finds Linux kernels
  30_os-prober     # Detects other OSes
  40_custom        # User entries
  41_custom        # More user entries
# /etc/default/grub example
GRUB_DEFAULT=0               # Default menu entry
GRUB_TIMEOUT=5               # Seconds before boot
GRUB_TIMEOUT_STYLE=menu      # menu, countdown, hidden
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"  # Kernel params
GRUB_GFXMODE=1920x1080       # Graphics resolution
GRUB_DISABLE_OS_PROBER=false # Detect other OSes

# Apply changes
sudo update-grub  # or: sudo grub-mkconfig -o /boot/grub/grub.cfg

Initramfs & Kernel Parameters

The initramfs (initial RAM filesystem) contains kernel modules needed to mount the root filesystem:

# Rebuild initramfs
sudo mkinitcpio -P                     # Arch
sudo update-initramfs -u -k all        # Debian/Ubuntu
dracut --force                         # Fedora

# View initramfs contents
lsinitramfs /boot/initrd.img-$(uname -r)

# Kernel parameters (in GRUB_CMDLINE_LINUX)
# Add these to solve specific issues:

# Disable nouveau (NVIDIA) for safe boot
GRUB_CMDLINE_LINUX="modprobe.blacklist=nouveau"

# Memory limits
GRUB_CMDLINE_LINUX="mem=4G"

# Single-user mode (rescue)
GRUB_CMDLINE_LINUX="single"

# Disable ACPI (hardware debug)
GRUB_CMDLINE_LINUX="acpi=off"

UEFI, Secure Boot & Dual Boot

Modern systems use UEFI with GPT partitioning. Key points:

# Check boot mode
[ -d /sys/firmware/efi ] && echo "UEFI" || echo "BIOS/Legacy"

# EFI System Partition (ESP)
# Usually mounted at /boot/efi or /boot
ls /boot/efi/EFI/

# Install GRUB for UEFI
sudo grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=GRUB

# Secure Boot enrollment (using shim)
sudo apt install shim-signed
sudo grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=GRUB

# Dual boot: enable os-prober
GRUB_DISABLE_OS_PROBER=false
sudo update-grub

Troubleshooting Boot Issues

When your system won't boot, use these recovery techniques:

# At GRUB menu, press 'e' to edit boot entry
# Add these kernel parameters for rescue:

# Boot to emergency shell
systemd.unit=emergency.target

# Boot to single-user mode (root shell)
init=/bin/bash

# Skip systemd
init=/bin/sh

# After boot, view logs
journalctl -b -1        # Previous boot log
journalctl -xb          # Current boot with explanations

# Regenerate GRUB
sudo mount /dev/sda1 /mnt
sudo mount --bind /dev /mnt/dev
sudo chroot /mnt
grub-install /dev/sda
update-grub

Keep a live USB handy for recovery. Most distros offer a 'Try' mode that lets you chroot into your installed system.
