🧊 Linux Containers (LXC/LXD)

Lightweight system containers with LXC and LXD — create, manage, snapshot, and network containers.

Containers vs VMs

LXC (Linux Containers) provides OS-level virtualization. Unlike VMs, containers share the host kernel but run in isolated namespaces. LXD is a next-generation container manager built on LXC that adds a REST API, image management, and advanced features. Containers start in seconds and use far less RAM than VMs.

┌─────────────────────┬──────────────────┐
│ VMs         │ LXC Containers  │
├─────────────────────┼──────────────────┤
│ Full kernel per VM │ Shared host kernel│
│ Boot in minutes   │ Boot in seconds │
│ GBs of RAM each   │ MBs of RAM each │
│ Hardware emulation │ Native performance│
│ Hypervisor needed  │ Runs on bare metal│
└─────────────────────┴──────────────────┘

Installing LXD

LXD is available in most distro repositories. Install it and initialize the daemon. The lxd init wizard configures networking, storage, and clustering.

# Install LXD
sudo apt install lxd lxd-client  # Debian/Ubuntu
sudo pacman -S lxd        # Arch
sudo dnf install lxd       # Fedora

# Add your user to the lxd group
sudo usermod -aG lxd $USER

# Initialize LXD
sudo lxd init

# Check status
lxc list
lxc info

Creating Containers

LXD uses images from remote repositories (ubuntu, alpine, debian, fedora, etc.). Launch a container with lxc launch or create one with lxc init and start it later.

# List available images
lxc image list images:
lxc image list images:ubuntu

# Launch a container (create + start)
lxc launch ubuntu:24.04 my-container

# Launch a specific release
lxc launch ubuntu:22.04 focal-container

# Launch Alpine (very lightweight)
lxc launch alpine:3.19 alpine-box

# Create without starting
lxc init ubuntu:24.04 my-container
lxc start my-container

# Check container list
lxc list
Tip: Use Alpine images for minimal containers (~5 MB). Ubuntu images are good for general-purpose workloads.

Container Management

Manage running containers with lxc exec, lxc file, and lifecycle commands. The lxc CLI is intuitive and self-documenting.

# Execute commands inside a container
lxc exec my-container -- bash
lxc exec my-container -- apt update
lxc exec my-container -- ls -la /root

# Run a single command
lxc exec my-container -- cat /etc/os-release

# Copy files in/out
lxc file push myfile.conf my-container/etc/
lxc file pull my-container/var/log/syslog .

# Stop / start / restart
lxc stop my-container
lxc start my-container
lxc restart my-container

# Delete a container
lxc delete my-container

# Delete with force (even if running)
lxc delete --force my-container

Snapshots & Backups

Snapshots capture the container state instantly. Use them before risky operations or for backup. Snapshots can be restored, exported, or copied to other hosts.

# Create a snapshot
lxc snapshot my-container pre-upgrade

# List snapshots
lxc info my-container

# Restore a snapshot
lxc restore my-container pre-upgrade

# Export a container (backup)
lxc export my-container my-container-backup.tar.gz

# Import a container
lxc import my-container-backup.tar.gz

# Snapshot all containers
for c in $(lxc list -c n --format csv); do
 lxc snapshot "$c" "backup-$(date +%Y%m%d)"
done

Networking

LXD creates a managed bridge (lxdbr0) by default. Containers get DHCP IPs on this bridge and can communicate with each other and the host. You can also attach containers to existing bridges or use macvlan.

# Default bridge setup
lxc network list
lxc network show lxdbr0

# Get container IP
lxc list
lxc info my-container | grep eth0

# Add a network device
lxc config device add my-container eth1 \
 nic nictype=bridged parent=br0

# Port forwarding (proxy device)
lxc config device add my-container proxy-tcp \
 proxy listen=tcp:0.0.0.0:8080 \
 connect=tcp:127.0.0.1:80

Resource Limits

LXD supports fine-grained resource controls — CPU pinning, memory limits, disk quotas, and process limits. These use cgroups under the hood.

# Set memory limit (hard + soft)
lxc config set my-container limits.memory=512MB
lxc config set my-container limits.memory.enforce=hard

# CPU limits
lxc config set my-container limits.cpu=2    # 2 cores
lxc config set my-container limits.cpu.allowance=50% # 50% of 1 CPU

# Disk quota
lxc config device override my-container root size=10GB

# Process limit
lxc config set my-container limits.processes=100

# Apply to running container (no restart needed)
lxc config show my-container

# Remove a limit
lxc config unset my-container limits.memory
Tip: Resource limits take effect immediately on running containers. Use lxc config show to view the current configuration.
