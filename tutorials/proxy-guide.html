🔀 Proxy Guide

Proxy Types

TypeProtocolUse Case
Forward proxyHTTP/HTTPSClient-side proxying — browser or app routes traffic through it
Reverse proxyHTTP/HTTPSServer-side — sits in front of web servers, handles SSL, load balancing
SOCKS5TCP/UDPGeneric proxy for any TCP/UDP traffic, not just HTTP
TransparentAnyIntercepts traffic without client configuration (gateway)

HTTP/HTTPS Proxies

Configure an HTTP proxy for command-line tools or browsers:

# Environment variables for CLI
export HTTP_PROXY=http://proxy.example.com:8080
export HTTPS_PROXY=http://proxy.example.com:8080
export NO_PROXY=localhost,127.0.0.1,10.0.0.0/8

# Use with curl
curl -x http://proxy.example.com:8080 https://api.example.com

# Authenticated proxy
curl -x http://user:pass@proxy.example.com:8080 https://example.com

SOCKS5 Proxy

SOCKS5 proxies handle any TCP connection and can relay UDP. Useful for SSH tunnels, torrents, and general privacy.

# Create a SOCKS5 tunnel via SSH
ssh -D 1080 user@server

# Use with curl
curl --socks5 localhost:1080 https://example.com

# Use with an application
# Configure app to use SOCKS5 proxy at 127.0.0.1:1080
Tip: SOCKS5 proxies do not modify or inspect traffic — they just forward packets. This makes them faster and more private than HTTP proxies for non-web traffic.

Reverse Proxy — Nginx

Route traffic to multiple backend services from a single domain. Also handles SSL termination, caching, and load balancing.

# /etc/nginx/sites-available/default
server {
  listen 80;
  server_name example.com;

  location / {
    proxy_pass http://localhost:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  }

  location /api {
    proxy_pass http://localhost:4000;
  }
}

# Enable SSL with certbot:
sudo certbot --nginx -d example.com

Reverse Proxy — Caddy

Caddy automates SSL via Let's Encrypt and has a simpler configuration syntax than Nginx.

# Caddyfile
example.com {
  reverse_proxy localhost:3000
}

api.example.com {
  reverse_proxy localhost:4000
}

# Run:
caddy run

Caddy automatically obtains and renews TLS certificates — no manual Certbot step needed.

Proxy Chaining

Route traffic through multiple proxies for layered anonymity (e.g., Tor -> VPN -> proxy).

# Using proxychains
sudo apt install proxychains4

# /etc/proxychains4.conf:
# strict_chain
# [ProxyList]
# socks4 127.0.0.1 9050  # Tor
# http  proxy.example.com 8080

# Run any command through the chain:
proxychains4 curl https://check.torproject.org

Privacy Considerations

  • HTTP proxies see everything — only use HTTPS proxies for web traffic
  • SOCKS5 does not encrypt — combine with SSH tunnel or VPN for encryption
  • DNS leaks — ensure DNS queries go through the proxy (use --dns flags or proxychains)
  • WebRTC leaks — disable WebRTC in browser or use a proxy browser extension
  • IP leaks — verify with curl ifconfig.me before and after
  • Proxy logging — self-host proxies to avoid third-party logging
