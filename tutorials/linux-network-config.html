Linux Network Configuration

Configure networking on Linux — interfaces, bridges, routing, DNS, and network namespaces.

Network Interface Management

Modern Linux uses systemd-networkd or NetworkManager for network management. ip command replaces deprecated ifconfig:

# List interfaces
ip link show
ip addr show
ip route show

# Configure interface statically (systemd-networkd)
# /etc/systemd/network/20-wired.network
[Match]
Name=enp3s0

[Network]
Address=192.168.1.100/24
Gateway=192.168.1.1
DNS=1.1.1.1
DNS=8.8.8.8

# Apply
sudo systemctl restart systemd-networkd

NetworkManager CLI: nmcli device status, nmcli connection show, nmcli connection up id "My WiFi".

Bridging & Bonding

Network bridge (for VMs/containers):

# Create bridge with nmcli
nmcli connection add type bridge ifname br0
nmcli connection add type ethernet slave-type bridge ifname enp3s0 master br0

# Or with systemd-networkd
# /etc/systemd/network/br0.netdev
[NetDev]
Name=br0
Kind=bridge

# /etc/systemd/network/br0.network
[Match]
Name=br0
[Network]
Address=192.168.1.100/24
Gateway=192.168.1.1
DNS=1.1.1.1

Bonding (link aggregation): mode 4 (LACP with switch support), mode 1 (active-backup failover).

Firewall with nftables

nftables replaces iptables. Example ruleset:

# /etc/nftables.conf
table inet filter {
    chain input {
        type filter hook input priority 0; policy drop;
        ct state established,related accept
        ct state invalid drop
        lo accept
        icmp type echo-request accept
        tcp dport 22 accept       # SSH
        tcp dport 80 accept       # HTTP
        tcp dport 443 accept      # HTTPS
    }
    chain forward {
        type filter hook forward priority 0; policy drop;
    }
    chain output {
        type filter hook output priority 0; policy accept;
    }
}
sudo nft -f /etc/nftables.conf
sudo systemctl enable --now nftables

Use nft list ruleset to view current rules. Fail2ban can write nftables blocks.

DNS & Domain Resolution

DNS configuration on Linux:

# /etc/systemd/resolved.conf
[Resolve]
DNS=1.1.1.1 9.9.9.9
FallbackDNS=8.8.8.8
DNSSEC=yes
DNSOverTLS=yes
Cache=yes

# Per-interface DNS (systemd-networkd)
# In .network file:
[Network]
DHCP=yes
[DHCP]
UseDNS=false  # Ignore DHCP DNS

# /etc/hosts for static entries
127.0.0.1 localhost
192.168.1.50 nas.local nas

# Use local DNS cache: systemd-resolve or dnsmasq
sudo systemctl enable --now systemd-resolved

Network Namespaces

Network namespaces isolate network stacks — useful for VPNs, containers, and testing:

# Create namespace
sudo ip netns add blue
sudo ip netns add red

# Connect them with a virtual ethernet pair
sudo ip link add veth-b type veth peer name veth-r
sudo ip link set veth-b netns blue
sudo ip link set veth-r netns red

# Assign IPs and bring up
sudo ip netns exec blue ip addr add 10.0.0.1/24 dev veth-b
sudo ip netns exec blue ip link set veth-b up
sudo ip netns exec red ip addr add 10.0.0.2/24 dev veth-r
sudo ip netns exec red ip link set veth-r up

# Test connectivity
sudo ip netns exec blue ping 10.0.0.2
