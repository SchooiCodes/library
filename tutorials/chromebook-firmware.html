Chromebook Firmware Overview

Chromebooks do not use a traditional BIOS or UEFI like standard PCs. Instead, Google uses a custom UEFI implementation combined with a verified boot chain. Every time the system boots, each stage of the boot process cryptographically verifies the next stage before executing it. This verified boot chain is what makes Chromebooks so secure — and also what makes installing alternative operating systems a challenge.

The firmware consists of several components: the read-only (RO) firmware that cannot be modified without physical access, and the read-write (RW) firmware that contains the boot splash, recovery mode, and developer mode features. Understanding this split is critical before you start flashing.

Why Flash Chromebook Firmware?

There are several reasons you might want to replace or modify the stock firmware:

  • Install Linux or Windows — The stock firmware will only boot ChromeOS. Custom firmware allows you to boot standard operating systems from USB or internal storage.
  • Remove ChromeOS — If you have no use for ChromeOS and want a full Linux or Windows machine, wiping ChromeOS and flashing custom firmware gives you a standard PC experience.
  • Extend device life after AUE — Google's Auto Update Expiration (AUE) means your Chromebook stops receiving security updates after a certain date. Flashing custom firmware lets you run a supported OS long after Google drops support.
  • Dual boot — With RW_LEGACY firmware, you can keep ChromeOS installed and boot into Linux when needed.

Safety First: Backup Your Stock BIOS

Critical: Before making any changes, you must back up your stock firmware. If you skip this step and something goes wrong, you could brick your device. The Google Fleex ROMs you found on your USB are exactly this — backups of Chromebook firmware, saved for safekeeping.

Always store the backup on a separate drive, in cloud storage, or both. Label it with the device model and date so you know exactly what it is when you need to restore it months or years later.

Tools Needed

  • USB drive with ChromeOS recovery image — Create one using the Chromebook Recovery Utility or the chromeos-recovery tool. This is your lifeline if you need to restore stock firmware.
  • Linux live USB — Ubuntu, Mint, or any live Linux environment where you can run flashrom. Many flashing tools require Linux.
  • flashrom — The open-source flashing utility. Most Linux live USBs include it, or you can install it with sudo apt install flashrom.
  • Screwdriver — You will need to open the Chromebook chassis and remove the write-protect screw. A small Phillips head (#0 or #00) is usually required.
  • Internet connection — The MrChromebox script downloads the latest firmware binaries, so you need Wi-Fi or Ethernet.

Write-Protect Screw

Most Chromebooks have a firmware write-protect (WP) screw on the motherboard. When this screw is in place, the RO firmware region is locked and flashrom cannot write to it. You must remove this screw before you can flash custom firmware.

  • The WP screw is usually silver or gold and located near the firmware SPI flash chip.
  • On some models it is a physical screw; on others it is a jumper.
  • After removing the screw, the system will show a OS verification is OFF screen at boot — this is normal.
  • Keep the screw in a safe place. You can reinstall it after flashing to restore write protection.

Consult the Chromium OS firmware porting guide or search for your specific model on the Chromium Developer Information page to locate the WP screw on your device.

Backup Stock Firmware

Follow these steps to dump your stock firmware before making any changes:

  1. Enter Developer Mode: Power off the Chromebook. Press and hold Esc + Refresh (F3), then press Power. When the recovery screen appears, press Ctrl + D to enable developer mode, then press Enter to confirm. The device will wipe and reboot — this takes a few minutes.
  2. Open a Shell: Once in developer mode, press Ctrl + Alt + F2 (or Ctrl + Alt + T and type shell).
  3. Read the firmware with flashrom: 
    # Check that flashrom can see the SPI chip
sudo flashrom -p internal

# Dump the full firmware to a file
sudo flashrom -p internal -r backup.rom
  4. Store the backup safely! Copy backup.rom to a USB drive, cloud storage, and/or another machine. Name it something like google-fleex-backup-2026-05-27.rom so you know exactly what it is.

Flashing Custom Firmware with MrChromebox

MrChromebox provides the most popular and well-maintained firmware utility for Chromebooks. The script handles downloading, verifying, and flashing firmware for hundreds of Chromebook models.

  1. Boot ChromeOS or Linux on your Chromebook. If you are using a live USB, boot into it now.
  2. Run the firmware utility script: 
    cd; curl -LO https://mrchromebox.tech/firmware-util.sh \
  && sudo bash firmware-util.sh
  3. Choose your firmware type:
    • RW_LEGACY — Installs a custom UEFI payload into the RW region. ChromeOS remains the primary boot option, but you can press Ctrl + L at the developer screen to boot Linux from USB. This is the safest option because you keep the stock RO firmware intact.
    • UEFI (Full ROM) — Replaces the entire firmware with a UEFI implementation. ChromeOS is no longer bootable. Your Chromebook behaves like a standard UEFI PC and can boot Windows, Linux, or any UEFI-compatible OS. This is irreversible without restoring from backup.
    • SeaBIOS — A legacy BIOS payload. Useful for older operating systems or specific use cases. Less common than UEFI for modern installations.
Recommendation: If you are unsure, start with RW_LEGACY. It preserves the ability to recover ChromeOS and lets you test Linux compatibility before committing to a full UEFI replacement.

After Flashing: Install an OS

Once you have custom firmware installed, your Chromebook can boot standard operating systems:

  • Linux — Ubuntu, Linux Mint, Fedora, Arch Linux, Debian, and most mainstream distributions work well. Use a tool like Rufus or dd to create a bootable USB, then boot from it. Performance is generally excellent since Chromebook hardware is well-supported by the Linux kernel.
  • Windows — Possible on some Intel/AMD Chromebook models with full UEFI firmware. Note that Windows drivers for Chromebook hardware are limited (audio, touchpad, and keyboard firmware keys may not work perfectly). ARM-based Chromebooks (like Google Fleex) cannot run Windows.
  • ChromeOS Flex — If you want to stay in the Google ecosystem but need a supported OS after AUE, ChromeOS Flex can be installed on devices with UEFI firmware.

Restoring Stock Firmware

If you ever need to go back to stock, you have two options:

  • ChromeOS Recovery USB — Boot from a recovery USB created with the Chromebook Recovery Utility. This restores both ChromeOS and the stock firmware to factory condition. Works regardless of what firmware you flashed (as long as the RO firmware is intact).
  • Flash backup.rom back — If you saved your stock firmware backup, you can flash it back using the MrChromebox script (choose the restore option) or manually with: 
    sudo flashrom -p internal -w backup.rom

Chromebook Models: ARM vs x86

It is important to know what architecture your Chromebook uses, because it determines what OS options are available:

ArchitectureCommon CPUsOS SupportExample Models
x86 (Intel/AMD)Intel Celeron/Pentium/Core, AMD RyzenLinux, Windows, ChromeOS FlexHP Chromebook x360, Acer Chromebook Spin 713, Lenovo ThinkPad C13 Yoga
ARMMediaTek MT8173/MT8192, Rockchip RK3399, Qualcomm SnapdragonLinux (ARM builds only), no WindowsGoogle Fleex (MediaTek), Lenovo Duet (MediaTek P60T), Samsung Galaxy Chromebook

Google Fleex is an ARM-based Chromebook (MediaTek MT8173). This means you cannot install Windows on it. You are limited to Linux distributions that offer ARM builds, such as Ubuntu ARM, Debian ARM, or Arch Linux ARM. The MrChromebox script does support Fleex for UEFI firmware, but always verify compatibility on the MrChromebox device compatibility list before flashing.

Pro Tip: If you have an ARM Chromebook like Fleex, consider using RW_LEGACY firmware and installing an ARM build of Ubuntu or Debian. This gives you a full Linux desktop while keeping the ability to boot ChromeOS if needed. For x86 Chromebooks, full UEFI replacement is generally the best path for a long-term Linux or Windows machine.
