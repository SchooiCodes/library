File Transfer

Croc

CLI tool for encrypted peer-to-peer file transfers between any two computers.

Overview

Croc is a command-line tool that lets any two computers transfer files and folders securely using a relay server to establish a peer-to-peer connection. It uses PAKE (Password Authenticated Key Exchange) for end-to-end encryption, meaning the relay server cannot decrypt the file contents even though it facilitates the connection.

Unlike tools like scp or rsync that require pre-shared SSH keys or server configuration, Croc needs only a single-use "code phrase" — a randomly generated word like pink-flamingo-37. The sender generates it, the receiver types it in, and the transfer begins. Croc supports resumable transfers, folder sending, text piping, and custom relay servers for air-gapped environments.

Installation

Quick Install (curl piped to bash)

curl https://getcroc.schollz.com | bash

Homebrew (macOS / Linux)

brew install croc

Go Install

go install github.com/schollz/croc/v10/cmd/croc@latest

Cargo (Rust)

cargo install croc

Binary Download

# Linux amd64
curl -LO https://github.com/schollz/croc/releases/latest/download/croc_linux-amd64.tar.gz
tar -xzf croc_linux-amd64.tar.gz
sudo mv croc /usr/local/bin/

Windows

# Using Scoop
scoop install croc

# Or download the Windows exe from GitHub releases

Verify installation:

croc --version

Basic Usage

The workflow is simple. On the sending machine:

croc send file.zip

Output:

Sending 'file.zip' (42 MB)
Code is: pink-flamingo-37
On the other machine, run: croc pink-flamingo-37

On the receiving machine:

croc pink-flamingo-37

That's it. The file transfers directly (or through the relay if a direct P2P connection can't be established). The code phrase is single-use and expires after the transfer.

Sending Files

Send a Single File

croc send report.pdf

Send a Folder

croc send ~/Documents/project/

Folders are automatically zipped before sending and unzipped on the receiving end.

Send with a Custom Code

croc send --code my-secret-99 file.zip

Use a memorable code so you can communicate it verbally. Codes must be at least 6 characters and contain a number.

Send Text from stdin

cat secret.txt | croc send --text
echo "Hello, world!" | croc send --text

Pipe a File from Another Command

pg_dump mydb | gzip | croc send --code db-backup-99 --text

Send with Encryption Passphrase

croc send --passphrase "my-strong-password" sensitive.doc

This adds an extra layer of encryption using the passphrase as a key, on top of the PAKE-derived key.

Disable P2P (Force Relay)

croc send --relay-only largefile.iso

Useful when both peers are behind symmetric NAT that prevents direct connections.

Receiving Files

Basic Receive

croc pink-flamingo-37

Receive to a Specific Directory

croc --out ~/Downloads pink-flamingo-37

Receive and Overwrite Existing Files

croc --yes --out ~/Downloads pink-flamingo-37

The --yes flag skips the confirmation prompt for accepting the file.

Receive Multiple Transfers in Sequence

croc --multiple

This keeps Croc listening for multiple incoming transfers one after another.

Custom Relay

By default, Croc uses a public relay at croc6.schollz.com. For privacy, performance, or air-gapped networks, you can run your own relay:

Start a Relay Server

# Install the relay
go install github.com/schollz/croc/v10/cmd/croc-relay@latest

# Run it
croc-relay --port 9009 --password "relay-secret"

Using Docker for the Relay

docker run -d --name croc-relay \
  -p 9009:9009 \
  -p 9010:9010 \
  schollz/croc-relay

Send via Custom Relay

croc send --relay "192.168.1.50:9009" file.zip

Receive via Custom Relay

croc --relay "192.168.1.50:9009" pink-flamingo-37

Environment Variables for Relay

export CROC_RELAY="192.168.1.50:9009"
export CROC_RELAY_PASSWORD="relay-secret"
croc send file.zip

Resume Transfers

Croc supports resuming interrupted transfers, which is critical for large files over unreliable connections:

# Sender (interrupted)
croc send bigfile.iso

# If it fails, just send again with the same code:
croc send --code same-code-99 bigfile.iso

# The receiver runs the same command; Croc detects partial data
# and resumes from where it left off.

Resume works by storing a hash of partial data on disk at ~/.cache/croc/. When a transfer with the same code and same file hash is detected, Croc skips already-transmitted chunks.

Note: Resume requires the relay to stay online. If the relay is restarted, the partial data on the cache is lost and the transfer must start from scratch.

Aliases

Common shell aliases to speed up daily use:

# Sending shortcut
alias send='croc send --code mycode99'

# Receive shortcut with output directory
alias get='croc --out ~/Downloads'

# Send with compression
alias sendzip='croc send --zip'

# Send text directly
alias sendtxt='croc send --text'

# Custom relay alias
alias croc-local='croc --relay 192.168.1.50:9009'

Add these to ~/.bashrc or ~/.zshrc and reload with source ~/.bashrc.

Configuration File

Croc reads ~/.config/croc/croc.yaml if it exists:

# ~/.config/croc/croc.yaml
relay: "192.168.1.50:9009"
relay_password: "relay-secret"
out: "~/Downloads"
yes: true
internal: false
disable_local: false

Troubleshooting

IssueSolution
Can't connect to relayCheck if croc6.schollz.com:9009 is reachable. Try nc -zv croc6.schollz.com 9009. Use a custom relay if the public one is blocked.
Code phrase rejectedCodes require at least 6 characters, at least one number. Avoid special characters that might be hard to communicate verbally.
Slow transfer with relayP2P failed, falling back to relay. Check NAT situation: croc send --no-p2p will use relay only. For large files, run a relay on a machine with good bandwidth.
File corruptionCroc verifies SHA-256 checksums. If mismatch, re-run. Check disk space on both ends.
Transfer stops at X%Either peer lost connectivity. Re-run with the same code; Croc will resume if the cache is intact.
Firewall blocks the transferCroc uses TCP ports 9009 (relay) and random high ports for P2P. Ensure outbound TCP is allowed. Use --relay-only if P2P UDP is blocked.
