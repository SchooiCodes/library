Advanced Linux Networking Tools

ip, ss, nmcli, bridge, tc -- advanced network configuration and troubleshooting on Linux.

iproute2: The Modern Network Stack

The iproute2 suite replaces legacy tools (ifconfig, route, arp, netstat) with a unified interface. The ip command manages addresses, routes, links, and neighbors:

# Address management
ip addr show                     # List all interfaces
ip addr add 192.168.1.10/24 dev eth0  # Assign IP
ip addr del 192.168.1.10/24 dev eth0  # Remove IP

# Link management
ip link set eth0 up/down         # Enable/disable interface
ip link set eth0 mtu 9000        # Jumbo frames

# Route management
ip route show                    # View routing table
ip route add default via 192.168.1.1
ip route add 10.0.0.0/8 via 192.168.1.254

# Neighbor (ARP) table
ip neigh show
ip neigh del 192.168.1.5 dev eth0

Socket Statistics with ss

ss is the modern replacement for netstat. It reads socket information directly from the kernel for faster, more detailed output:

# List all listening TCP sockets
ss -tlnp

# List all established connections
ss -tunap

# Filter by port
ss -tlnp 'sport = :22'
ss -tunap 'dport = :443'

# Show process using socket
ss -tlnp | grep nginx

# Unix domain sockets
ss -xlp

# Socket statistics by state
ss -s
# Total: 1234 (kernel 5678)
# TCP:   456 (estab 234, closed 123, orphaned 5, synrecv 0, ...)

Network Management with nmcli

nmcli is the command-line interface to NetworkManager. It controls network connections and devices:

# Device status
nmcli device status
nmcli device show wlan0

# Connection management
nmcli connection show
nmcli connection up "My WiFi"
nmcli connection down "My WiFi"

# Create new connection
nmcli connection add type wifi con-name "Office" \
    ifname wlan0 ssid "Office-5G" \
    wifi-sec.key-mgmt wpa-psk wifi-sec.psk "password"

# Modify connection
nmcli connection modify "Office" ipv4.method manual \
    ipv4.addresses 192.168.1.100/24 \
    ipv4.gateway 192.168.1.1 \
    ipv4.dns 8.8.8.8

# Hotspot
nmcli device wifi hotspot ifname wlan0 ssid "MyHotspot" password "guest1234"

Linux Bridge & Bonding

Create virtual bridges for VMs/containers and bond interfaces for redundancy:

# Create a bridge
ip link add name br0 type bridge
ip link set br0 up
ip addr add 10.0.100.1/24 dev br0
ip link set eth0 master br0
ip link set tap0 master br0

# Bridge management
bridge link show
bridge fdb show  # Forwarding database

# Bonding (NIC teaming)
ip link add bond0 type bond mode 802.3ad
ip link set eth0 master bond0
ip link set eth1 master bond0
ip addr add 192.168.1.10/24 dev bond0

# View bond status
cat /proc/net/bonding/bond0

Traffic Control (tc) -- Shaping & Queuing

Traffic control manages packet queuing, shaping, and scheduling:

# Add root qdisc (HTB -- Hierarchical Token Bucket)
tc qdisc add dev eth0 root handle 1: htb default 30

# Create classes
tc class add dev eth0 parent 1: classid 1:1 htb rate 100mbit
tc class add dev eth0 parent 1:1 classid 1:10 htb rate 10mbit ceil 20mbit
tc class add dev eth0 parent 1:1 classid 1:20 htb rate 30mbit ceil 40mbit

# Add filters to classify traffic
tc filter add dev eth0 protocol ip parent 1:0 prio 1 \
    u32 match ip dport 80 0xffff flowid 1:10
tc filter add dev eth0 protocol ip parent 1:0 prio 2 \
    u32 match ip dst 192.168.1.0/24 flowid 1:20

# Show tc stats
tc -s qdisc show dev eth0
tc -s class show dev eth0
