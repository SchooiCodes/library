Kubernetes: Pods, Services & Deployments

Deep dive into core Kubernetes objects with practical YAML examples.

Core Kubernetes Objects

Pod — Smallest deployable unit. Service — Stable network endpoint. Deployment — Declarative updates. ConfigMap — Non-sensitive config. Secret — Sensitive data.

Pods

apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
  labels:
    app: nginx
spec:
  containers:
  - name: nginx
    image: nginx:1.25
    ports:
    - containerPort: 80

Deployments

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.25

Services

ClusterIP (internal), NodePort (on each node), LoadBalancer (cloud LB), ExternalName (DNS alias).

Ingress Controller

Ingress routes HTTP/S traffic to services. Popular controllers: nginx-ingress, traefik, HAProxy, AWS ALB.

ConfigMaps & Secrets

ConfigMaps store non-sensitive config (env vars, config files). Secrets store sensitive data (base64 encoded). Both are mounted as volumes or env vars.
