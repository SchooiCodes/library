802.1Q VLAN tagging, trunk ports, access ports, inter-VLAN routing, and troubleshooting.

What are VLANs?

A Virtual Local Area Network (VLAN) is a logical grouping of devices on the same broadcast domain, even if they are physically connected to different switches. VLANs segment networks without requiring separate physical infrastructure, improving security, performance, and manageability.

Key benefits:

  • Traffic isolation -- broadcast traffic stays within the VLAN
  • Security -- sensitive devices isolated from general traffic
  • Simplified moves and changes -- re-VLANing a port is a configuration change, not a physical re-cable
  • Cost savings -- fewer switches and routers needed

802.1Q Tagging & Trunking

802.1Q is the IEEE standard for VLAN tagging. Frames traversing a trunk port carry a 4-byte tag identifying the VLAN:

# Access port -- untagged, assigned to single VLAN
interface GigabitEthernet0/1
 switchport mode access
 switchport access vlan 10
 spanning-tree portfast

# Trunk port -- carries multiple VLANs with tags
interface GigabitEthernet0/24
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk native vlan 99  # Untagged traffic (management)
 switchport trunk allowed vlan 10,20,30,100-200
 switchport nonegotiate  # Disable DTP

# Native VLAN -- frames without tags belong to this VLAN
# Important: Native VLAN must match on both ends of trunk

# Check trunk status
show interfaces trunk
show interfaces GigabitEthernet0/24 switchport

Inter-VLAN Routing (Router-on-a-Stick & Layer 3 Switch)

Devices in different VLANs must route through a Layer 3 device:

# Router-on-a-Stick (subinterfaces on router)
interface GigabitEthernet0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0
!
interface GigabitEthernet0/0.20
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0

# Layer 3 Switch (SVI -- Switched Virtual Interface)
interface Vlan10
 ip address 192.168.10.1 255.255.255.0
 no shutdown
!
interface Vlan20
 ip address 192.168.20.1 255.255.255.0
 no shutdown
!
ip routing  # Enable IP routing on the switch

VLAN Design Best Practices

Design a scalable VLAN architecture:

# Recommended VLAN numbering scheme:
# 1      -- Default VLAN (untagged, avoid using)
# 10-19  -- User VLANs (data)
# 20-29  -- Voice VLANs (VoIP)
# 30-39  -- Management (switches, APs, IPMI)
# 40-49  -- Guest/Public WiFi
# 50-59  -- Server/DMZ
# 60-69  -- IoT/OT devices
# 99     -- Native/Blackhole VLAN
# 100-199 -- Customer VLANs (MSP)
# 999    -- Unused ports (shut down)

# VTP (VLAN Trunking Protocol)
vtp mode transparent  # Best practice: transparent mode

# Private VLANs (PVLAN) -- micro-segmentation within a VLAN
# Primary VLAN: carries all traffic
# Community VLAN: ports can communicate within the community
# Isolated VLAN: ports can only communicate with promiscuous port

VLAN Troubleshooting

Common VLAN issues and diagnostic commands:

# Check VLAN database
show vlan brief
show vlan id 10

# Verify trunk status
show interfaces trunk
show interfaces GigabitEthernet0/24 trunk

# Check MAC address table
show mac address-table vlan 10

# Common issues:
# 1. Trunk not forming -- check native VLAN mismatch
# 2. Port stuck in blocking -- RSTP: verify port roles
# 3. VLAN not in database -- create VLAN: vlan 10
# 4. Allowed VLAN list missing -- check "switchport trunk allowed vlan"
# 5. ACL blocking traffic -- verify inter-VLAN ACLs

# Debug spanning tree
show spanning-tree vlan 10
debug spanning-tree events
