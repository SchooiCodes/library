Driver Updates on Linux

Manage hardware drivers on Linux — kernel modules, firmware, NVIDIA/AMD GPUs, and peripheral drivers.

Table of Contents (5 sections)

  • GPU Drivers: NVIDIA
  • GPU Drivers: AMD

Linux Driver Model

Linux drivers are either built into the kernel, loaded as kernel modules, or provided by firmware blobs. Most hardware works out-of-the-box thanks to in-kernel drivers.

Key locations: /lib/modules/$(uname -r)/ (kernel modules), /lib/firmware/ (firmware blobs), /etc/modprobe.d/ (module configuration).

List loaded modules: lsmod. Get hardware info: lspci -k, lsusb, lscpu. Check driver in use: lspci -v -s 01:00.0.

GPU Drivers: NVIDIA

NVIDIA offers proprietary and open-source (nvidia-open) drivers:

# Arch — nvidia or nvidia-open
sudo pacman -S nvidia nvidia-utils nvidia-settings
# For Wayland, install:
sudo pacman -S egl-wayland

# Ubuntu/Debian
# Use the graphics-drivers PPA
sudo add-apt-repository ppa:graphics-drivers/ppa
sudo apt install nvidia-driver-570

# Check which driver your GPU uses
nvidia-smi  # GPU name, driver version, CUDA version

For laptop Optimus setups: nvidia-prime (Arch), nvidia-prime-select (Ubuntu), or supergfxctl (ASUS ROG).

GPU Drivers: AMD

AMD GPU drivers are open-source and part of the kernel (amdgpu). Just install Mesa:

# Arch
sudo pacman -S mesa lib32-mesa vulkan-radeon lib32-vulkan-radeon
# Ubuntu/Debian
sudo apt install mesa-vulkan-drivers mesa-utils

# For ROCm (AMD's CUDA alternative for compute)
yay -S rocm-hip-sdk rocm-opencl-sdk
# Verify
rocm-smi  # GPU info and metrics

AMD GPUs are plug-and-play — no proprietary driver needed for most use cases.

Wi-Fi, Bluetooth & Peripheral Drivers

Intel Wi-Fi: works out-of-box (iwlwifi module). Realtek: may need rtl88x2bu-dkms or rtl88x2cu-dkms from AUR/git.

Bluetooth: bluez (daemon), bluez-utils (bluetoothctl), blueman (GUI). Enable: sudo systemctl enable --now bluetooth.

Printers: CUPS + cups-pdf + manufacturer PPD. Scanner: sane-utils, simple-scan. Gamepads: joycond (Switch Pro), xpadneo (Xbox).

Firmware Updates & DKMS

Linux firmware is distributed via linux-firmware package. Update system firmware with fwupd:

sudo pacman -S fwupd  # Install
sudo fwupdmgr refresh  # Update metadata
sudo fwupdmgr get-updates  # List available firmware updates
sudo fwupdmgr update  # Apply updates

# DKMS — auto-rebuild modules on kernel update
sudo pacman -S dkms
# Install a DKMS module (e.g., nvidia-dkms, virtualbox-dkms)
sudo pacman -S nvidia-dkms

DKMS ensures third-party kernel modules are rebuilt automatically when the kernel updates.
