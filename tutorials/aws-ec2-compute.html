AWS EC2 & Compute

Deep dive into AWS EC2 -- instance types, storage, networking, and optimization.

Table of Contents (2 sections)

  • Instances & AMIs
  • EBS & Networking

Instances & AMIs

EC2 instance types: naming (m5.xlarge -- family, generation, size). Families: general (t3, m5, m6i), compute (c5, c6i), memory (r5, r6i, x2gd), storage (i3, d2), GPU (p3, p4, g5). Pricing: On-Demand, Reserved (1/3yr, partial/upfront), Spot (up to 90% off, interruptions). Placement groups: cluster (low latency), spread (hardware isolation), partition (distributed). AMI: Amazon Machine Image -- EBS-backed vs instance-store. User data: scripts run at launch (cloud-init). Hibernate: preserve RAM state. Nitro: next-gen hypervisor (better perf, security).

EBS & Networking

EBS: block storage -- gp3 (baseline 3000 IOPS, 125 MB/s), io2 (provisioned IOPS, 64k max), st1 (throughput, HDD), sc1 (cold HDD). Snapshots: incremental, EBS Snapshots Archive. Instance store: ephemeral, lost on stop/terminate. ENI: elastic network interface -- separate subnets, failover. ENA: enhanced networking (up to 100 Gbps). EFA: elastic fabric adapter (HPC/ML, OS bypass). Elastic IP: static public IPv4. EC2 Auto Scaling: launch template, min/max/desired, scaling policies (target tracking, step, simple). Lifecycle hooks: pause instances during scale events. EC2 Image Builder: automate AMI creation. Instance metadata: http://169.254.169.254/latest/meta-data/. Systems Manager: SSM agent, Run Command, Patch Manager, Session Manager.
