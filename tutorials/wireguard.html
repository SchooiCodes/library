What is WireGuard?

WireGuard is a modern VPN protocol designed for simplicity, speed, and security. It was created by Jason A. Donenfeld and merged into the Linux kernel in 2020. It consists of only ~4,000 lines of code — compared to ~400,000+ for OpenVPN and ~600,000+ for IPsec.

Key advantages:

  • Extremely fast — minimal overhead, kernel-level performance
  • Cryptographically sound — uses modern primitives (Curve25519, ChaCha20, Poly1305, BLAKE2s)
  • Auditable — tiny codebase makes security auditing practical
  • Simple configuration — no certificates, no CA, no complex config
  • Built into Linux — part of the kernel since 5.6

How It Works

WireGuard uses a simple concept: each peer has a private key and a public key. Peers authenticate each other by exchanging public keys (like SSH). There is no central server — WireGuard is peer-to-peer by design.

A WireGuard interface is a virtual network adapter with an IP address and a private key. Traffic is encrypted and sent to other peers based on their public key and endpoint (IP:port).

Key concepts:

  • Interface — the virtual VPN adapter (e.g., wg0)
  • Peer — another device you connect to
  • Endpoint — the public IP:port of a peer
  • AllowedIPs — which IP ranges to route through the tunnel
  • PersistentKeepalive — keeps NAT mappings alive (needed for peers behind NAT)

Server Configuration

Generate keys and create the server config:

# Generate keys
wg genkey | tee server_private.key | wg pubkey > server_public.key
chmod 600 server_private.key

# /etc/wireguard/wg0.conf
[Interface]
Address = 10.0.0.1/24
ListenPort = 51820
PrivateKey = <contents of server_private.key>

# Enable IP forwarding
sudo sysctl -w net.ipv4.ip_forward=1
sudo sysctl -w net.ipv6.conf.all.forwarding=1

# Make forwarding permanent
echo "net.ipv4.ip_forward = 1" | sudo tee -a /etc/sysctl.conf
echo "net.ipv6.conf.all.forwarding = 1" | sudo tee -a /etc/sysctl.conf
Tip: Use a different subnet than your LAN to avoid routing conflicts. Common choices: 10.0.0.0/24, 172.16.0.0/24, or 192.168.2.0/24.

Client Configuration

First add the client as a peer on the server:

# Append to /etc/wireguard/wg0.conf (on the server)
[Peer]
# Client 1 — laptop
PublicKey = <client1_public.key>
AllowedIPs = 10.0.0.2/32

[Peer]
# Client 2 — phone
PublicKey = <client2_public.key>
AllowedIPs = 10.0.0.3/32

Then create the client config (on each client):

# /etc/wireguard/wg0.conf (on the client)
[Interface]
Address = 10.0.0.2/24
PrivateKey = <client1_private.key>
DNS = 1.1.1.1

[Peer]
PublicKey = <server_public.key>
Endpoint = your-server.com:51820
AllowedIPs = 0.0.0.0/0, ::/0
PersistentKeepalive = 25

Routing & NAT

To route all internet traffic through the VPN, enable masquerading on the server:

# Add iptables rules for NAT (in wg0.conf)
# These go in the [Interface] section:
PostUp = iptables -A FORWARD -i wg0 -j ACCEPT
PostUp = iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
PostDown = iptables -D FORWARD -i wg0 -j ACCEPT
PostDown = iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE

If you want to route only local network traffic (split tunnel), change AllowedIPs on the client:

# Only route 10.0.0.0/24 through the VPN
AllowedIPs = 10.0.0.0/24

# Route home LAN (192.168.1.0/24) through VPN
AllowedIPs = 10.0.0.0/24, 192.168.1.0/24

Quick Start with wg-quick

# Start the VPN
sudo wg-quick up wg0

# Check status
sudo wg show

# Stop
sudo wg-quick down wg0

# Enable at boot
sudo systemctl enable wg-quick@wg0 --now

# View all peers and their transfer stats
sudo wg show wg0
# Output:
# interface: wg0
#   public key: ...
#   private key: (hidden)
#   listening port: 51820
#
# peer: (client1 public key)
#   endpoint: 203.0.113.5:51820
#   allowed ips: 10.0.0.2/32
#   latest handshake: 1 minute ago
#   transfer: 1.2 GiB received, 3.4 GiB sent

Mobile Clients

WireGuard has official apps for iOS and Android. Configuration is done by QR code or file:

# Generate a QR code for a client config
sudo apt install qrencode
qrencode -t ansiutf8 < client.conf

# Or save to file
qrencode -o client.png < client.conf

The WireGuard mobile app can scan the QR code to import the full configuration instantly. You can also share config files via AirDrop, email, or a temporary web server.

Troubleshooting

IssueFix
Handshake not happeningCheck firewall: UDP port 51820 must be open. sudo ufw allow 51820/udp.
Can't reach internet through VPNEnsure IP forwarding is enabled and NAT rules are correct. Check PostUp in server config.
DNS not workingAdd DNS = 1.1.1.1 to the client [Interface] and ensure resolvconf or systemd-resolved is installed.
Peer behind NATAdd PersistentKeepalive = 25 to the client config to keep the connection alive.
Key mismatchRegenerate keys with umask 077; wg genkey | tee privatekey | wg pubkey > publickey.
