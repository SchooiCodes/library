🎨 CSS Animations Guide

Create smooth CSS animations — transitions, keyframes, transforms, performance tips, and practical examples.

CSS Transitions

Transitions smoothly change property values over a specified duration. Triggered by state changes like hover or focus.

.button {
 background-color: blue;
 color: white;
 transition: background-color 0.3s ease, transform 0.2s ease;
}

.button:hover {
 background-color: darkblue;
 transform: scale(1.05);
}

/* Shorthand: property duration timing-function delay */
transition: all 0.3s ease-in-out 0s;

CSS Transforms

Transforms modify elements in 2D or 3D space without affecting layout:

.box {
 transform: translateX(50px) rotate(45deg) scale(1.2);
 /* translate — move element */
 /* rotate — spin (deg) */
 /* scale — grow/shrink */
 /* skew — slant */
}

/* 3D transforms */
.card {
 transform: perspective(800px) rotateY(15deg);
 transform-style: preserve-3d;
}

/* Multiple transforms combine in order */
transform: translate(10px, 20px) rotate(45deg) scale(0.9);

Keyframe Animations

Define complex multi-step animations using @keyframes:

@keyframes slide-in {
 from {
  opacity: 0;
  transform: translateX(-100px);
 }
 to {
  opacity: 1;
  transform: translateX(0);
 }
}

@keyframes bounce {
 0%, 100% { transform: translateY(0); }
 50% { transform: translateY(-30px); }
}

.animated-element {
 animation: slide-in 0.5s ease-out forwards;
 /* animation: name duration timing-function fill-mode */
}

.bounce-element {
 animation: bounce 1s ease-in-out infinite;
}
Tip: Use forwards as the fill-mode to keep the end state of the animation after it completes. Use infinite sparingly — constant motion can be distracting.

Animation Properties

PropertyDescription
animation-nameName of the @keyframes rule
animation-durationHow long one cycle takes (e.g., 0.5s, 2s)
animation-timing-functionSpeed curve (ease, linear, ease-in-out, cubic-bezier)
animation-delayDelay before starting
animation-iteration-countNumber of cycles (1, 3, infinite)
animation-directionnormal, reverse, alternate, alternate-reverse
animation-fill-modenone, forwards, backwards, both
animation-play-staterunning, paused (for play/pause controls)

Performance Tips

For smooth 60fps animations, only animate these properties:

/* GPU-accelerated (cheap): */
transform: translate, scale, rotate, skew
opacity

/* Expensive (triggers layout/reflow): avoid */
width, height, margin, padding, top, left, font-size
/* These trigger layout + paint — janky animations */

/* Example: animate position with transform instead of left/top */
/* Bad: */
.box { left: 0; transition: left 0.3s; }
.box.active { left: 100px; }

/* Good: */
.box { transform: translateX(0); transition: transform 0.3s; }
.box.active { transform: translateX(100px); }

Other performance tips:

  • Use will-change to hint GPU acceleration: will-change: transform, opacity
  • Keep animations under ~500ms for UI feedback, up to 2s for decorative effects
  • Test on low-end devices — a smooth animation on desktop may stutter on mobile
  • Use requestAnimationFrame for JS-driven animations instead of setInterval

Practical Examples

/* Fade in on load */
.fade-in {
 opacity: 0;
 animation: fadeIn 0.5s ease forwards;
}
@keyframes fadeIn {
 to { opacity: 1; }
}

/* Skeleton loading shimmer */
.skeleton {
 background: linear-gradient(90deg, #eee 25%, #f5f5f5 50%, #eee 75%);
 background-size: 200% 100%;
 animation: shimmer 1.5s infinite;
}
@keyframes shimmer {
 0% { background-position: 200% 0; }
 100% { background-position: -200% 0; }
}

/* Pulse ring (for notifications) */
.pulse-ring {
 animation: pulse 2s infinite;
}
@keyframes pulse {
 0% { box-shadow: 0 0 0 0 rgba(0, 150, 255, 0.4); }
 70% { box-shadow: 0 0 0 15px rgba(0, 150, 255, 0); }
 100% { box-shadow: 0 0 0 0 rgba(0, 150, 255, 0); }
}
