Chitralaya

Self-hosted lightweight image gallery and media server for organizing and sharing photos.

Overview

Chitralaya (Sanskrit for "picture house") is a lightweight, self-hosted image gallery server designed for individuals who want full control over their media. Unlike cloud services like Google Photos, Chitralaya runs on your own hardware — a Raspberry Pi, a VPS, or a local server — and serves your photo library over the web with zero tracking, no ads, and no data mining.

It supports common image formats (JPEG, PNG, WebP, AVIF, GIF), video thumbnails, EXIF metadata parsing, album organization, and responsive streaming. The entire server can run in under 64 MB of RAM, making it ideal for low-power devices.

Installation

Via Go (Recommended)

go install github.com/chitralaya/gallery@latest

Ensure $GOPATH/bin is in your PATH. Verify with:

chitralaya --version

Via pip (Python)

pip install chitralaya

Requires Python 3.9+. On Linux, you may need pip3. For a virtual environment:

python3 -m venv chitralaya-env
source chitralaya-env/bin/activate
pip install chitralaya

Prebuilt Binaries

Download the latest release from the GitHub releases page. Choose your platform:

# Linux amd64
wget https://github.com/chitralaya/gallery/releases/latest/download/chitralaya-linux-amd64.tar.gz
tar -xzf chitralaya-linux-amd64.tar.gz
sudo mv chitralaya /usr/local/bin/

Configuration

Chitralaya uses a YAML config file. By default it looks for ~/.config/chitralaya/config.yaml. Create it:

mkdir -p ~/.config/chitralaya
touch ~/.config/chitralaya/config.yaml

Example configuration:

server:
  host: "0.0.0.0"
  port: 8080
  base_url: "https://gallery.example.com"

storage:
  data_dir: "/var/lib/chitralaya"
  thumbnails_dir: "/var/lib/chitralaya/thumbnails"
  originals_dir: "/var/lib/chitralaya/originals"

database:
  type: "sqlite"
  path: "/var/lib/chitralaya/gallery.db"

gallery:
  title: "My Photo Gallery"
  description: "Self-hosted photo collection"
  albums_per_page: 20
  enable_uploads: true
  max_file_size: 50  # MB

thumbnails:
  quality: 85
  max_width: 400
  max_height: 300
  format: "webp"

auth:
  enabled: true
  method: "htpasswd"
  htpasswd_file: "/etc/chitralaya/.htpasswd"
Important: Always set auth.enabled: true when exposing Chitralaya to the internet. Use htpasswd to generate credentials: htpasswd -c /etc/chitralaya/.htpasswd admin.

Reverse Proxy with Nginx

server {
    listen 443 ssl;
    server_name gallery.example.com;

    ssl_certificate /etc/letsencrypt/live/gallery.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/gallery.example.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        client_max_body_size 100M;
    }
}

Uploading Media

Web Interface

Navigate to http://your-server:8080 and log in. Use the Upload button in the top-right corner to select files. You can drag-and-drop entire folders.

CLI Upload

chitralaya upload --album "Vacation 2026" ~/Photos/beach.jpg

Upload a directory recursively:

chitralaya upload --album "Vacation 2026" --recursive ~/Photos/vacation/

Supported flags:

FlagDescription
--albumTarget album name (created if missing)
--recursiveWalk directories recursively
--qualityJPEG export quality (1-100, default 90)
--resizeMax dimension for resize (e.g. 1920)
--thumb-onlyRegenerate thumbnails only

API Upload (curl)

curl -X POST https://gallery.example.com/api/upload \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -F "album=Vacation 2026" \
  -F "file=@~/Photos/sunset.jpg"

Accessing Gallery

Once running, open http://localhost:8080 in your browser. Features include:

  • Album view — Grid thumbnails with lazy loading
  • Slideshow — Full-screen with keyboard navigation
  • EXIF data — Camera model, aperture, ISO, GPS coordinates
  • Search — By filename, date, album, or EXIF tags
  • Share links — Generate expiring public links for individual photos
  • Responsive — Mobile-first design, touch gestures

To access from other devices on your LAN, find your server's local IP:

ip addr show | grep 'inet '

# Example output
# inet 192.168.1.100/24 brd 192.168.1.255 scope global eth0

Then visit http://192.168.1.100:8080 from any device on your network.

Docker Setup

docker run -d \
  --name chitralaya \
  -p 8080:8080 \
  -v /path/to/photos:/var/lib/chitralaya/originals \
  -v /path/to/config:/etc/chitralaya \
  -v /path/to/thumbnails:/var/lib/chitralaya/thumbnails \
  -e CHITRALAYA_CONFIG=/etc/chitralaya/config.yaml \
  chitralaya/gallery:latest

Docker Compose

version: "3.8"
services:
  chitralaya:
    image: chitralaya/gallery:latest
    container_name: chitralaya
    ports:
      - "8080:8080"
    volumes:
      - ./photos:/var/lib/chitralaya/originals
      - ./config:/etc/chitralaya
      - ./thumbnails:/var/lib/chitralaya/thumbnails
      - chitralaya-db:/var/lib/chitralaya
    environment:
      - CHITRALAYA_CONFIG=/etc/chitralaya/config.yaml
    restart: unless-stopped

volumes:
  chitralaya-db:

CLI Tools

Chitralaya ships with a handful of CLI utilities for batch operations:

# Rebuild all thumbnails after changing config
chitralaya thumbnails rebuild

# Import from a Google Photos Takeout export
chitralaya import google-takeout ~/Downloads/takeout.zip --album "Google Import"

# Export an album as zip
chitralaya export --album "Vacation 2026" --output vacation.zip

# List all albums and photo counts
chitralaya albums list

# Delete album and contents
chitralaya albums delete --album "Old Backup" --confirm

# Database backup
chitralaya db backup --output ~/backups/chitralaya-$(date +%F).sqlite

# User management
chitralaya users add --username alice --password
chitralaya users list
chitralaya users remove --username bob

Backup Your Gallery

#!/bin/bash
# Full backup script
BACKUP_DIR=~/backups/chitralaya
DATE=$(date +%F)

mkdir -p "$BACKUP_DIR"

# Backup database
chitralaya db backup --output "$BACKUP_DIR/db-$DATE.sqlite"

# Backup config
cp ~/.config/chitralaya/config.yaml "$BACKUP_DIR/config-$DATE.yaml"

# Rsync originals
rsync -avh /var/lib/chitralaya/originals/ "$BACKUP_DIR/originals-$DATE/"

# Compress
tar -czf "$BACKUP_DIR/chitralaya-full-$DATE.tar.gz" \
  "$BACKUP_DIR/db-$DATE.sqlite" \
  "$BACKUP_DIR/config-$DATE.yaml" \
  "$BACKUP_DIR/originals-$DATE/"

echo "Backup complete: $BACKUP_DIR/chitralaya-full-$DATE.tar.gz"

Troubleshooting

IssueSolution
Server won't start — port in useChange port in config.yaml or run sudo lsof -i :8080 to find the process
Thumbnails not generatingCheck ~/.config/chitralaya/config.yaml thumbnails section. Ensure ffmpeg is installed for video support.
Upload fails — file too largeIncrease max_file_size in config and client_max_body_size in nginx
Images load slowlyEnable thumbnail caching. Set thumbnails.format: "webp" for smaller files.
Login prompt loopsThe base_url in config must match the URL you're accessing. Check reverse proxy headers.
EXIF data missingInstall libimage-exiftool-perl: sudo apt install exiftool
Docker permissions errorEnsure mounted volumes have proper ownership: chown -R 1000:1000 /path/to/photos
