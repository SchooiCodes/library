Redis Deep Dive

Advanced Redis — data structures, persistence, clustering, and caching patterns.

Table of Contents (2 sections)

  • Data Structures
  • Persistence & Clustering

Data Structures

Redis data structures: String (caching, counters), List (queues, stacks — LPUSH, RPOP, BLPOP blocking), Set (unique items — SADD, SISMEMBER, SUNION), Sorted Set (leaderboards — ZADD, ZRANGEBYSCORE, ZREVRANK), Hash (objects — HSET, HGETALL), Bitmap (tracking — SETBIT, BITCOUNT), HyperLogLog (cardinality — PFADD, PFCOUNT), Stream (message queue — XADD, XREAD, XREADGROUP). Geospatial: GEOADD, GEORADIUS for location-based queries.

Persistence & Clustering

Persistence: RDB (snapshots — save 900 1), AOF (append-only log — everysec, always), or both. Cluster: automatic sharding (16384 slots), replication (master-replica per shard), no centralized proxy. Sentinel: HA monitoring — auto-failover, master detection, client redirection. Redis Stack: RediSearch (full-text search), RedisJSON (native JSON), RedisTimeSeries, RedisBloom (probabilistic data structures — bloom filter, cuckoo filter).
