🧪 API Testing Guide

Test REST and GraphQL APIs — curl, Postman/Insomnia, HTTPie, status codes, auth testing, and automation.

REST API Basics

REST APIs use HTTP methods to perform CRUD operations on resources:

MethodActionExample
GETRead a resourceGET /api/users
POSTCreate a resourcePOST /api/users
PUTUpdate a resource (full)PUT /api/users/1
PATCHPartial updatePATCH /api/users/1
DELETEDelete a resourceDELETE /api/users/1

Responses include a status code, headers, and a body (usually JSON).

Testing with curl

curl is the universal command-line HTTP client. Installed on almost every system.

# GET request
curl https://api.github.com/users/octocat

# POST with JSON body
curl -X POST https://jsonplaceholder.typicode.com/posts \
 -H "Content-Type: application/json" \
 -d '{"title":"foo","body":"bar","userId":1}'

# Include headers in response
curl -i https://api.example.com/users

# Follow redirects
curl -L https://example.com

# Silent mode (no progress bar)
curl -s https://api.example.com

# Output to file
curl -o output.json https://api.example.com/data

Postman & Insomnia

GUI tools for building, testing, and documenting APIs. Both support:

  • Collections — organize requests by project
  • Environments — switch between dev/staging/production variables
  • Pre-request & test scripts (JavaScript)
  • Automated test runs (Newman for Postman, CLI for Insomnia)
  • Code generation — export requests as curl, Python, JS, etc.
Tip: Insomnia is lighter and faster than Postman, and it handles GraphQL natively. Postman has more features for team collaboration and test automation.

HTTPie

A user-friendly curl alternative with a cleaner syntax and colored output:

# Install: apt install httpie or brew install httpie

# GET (http is shorthand)
http https://api.github.com/users/octocat

# POST with JSON
http POST https://jsonplaceholder.typicode.com/posts \
 title=foo body=bar userId:=1

# Use --json or --form for content type
http --form POST https://example.com/login \
 username=admin password=secret

GraphQL Testing

GraphQL uses a single endpoint with POST queries. Test with curl or a dedicated client:

# curl GraphQL query
curl -X POST https://api.github.com/graphql \
 -H "Authorization: Bearer YOUR_TOKEN" \
 -H "Content-Type: application/json" \
 -d '{"query":"query { viewer { login } }"}'

# With HTTPie
http POST https://api.github.com/graphql \
 Authorization:"Bearer YOUR_TOKEN" \
 query="query { viewer { login } }"

Use GraphiQL or Altair for interactive GraphQL exploration with auto-completion and schema docs.

Auth Testing

Test different authentication methods:

# Basic Auth
curl -u username:password https://api.example.com/secure

# Bearer token
curl -H "Authorization: Bearer YOUR_TOKEN" \
 https://api.example.com/secure

# API key in header
curl -H "X-API-Key: YOUR_KEY" \
 https://api.example.com/secure

# Cookie-based
curl -b "session=abc123" https://api.example.com/dashboard

Automation

Automate API tests with scripts or dedicated tools:

#!/bin/bash
# Simple bash test script
BASE="https://jsonplaceholder.typicode.com"

echo "=== Test GET users ==="
response=$(curl -s -o /dev/null -w "%{http_code}" $BASE/users)
[ "$response" = "200" ] && echo "PASS" || echo "FAIL (got $response)"

echo "=== Test POST ==="
response=$(curl -s -o /dev/null -w "%{http_code}" \
 -X POST -H "Content-Type: application/json" \
 -d '{"title":"test"}' $BASE/posts)
[ "$response" = "201" ] && echo "PASS" || echo "FAIL (got $response)"

For serious automation, use Newman (Postman CLI), Playwright, or pytest + requests.
