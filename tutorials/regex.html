🔍 Regular Expressions Guide

A practical guide to regex — patterns, syntax, tools, and real-world examples.

What are Regular Expressions?

A regular expression (regex or regexp) is a sequence of characters that defines a search pattern. Typically used for pattern matching, validation, text extraction, and find-and-replace operations. Regex is supported in nearly every modern programming language and in many command-line tools.

Regex engines come in two main flavors: PCRE (Perl Compatible Regular Expressions) used by most languages, and POSIX (used by tools like grep, sed, awk in basic mode). Most modern tools support PCRE with the -P flag.

Character Classes, Quantifiers, Anchors

# Character classes
[aeiou]     # Matches any vowel
[0-9]       # Matches any digit (same as \d)
[A-Za-z]    # Matches any letter
[^0-9]      # Matches anything NOT a digit

# Predefined classes
\d          # Digit [0-9]
\w          # Word character [A-Za-z0-9_]
\s          # Whitespace (space, tab, newline)
\D, \W, \S  # Negations of the above

# Quantifiers
{3}         # Exactly 3 times
{2,5}       # Between 2 and 5 times
{3,}        # 3 or more times
*           # 0 or more (same as {0,})
+           # 1 or more (same as {1,})
?           # 0 or 1 (same as {0,1})

# Anchors
^           # Start of string
$           # End of string
\b          # Word boundary
\B          # Non-word boundary

Groups, Alternation, Backreferences

# Groups: capture part of the match
(\d{3})-(\d{4})   # Captures area code and number separately

# Non-capturing group
(?:\d{3})-(\d{4}) # Only captures the second part

# Alternation (OR)
cat|dog           # Matches "cat" or "dog"

# Backreferences: refer to captured groups
(\w+)\s+\1        # Matches repeated words like "hello hello"

# Named groups (modern regex, e.g. Python, PCRE)
(?<year>\d{4})-(?<month>\d{2})

Backreferences are powerful for finding duplicates, matching paired tags, and parsing structured text:

<(\w+)>.*</\1>   # Matches paired HTML tags like <div>...</div>

Lookahead & Lookbehind

Lookaround assertions match a position without consuming characters. They come in positive and negative flavors:

# Positive lookahead: match only if followed by X
\d(?=px)    # Digit followed by "px" (matches "2" in "2px")

# Negative lookahead: match only if NOT followed by X
\d(?!px)    # Digit NOT followed by "px"

# Positive lookbehind: match only if preceded by X
(?<=\$)\d+  # Number preceded by $ (matches "100" in "$100")

# Negative lookbehind: match only if NOT preceded by X
(?<!\$)\d+  # Number NOT preceded by $

Example: Extract prices from text — (?<=\$)\d+(?:\.\d{2})? matches "$19.99" and "$5".

Tools & Commands

  • regex101.com — The best interactive regex tester with real-time explanation, syntax highlighting, and multiple engine support.
  • grep -P — Uses PCRE syntax for advanced pattern matching: grep -P '\d{3}-\d{4}' file.txt
  • sed — Stream editor for find-and-replace: sed -E 's/(\w+)@(\w+)/\1 at \2/g' file.txt
  • awk — Pattern scanning and processing: awk '/(error|fail)/ {print $0}' log.txt
  • ripgrep (rg) — Faster grep with PCRE support: rg -P 'pattern'
  • Python re module — Full-featured regex: import re; re.findall(r'\d+', text)
Best practice: Write and test your regex on regex101.com before using it in production. Always consider edge cases — what if the string is empty? What if there are multiple matches? What if the input contains special characters?

Regex in Programming Languages

Different languages have slight variations in regex syntax and API. Here is how to use regex in the most common languages:

JavaScript

const pattern = /^[a-z]+@[a-z]+\.[a-z]{2,}$/i;
pattern.test('Alice@Example.com'); // true

const text = 'My email is alice@example.com and bob@test.org';
const emails = text.match(/[\w.+-]+@[\w-]+\.[\w.]+/g);
// ['alice@example.com', 'bob@test.org']

// Replace
text.replace(/alice/gi, 'CHARLIE');

Python

import re

pattern = r'^[a-z]+@[a-z]+\.[a-z]{2,}$'
re.match(pattern, 'alice@example.com', re.IGNORECASE)

text = 'My email is alice@example.com'
re.findall(r'[\w.+-]+@[\w-]+\.[\w.]+', text)
# ['alice@example.com']

# Substitution
re.sub(r'alice', 'bob', text, flags=re.IGNORECASE)

Go

package main
import "regexp"

func main() {
    re := regexp.MustCompile(`^[a-z]+@[a-z]+\.[a-z]{2,}$`)
    re.MatchString("alice@example.com") // true
}

Rust

use regex::Regex;

let re = Regex::new(r"^[a-z]+@[a-z]+\.[a-z]{2,}$").unwrap();
re.is_match("alice@example.com"); // true

Important differences: JavaScript uses /pattern/flags syntax, Python uses r'...' raw strings, Java requires double escaping (\\d), and PCRE (PHP, R) supports features like recursive patterns ((?R)) that other engines do not.
