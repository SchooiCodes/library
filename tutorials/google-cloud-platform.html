Google Cloud Platform

GCP services — Compute Engine, Cloud Storage, BigQuery, and Kubernetes.

Compute Engine & GKE

Compute Engine provides VMs on GCP. Machine families: E2 (cost-optimized), N2/N4 (standard), C3 (compute), M3 (memory). Sole-tenant nodes for licensing compliance. Preemptible (24h max, no SLA) and Spot instances (no max time).

GKE (Google Kubernetes Engine) is the most advanced managed Kubernetes service. Features: Autopilot (fully managed, no node management), GKE clusters (standard), GKE Enterprise (advanced security, multi-cluster). Horizontal/Vertical Pod Autoscaler, Workload Identity (IAM for K8s), and Anthos (multi-cloud/hybrid).

Cloud Storage & Buckets

Cloud Storage is GCP's object storage. Storage classes: Standard, Nearline, Coldline, Archive (decreasing cost, increasing access latency). Key features: object versioning, lifecycle policies, IAM integration, customer-managed encryption keys (CMEK), and object holds.

Transfer Service moves data from AWS S3, HTTP(S), or on-premises. Cloud Storage FUSE mounts buckets as filesystems. gsutil is the CLI tool. Uniform bucket-level access replaces ACLs for simpler permission management.

BigQuery & Data Analytics

BigQuery is GCP's serverless, petabyte-scale data warehouse. It uses columnar storage, automatic sharding, and distributed query execution. SQL (standard SQL, GoogleSQL) is the query language. BigQuery separates storage and compute — you pay for storage separately from query processing.

Key features: BI Engine (in-memory acceleration), BigQuery ML (ML models in SQL), BigQuery Omni (cross-cloud queries), streaming inserts, and automatic expiration. Partition (by date) and cluster (by columns) tables for optimal query performance and cost.

Cloud Run & Serverless

Cloud Run is GCP's fully managed serverless container platform. Deploy any containerized application (HTTP/S) — it auto-scales to zero when idle and handles millions of requests. Pay only for CPU/memory used during request processing.

Cloud Functions (1st gen: Node.js, Python, Go; 2nd gen: Cloud Run-based, any runtime) provide event-driven serverless compute. Eventarc routes events from 60+ sources (Cloud Storage, Pub/Sub, Firestore) to Cloud Run or Cloud Functions.

GCP Networking & Security

VPC (Virtual Private Cloud) provides global networking. VPCs span regions globally — subnets are regional resources. Cloud Load Balancing (global anycast, one IP worldwide), Cloud CDN (global content delivery), and Cloud Armor (WAF, DDoS protection, OWASP rules).

Security: Cloud IAM (roles/permissions), Cloud KMS (key management), Secret Manager (API keys, passwords), VPC Service Controls (data exfiltration prevention), and Security Command Center (threat detection, vulnerability scanning).
