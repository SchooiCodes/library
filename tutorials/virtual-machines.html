💻 Virtual Machines Guide

A beginner's guide to virtual machines with VirtualBox and VMware.

What is a Virtual Machine?

A software-based computer that runs inside your real computer. A VM runs its own operating system, is isolated from the host, and is perfect for testing, learning, running different OSes, malware analysis (safely), and legacy software.

Why Use VMs?

  • Test Linux without leaving Windows
  • Safely try suspicious software
  • Run old Windows apps on new hardware
  • Learn about networking and servers
  • Create isolated development environments
  • Take snapshots and roll back changes

Option 1: VirtualBox (Free, Open-Source)

VirtualBox is a powerful x86 and AMD64/Intel64 virtualization product for enterprise as well as home use.

  • Download: https://www.virtualbox.org
  • Install: Default settings (may temporarily lose network)
  • Also download: Extension Pack (USB support, RDP)

Option 2: VMware Workstation Player (Free for Personal Use)

A free virtualization tool that's faster than VirtualBox with better GPU support and a simpler interface.

  • Download: https://www.vmware.com/products/workstation-player.html
  • Faster than VirtualBox, better GPU support
  • Simpler interface

Creating Your First VM (Linux Ubuntu Example)

  1. Download Ubuntu ISO: https://ubuntu.com/download
  2. In VirtualBox: Click "New"
  3. Name: "Ubuntu Test", Type: Linux, Version: Ubuntu (64-bit)
  4. Memory: 2048–4096 MB (2–4 GB)
  5. Hard disk: Create VDI, 20–50 GB, dynamically allocated
  6. Settings → Storage → Add ISO as optical drive
  7. Settings → Network → NAT (default, internet access)
  8. Start the VM, follow Ubuntu installer
  9. After install: Settings → General → Advanced → Shared Clipboard: Bidirectional. Settings → Shared Folders → Add shared folder.

Essential VM Settings

  • RAM: Minimum 2 GB for lightweight Linux, 4–8 GB for Windows guest
  • CPU: 2–4 cores depending on host
  • Video Memory: 128 MB minimum for smooth UI
  • Network: NAT (default, internet access), Bridged (VM gets own IP on network), Host-only (VM-to-host only), Internal (VM-to-VM only)
  • Storage: Dynamic allocation (grows as needed) vs Fixed (better performance, pre-allocates)
  • USB: Enable USB controller, add USB devices
  • Shared Folders: Folder accessible from both host and guest
  • Drag & Drop: Enable bidirectional for easy file transfer

Snapshots

Take a snapshot before making changes. Restore if something breaks. A snapshot acts as a restore point for your VM.

Useful VM Use Cases

  • Linux Learning: Install Ubuntu, Fedora, Arch to learn Linux
  • Windows Testing: Test Windows 11 preview builds
  • Malware Analysis: Isolated environment to safely run suspicious files (take snapshot first)
  • Development: Isolated dev environments per project
  • Server Simulation: Run multiple VMs to simulate a network
  • Old Software: Run Windows XP/7 for legacy applications

Performance Tips

  • Enable hardware virtualization in BIOS (VT-x/AMD-V)
  • Use SSD for VM storage
  • Allocate enough RAM (host should keep at least 4 GB)
  • Use Paravirtualization interface (VirtualBox: default is KVM)
  • Enable 3D acceleration if guest supports it
  • Install Guest Additions (VirtualBox) or VMware Tools for drivers and performance
  • Close unnecessary host apps while running VMs
