APK Sideloading Guide

Everything you need to know about installing Android apps from outside the Play Store — methods, tools, safety, and best practices.

Overview

APK sideloading means installing an Android application from a file (.apk or split APK bundle) rather than from an app store like Google Play. This is the foundation of Android modding: every modded app, every custom app store, and every tool in this library depends on sideloading.

Android has made sideloading progressively easier over the years. On Android 8+, you grant install permission per-app rather than flipping a global switch. On Android 14+, you can even sideload split APKs without third-party tools using the system's built-in package installer.

Safety Warning. Sideloading bypasses Google's Play Protect scanning. Always download APKs from trusted sources, verify signatures when possible, and scan files with an antivirus app before installing. This guide shows you how to stay safe.

Enabling Unknown Sources

The method for enabling sideloading depends on your Android version:

Android 8.0 — 9.0 (Oreo/Pie)

Settings > Apps & notifications > Special app access > Install unknown apps
Select the app you want to allow (e.g., Chrome, Files, Telegram)
Toggle "Allow from this source" ON

Android 10 — 13

Settings > Apps > Special app access > Install unknown apps
Or: Settings > Security > Install unknown apps
Select the app and toggle ON

Android 14+

Settings > Apps > Special app access > Install unknown apps
Same as Android 10-13, but with additional "Split APK install" support built into
the system installer for seamless bundle installation.

Samsung One UI

Settings > Biometrics and security > Install unknown apps
Or, when you try to install an APK, Samsung will prompt you to grant permission
to the file manager app you are using.

Xiaomi MIUI / HyperOS

Settings > Security & privacy > Unknown source installation
Or: Settings > Apps > Manage apps > [Your file manager] > Install via USB
MIUI may also require you to enable "Install via USB" in Developer options.
Tip. Grant install permission only to apps that need it (file managers, browsers, Telegram). Revoke it for apps that should not install APKs. This is a good security practice.

Split APK Installers (SAI)

Many apps from APKMirror, APKPure, and Aurora Store are distributed as split APKs (multiple .apk files or a single .apks bundle). The standard "Install" button cannot handle these — you need a split APK installer.

SAI (Split APKs Installer)

SAI is the most popular open-source split APK installer. It supports APKS, XAPK, and APKM formats.

# Download SAI from F-Droid:
# Search for "SAI" and install.

# Or from GitHub:
# https://github.com/Aefyr/SAI/releases

How to Use SAI

  1. Open SAI. It will show a list of your installed apps.
  2. Tap the Install APK button (folder icon) at the bottom.
  3. Navigate to the APK file(s). For bundles:
    • .apks files — Select the single .apks file.
    • .xapk files — Select the .xapk file (SAI will extract it).
    • Multiple .apk files — Select all parts together (they are usually named base.apk, split_config.xx.apk, etc.).
  4. Tap Install and confirm the system dialog.

SAI with Shizuku

If Shizuku is running, SAI can install silently without showing a system dialog for each installation:

# In SAI settings:
# Enable "Use Shizuku"
# Ensure Shizuku is running before installing.

ADB Install Method

ADB (Android Debug Bridge) lets you install APKs from your computer. This bypasses all "unknown sources" restrictions.

Prerequisites

# Enable Developer options and USB debugging on your phone.
# Install ADB on your computer.

# Windows: Download platform-tools from developer.android.com
# Linux: sudo apt install adb
# macOS: brew install android-platform-tools

# Verify the connection:
adb devices
# Your device should show as "device" (not "unauthorized").

Installing an APK

# Basic install:
adb install app.apk

# Install split APK bundle:
adb install-multiple base.apk split_config.xx.apk split_config.xxxhdpi.apk

# Replace an existing app (keep data):
adb install -r app.apk

# Install to SD card:
adb install -s app.apk

# Downgrade (install older version):
adb install -d app.apk

# Install with all permissions granted:
adb install -g app.apk

Wireless ADB (Android 11+)

# No USB cable needed:
adb connect IP_ADDRESS:PORT
# You get the IP and port from Developer options > Wireless debugging
adb install app.apk

Batch Install with ADB

# Install all APKs in a folder:
for f in *.apk; do adb install "$f"; done

# Or use a one-liner:
adb install-multiple *.apk

How to Verify an APK

  1. Check the signature. Compare the APK's signing certificate fingerprint with the official one (if available from the developer).
  2. Use APK Analyzer. Android Studio includes APK Analyzer. Online tools like www.apkanalyzer.net can also inspect APK contents.
  3. Check permissions. If a flashlight app requests "Read Contacts" and "Read SMS," do not install it.
  4. Scan with antivirus. Use Malwarebytes, Bitdefender, or Hypatia (F-Droid) to scan APK files before installing.
  5. Check the package name. Official WhatsApp is com.whatsapp. If an app claiming to be WhatsApp has package com.whatsapp.free, it is fake.

X-plore File Manager Method

X-plore is a dual-pane file manager for Android that includes a built-in APK installer with advanced features.

Installing APKs with X-plore

  1. Install X-plore from the Play Store or directly from https://www.lonelycatgames.com/xplore.
  2. Navigate to the folder containing your APK file(s).
  3. Tap the APK file. X-plore will show its details (permissions, package name, version).
  4. Tap Install.
  5. If you have Shizuku enabled, X-plore can install silently.

Batch Install in X-plore

  1. In X-plore, long-press the first APK to enter selection mode.
  2. Tap additional APKs to select them.
  3. Tap the APK icon in the bottom toolbar.
  4. Select "Install all".
  5. Each APK will install sequentially.

Verifying Signatures

Every APK is digitally signed by its developer. The signature proves the APK has not been tampered with. You can verify signatures to check authenticity.

Using ADB to Check Signature

# Check the signature of an installed app:
adb shell dumpsys package com.example.app | grep "signatures"

# Check signature of an APK file (using apksigner):
apksigner verify --print-certs app.apk

# Download apksigner from Android SDK build-tools.

Using APK Signer App on Android

# Install "APK Signer" from F-Droid or GitHub.
# Open the app and select an APK file.
# Tap "View Certificate" to see the MD5, SHA-1, and SHA-256 fingerprints.
# Compare these with the official developer's published fingerprints.

What to Compare

  • Google apps — Signed by "Google Inc." with a specific certificate fingerprint.
  • WhatsApp — Signed by "WhatsApp Inc."
  • Signal — Signed by "Signal Messenger LLC."
  • If you download from APKMirror, their site lists the expected signature for each APK. Compare against that.
Split APK Signatures. In a split APK bundle, all parts must share the same signature. SAI and other tools verify this automatically and will refuse to install if they do not match.

Troubleshooting

"App Not Installed" Error

  • The APK is corrupted. Re-download it.
  • The app is already installed with a different signature. You must uninstall the existing version first.
  • Your Android version is too old. Check the app's minimum SDK.
  • The APK uses a split format. Use SAI instead of the default installer.
  • Insufficient storage space. Free up space and try again.

"Parse Error"

  • The APK file is incomplete or damaged. Re-download it.
  • The APK was not downloaded properly. Try downloading with a different browser.
  • The file extension is wrong. Rename it to .apk if needed.

"App Not Installed — Package conflicts with existing package"

  • You are trying to install a modded version of an app you already have installed. Uninstall the official version first.
  • If you cannot uninstall (system app), use Hail to freeze it or use ADB to uninstall it.
  • Use the "clone" variant of the mod if available.

ADB Says "More than one device/emulator"

# List connected devices:
adb devices

# Target a specific device:
adb -s DEVICE_ID install app.apk

# Or use the -d flag for the only USB device:
adb -d install app.apk

SAI Says "APK is not signed"

Some split APK bundles include unsigned base APKs. This usually means the bundle is incomplete or corrupted. Download it again from a different source.

Tips & Best Practices

  • Use APKMirror as your primary source. It has the strictest verification process among third-party APK sites.
  • Always check the signature. Especially for sensitive apps like banking, social media, and messaging.
  • Keep Aurora Store for Play Store downloads. It downloads directly from Google's servers, so the signature is guaranteed correct.
  • Use SAI with Shizuku. It makes split APK installation seamless and silent.
  • Enable Play Protect. Even if you sideload, keep Google Play Protect enabled. It scans all installed apps, including sideloaded ones.
  • Avoid "APK downloader" websites. Sites that claim to let you download paid apps for free are almost always scams or malware distributors.
  • Keep APK backups. After installing a sideloaded app, keep the APK file in a backup folder. If you need to reinstall, you will not need to download it again.
  • Use a dedicated download manager. Apps like "1DM" or "Advanced Download Manager" can resume interrupted downloads, preventing corrupt APK files.
  • Scan before installing. Use a local malware scanner (Hypatia, Malwarebytes) on the APK file before you install it.
  • Know how to uninstall. Sideloaded apps uninstall like any other app — drag to uninstall or go to Settings > Apps > [App] > Uninstall.
  • Do not install unknown APKs from modded games. Many modded game APKs contain aggressive adware or spyware. Only use well-known modding communities.
  • Use a second device for testing. If you have an old phone, use it as a test device for suspicious APKs before installing them on your daily driver.
