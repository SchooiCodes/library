🍓 Raspberry Pi Setup Guide

Set up a Raspberry Pi from scratch — headless install, SSH, GPIO, config, useful projects, and troubleshooting.

Table of Contents

Hardware Needed

  • Raspberry Pi board (3B+, 4B, 5, or Zero 2 W)
  • MicroSD card (16GB+, class 10 / A2 recommended)
  • USB-C or micro-USB power supply (5V/3A+ for Pi 4/5)
  • Optional: case, heatsinks, fan, HDMI cable, USB keyboard/mouse

For a headless setup you only need the board, SD card, and power. Everything else is done over the network.

OS Installation with Raspberry Pi Imager

Download Raspberry Pi Imager for your desktop OS. It writes the image and lets you preconfigure SSH, Wi-Fi, and user settings.

# Advanced options (Ctrl+Shift+X or Cmd+Shift+X):
# - Set hostname: raspberrypi.local
# - Enable SSH with password or public key
# - Set username/password (default: pi/raspberry)
# - Configure Wi-Fi SSID and password
# - Set locale
Tip: Use the Raspberry Pi OS Lite image (no desktop) for servers and headless projects — it uses far less RAM and storage.

Headless Setup (SSH & Wi-Fi)

After writing the SD card, insert it into the Pi and power it on. Find it on your network:

# Ping the hostname (if mDNS is supported)
ping raspberrypi.local

# Or scan your local network
nmap -sn 192.168.1.0/24

# SSH into it
ssh pi@raspberrypi.local

First-time setup commands:

sudo raspi-config
# -> 1 System Options -> S5 Boot / Auto Login -> Console
# -> 3 Interface Options -> I1 Legacy Camera (if needed)
# -> 6 Advanced Options -> A1 Expand Filesystem
# -> Update

sudo apt update && sudo apt upgrade -y
sudo reboot

GPIO Basics

The 40-pin GPIO header lets you connect LEDs, sensors, motors, and more. Use the RPi.GPIO library in Python or gpiod in C/Rust.

import RPi.GPIO as GPIO
import time

GPIO.setmode(GPIO.BCM)
GPIO.setup(18, GPIO.OUT)

while True:
    GPIO.output(18, GPIO.HIGH)
    time.sleep(1)
    GPIO.output(18, GPIO.LOW)
    time.sleep(1)

Pin reference: pinout command on the Pi shows a graphical pinout.

Troubleshooting

ProblemSolution
No green LED activityCheck power supply (5V/3A minimum). Corrupted SD card — reflash.
Cannot SSHEnsure SSH is enabled via raspi-config or by placing ssh file on boot partition.
Pi keeps rebootingUnderpowered PSU or overheating. Add heatsinks and check voltage.
Wi-Fi disconnectsDisable power saving: iwconfig wlan0 power off
