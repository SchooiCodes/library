AWS S3 & Storage

Deep dive into S3 -- storage classes, security, lifecycle, and performance.

Table of Contents (2 sections)

  • S3 Basics & Classes
  • Advanced & Performance

S3 Basics & Classes

S3: objects in buckets (flat namespace, max 100 buckets/account soft). Objects: key (path), version ID, metadata, tags. Storage classes: S3 Standard (frequent access), Intelligent-Tiering (auto-tier), Standard-IA (infrequent), One Zone-IA (lower cost), Glacier Instant Retrieval (archive ms), Glacier Flexible (min 90 days), Glacier Deep Archive (min 180 days). Lifecycle policies: transition objects between classes, expire (delete). Multipart upload: large objects >100MB. S3 Transfer Acceleration: global upload speed. Pre-signed URLs: temporary access. SSE: server-side encryption (SSE-S3, SSE-KMS, SSE-C). Bucket policies: resource-based IAM policies. Block Public Access: account/bucket settings.

Advanced & Performance

S3 Event Notifications: SQS, SNS, Lambda triggers. S3 Replication: CRR (cross-region), SRR (same-region), batch replication, delete marker replication. S3 Object Lock: WORM (write once read many) -- retention period, legal hold. S3 Select: SQL-like queries on objects. S3 Glacier Select: query archived data. S3 Batch Operations: bulk actions (copy, invoke Lambda, restore). S3 Inventory: audit object metadata. S3 Access Points: named network endpoints with policies. Performance: 3500 PUT/5500 GET per prefix per second. Multi-region access points: global endpoint with routing. VPC endpoint: S3 Gateway endpoint (free) or Interface endpoint (PrivateLink). Requester Pays: bucket owner transfers costs. S3 on Outposts: on-prem S3.
