🔒 VPN Setup Guide

Set up WireGuard and OpenVPN — server configuration, client setup, key generation, routing, and troubleshooting.

VPN Types

WireGuard — Modern, fast, simple. Uses UDP, built into Linux kernel 5.6+. Minimal config, state-of-the-art cryptography (Curve25519, ChaCha20, BLAKE2).

OpenVPN — Mature, highly configurable, TCP+UDP. Uses TLS handshake, supports PKI with CA certs. Heavier but extremely flexible.

IPsec/IKEv2 — Native on many devices (Windows, iOS). Good for mobile, but complex to configure manually.

WireGuard — Server Setup (Linux)

# Install
sudo apt install wireguard

# Generate keys
wg genkey | tee server-private.key | wg pubkey > server-public.key
chmod 600 server-private.key

# Create config /etc/wireguard/wg0.conf
[Interface]
Address = 10.0.0.1/24
ListenPort = 51820
PrivateKey = 

# Client peer
[Peer]
PublicKey = 
AllowedIPs = 10.0.0.2/32

# Start
sudo wg-quick up wg0
sudo systemctl enable wg-quick@wg0

WireGuard — Client Setup (Linux)

# Client config /etc/wireguard/wg0.conf
[Interface]
Address = 10.0.0.2/24
PrivateKey = 
DNS = 1.1.1.1

[Peer]
PublicKey = 
Endpoint = your-server.com:51820
AllowedIPs = 0.0.0.0/0  # route all traffic through VPN
PersistentKeepalive = 25

# Connect
sudo wg-quick up wg0
Tip: Set AllowedIPs = 0.0.0.0/0 for full-tunnel (all traffic through VPN). Use 10.0.0.0/24, 192.168.1.0/24 for split-tunnel (only local LAN + VPN subnet).

WireGuard on Windows

Download from wireguard.com/install. Create a tunnel with the same client config. Use the GUI to import .conf files. Toggle activation with one click.

To forward traffic on Windows, enable Internet Connection Sharing or set up routing manually via route add.

OpenVPN Setup

# Install
sudo apt install openvpn easy-rsa

# Set up CA
make-cadir ~/openvpn-ca
cd ~/openvpn-ca
./easyrsa init-pki
./easyrsa build-ca nopass
./easyrsa gen-req server nopass
./easyrsa sign-req server server
./easyrsa gen-dh

# Server config
port 1194
proto udp
dev tun
ca ca.crt
cert server.crt
key server.key
dh dh.pem
server 10.8.0.0 255.255.255.0
push "redirect-gateway def1 bypass-dhcp"
push "dhcp-option DNS 1.1.1.1"
keepalive 10 120

# Start
sudo systemctl enable --now openvpn@server

Kill Switch

Prevents traffic leaks if the VPN drops.

# WireGuard kill switch (nftables)
sudo nft add table inet killswitch
sudo nft add chain inet killswitch input { type filter hook input priority 0\; }
sudo nft add rule inet killswitch input iif != wg0 ip daddr != 10.0.0.0/24 drop

# OpenVPN: add in server config
push "route-up /etc/openvpn/route-up.sh"
# route-up.sh kills default route if VPN goes down

Split Tunneling

Route only specific traffic through the VPN.

# WireGuard — only route specific subnets
AllowedIPs = 10.0.0.0/24, 192.168.1.0/24

# OpenVPN — bypass specific routes
route-nopull
route 10.8.0.0 255.255.255.0
# Then add routes for apps that need VPN

Performance Tuning

WireGuard is already fast, but these help:

  • Use a server close to you (low latency).
  • Set MTU = 1420 to avoid fragmentation.
  • Increase kernel UDP buffers: net.core.rmem_max = 2500000.
  • Enable TCP BBR congestion control: net.core.default_qdisc=fq; net.ipv4.tcp_congestion_control=bbr.
  • Disable IPv6 if not needed on both ends.
Tip: Always test your VPN via a torrent IP check or curl ifconfig.me — ensure the public IP matches your VPN server, not your ISP.
