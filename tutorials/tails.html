🕵️ Tails OS Guide

A complete walkthrough of Tails — the amnesic incognito live system that leaves no trace and routes everything through Tor.

What is Tails?

Tails (The Amnesic Incognito Live System) is a Debian-based Linux distribution designed for privacy and anonymity. It routes all internet traffic through the Tor network and leaves no trace on the host computer after shutdown. Every time you reboot, Tails returns to a clean, fresh state — hence "amnesic."

Edward Snowden famously used Tails to communicate securely with journalists. It is the gold standard for anyone who needs strong operational security: whistleblowers, journalists, activists, or anyone living under a repressive regime.

Installation

1. Download the ISO

Head to tails.net and download the latest ISO image. Always download over Tor if possible.

2. Verify the Signature

Download the .sig file that matches your ISO. Import the Tails signing key and verify:

gpg --auto-key-locate keys.openpgp.org --fetch-key 0xDBB802B258ACD84F
gpg --verify tails-amd64-*.img.sig tails-amd64-*.img

Never skip this step. If the signature does not match, the ISO may have been tampered with.

3. Write to USB

You have two good options:

  • Ventoy — Copy the ISO to a Ventoy-formatted USB. Reboot and select it from the Ventoy menu. Simple and keeps your USB usable for other files.
  • Balena Etcher — Flash the ISO directly. Works on Windows, macOS, and Linux. Wipes the entire USB.

4. Configure Persistent Storage

After booting Tails for the first time, go to Applications → Tails → Configure persistent storage. This creates an encrypted LUKS partition on your USB that survives reboots. You choose what to persist (see below).

First Boot

When you boot Tails for the first time you will see a startup screen with a few options:

  • Administration Password — Leave blank unless you need root access (e.g., to install drivers or additional software). If you set it, make it strong.
  • Tor Connection — Connect directly to Tor by default. If you are in a country that censors Tor (China, Iran, etc.), select "Configure a bridge" to bypass censorship.
  • Language & Keyboard — Set your locale. Changing the keyboard layout is essential for typing passwords comfortably.

Once booted you land on the GNOME desktop. It is stripped down compared to a full GNOME install — no unnecessary applications, no bloat. The top bar shows a Tor status icon (green onion = connected, red = disconnected).

Tor Browser & Onion Routing

Tails comes with Tor Browser (a hardened Firefox) pre-installed and pre-configured. Every tab is isolated by default.

Key concepts:

  • Tor Circuit — Your traffic passes through three relays (entry, middle, exit). Each relay only knows the one before and after it, so no single relay can link you to your destination.
  • New Circuit — Click the shield icon next to the URL bar and select "New Circuit for this site" to change your exit node. Use this if a site is rate-limiting you or showing the wrong geolocation.
  • .onion Sites — Tor-hidden services with addresses ending in .onion. They are only accessible through Tor. No exit node is needed — traffic stays inside the Tor network, which means end-to-end encryption and no clearnet exposure.
Tip: Never resize the Tor Browser window to your screen's full resolution. Maximized browsers are one of the most common fingerprinting vectors. Keep it smaller or use a standard window size.

Persistent Encrypted Storage

Persistent storage is an encrypted LUKS partition on your USB. When you boot Tails you are prompted for the passphrase to unlock it. You choose which data to persist by toggling features in the configuration:

  • Personal Documents — Files stored in the Persistent folder on the desktop.
  • Tor Browser bookmarks and browser history.
  • GnuPG keys — Your PGP public and private keys (used by Thunderbird/Enigmail).
  • Bitcoin wallet — Electrum wallet data.
  • KeePassXC databases — Password databases.
  • Network connections — Saved Wi-Fi passwords and network config.
  • Additional software — Packages you install persist across reboots if you enable this.

Be deliberate about what you persist. Every persisted item creates a breadcrumb. The golden rule: persist only what you absolutely need.

Security Features

  • MAC Address Spoofing — Tails randomizes your MAC address on every boot by default. This prevents your hardware address from being tracked across networks. You can disable it for a specific network in the Wi-Fi settings if the network uses MAC-based authentication.
  • Clearnet Blocker — All non-Tor network traffic is blocked at the firewall level. If an application tries to connect directly to the internet (not through Tor), it will fail. This prevents accidental IP leaks.
  • No Swap — Swap partitions and swap files are disabled. Data written to swap could survive a reboot and reveal sensitive information. Tails runs entirely in RAM.
  • Amnesic File System — Everything is stored in memory (tmpfs). When you shut down, the memory is wiped. No trace remains on the host machine's hard drive.

Killing the Connection vs Shutting Down

These are two different actions with different purposes:

  • Kill the connection ( Tor icon → "Kill the connection") — Immediately cuts all Tor circuits. Use this if you feel compromised or need to force a new identity. The system stays running, but no network traffic flows until you reconnect.
  • Shut down ( System menu → "Shut Down") — Wipes all memory and powers off the machine. All data that was not saved to persistent storage is gone forever. Use this when you are done working.

There is also a New Identity button in the Tor Browser toolbar that closes all open tabs, clears all browser state (cookies, cache, history), and opens a fresh Tor circuit.

Practical Safeguards

  • Do not log into personal accounts. Tails is not for casual browsing. Logging into Facebook, Google, or your personal email ties your real identity to your Tor exit node and defeats the purpose of using Tails.
  • Do not install extra software. Tails already includes everything you need. Installing additional packages increases your attack surface and may create identifiable artifacts. If you absolutely must, enable the persistent "Additional Software" feature — but be very selective.
  • Use bridges in censored countries. If Tor is blocked where you are (China, Iran, Russia, etc.), configure a bridge at the Tor Connection screen on boot. Bridges are non-public Tor relays that are harder to block. You can request bridges at bridges.torproject.org or email bridges@torproject.org.
  • Watch for physical surveillance. Tails protects you online, but it cannot protect you from someone looking over your shoulder or confiscating your USB. Use a privacy screen filter and never leave your USB unattended.
  • Keep Tails updated. Subscribe to the Tails release announcements. Old versions have known vulnerabilities. Upgrade by cloning the latest ISO to your USB.
Getting Started: Download Tails from tails.net, verify the signature, flash it to a USB, and reboot. Start with the persistent storage configured — you will appreciate having your bookmarks and keys survive reboots.
