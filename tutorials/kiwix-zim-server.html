Kiwix & ZIM Offline Server

Serve Wikipedia, WikiHow, Stack Exchange, and other knowledge bases entirely offline using Kiwix and ZIM files. Perfect for areas with limited or no internet.

ZIM File Download Sources

ZIM files contain compressed offline content from wikis and other websites. Download from these sources:

  • Kiwix Library — Official repository, searchable web interface.
  • download.kiwix.org/zim/ — Direct download directory (wget-friendly).
  • FAU Mirror (Germany) — Fast European mirror.
  • Accum Mirror (Sweden) — Nordic mirror.
# Download Wikipedia (English, mini) via wget
wget https://download.kiwix.org/zim/wikipedia/wikipedia_en_all_mini_2025-01.zim

# List available ZIM files
curl -s https://download.kiwix.org/zim/wikipedia/ | grep -oP 'href="[^"]+\.zim"' | head -20
Tip: "mini" ZIMs are smaller (~2 GB) and contain article summaries without images. Full ZIMs with images can be 40-100 GB.

kiwix-serve Setup & Configuration

Kiwix-serve is the server component that serves ZIM files over HTTP.

Installation

# Linux (static binary, no deps required)
wget https://download.kiwix.org/release/kiwix-tools/kiwix-tools_linux-x86_64.tar.gz
tar -xzf kiwix-tools_linux-x86_64.tar.gz
sudo cp kiwix-tools*/kiwix-serve /usr/local/bin/

# Or via Docker
docker pull kiwix/kiwix-serve

Basic Server Command

# Serve a single ZIM file on port 8080
kiwix-serve --port=8080 wikipedia_en_all_mini_2025-01.zim

# Serve multiple ZIM files
kiwix-serve --port=8080 *.zim

# Serve from library XML file
kiwix-serve --port=8080 --library=library.xml

Docker Command

docker run -d \
  --name kiwix \
  -p 8080:8080 \
  -v /path/to/zim:/data \
  kiwix/kiwix-serve \
  *.zim
Memory: Each ZIM file served increases RAM usage. A full Wikipedia ZIM may use 1-2 GB of RAM. Serve only what you need.

systemd Service for Auto-Start

Create a systemd service to start kiwix-serve automatically on boot.

sudo tee /etc/systemd/system/kiwix-serve.service << 'EOF'
[Unit]
Description=Kiwix ZIM server
After=network.target

[Service]
Type=simple
User=kiwix
Group=kiwix
ExecStart=/usr/local/bin/kiwix-serve --port=8080 /opt/zim/*.zim
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF
# Create the kiwix user
sudo useradd -r -s /bin/false kiwix

# Set permissions on ZIM directory
sudo chown -R kiwix:kiwix /opt/zim

# Enable and start
sudo systemctl daemon-reload
sudo systemctl enable --now kiwix-serve

# Check status
sudo systemctl status kiwix-serve

6-Tier Priority Table for ZIM Archives

TierContentSizeWhy
1 — CriticalWikipedia (English, mini)~2 GBUniversal knowledge base, text-only
2 — HighWikipedia (English, full no-images)~10 GBFull articles with formatting
3 — MediumWikiHow~1 GBPractical how-to guides, essential
4 — MediumStack Overflow~80 GBProgramming Q&A, huge but invaluable
5 — LowWikipedia (images)~40 GBNice for visual reference
6 — OptionalWiktionary, Wikibooks, Wikisource~5 GB eachLanguage references, textbooks
Strategy: Start with Tier 1. Add more as storage permits. The mini Wikipedia + WikiHow + Stack Overflow covers 90%+ of offline knowledge needs.

Accessing Offline Content

Once kiwix-serve is running, access content from any device on your local network:

  • Open a browser and navigate to http://<server-ip>:8080
  • Search across all loaded ZIM files from the homepage
  • Browse by category or full-text search
  • Access specific ZIMs directly via the library view

Offline Wikipedia

Full-text search of all Wikipedia articles. Links between articles work within the ZIM. Images load if present.

Offline WikiHow

Step-by-step guides for thousands of practical tasks — cooking, repairs, survival skills, and more.

Offline Stack Exchange

Stack Overflow and other Stack Exchange sites. Search for programming solutions, engineering answers, and community Q&A.

# Quick test that the server is working
curl -s http://localhost:8080 | head -5

# Search Wikipedia from the command line
curl -s "http://localhost:8080/search?content=wikipedia_en_all_mini&pattern=offline+server"
