🐚 Bash Scripting Guide

Learn Bash scripting from the ground up — variables, conditionals, loops, functions, error handling, and automation.

What is Bash?

Bash (Bourne Again SHell) is the default shell on most Linux distributions. A Bash script is a plain-text file containing a sequence of commands that the shell interprets. Scripts are used for task automation, system administration, and pipeline orchestration.

#!/bin/bash
# This is a comment — the shebang tells the kernel to use Bash
echo "Hello, world!"

Make a script executable with chmod +x script.sh and run it with ./script.sh.

Variables & Strings

Variables are assigned without spaces around = and referenced with $. Use curly braces to delimit variable names in strings.

#!/bin/bash
name="Alice"
greeting="Hello, ${name}!"
echo "$greeting"

# Command substitution
now=$(date)
echo "The time is $now"

# Arrays
fruits=("apple" "banana" "cherry")
echo "${fruits[0]}" # apple
echo "${fruits[@]}" # all items
Tip: Always quote variables ("$var") to prevent word-splitting and glob expansion.

Conditionals (if/else, test)

Use if, elif, and else with the test command (or [ ] / [[ ]]). Double brackets [[ ]] are Bash-specific and more powerful.

#!/bin/bash
score=85

if [[ $score -ge 90 ]]; then
 echo "Grade: A"
elif [[ $score -ge 80 ]]; then
 echo "Grade: B"
else
 echo "Grade: C"
fi

# File tests
if [[ -f "/etc/passwd" ]]; then
 echo "File exists"
fi

# String comparison
if [[ "$USER" == "root" ]]; then
 echo "You are root"
fi

Loops (for/while/until)

Loop over lists, ranges, command output, or files. while runs while a condition is true; until runs until it becomes true.

#!/bin/bash
# For loop over a list
for fruit in apple banana cherry; do
 echo "I like $fruit"
done

# For loop with range (Bash 4+)
for i in {1..5}; do
 echo "Count: $i"
done

# While loop reading a file
while IFS= read -r line; do
 echo "$line"
done < "input.txt"

# Until loop
count=5
until [[ $count -eq 0 ]]; do
 echo "$count"
 ((count--))
done

Functions

Functions group commands for reuse. Define them before use. Parameters are accessed as $1, $2, etc. $? captures the last exit code.

#!/bin/bash
greet() {
 local name="$1"
 echo "Hello, $name!"
}

greet "World"

# Return values via exit code or echo
is_even() {
 if (( $1 % 2 == 0 )); then
  return 0
 else
  return 1
 fi
}

if is_even 4; then
 echo "4 is even"
fi

Error Handling

Use set -e to exit on error, set -u to error on undefined variables, and trap to catch signals or cleanup.

#!/bin/bash
set -euo pipefail

cleanup() {
 echo "Cleaning up..."
 rm -f /tmp/tempfile
}
trap cleanup EXIT

# Check if a command succeeded
if ! mkdir /some/dir; then
 echo "Failed to create directory" >&2
 exit 1
fi
Tip: Always use set -euo pipefail at the top of production scripts to catch errors early.

Scheduling with Cron

Cron runs scripts on a schedule. Edit your crontab with crontab -e. Format: minute, hour, day, month, weekday.

# ┌──────── minute (0-59)
# │ ┌────── hour (0-23)
# │ │ ┌──── day of month (1-31)
# │ │ │ ┌── month (1-12)
# │ │ │ │ ┌─ weekday (0-7, Sun=0)
# * * * * * command

# Run backup every day at 2 AM
0 2 * * * /home/user/scripts/backup.sh

# Run every 15 minutes
*/15 * * * * /home/user/scripts/healthcheck.sh

Practical Examples

A backup script with rotation:

#!/bin/bash
set -euo pipefail
BACKUP_DIR="/backups"
SOURCE="/home/user/data"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

mkdir -p "$BACKUP_DIR"
tar czf "${BACKUP_DIR}/backup_${TIMESTAMP}.tar.gz" "$SOURCE"

# Keep only last 7 backups
ls -t "$BACKUP_DIR"/*.tar.gz | tail -n +8 | xargs rm -f
echo "Backup complete: backup_${TIMESTAMP}.tar.gz"
