Setting Up Offline Servers

Run local servers for offline use — DHCP, DNS, file sharing, and media streaming without internet.

Table of Contents (3 sections)

  • Local Network Services
  • Raspberry Pi as Server
  • Mesh Networking

Local Network Services

Offline servers run on a LAN without internet. DHCP server: assign IPs automatically (dnsmasq or your router), DNS server: resolve local hostnames (dnsmasq with /etc/hosts or Pi-hole), NTP server: keep devices' clocks synchronized (chrony). File sharing: Samba (Windows file shares), NFS (Linux), SFTP/SSH (cross-platform). Media streaming: Jellyfin (open-source Plex alternative) serves your movie/TV/music library offline.

Raspberry Pi as Server

Raspberry Pi 4/5: the perfect offline server. Power consumption: ~5W (runs 24/7). Services to run: Pi-hole (DNS-level ad blocking and caching), Samba (file server with external HDD), Jellyfin (media server with hardware transcoding on Pi 5), Syncthing (offline file sync between devices), Kiwix-serve (serve ZIM content over LAN). Install with Raspberry Pi OS Lite (no desktop — saves resources). Use PoE (Power over Ethernet) for single-cable deployment.

Mesh Networking

When infrastructure fails, mesh networks keep devices connected. BATMAN-adv: kernel-level mesh protocol for Wi-Fi — nodes relay data for each other. Yggdrasil: IPv6 mesh network with end-to-end encryption — auto-configures, can route over the internet too. CJDNS: encrypted IPv6 mesh. Setup: flash LibreMesh on compatible routers (TP-Link, Ubiquiti), configure mesh interfaces, and deploy multiple nodes for coverage. Each new node strengthens the network.
