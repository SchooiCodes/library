📶 Wi-Fi Security Guide

Secure your Wi-Fi network — WPA2/WPA3, 802.1X, guest networks, deauth protection, and auditing.

Table of Contents

  • Encryption Standards
  • Router Configuration
  • Guest Networks
  • Deauth Protection

Encryption Standards

StandardSecurityNotes
WEPBroken — do not useCan be cracked in minutes
WPA (TKIP)WeakVulnerable to multiple attacks
WPA2 (AES)StrongMinimum acceptable standard. Still widely used.
WPA3 (SAE)Very strongBest available. Required for Wi-Fi 6/6E certification.

Always use WPA2 with AES (CCMP) at minimum. If your hardware supports it, enable WPA3 for stronger handshake protection against brute force.

Router Configuration

  • Change the default admin username and password
  • Disable WPS (Wi-Fi Protected Setup) — it is trivially bruteforceable
  • Disable UPnP unless absolutely needed
  • Use a strong, unique Wi-Fi password (16+ chars, mixed case + symbols)
  • Disable SSID broadcast if you want "security by obscurity" (minimal effect but reduces noise)
  • Keep router firmware updated — enable auto-update if available
# Example strong Wi-Fi password generation:
openssl rand -base64 24
# Returns something like: 8Bk2xPq3R9vLmN7fJ4tW6yZ1
Tip: Use a separate SSID for 2.4 GHz and 5 GHz bands if your router allows it. This helps with troubleshooting and lets you assign devices to the appropriate band.

Guest Networks

Set up a separate guest Wi-Fi network with its own SSID and password. Most modern routers support this natively. Guest networks should:

  • Have internet access only (no access to your LAN)
  • Use a different subnet/VLAN
  • Have bandwidth limits (optional)
  • Be isolated from other guest clients (AP isolation / client isolation)

Deauth Protection

Deauthentication attacks disconnect clients by spoofing deauth frames. Modern WPA3 mitigates this, but WPA2 networks are vulnerable. Protections:

  • Use WPA3 — SAE handshake is not vulnerable to deauth-based PSK capture
  • Enable "Management Frame Protection (802.11w)" if available
  • Monitor for deauth floods using WIDS/WIPS
  • Use wired connections for critical devices
# Detect deauth attacks with airodump-ng
sudo airodump-ng wlan0 --channel 6

# Look for repeated "Deauth" in the output
# A sudden spike in deauth packets indicates an attack

Auditing Your Wi-Fi

Regularly audit your Wi-Fi network for vulnerabilities:

# Use nmap to discover devices on your network
nmap -sn 192.168.1.0/24

# Check for weak encryption with airodump-ng
sudo airodump-ng wlan0

# Or use a dedicated tool like Wifite or Kismet
# For professional audits, use WPA3-Enterprise

Enterprise Wi-Fi (802.1X)

For organizations, use WPA2/WPA3-Enterprise with 802.1X/RADIUS authentication. Each user gets unique credentials and TLS certificates.

# Components:
# - RADIUS server (FreeRADIUS, Windows NPS)
# - Certificate Authority (for EAP-TLS)
# - Supplicant (client device)

# EAP methods (from least to most secure):
# - PEAP/MSCHAPv2 (password-based, common)
# - EAP-TLS (certificate-based, strongest)

Enterprise Wi-Fi with EAP-TLS is the most secure option — it uses client certificates and is resistant to evil twin attacks.
