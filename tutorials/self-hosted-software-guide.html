Self-Hosted Software Guide

Self-host your own services — media, cloud storage, password manager, monitoring, and more.

Why Self-Host?

Self-hosting gives you full control over your data, no subscription fees, unlimited storage, privacy from cloud providers, and excellent learning opportunities.

Requirements: a dedicated machine (old laptop, Raspberry Pi, or mini PC), a domain name (or Tailscale for private access), and basic Docker knowledge.

Self-hosted stack examples: media (Jellyfin, Sonarr, Radarr), cloud storage (Nextcloud), password manager (Vaultwarden), DNS (AdGuard Home), monitoring (Uptime Kuma).

Essential Self-Hosted Services

Must-have self-hosted apps:

# Media Stack (Docker Compose)
services:
  jellyfin:
    image: jellyfin/jellyfin
    ports: ["8096:8096"]
  sonarr:
    image: lscr.io/linuxserver/sonarr
  radarr:
    image: lscr.io/linuxserver/radarr
  prowlarr:
    image: lscr.io/linuxserver/prowlarr
  qbittorrent:
    image: lscr.io/linuxserver/qbittorrent

Reverse Proxy & Domain Setup

Access services with nice URLs instead of port numbers:

# Nginx Proxy Manager — web UI for reverse proxy
services:
  npm:
    image: jc21/nginx-proxy-manager
    ports:
      - "80:80"
      - "443:443"
      - "81:81"  # Admin UI
    volumes:
      - ./data:/data

# Then in NPM admin UI (port 81):
# Add proxy host → jellyfin.domain.com → http://jellyfin:8096
# SSL → Let's Encrypt → enable force SSL

DNS: point your domain's A record to your server's IP. Use Cloudflare (free CDN + DNS) or DuckDNS (free dynamic DNS).

Backup & Maintenance

Automate backups for self-hosted services:

# Automated backup script
#!/bin/bash
BACKUP_DIR="/mnt/backup/selfhosted"
DATE=$(date +%Y-%m-%d)

# Backup Docker volumes
docker run --rm -v nextcloud_data:/data -v $BACKUP_DIR:/backup alpine tar czf /backup/nextcloud-$DATE.tar.gz -C /data .
docker run --rm -v vaultwarden_data:/data -v $BACKUP_DIR:/backup alpine tar czf /backup/vaultwarden-$DATE.tar.gz -C /data .

# Backup Docker Compose configs
cp -r /opt/docker $BACKUP_DIR/configs-$DATE

# Sync to off-site
rclone sync $BACKUP_DIR remote:b2-bucket --progress

Self-Hosted Security Checklist

☐ Use SSH keys, disable password auth
☐ Fail2ban for SSH and web services
☐ Automatic SSL via Let's Encrypt (NPM, Caddy, Traefik)
☐ Regular security updates (unattended-upgrades/watchtower)
☐ Separate VLAN for untrusted IoT devices
☐ Use CrowdSec or Cloudflare WAF
☐ Enable 2FA on all user-facing services
☐ Regular backups with off-site copy
☐ Monitor with Uptime Kuma + healthcheck endpoints
☐ Read-only root filesystem for containers where possible
☐ Regular log review: journalctl, lnav, goaccess
