Helm Charts

Kubernetes package manager — charts, templates, values, and repositories.

Table of Contents (2 sections)

  • Chart Structure & Templates
  • Dependencies & Hooks

Chart Structure & Templates

Helm chart directory: Chart.yaml (metadata), values.yaml (default config), templates/ (Go templates + YAML), charts/ (subcharts), crds/ (CRDs). Templates: apiVersion: apps/v1 kind: Deployment metadata: name: {{ include "myapp.fullname" . }}. Built-in objects: .Values, .Release, .Chart, .Files, .Capabilities. Functions: {{ .Values.replicaCount | default 1 }}, {{ quote .Values.name }}, {{ upper .Values.env }}. Flow control: {{- if .Values.ingress.enabled }} ... {{- end }}, {{- range .Values.containers }} ... {{- end }}.

Dependencies & Hooks

Chart dependencies: Chart.yaml — dependencies: [{ name: "redis", version: "17.x", repository: "https://charts.bitnami.com/bitnami" }]. helm dependency update. Hooks: pre-install, post-install, pre-upgrade, post-upgrade, pre-delete, post-delete — defined with annotations (helm.sh/hook). Testing: helm test — pods with test hook. Repository: helm repo add bitnami https://charts.bitnami.com/bitnami. Chart Museum: self-hosted chart repository. Helmfile: declarative management of multiple Helm releases.
