MPLS Networking Basics

Labels, LSPs, LDP, RSVP-TE, MPLS VPNs, and traffic engineering fundamentals.

What is MPLS?

Multiprotocol Label Switching (MPLS) is a data-carrying technique that directs data from one network node to the next based on short path labels rather than long network addresses. It sits between Layer 2 (data link) and Layer 3 (network) — often called Layer 2.5. MPLS is widely used by ISPs and large enterprises for VPNs, traffic engineering, and QoS.

Key concepts:

  • Label -- 20-bit identifier (1,048,576 possible values), local significance per hop
  • LSP (Label Switched Path) -- the path packets traverse through the MPLS network
  • LER (Label Edge Router) -- ingress/egress router that pushes/removes labels
  • LSR (Label Switch Router) -- core router that swaps labels

Label Distribution: LDP vs RSVP-TE

MPLS labels can be distributed using different protocols:

# LDP (Label Distribution Protocol) -- basic, hop-by-hop
# Automatically distributes labels for IGP routes
# Each LSR assigns label for each prefix in routing table

# Cisco LDP configuration
mpls label protocol ldp
interface GigabitEthernet0/0
 mpls ip
!
router ospf 1
 mpls ldp autoconfig

# RSVP-TE (Resource Reservation Protocol - Traffic Engineering)
# Explicit path setup, bandwidth reservation
# Used for traffic engineering MPLS-TE

# Cisco RSVP-TE configuration
ip explicit-path name PATH-TO-CORE
 next-address 10.0.1.1
 next-address 10.0.2.1
!
interface Tunnel100
 ip unnumbered Loopback0
 tunnel mode mpls traffic-eng
 tunnel destination 192.168.255.1
 tunnel mpls traffic-eng path-option 1 explicit name PATH-TO-CORE
!

MPLS VPNs: Layer 3 VPN & Layer 2 VPN

MPLS is widely used to provide VPN services:

# Layer 3 MPLS VPN (RFC 4364)
# Provider Edge (PE) routers contain separate VRF per customer
# Customer Edge (CE) router peers with PE

# VRF configuration (Cisco)
ip vrf CUSTOMER-A
 rd 64500:100
 route-target export 64500:100
 route-target import 64500:100
!
interface GigabitEthernet0/1
 ip vrf forwarding CUSTOMER-A
 ip address 192.168.10.1 255.255.255.252
!
router bgp 64500
 address-family ipv4 vrf CUSTOMER-A
  neighbor 192.168.10.2 remote-as 65001
  neighbor 192.168.10.2 activate

# Layer 2 VPN (VPWS -- Virtual Private Wire Service, VPLS)
# Point-to-point (VPWS) or multipoint (VPLS) L2 connectivity
# Pseudowire: emulates a wire between two sites

# Pseudowire configuration
interface Pseudowire1
 encapsulation mpls
 neighbor 192.168.255.1 100

MPLS Traffic Engineering (MPLS-TE)

MPLS-TE provides explicit path control and bandwidth management:

# Traffic engineering database (TED) via OSPF-TE or IS-IS TE
# Links advertise available bandwidth, TE metric, admin groups

# Constraint-based path computation (CSPF)
# Finds path meeting constraints (bandwidth, latency, affinity)

# Fast Reroute (FRR) -- sub-50ms protection switching
# Protects link or node failures via pre-computed backup paths

# DiffServ-Aware TE (DS-TE)
# Different bandwidth pools for different traffic classes

# View MPLS-TE tunnels
show mpls traffic-eng tunnels brief
show mpls traffic-eng topology

# LSP ping and traceroute
ping mpls ipv4 192.168.255.1/32
traceroute mpls ipv4 192.168.255.1/32

Segment Routing (SR-MPLS)

Segment Routing simplifies MPLS by using IGP-based labels without LDP:

# SR-MPLS benefits:
# - No LDP or RSVP-TE needed (uses IGP extensions)
# - Source routing: ingress node encodes path as label stack
# - TI-LFA (Topology-Independent LFA) for sub-50ms FRR

# Cisco SR-MPLS (IS-IS) configuration
router isis 1
 net 49.0001.1921.6800.1001.00
 is-type level-2-only
 metric-style wide
 segment-routing mpls
!
interface Loopback0
 ip address 192.168.255.1 255.255.255.255
 ip router isis 1
 isis passive
 isis prefix-sid index 101
!
interface GigabitEthernet0/0
 ip address 10.0.1.1 255.255.255.252
 ip router isis 1
 isis circuit-type level-2-only
 isis adjacency-sid label 16001
