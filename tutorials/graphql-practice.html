GraphQL in Practice

GraphQL -- schema design, resolvers, subscriptions, and performance.

Table of Contents (2 sections)

  • Schema & Queries
  • Resolvers & Perf

Schema & Queries

GraphQL: query language, single endpoint. Schema: type Query { users: [User!]! user(id: ID!): User type User { id: ID! name: String posts: [Post!]! } }. Queries: { users { id name } }. Arguments: { user(id: "1") { name email } }. Mutations: mutation { createUser(input: { name: "Alice" }) { id name } }. Input types: input CreateUserInput { name: String! email: String! }. Subscriptions: real-time via WebSocket. Fragments: reusable field sets. Variables: query($id: ID!) { user(id: $id) { name } }. Directives: @include(if:), @skip(if:), @deprecated. Interfaces: shared fields across types. Unions: type SearchResult = User | Post. Enums: enum Role { ADMIN USER }.

Resolvers & Perf

Resolvers: functions per field -- root types (Query, Mutation, Subscription) and field resolvers. DataLoader: batching + caching (solve N+1 problem). Batching: DataLoader loads multiple keys in one batch. Caching: per-request cache. Mutations: return updated data. Error handling: errors array in response, partial success. Auth: context (auth token), field-level permissions (directives or wrapper). Complexity analysis: limit query depth/complexity to prevent abuse. Query cost: weight fields, reject expensive queries. Pagination: connection pattern (edges, node, pageInfo, cursor). Relay spec: global IDs, cursor-based pagination, mutations. Persisted queries: whitelist allowed queries. Apollo: client (cache, optimistic UI, subscriptions), server (schema stitching, federation). Federation: compose multiple GraphQL services. GraphQL vs REST: GraphQL (client-specified fields, single endpoint), REST (server-defined responses, multiple endpoints). Tools: GraphiQL, Apollo Studio, GraphQL Playground. Codegen: generate TypeScript types from schema. Security: depth limiting, rate limiting, auth, validation.
