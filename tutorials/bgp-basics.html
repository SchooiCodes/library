BGP Routing Basics

Border Gateway Protocol, AS numbers, peering, route selection, and BGP security with RPKI.

What is BGP?

Border Gateway Protocol (BGP) is the routing protocol that makes the internet work. It is a path-vector protocol that exchanges routing and reachability information between autonomous systems (ASes). BGP replaces earlier EGP and has been the internet's core routing protocol since 1993.

Key concepts:

  • Autonomous System (AS) -- a network under a single administrative domain, identified by a unique AS number (ASN)
  • eBGP -- external BGP; routes exchanged between different ASes
  • iBGP -- internal BGP; routes exchanged within the same AS
  • CIDR notation -- BGP advertises IP prefixes with subnet masks (e.g., 203.0.113.0/24)

BGP Path Attributes & Route Selection

BGP uses path attributes to make routing decisions. When multiple paths exist, the router selects the best path using this order:

1. Highest Weight (Cisco proprietary, local to router)
2. Highest Local Preference (local to AS)
3. Prefer locally originated routes
4. Shortest AS_PATH
5. Lowest ORIGIN type (IGP < EGP < INCOMPLETE)
6. Lowest MED (Multi-Exit Discriminator)
7. eBGP preferred over iBGP
8. Shortest IGP metric to next-hop
9. Oldest route (eBGP) or lowest Router ID (iBGP)

# Common AS_PATH prepending example:
# route-map SET_PATH permit 10
#   set as-path prepend 64510 64510 64510

# BGP community string example:
# 64510:80  -- prepend to upstream peers
# 64510:90  -- announce to all peers

BGP Peering & Configuration

Configure a basic BGP peering on a Cisco router or Linux with FRR/BIRD:

# Cisco IOS XE
router bgp 64500
 bgp router-id 192.0.2.1
 neighbor 203.0.113.1 remote-as 64501
 neighbor 203.0.113.1 description TRANSIT_PROVIDER
 neighbor 203.0.113.1 password my-secret
 !
 address-family ipv4 unicast
  network 192.0.2.0 mask 255.255.255.0
  neighbor 203.0.113.1 activate
  neighbor 203.0.113.1 route-map INBOUND in
  neighbor 203.0.113.1 route-map OUTBOUND out
 exit-address-family

# BIRD (Linux)
protocol bgp upstream {
    local as 64500;
    neighbor 203.0.113.1 as 64501;
    source address 192.0.2.1;
    password "my-secret";
    ipv4 {
        export filter {
            if net ~ [192.0.2.0/24+] then accept;
            reject;
        };
        import all;
    };
}

BGP Security: RPKI, BGPsec & Prefix Filtering

BGP was designed without built-in security. Modern mitigation techniques include:

# RPKI (Resource Public Key Infrastructure)
# Validates that an AS is authorized to announce a prefix
# Three states: VALID, INVALID, NOT_FOUND

# Configure ROV (Route Origin Validation) on Cisco:
router bgp 64500
 bgp rpki server tcp 192.0.2.10 port 323 refresh 600
 route-map RPKI-FILTER permit 10
  match rpki valid
 route-map RPKI-FILTER deny 10
  match rpki invalid

# IRR (Internet Routing Registry) prefix filtering
# Create prefix-list based on registered routes
ip prefix-list ALLOWED seq 5 permit 192.0.2.0/24
neighbor 203.0.113.1 prefix-list ALLOWED in

# BGPsec (draft standard, limited adoption)
# Cryptographicly signs BGP updates using router keys

# TTL Security (GTSM)
neighbor 203.0.113.1 ttl-security hops 1

Troubleshooting BGP

Common BGP troubleshooting commands:

# Cisco
show ip bgp summary
show ip bgp neighbors 203.0.113.1 advertised-routes
show ip bgp neighbors 203.0.113.1 received-routes
show ip bgp 192.0.2.0/24
show ip route bgp
ping 203.0.113.1 source 192.0.2.1

# Linux (FRR)
vtysh
router# show ip bgp summary
router# show ip bgp 192.0.2.0/24
router# show ip bgp neighbors 203.0.113.1

# Linux (BIRD)
sudo birdc show status
sudo birdc show protocols
sudo birdc show route for 192.0.2.0/24
sudo birdc show route all
