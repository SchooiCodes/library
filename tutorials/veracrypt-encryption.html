VeraCrypt Full Disk Encryption

Secure your data with VeraCrypt — full disk, partition, and container encryption for all platforms.

Table of Contents (5 sections)

  • Full Disk & System Encryption

What is VeraCrypt?

VeraCrypt is a free, open-source disk encryption tool based on TrueCrypt 7.1a. It provides on-the-fly encryption (OTFE), meaning data is automatically encrypted before saving and decrypted before loading.

Key features: full disk/partition encryption, encrypted containers (virtual drives), hidden volumes (plausible deniability), system encryption (pre-boot authentication), and cross-platform (Windows, macOS, Linux).

VeraCrypt uses AES, Serpent, and Twofish ciphers (singly or cascaded), with XTS mode. It's more secure than TrueCrypt — it fixed vulnerabilities and added 1000+ iterations of PBKDF2 for stronger brute-force resistance.

Creating Encrypted Containers

Encrypted containers are files that act as virtual encrypted disks. Create one:

# Install VeraCrypt
sudo pacman -S veracrypt  # Arch
sudo apt install veracrypt  # Debian/Ubuntu (from repo or download binary)

# Create a 1GB container (interactive wizard)
veracrypt --create
# Or command-line
veracrypt --create /path/to/container --size=1G --encryption=AES --hash=SHA-512 --filesystem=ext4 --non-interactive

Mount: veracrypt /path/to/container /mnt/secure. Dismount: veracrypt --dismount.

Full Disk & System Encryption

System encryption encrypts the entire OS partition with pre-boot authentication (PBA).

On Linux: encrypt the root partition during OS installation (e.g., Ubuntu's LUKS option) or use VeraCrypt's system encryption for Windows dual-boot setups. VeraCrypt can encrypt a Linux system partition via veracrypt --encrypt-system but LUKS is the native Linux approach.

Cross-platform tip: create a VeraCrypt container on an external drive to share encrypted files between Windows, macOS, and Linux.

Hidden Volumes & Plausible Deniability

VeraCrypt supports hidden volumes — a second encrypted volume inside the free space of a normal (outer) volume. If forced to reveal your password, you give the outer volume password; the hidden volume remains undetectable.

veracrypt --create /path/to/container --size=10G --encryption=AES  # Outer
# Then create hidden volume within free space
veracrypt --create /path/to/container --size=2G --encryption=Serpent --hidden

Mount a hidden volume: veracrypt --mount /path/to/container /mnt/hidden. Use a different password than the outer volume.

Best Practices & Security Tips

Use strong passwords (15+ chars, mixed case + numbers + symbols). Store recovery keys offline (printed, in a safe). Never leave mounted volumes unattended.

For maximum security: cascade ciphers (AES-Twofish-Serpent), SHA-512 hash, PIM (Personal Iterations Multiplier) of 500+ for slower brute-force attempts.

Always use the latest VeraCrypt version (1.26+). Verify PGP signatures on downloaded binaries. VeraCrypt source code is publicly auditable.
