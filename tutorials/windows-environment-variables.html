⚙️ Windows Environment Variables

Understand and manage Windows environment variables — PATH, system vs user, setx, PowerShell, and common use cases.

What Are Environment Variables?

Environment variables are key-value pairs that store configuration data for the operating system and applications. They control things like where programs are located (PATH), where temp files go (TEMP), and user-specific paths (USERPROFILE).

# Examples of environment variables:
%PATH%    — List of directories to search for executables
%USERPROFILE% — Current user's home folder (C:\Users\username)
%TEMP%    — Temp file directory
%APPDATA%  — Application data folder
%SYSTEMROOT% — Windows installation folder (C:\Windows)

System vs User Variables

ScopeVisibilityOverride
SystemAll usersApply to every user on the machine
UserCurrent user onlyOverride system variables of the same name

System variables require administrator privileges to modify. User variables can be edited by the user without admin rights.

The PATH Variable

The most important env var. When you type a command in the terminal, Windows searches each directory in %PATH% for an executable with that name.

# Common PATH entries:
C:\Windows\system32
C:\Users\username\AppData\Local\Programs\Python\Python312\Scripts\
C:\Program Files\Nodejs\
C:\Program Files\Git\cmd\

# View PATH in cmd:
echo %PATH%
# In PowerShell:
$env:PATH

# PATH is semicolon-separated on Windows
Tip: When editing PATH, use the GUI editor (System Properties -> Advanced -> Environment Variables) to avoid accidentally breaking the string. Always keep a backup of the original PATH value.

setx & PowerShell

Modify environment variables from the command line:

:: CMD — setx (persistent, survives reboot)
setx MY_VAR "Hello World"
setx PATH "%PATH%;C:\MyTools"

:: Set a system variable (requires admin)
setx MY_VAR "Value" /M

:: CMD — temporary (current session only)
set MY_VAR=tempvalue
# PowerShell — temporary
$env:MY_VAR = "tempvalue"

# PowerShell — persistent (user)
[Environment]::SetEnvironmentVariable("MY_VAR", "Value", "User")

# PowerShell — persistent (system, admin required)
[Environment]::SetEnvironmentVariable("MY_VAR", "Value", "Machine")

# Remove variable:
[Environment]::SetEnvironmentVariable("MY_VAR", $null, "User")

Note: setx changes take effect in new terminal windows, not the current one.

Practical Uses

  • Add tools to PATH: After installing Python, Node.js, or Git, ensure their directories are in PATH
  • Set API keys: Store API keys as variables (GITHUB_TOKEN, OPENAI_API_KEY) instead of hardcoding them
  • Build shortcuts: %USERPROFILE%\Projects as a quick-access env var
  • CI/CD: Set environment-specific variables in different build pipelines
  • Portable apps: Set custom paths so portable software can find its dependencies
:: Example: Quick access to projects
setx PROJECTS "%USERPROFILE%\Documents\Projects"

:: Then from anywhere:
cd %PROJECTS%
