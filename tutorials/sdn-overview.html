What is SDN?

Software-Defined Networking (SDN) decouples the network control plane from the data plane, enabling centralized network management and programmatic control. In traditional networking, each device makes independent forwarding decisions. In SDN, a central controller computes paths and installs flow entries into switches.

Key architecture layers:

  • Infrastructure Layer -- physical/virtual switches and routers (data plane)
  • Control Layer -- SDN controller that manages forwarding state
  • Application Layer -- network services (firewalling, load balancing, visibility)

The controller communicates with switches via Southbound APIs (primarily OpenFlow) and with applications via Northbound APIs (REST, gRPC).

OpenFlow Protocol

OpenFlow is the first standard SDN protocol. It allows the controller to install flow entries in switches:

# OpenFlow flow entry components:
# - Match fields: ingress port, MAC src/dst, IP src/dst, TCP/UDP ports, VLAN, MPLS
# - Priority: higher priority matches first
# - Instructions: actions or goto-table
# - Actions: output, drop, set-field, push-vlan, group
# - Counters: packets, bytes, duration

# Example flow entry (OVS commands):
# Match all HTTP traffic from 10.0.0.0/8, send to controller
ovs-ofctl add-flow br0 \
    "priority=100,tcp,nw_src=10.0.0.0/8,tp_dst=80,actions=controller"

# Match all ARP, flood via all ports
ovs-ofctl add-flow br0 \
    "priority=10,arp,actions=FLOOD"

# Match IPv4, apply normal L2/L3 switching
ovs-ofctl add-flow br0 \
    "priority=1,ip,actions=NORMAL"

SDN Controllers & Platforms

Popular SDN controllers and platforms:

# OpenDaylight -- Linux Foundation, Java, modular
# Features: OpenFlow, NETCONF, OVSDB, RESTCONF

# ONOS (Open Network Operating System) -- Java, carrier-grade
# Features: Intent-based networking, distributed core

# Floodlight (OpenFlow 1.3) -- Java, OpenFlow-only

# Ryu -- Python-based, lightweight
pip install ryu
ryu-manager ~/simple_switch.py

# POX -- Python, educational/research

# OVS (Open vSwitch) -- production virtual switch
sudo apt install openvswitch-switch
sudo ovs-vsctl add-br br0
sudo ovs-vsctl add-port br0 eth0
sudo ovs-vsctl set-controller br0 tcp:192.168.1.100:6633

Network Virtualization & Overlays

SDN enables network virtualization through overlay tunnels:

# VXLAN (Virtual Extensible LAN)
# Encapsulates L2 frames in UDP (port 4789)
# 24-bit VNI (VXLAN Network Identifier) -- 16 million segments

# OVS VXLAN configuration
sudo ovs-vsctl add-br vxlan-br
sudo ovs-vsctl add-port vxlan-br vxlan1 -- set interface vxlan1 \
    type=vxlan options:remote_ip=192.168.2.100 options:key=5000

# VTEP (VXLAN Tunnel Endpoint) -- switches or hosts

# Geneve -- modern overlay protocol (Linux kernel 3.18+)
# Similar to VXLAN but with variable-length options

# Network Service Mesh (NSM) -- cloud-native SDN for Kubernetes

# EVPN (Ethernet VPN) -- BGP-based control plane for VXLAN
# Standardized in RFC 7432

Practical SDN: Use Cases & Getting Started

Real-world SDN applications and lab setup:

# Mininet -- network emulator for SDN testing
sudo apt install mininet
sudo mn --topo tree,3 --controller=remote,ip=127.0.0.1

# Hands-on SDN scenario:
# 1. Start Mininet with remote controller
# 2. Run Ryu controller with simple_switch
# 3. Verify connectivity between hosts

# SDN use cases:
# - Traffic engineering: dynamic path computation based on utilization
# - Service chaining: steer traffic through firewall > LB > WAN optimizer
# - Network slicing: dedicated virtual networks for tenants
# - Intent-based networking: declare what you want, controller figures out how
# - Cloud networking: OpenStack Neutron, OpenShift SDN, Kubernetes CNI
