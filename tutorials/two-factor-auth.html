Two-Factor Authentication Guide

Everything you need to know about 2FA, TOTP, hardware keys, passkeys, and securing your accounts with a second factor.

Overview

Two-factor authentication (2FA) is one of the most effective measures you can take to protect your online accounts. A password alone can be compromised through phishing, data breaches, or credential stuffing. Adding a second factor makes it drastically harder for an attacker to gain access. This guide breaks down every type of 2FA, the best tools, and how to set them up safely.

What is 2FA/MFA?

Multi-factor authentication (MFA) combines two or more of these categories:

  • Something you know — password, PIN, security question
  • Something you have — phone, hardware key, authenticator app
  • Something you are — fingerprint, face scan, iris scan

Avoid SMS 2FA: SMS codes can be intercepted via SIM swap attacks, SS7 protocol exploits, or phishing. Only use SMS if it is the only option available, and push for TOTP or hardware key support.

Authenticator Apps

Authenticator apps generate time-based or counter-based codes offline on your device. They are more secure than SMS because the secret key never leaves your phone.

Hardware Keys

Hardware security keys are physical devices that authenticate using public-key cryptography. They are phishing-resistant because the key validates the domain before signing.

Registering a Hardware Key

  1. Go to the service's security settings (e.g., Google: myaccount.google.com/security)
  2. Find "Security Key" or "2-Step Verification" section
  3. Plug in your hardware key when prompted
  4. Touch the key's button or sensor to confirm
  5. Name the key (e.g., "Primary YubiKey")
  6. Register at least two keys — one primary and one backup stored separately
Backup Keys: Always register a second hardware key and store it in a safe location (e.g., a fireproof safe). If you lose your only key and have no backup codes, you can be permanently locked out.

TOTP Setup Guide

Time-based One-Time Password (TOTP) is the most common 2FA method. Here is how to set it up step by step.

  1. Enable 2FA on your account (usually under Security Settings)
  2. Choose "Authenticator App" as the method
  3. Scan the QR code with your authenticator app — this contains the secret seed
  4. Verify by entering a 6-digit code generated by the app
  5. Save backup codes immediately (see next section)

The QR code encodes the TOTP secret as a URI:

otpauth://totp/Example:user@email.com?secret=JBSWY3DPEHPK3PXP&issuer=Example&algorithm=SHA1&digits=6&period=30

Most apps also allow manual entry of the secret key (a base32 string) — useful if the QR code does not scan.

Backup Codes

When you enable 2FA, most services provide one-time backup codes. These are crucial if you lose access to your authenticator app or hardware key.

Best Practices for Backup Codes

  • Print them and store in a secure physical location
  • Encrypt and store digitally in a password manager (Bitwarden, KeePassXC)
  • Do not store them in cloud storage unencrypted
  • Test one code to ensure they work, then mark it as used
  • Regenerate codes if you suspect they have been compromised

Recovery Strategies

Locking yourself out of your accounts is a real risk. Plan for recovery before it happens.

Multi-Layered Recovery

  1. Hardware key — keeps at home and in a separate location
  2. Authenticator app backup — Aegis and Ente Auth support encrypted exports
  3. Backup codes — printed and in password manager
  4. Recovery email — a separate email account with its own 2FA
  5. Account recovery — services like Google offer account recovery via trusted devices
Pro Tip: Create a dead man's switch document stored with a lawyer or trusted person. It should contain instructions for accessing your digital accounts in an emergency.

Passkeys

Passkeys are a modern replacement for passwords based on WebAuthn (Web Authentication). Instead of a password, you authenticate with a cryptographic key pair stored on your device.

  • The private key never leaves your device
  • The public key is stored on the server
  • Authentication uses biometrics (fingerprint, face) or device PIN
  • Phishing-resistant by design — the key is bound to the origin domain

Passkeys sync across devices via platform providers:

Platform Sync Method Security
Apple iCloud KeychainiCloud (E2EE)Strong (hardware-backed)

To set up a passkey: go to the service's security settings, look for "Passkeys" or "Security Keys", and follow the device prompt. Examples: accounts.google.com, github.com/settings/security.
