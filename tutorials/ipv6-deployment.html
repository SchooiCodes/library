IPv6 Deployment Guide

Address types, stateless autoconfig (SLAAC), DHCPv6, transition mechanisms, and dual-stack setup.

IPv6 Addressing & Header

IPv6 uses 128-bit addresses (vs IPv4's 32-bit), providing 340 undecillion addresses. It was designed to replace IPv4 completely, but the transition has been gradual through dual-stack deployment.

IPv6 address types:

  • Global Unicast (2000::/3) -- globally routable, similar to IPv4 public addresses
  • Link-Local (fe80::/10) -- automatically assigned, used for local subnet communication
  • Unique Local (fc00::/7) -- private addresses, similar to RFC 1918
  • Multicast (ff00::/8) -- replaces IPv4 broadcast
  • Anycast -- multiple interfaces share one address, traffic goes to nearest

An IPv6 address example: 2001:db8:1234:abcd:5678:90ab:cdef:1234

Address Autoconfiguration: SLAAC vs DHCPv6

IPv6 supports both stateless and stateful address configuration:

# SLAAC (Stateless Address Autoconfiguration)
# Router Advertisement (RA) contains prefix + prefix length
# Host generates its own interface ID (EUI-64 or privacy extension)
# Flags in RA: A=auto, M=managed, O=other

# Router configuration (Cisco)
interface GigabitEthernet0/0
 ipv6 address 2001:db8:1:10::1/64
 ipv6 enable
 ipv6 nd prefix 2001:db8:1:10::/64  # SLAAC advertisement
 ipv6 nd ra-interval 30

# DHCPv6 (stateful) -- provides addresses and options
ipv6 dhcp pool MYPOOL
 prefix-delegation pool MYPOOL-PD
 dns-server 2001:4860:4860::8888
 domain-name example.com

interface GigabitEthernet0/0
 ipv6 dhcp server MYPOOL
 ipv6 nd other-config-flag  # M=0, O=1 (DHCP for options only)

Dual-Stack & Transition Mechanisms

Dual-stack runs IPv4 and IPv6 simultaneously; various other transition techniques exist:

# Dual-stack -- best practice: run both protocols
interface GigabitEthernet0/0
 ip address 192.168.1.1 255.255.255.0
 ipv6 address 2001:db8:1::1/64
 ipv6 enable

# DNS64 + NAT64 -- connect IPv6-only clients to IPv4 servers
# Uses DNS rewriting: AAAA queries synthesized from A records

# Tunneling mechanisms (legacy):
# 6to4 -- automatic tunnel using 2002::/16
# Teredo -- UDP tunnel through NAT devices
# ISATAP -- tunnel over IPv4 intranet

# DS-Lite (Carrier-Grade NAT)
# CPE builds IPv4-in-IPv6 tunnel to ISP's AFTR

# Check dual-stack operation
ping google.com            # IPv4 test
ping -6 google.com         # IPv6 test
nslookup -type=AAAA google.com
curl -6 https://ipv6.google.com

IPv6 Security Considerations

IPv6 introduces new security challenges beyond IPv4:

# ICMPv6 is essential (unlike ICMPv4 which can be blocked)
# Neighbor Discovery Protocol (NDP) replaces ARP
# Must allow: Neighbor Solicitation (135), Neighbor Advertisement (136)
# Router Solicitation (133), Router Advertisement (134)

# RA Guard -- prevent rogue router advertisements
ipv6 nd raguard

# DHCPv6 Guard -- prevent rogue DHCPv6 servers
ipv6 dhcp guard

# Source address validation (First Hop Security)
ipv6 source-guard

# Privacy Extensions (temporary addresses)
# Windows: Enabled by default
# Linux: /proc/sys/net/ipv6/conf/all/use_tempaddr = 2
# macOS: Enabled by default

# Firewall example (nftables)
table ip6 filter {
    chain input {
        type filter hook input priority 0
        icmpv6 type { nd-neighbor-solicit, nd-neighbor-advert, nd-router-solicit, nd-router-advert } accept
        ct state established,related accept
    }
}

Deployment Planning & Testing

Plan and test your IPv6 deployment:

# IPv6 readiness checklist:
# [ ] ISP provides IPv6 prefix (check with your provider)
# [ ] Customer Premise Equipment (CPE) supports IPv6
# [ ] Core routers/switches have IPv6 enabled
# [ ] DNS infrastructure serves AAAA records
# [ ] Firewall rules include IPv6 (ip6tables or nftables ip6 family)
# [ ] Applications tested on IPv6
# [ ] Monitoring systems track IPv6 metrics

# Test tools:
ip -6 addr show
ip -6 route show
ip -6 neigh show

# Online IPv6 tests:
# https://test-ipv6.com/
