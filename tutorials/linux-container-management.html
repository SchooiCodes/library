Linux Container Management

Docker, Podman, LXC, and container orchestration for Linux power users.

Docker vs Podman vs LXC

Three main container technologies on Linux:

  • Docker — most popular, daemon-based, root privilege required. docker command, dockerd daemon. Docker Compose for multi-container apps.
  • Podman — daemonless (child process of your terminal), can run rootless, supports same OCI images. Drop-in replacement: alias docker=podman.
  • LXC/LXD — system containers (full OS environment, not just apps). More like lightweight VMs. Good for running different distros simultaneously.

For most users: Docker for compatibility (largest ecosystem). Podman for better security (rootless by default). LXC for full-system isolation.

Docker Installation & Essentials

# Install Docker
sudo pacman -S docker docker-compose  # Arch
sudo usermod -aG docker $USER        # No sudo needed for docker
newgrp docker                        # Reload group
sudo systemctl enable --now docker

# Basic commands
docker pull nginx:alpine
docker run -d --name web -p 80:80 nginx:alpine
docker ps          # Running containers
docker images      # Downloaded images
docker exec -it web sh  # Shell into container
docker logs -f web      # Follow container logs
docker compose up -d    # Start compose project

# Cleanup
docker system prune -a  # Remove unused images, containers, networks

Docker Compose & Networking

Docker Compose defines multi-service apps in YAML:

# docker-compose.yml
version: "3.9"
services:
  web:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
  db:
    image: postgres:16
    environment:
      POSTGRES_PASSWORD: changeme
    volumes:
      - pgdata:/var/lib/postgresql/data
  app:
    build: .
    depends_on:
      - db

volumes:
  pgdata:

Networking: default bridge (isolated), host (share host network), overlay (Swarm). Custom networks: docker network create mynet.

Container Storage & Volumes

Persist data with volumes and bind mounts:

# Named volume (managed by Docker)
docker volume create nginx-data
docker run -v nginx-data:/usr/share/nginx/html nginx

# Bind mount (directory on host)
docker run -v /host/path:/container/path nginx

# tmpfs mount (in-memory, for sensitive data)
docker run --tmpfs /tmp:noexec,nosuid,size=64M nginx

# Back up a volume
docker run --rm -v nginx-data:/source -v $(pwd):/backup alpine tar czf /backup/nginx-backup.tar.gz -C /source .

Volumes are the preferred way to persist data. They're faster on macOS/Windows and manageable via docker volume commands.

Podman Rootless Containers

Podman runs rootless and integrates with systemd:

# Podman equivalent commands (same interface)
podman pull nginx
podman run -d --name web -p 8080:80 nginx
podman ps

# Generate systemd service for a container
podman generate systemd --name web --files --new
# This creates container-web.service
systemctl --user enable --now container-web.service

# Podman Compose (alternative to Docker Compose)
podman-compose up -d

# Pod Machine (VM-based for macOS/Windows compatibility)
podman machine init
podman machine start

Podman's key advantage: no daemon, better security with rootless, systemd integration by default, and podman play kube for deploying Kubernetes YAML directly.
