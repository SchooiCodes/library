Load Balancing

Load balancing — algorithms, reverse proxies, health checks, and global server load balancing.

Table of Contents (2 sections)

  • Algorithms & Reverse Proxy
  • Health Checks & GSLB

Algorithms & Reverse Proxy

Load balancing: distribute traffic across multiple servers. Algorithms: round-robin (sequential), least-connections (to fewest active), IP hash (stickiness — session persistence), weighted round-robin (different capacities), random, least-response-time. Reverse proxy: nginx (upstream block), HAProxy (frontend/backend), Traefik (auto-discovery, Let's Encrypt), Caddy (automatic HTTPS). Config (nginx): upstream backend { server 10.0.0.1:3000 weight=3; server 10.0.0.2:3000; } server { location / { proxy_pass http://backend; } }. Session persistence: sticky cookies (Set-Cookie with backend ID), IP hash, Redis-backed sessions.

Health Checks & GSLB

Health checks: passive (proxy detects failure by connection errors), active (periodic probe — HTTP /health, TCP connect). Unhealthy: remove from pool until healthy again. Circuit breaker: stop sending after N failures. Rate limiting: nginx limit_req_zone, limit_conn_zone. GSLB (Global Server Load Balancing): DNS-based (geo-aware DNS, weighted records), Anycast (same IP, routed to nearest datacenter). Cloud LBs: AWS ALB (HTTP/HTTPS), NLB (TCP/UDP), GCLB (global HTTP), Azure Load Balancer. Layer 4 (transport) vs Layer 7 (application): L4 faster, L7 smarter (header-based routing, content switching).
