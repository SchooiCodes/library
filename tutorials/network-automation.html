Network Automation with Ansible & Python

Automate network configuration, Netmiko, NAPALM, Ansible network modules, and CI/CD for networks.

Why Network Automation?

Manual network configuration is slow, error-prone, and difficult to audit at scale. Network automation replaces CLI-based, human-intensive workflows with programmatic, version-controlled, and repeatable processes. Benefits include reduced configuration drift, faster deployments, consistent multi-vendor management, and self-documenting infrastructure.

Key tools in the network automation ecosystem:

  • Netmiko -- Python library for SSH-based network device interaction
  • NAPALM -- multi-vendor abstraction layer with consistent API
  • Ansible -- agentless automation with declarative YAML playbooks
  • Nornir -- Python-based task automation framework
  • pyATS -- Cisco test automation system

Netmiko: SSH Automation in Python

Netmiko simplifies SSH connections to network devices:

from netmiko import ConnectHandler

# Define device
cisco_device = {
    'device_type': 'cisco_ios',
    'host': '192.168.1.1',
    'username': 'admin',
    'password': 'password',
    'secret': 'enable_password',
}

# Connect and run commands
connection = ConnectHandler(**cisco_device)
connection.enable()

# Send one command
output = connection.send_command('show ip interface brief')
print(output)

# Send configuration
config_commands = [
    'interface GigabitEthernet0/1',
    'description LAN Access Port',
    'switchport mode access',
    'switchport access vlan 10',
    'no shutdown',
]
output = connection.send_config_set(config_commands)

# Save config
connection.save_config()
connection.disconnect()

NAPALM: Multi-Vendor Abstraction

NAPALM provides a consistent API across different network vendors:

from napalm import get_network_driver

# Connect to device
driver = get_network_driver('ios')
device = driver(
    hostname='192.168.1.1',
    username='admin',
    password='password',
    optional_args={'secret': 'enable'},
)
device.open()

# Consistent methods across vendors
facts = device.get_facts()
interfaces = device.get_interfaces()
arp_table = device.get_arp_table()
ntp_peers = device.get_ntp_peers()

# Deploy configuration
config = """
interface GigabitEthernet0/1
 description Automated port
 switchport mode access
 switchport access vlan 20
"""
device.load_merge_candidate(config=config)

# Compare and commit
print(device.compare_config())
device.commit_config()
device.close()

Ansible Network Automation

Ansible manages network devices using agentless YAML playbooks:

# inventory/hosts.yml
all:
  hosts:
    core-01:
      ansible_host: 192.168.1.1
      ansible_network_os: cisco.ios.ios
    core-02:
      ansible_host: 192.168.1.2
      ansible_network_os: cisco.ios.ios

# playbooks/configure-vlans.yml
---
- name: Configure VLANs on Core Switches
  hosts: core
  gather_facts: false
  tasks:
    - name: Ensure VLAN 10 exists
      cisco.ios.ios_vlan:
        vlan_id: 10
        name: USERS
        state: present

    - name: Apply interface config
      cisco.ios.ios_interfaces:
        config:
          - name: GigabitEthernet0/1
            description: "User Access Port"
            mode: access
            enabled: true

CI/CD for Network & Validation Tools

Integrate network automation with GitOps pipelines:

# GitHub Actions example
# .github/workflows/network-deploy.yml
name: Deploy Network Config
on:
  push:
    branches: [main]
    paths:
      - 'network-configs/**'

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Validate and deploy
        run: |
          ansible-playbook playbooks/validate.yml
          ansible-playbook playbooks/deploy.yml --check
          ansible-playbook playbooks/deploy.yml

# Network validation tools:
# Batfish -- static config analysis (find ACL errors, routing issues)
# pytest + pyATS -- automated test suites for network states
# Ansible --assert -- ensure post-deployment state

# Backup automation (run daily):
# ansible-playbook playbooks/backup-configs.yml
# Stores configs in version control
