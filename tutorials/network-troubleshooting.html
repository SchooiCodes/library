Practical guide to diagnosing and fixing network issues using command-line tools.

📖 Introduction

Network problems are common. This guide covers command-line tools and techniques to diagnose and fix them. All commands work on Windows, most work on Linux/macOS.

1. Ping (Connectivity Test)

ping is the most basic network diagnostic tool. It sends ICMP echo requests to a target and waits for a reply.

ping google.com
ping -t google.com        # Continuous ping (Ctrl+C to stop)
ping -n 10 google.com      # Send 10 pings (Windows)
ping -c 10 google.com      # Send 10 pings (Linux/macOS)

What to look for: Reply vs Request timed out. Consistent vs fluctuating response times. Packet loss percentage.

No reply? Could be: Internet down, DNS issue, firewall blocking, target blocking ICMP.

2. Tracert / Traceroute (Path Discovery)

Shows each hop between you and the target — great for identifying where packets are being dropped or slowed down.

tracert google.com        # Windows
traceroute google.com      # Linux/macOS

* * * means no response from that hop (could be firewall or congestion). High latency at a specific hop? That router might be the bottleneck.

3. IPConfig (Windows Network Config)

View and manage your network configuration on Windows.

ipconfig             # Show IP, subnet mask, default gateway
ipconfig /all          # Detailed config: MAC, DHCP, DNS servers
ipconfig /flushdns        # Clear DNS cache (fixes DNS issues)
ipconfig /release        # Release DHCP lease
ipconfig /renew         # Get new DHCP lease (fixes IP conflicts)
ipconfig /displaydns       # Show cached DNS entries

4. NSLookup (DNS Lookup)

Query DNS servers to verify domain resolution.

nslookup google.com       # Find IP for a domain
nslookup google.com 8.8.8.8   # Query a specific DNS server
nslookup -type=mx google.com   # Find mail servers

Compare results from different DNS servers (your ISP, 8.8.8.8, 1.1.1.1) to pinpoint DNS issues.

5. Netstat (Connection Viewer)

List active connections, listening ports, and process ownership.

netstat -an           # All active connections and listening ports
netstat -b            # Show programs owning connections (admin)
netstat -ano           # Show connections with process IDs
netstat -rn           # Show routing table

Great for: Checking if a service is listening, finding suspicious connections, seeing what's using your network.

6. PathPing (Windows Advanced Traceroute)

Combines ping and traceroute. Shows per-hop latency and packet loss statistics.

pathping google.com

Takes several minutes to complete. More detailed than tracert.

8. Router Troubleshooting

  • Power cycle: Unplug for 30 seconds, plug back in
  • Check lights: Power, Internet/WAN, WiFi/LAN
  • Access admin panel: 192.168.0.1 or 192.168.1.1 in browser
  • Update firmware
  • Change DNS: Set to 1.1.1.1 (Cloudflare) or 8.8.8.8 (Google)
  • Enable/disable QoS for gaming/streaming

9. Windows Network Reset

Settings → Network & Internet → Advanced network settings → Network reset

Or via command line:

netsh int ip reset
netsh winsock reset

Then restart your computer. This resets all network adapters to default.
