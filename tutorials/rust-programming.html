Rust Programming

Systems programming with Rust -- ownership, borrowing, lifetimes, and concurrency.

Ownership & Borrowing

Rust: memory-safe without GC. Ownership: each value has one owner. Move: assigning transfers ownership. Borrowing: &T (immutable reference), &mut T (mutable reference, exclusive). Lifetimes: 'a -- annotate how long references are valid. Lifetime elision: compiler infers common patterns. String vs &str: String (owned, heap), &str (slice, reference). Struct: struct User { name: String, age: u32 }. Enum: enum Option { None, Some(T) }, enum Result { Ok(T), Err(E) }. Pattern matching: match, if let. Methods: impl User { fn new(name: &str) -> Self { ... } }.

Concurrency & Eco

Threads: std::thread::spawn, channels (std::sync::mpsc), Mutex, Arc (atomic reference counting). Send + Sync traits: auto-implemented for types safe to transfer/share between threads. Async: async/await with tokio or async-std. Cargo: package manager -- cargo new, build, run, test, clippy (linter), rustfmt (formatter). Crates: crates.io -- community packages. serde: serialization/deserialization. rayon: data parallelism. error handling: Result, ? operator. panic! vs Result. Testing: #[test], assert!, #[should_panic]. Popular crates: clap (CLI), actix-web/axum (web), diesel/sqlx (DB), tokio (async). Unsafe: unsafe code for FFI, raw pointers. FFI: extern C. Build systems: Cargo workspaces, build.rs. Embedded: no_std for bare-metal. WASM: wasm-pack for web.
