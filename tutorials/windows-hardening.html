ASR Rules (Attack Surface Reduction)

ASR rules are part of Windows Defender Exploit Guard. They block behaviours commonly used by malware — script execution, Office child processes, USB-borne threats, and credential theft. Rules are managed via Group Policy, Intune, or PowerShell:

# List all ASR rules and their current state
Get-MpPreference | Select-Object -ExpandProperty AttackSurfaceReductionRules_Ids

# Enable a specific ASR rule (GUID-based)
Add-MpPreference -AttackSurfaceReductionRules_Ids "b2b3f03d-671d-4e71-a0c8-2f49a8bdd3c1" -AttackSurfaceReductionRules_Actions Enabled

# Block Office applications from creating child processes
# GUID: d4e940f7-3cbf-4b4c-b5b1-2d6c1f7b3c9a
Add-MpPreference -AttackSurfaceReductionRules_Ids "d4e940f7-3cbf-4b4c-b5b1-2d6c1f7b3c9a" -AttackSurfaceReductionRules_Actions Enabled

# Enable all ASR rules in audit mode first (test before blocking)
Set-MpPreference -AttackSurfaceReductionRules_Actions AuditMode
Get-MpPreference -ASRRulesExclusions
Tip: Always run ASR rules in audit mode for a week before enabling them. Check the Microsoft-Windows-Windows Defender/Operational event log (Event ID 1121 block, 1122 audit) to see what would have been blocked.

Windows Defender Configuration

Windows Defender (Microsoft Defender Antivirus) is surprisingly effective when properly configured. Disable it only if you have a third-party AV — never leave your system unprotected:

# Enable cloud-delivered protection and automatic sample submission
Set-MpPreference -CloudBlockLevel High -CloudTimeout 50
Set-MpPreference -SubmitSamplesConsent Always
Set-MpPreference -PUAProtection Enabled
Set-MpPreference -DisableArchiveScanning $false
Set-MpPreference -DisableRemovableDriveScanning $false

# Run a full scan
Start-MpScan -ScanType FullScan

# Update definitions
Update-MpSignature

# Check current protection status
Get-MpComputerStatus | Select-Object AMProductVersion, AMServiceEnabled,
  AntivirusEnabled, RealTimeProtectionEnabled, PUAEnabled

AppLocker

AppLocker controls exactly which executables, scripts, installers, and DLLs can run. It is far more effective than simple antivirus for preventing unauthorized software:

# Create default AppLocker rules (allow %PROGRAMFILES%, %WINDIR%, %SYSTEM32%)
New-AppLockerPolicy -RuleType Exe,Script,Dll,Msi -User Everyone -Xml > C:\AppLocker\default-policy.xml

# Apply the policy
Set-AppLockerPolicy -XmlPolicy C:\AppLocker\default-policy.xml -Merge

# Add a publisher rule for a specific app
$rule = New-AppLockerPolicy -RuleType Exe -Path "C:\Program Files\VerifiedApp\*.exe" -User Everyone
Set-AppLockerPolicy -XmlPolicy $rule -Merge

# Audit mode — log violations without blocking
Set-AppLockerPolicy -XmlPolicy C:\AppLocker\audit-policy.xml -Merge
Danger Zone: Test AppLocker policies in audit mode first. A misconfigured policy can lock you out of your own system. Always keep a secondary admin account or a recovery method. The Application Identity service must be running for AppLocker to function.

BitLocker

BitLocker provides full-disk encryption using AES (XTS-AES 128 or 256). It requires a TPM 2.0 module and a supported Windows edition (Pro, Enterprise, Education):

# Enable BitLocker on C: drive
Enable-BitLocker -MountPoint "C:" -RecoveryPasswordProtector -TpmProtector

# Add a PIN + TPM protector (requires pre-boot authentication)
Add-BitLockerKeyProtector -MountPoint "C:" -TpmAndPinProtector -Pin "123456"

# Backup recovery key to Active Directory or a file
Backup-BitLockerKeyProtector -MountPoint "C:" -KeyProtectorId (Get-BitLockerVolume -MountPoint "C:").KeyProtector[0].KeyProtectorId
(Get-BitLockerVolume -MountPoint "C:").KeyProtector[0].RecoveryPassword | Out-File C:\BitLocker-Recovery.txt

# Check BitLocker status
Get-BitLockerVolume -MountPoint "C:" | Format-List

# Suspend BitLocker (for firmware updates)
Suspend-BitLocker -MountPoint "C:" -RebootCount 1
Resume-BitLocker -MountPoint "C:"

User Account Security

The weakest link is almost always the user account. Apply these settings to every Windows machine:

  • Use a standard (non-admin) account for daily work. Only use the admin account for installations and configuration. Use runas or Shift+right-click → Run as different user for elevation.
  • Enable UAC at maximum — set to Always notify. This prevents silent elevation by malware.
  • Disable the built-in Administrator account unless needed — it is a prime target.
  • Use Windows Hello for Business (PIN, biometrics) instead of passwords where possible — PIN is device-bound and more secure.
  • Set account lockout policy via secpol.msc → Account Lockout Policy (5 attempts, 30 min lockout).
# Check UAC level
Get-ItemProperty -Path "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" -Name EnableLUA

# Disable the built-in Administrator account
Disable-LocalUser -Name "Administrator"

# List all local users
Get-LocalUser | Where-Object Enabled -eq $true

# Set account lockout policy via Net CLI
net accounts /lockoutthreshold:5 /lockoutduration:30 /lockoutwindow:30

Group Policy Hardening

Group Policy is the centralised way to harden Windows. Key hardening policies:

  • Computer Configuration → Windows Settings → Security Settings → Local Policies → Security Options:
    • Network access: Do not allow anonymous enumeration of SAM accounts → Enabled
    • Microsoft network server: Digitally sign communications (always) → Enabled
    • Shutdown: Allow system to be shut down without having to log on → Disabled
  • Windows Defender Firewall: Block inbound by default, allow only necessary outbound. Log dropped packets.
  • Restrict Remote Desktop to specific security groups and require NLA (Network Level Authentication).
  • Disable SMBv1 — it is ancient, insecure, and the vector for EternalBlue (WannaCry).
# Disable SMBv1 (requires reboot)
Set-SmbServerConfiguration -EnableSMB1Protocol $false -Force
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters" SMB1 -Value 0

# Block inbound RDP except from a specific IP
New-NetFirewallRule -DisplayName "Block RDP except VPN" -Direction Inbound -Protocol TCP -LocalPort 3389 -Action Block
New-NetFirewallRule -DisplayName "Allow RDP from VPN subnet" -Direction Inbound -Protocol TCP -LocalPort 3389 -RemoteAddress "10.0.0.0/8" -Action Allow

# Enable logging for Windows Defender Firewall
Set-NetFirewallProfile -Profile Domain,Public,Private -LogFileName "%SystemRoot%\System32\LogFiles\Firewall\pfirewall.log" -LogMaxSizeKilobytes 16384 -LogAllowed $true -LogBlocked $true

Third-party Tools

These tools complement built-in hardening features:

  • Sysinternals Suite (Microsoft) — Autoruns, Process Monitor, PsExec, AccessChk. Essential for analysing and securing a system.
  • HardeningKitty — Hardening assessment and automation script based on BSI and Microsoft benchmarks.
  • Strela — Community-driven Windows hardening tool. Applies CIS benchmarks automatically.
  • O&O ShutUp10++ — Free tool to toggle privacy and security settings in Windows 10/11.
  • WPD (Windows Privacy Dashboard) — Blocks telemetry, disables Cortana, and removes OneDrive integration.
Tip: The CIS Microsoft Windows Benchmark (free download from cisecurity.org) is the gold standard for Windows hardening. It covers 500+ configuration items with detailed remediation steps. Run a compliance scan with Invoke-HardeningKitty to measure your baseline.
