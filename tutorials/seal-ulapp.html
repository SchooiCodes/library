Seal & ulapp

Android video downloader (Seal) and LAN file sharing app (ulapp) for offline media.

Overview

Seal is an Android front-end for yt-dlp, the powerful command-line video downloader. It lets you download videos and audio from YouTube, Bilibili, Niconico, Twitter/X, Instagram, Reddit, and hundreds of other sites — all through a clean Material You interface. No ads, no tracking, no proprietary blobs.

ulapp is a lightweight file-sharing Android app that creates local Wi-Fi hotspots or connects to LAN networks to share files without internet. Think of it as a portable FTP server in your pocket, with QR codes for instant pairing.

Required Permissions

Seal needs Notifications (for download progress) and Storage (to save files). On Android 13+, grant Manage all files if you want to save to arbitrary directories.

Seal — Downloading Videos

Basic Download

  1. Copy a video URL from your browser or app (e.g. YouTube, Twitter/X, Instagram).
  2. Open Seal — the URL is auto-detected from your clipboard.
  3. Select format (MP4, WebM, MKV) and quality (best, 1080p, 720p, audio-only).
  4. Tap the download button. The video saves to Downloads/Seal/.

Batch Downloads

You can paste multiple URLs (one per line) to queue them. Seal processes them sequentially. Each URL can have its own format settings.

Audio-Only Extraction

Select Audio tab. Choose format (MP3, M4A, OPUS, FLAC). The app extracts audio using yt-dlp's --extract-audio option. For YouTube music, this is the easiest way to build a local music library.

Custom yt-dlp Arguments

In Seal's settings, you can pass raw yt-dlp flags for advanced use:

# Download only subtitles
--write-subs --skip-download

# Download playlist from index 5 to 10
--playlist-start 5 --playlist-end 10

# Limit download speed
--limit-rate 1M

# Download with cookies (for age-restricted content)
--cookies /path/to/cookies.txt

ulapp — Installation

F-Droid

ulapp is available on F-Droid at https://f-droid.org/packages/com.ul.app/.

GitHub Releases

curl -LO https://github.com/ulapp/releases/latest/download/ulapp-v2.1.0.apk
adb install ulapp-v2.1.0.apk

Permissions

  • Location — Required on Android 10+ for Wi-Fi scanning (no GPS used)
  • Storage — To read files you want to share and save received files
  • Nearby Devices — For Wi-Fi Direct peer discovery

ulapp — Sharing Files

Mode 1: Hotspot (No Router Needed)

  1. Sender opens ulapp → selects Create Hotspot.
  2. A Wi-Fi network is created (SSID: ulapp-XXXX).
  3. Receiver connects to that Wi-Fi network.
  4. Sender picks files and taps Send.
  5. Receiver gets a notification to accept.

Mode 2: LAN (Same Wi-Fi Network)

  1. Both devices connect to the same Wi-Fi network.
  2. Both open ulapp — devices appear in a peer list automatically.
  3. Tap a device, select files, send.

QR Code Pairing

ulapp can generate a QR code that encodes the device's IP and port. The receiver scans it to initiate a transfer immediately — no browsing through peer lists.

Creating Rooms

You can create a "room" (shared folder) that multiple people can join:

Sender: Room > Create Room > "Party Photos"
Others: Room > Join Room > enter room name or scan QR

Everyone in the room sees uploaded files in a shared gallery.

Speed

ulapp uses raw TCP sockets for transfer. On Wi-Fi 5 (802.11ac), expect 20-50 MB/s. On Wi-Fi Direct, up to 50 MB/s. On a hotspot (soft AP), about 10-20 MB/s due to the single-radio bottleneck.

Troubleshooting

IssueSolution
Seal: "This video is unavailable"The site may require cookies. In Seal settings, import cookies from your browser (use a cookies.txt extension).
Seal: Download stuck at 0%Check your internet connection. The video may be region-locked. Try a VPN. Update yt-dlp via Seal's settings.
Seal: No audio in downloaded videoSelect a format that includes audio (e.g. "bestvideo+bestaudio" or a container like MP4/MKV). Some sites separate streams.
ulapp: Devices not seeing each otherEnsure both devices are on the same Wi-Fi network. Check that hotspot mode is not conflicting. Try QR code as fallback.
ulapp: Transfer very slowHotspot mode is slower than LAN mode due to single-radio duplex. Switch to LAN (both on same router) for better speed.
ulapp: App crashes on openGrant all required permissions (Location, Storage, Nearby). On Android 12+, location permission is mandatory for Wi-Fi scanning.
