LXC vs Docker: System Containers vs Application Containers

LXC (Linux Containers) provides system containers -- lightweight virtual machines that run a full operating system with init, systemd, and multiple processes. Docker focuses on application containers -- single-process, ephemeral environments.

Key differences:

  • LXC/LXD -- boots a full OS (like a VM but using kernel namespaces); persistent, long-running
  • Docker -- runs a single application process; stateless, designed for orchestration
  • Use LXC when -- you need a full Linux environment (development VM, CI runners, legacy apps)
  • Use Docker when -- microservices, stateless apps, Kubernetes orchestration

Installing LXD & Creating Containers

LXD is the container hypervisor that manages LXC containers:

# Install LXD
sudo snap install lxd
# OR from apt
sudo apt install lxd lxd-client

# Initialize LXD (interactive)
sudo lxd init

# Launch a container
lxc launch ubuntu:24.04 my-container
lxc launch images:centos/9/amd64 centos-dev

# List containers
lxc list

# Execute commands inside
lxc exec my-container -- bash
lxc exec my-container -- apt update

# Stop and delete
lxc stop my-container
lxc delete my-container

Container Profiles & Configuration

Profiles define container resource limits, network, and storage:

# Create a profile
lxc profile create dev-env

# Configure resource limits
lxc profile set dev-env limits.cpu 2
lxc profile set dev-env limits.memory 4GB
lxc profile set dev-env limits.disk.priority 10

# Configure networking
lxc profile device add dev-env eth0 nic nictype=bridged parent=lxdbr0

# Configure storage
lxc profile device add dev-env root disk path=/ pool=default size=20GB

# Apply profile
lxc launch ubuntu:24.04 dev-container --profile dev-env

# Edit profile YAML directly
lxc profile edit dev-env

Networking & Storage in LXD

LXD provides flexible networking options:

# Default bridge (NAT)
lxc network show lxdbr0

# Create a new bridge
lxc network create mybr0 --type=bridge \
    ipv4.address=10.10.100.1/24 ipv4.nat=true

# Attach container to network
lxc network attach mybr0 my-container eth0

# Create storage pool
lxc storage create ssd zfs source=/dev/sdb
lxc storage create data dir source=/mnt/storage

# Attach storage to container
lxc storage volume create ssd my-volume --size=10GB
lxc storage volume attach ssd my-volume my-container /mnt/data

Snapshots, Backup & Migration

LXD's snapshot and migration features make it great for development environments:

# Create snapshot
lxc snapshot my-container clean-install
lxc snapshot my-container before-upgrade

# List and restore
lxc list my-container/snapshots
lxc restore my-container clean-install

# Export/import container (full backup)
lxc export my-container /backup/my-container.tar.gz
lxc import /backup/my-container.tar.gz

# Live migration (between hosts)
lxc move my-container remote-host:

# Copy container
lxc copy my-container my-container-clone

# Publish as image
lxc publish my-container --alias my-custom-image --public
