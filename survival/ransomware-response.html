Ransomware Response

What to do when ransomware strikes - isolate, identify the strain, find decryption tools, and recover without paying criminals.

Table of Contents

  • Isolate the System
  • Identify the Variant
  • Decryption Tools (No More Ransom!)
  • Restore from Backup
  • Prevention

Isolate the System

Seconds matter. Ransomware encrypts files within minutes.

  • Pull Ethernet - Do not shut down (some accelerate on shutdown)
  • Disable Wi-Fi and Bluetooth
  • Document the ransom note - Photo, Bitcoin address, email
  • Do not power off - RAM may have encryption keys
Do not pay. Only 54% of victims who paid recovered data.

Identify the Variant

Identifying the strain determines if a decryption tool exists.

  • Check ransom note - ReadMe.txt, DECRYPTION_INSTRUCTIONS.html
  • Check file extensions - .lockbit, .alphv, .locked, .cry
  • Use ID Ransomware - id-ransomware.malwarehunterteam.com

Common variants: LockBit 3.0, BlackCat/ALPHV, Clop, Akira, Black Basta.

Decryption Tools (No More Ransom!)

No More Ransom! hosts free tools for 200+ ransomware variants.

Sources: No More Ransom (nomoreransom.org), Emsisoft Decryptor, Kaspersky RakhniDecryptor, Avast Ransomware Decryptor.

Pro tip: Before running a decryptor, back up encrypted files. If the tool damages them, you still have originals.

Restore from Backup

If you have backups, this is fastest.

# 1. Confirm backup is clean
# 2. Wipe infected system, reinstall Windows
# 3. Reinstall OS - apply updates - install AV
# 4. Connect backup - restore files
# Do NOT connect backup until OS is clean!

Shadow Copy recovery: Check vssadmin list shadows. Right-click folder - Properties - Previous Versions.

Prevention

  • Offline backups - Physically disconnected = immune
  • Limited user accounts - Daily work from standard account
  • Disable RDP - #1 entry vector
  • Patch aggressively - Apply patches within 48 hours

Immediate Response Steps Timeline

Every second counts during a ransomware attack. Follow this timeline:

Timeframe Actions
First 5 minutesPull Ethernet cable. Disable Wi-Fi/Bluetooth. Unplug external drives. Photograph the ransom note on screen. Take photo of file extensions. Do NOT shut down — RAM may hold encryption keys.
First hourIdentify the ransomware strain (file extension, ransom note, ID Ransomware). Check if decryption tools exist. Disconnect network shares. Change all admin passwords from a clean device. Document everything (timeline, files affected, ransom note details).
First 24 hoursCheck backups for integrity. Scan backup media for infection before connecting. If backups are clean, begin restore on wiped system. If no backups, attempt decryption tools. If no decryption possible, consider professional recovery. Report to authorities.

Disconnecting Infected Machines (Network Isolation)

Network isolation prevents ransomware from spreading to other machines and network shares:

# Network isolation procedure:
# 1. PHYSICALLY pull the Ethernet cable from infected machine
#    Do NOT rely on software disconnection — malware may ignore it
# 2. Disable Wi-Fi (hardware switch or BIOS disable)
# 3. Disable Bluetooth in Device Manager
# 4. Disconnect all external drives (USB, eSATA, Thunderbolt)
# 5. On the network switch/router:
#    - Identify the infected machine's port
#    - Disable the port or unplug the cable
#    - Check for other machines with similar symptoms
# 6. On other machines on the network:
#    - Disconnect from network shares immediately
#    - Run offline antivirus scans
#    - Check for encrypted files
# 7. Do NOT reconnect any machine until:
#    - All infected machines are identified and isolated
#    - Root cause is determined
#    - Cleanup is complete
#
# If ransomware spreads over SMB/RDP:
# - Block SMB ports (445, 139) at the firewall
# - Block RDP (3389) at the firewall
# - Disable Windows Remote Management (WinRM)

Identifying Ransomware Strain

Identifying the exact ransomware variant is critical for finding a decryption tool:

  • Ransom note patterns — Ransom notes have distinctive names: ReadMe.txt, DECRYPT_INSTRUCTION.html, HOW_TO_DECRYPT.hta, #DECRYPT#.txt. Note the exact filename and content — each ransomware family uses specific wording.
  • File extensions — Encrypted files get new extensions: .lockbit, .alphv, .clop, .akira, .basta, .cry, .encrypted. The extension often identifies the family.
  • ID Ransomware tool — Upload ransom note text and/or an encrypted file to id-ransomware.malwarehunterteam.com. It compares against a database of 1000+ variants and identifies the strain.
  • Desktop wallpaper — Many variants change the desktop wallpaper with instructions. Photograph this — it contains valuable identifying information.
  • Tor site — The ransom note usually includes a .onion URL. Visit it (with Tor) to understand the group's demands. This also helps confirm the variant.

Decryption Tools (NoMoreRansom, Emsisoft)

Free decryption tools exist for many ransomware variants. Here is where to find them:

# Decryption tool sources:
# 1. No More Ransom (nomoreransom.org)
#    - Largest collection: 200+ free decryption tools
#    - Upload encrypted file or ransom note to identify
#    - Download and run the matching decryptor
#    - Project by Europol, Kaspersky, and McAfee

# 2. Emsisoft Decrypter
#    - Free decryptors for 30+ ransomware families
#    - Run the tool, select encrypted files, choose output
#    - Works on: Stopped, Satana, GlobeImposter, etc.

# 3. Kaspersky RakhniDecryptor
#    - Targets: Rakhni, Trojan-Ransom.Win32.Rakhni
#    - Also works on some older CryptoWall variants

# 4. Avast Ransomware Decryptor
#    - Tools for: Avest, Alcatraz, BadEncrypt, Crypt888, etc.

# 5. Avira Ransomware Decryptor
#    - Targets: Phobos, Erebus, Xorist, etc.

# Before running any decryptor:
# - BACK UP encrypted files first
# - Run the decryptor on COPIES, not originals
# - Some decryptors may corrupt files if they guess wrong
# - Disconnect the machine from the network before decrypting

Backup Restoration Procedure (Clean Environment)

Restoring from backup is the fastest way to recover, but only if done correctly:

# Safe backup restoration procedure:
# 1. DO NOT connect backup media to infected machine
# 2. Boot from a clean live USB (SystemRescue or Ubuntu)
# 3. Scan backup media for infection:
#    - Mount backup drive read-only
#    - Run antivirus scan on backup files
#    - Check for encrypted files in backup
#    - Look for ransom notes in backup folders
# 4. If backup is clean:
#    - Wipe infected system completely (DBAN/nwipe)
#    - Reinstall OS from known-clean media
#    - Apply all security updates
#    - Install antivirus software
#    - Connect backup drive
# 5. Restore files:
#    - Scan each restored file with AV
#    - Do NOT restore .exe, .scr, .dll from backup
#    - Reinstall applications fresh, do not restore them
# 6. After restoration:
#    - Change all passwords
#    - Reset API keys and tokens
#    - Enable MFA on all accounts
#    - Review and apply security improvements

Paying Ransom (Legal Considerations, Negotiation, Crypto Payment)

Paying a ransom is controversial and has legal implications:

  • Legal status — Paying ransomware is NOT explicitly illegal in most countries, but it may violate sanctions laws. If the ransomware group is sanctioned (e.g., Conti/EvilCorp in the US), paying them is illegal. Check OFAC sanctions list before even considering payment.
  • Success rate — Only 54% of victims who paid recovered their data (Sophos 2024 study). 32% paid but did not get all data back. 11% paid and got nothing.
  • Negotiation points — Ransomware groups expect negotiation. Start at 10-20% of the demand. They rarely accept the first offer. Use a professional negotiator if the stakes are high.
  • Crypto basics — Payment is almost always in Bitcoin (BTC) or Monero (XMR). You will need to buy cryptocurrency from an exchange, which requires ID verification. This takes 1-3 days. Ransomware groups often set tight deadlines (48-72 hours) to pressure you.
  • Crypto payment procedure — Buy BTC on an exchange → transfer to a private wallet (e.g., Electrum) → pay ransom from private wallet (not directly from exchange). This provides some privacy. Transaction fees can be $5-50 depending on network congestion.
  • Decryption after payment — The group provides a decryptor tool. Run it on COPIES first. There is a 5-10% chance it corrupts data or does not work fully.

Post-Attack Hardening

After recovering from ransomware, harden your systems to prevent recurrence:

# Immediate hardening steps after ransomware recovery:
# 1. Change ALL passwords (admin, service, user, network)
# 2. Reset API keys, tokens, and secrets
# 3. Enable MFA on all accounts that support it
# 4. Disable RDP if not absolutely needed
# 5. If RDP is needed: use VPN first, change port, enable NLA
# 6. Review and tighten firewall rules
# 7. Enable Windows Attack Surface Reduction rules
# 8. Implement the 3-2-1 backup rule with offline backups
# 9. Deploy EDR (Endpoint Detection and Response) if budget allows
# 10. Create an incident response plan document
# 11. Train users: recognize phishing, dont enable macros
# 12. Apply patches within 48 hours of release
# 13. Segment network: isolate critical systems from user PCs
# 14. Monitor logs for unusual activity

Reporting to Authorities

Reporting ransomware attacks helps law enforcement track and disrupt criminal groups:

  • US: CISA and FBI — Report to CISA (cisa.gov/report) or local FBI field office. The IC3 (ic3.gov) accepts online complaints. Provide ransom note, encrypted files, communication with attackers.
  • UK: National Cyber Security Centre — report@ncsc.gov.uk. Also report to Action Fraud (actionfraud.police.uk).
  • EU: ENISA — Report through your national CSIRT. Each EU country has its own reporting mechanism.
  • Global: No More Ransom — nomoreransom.org accepts reports and samples. This helps them develop decryptors.
  • What to report: Ransom note (exact text), encrypted file sample, ransom amount demanded, wallet address from ransom note, timeline of events, any communication with attackers.

Forensic Preservation of Evidence

If you plan to involve law enforcement or pursue legal action, preserve forensic evidence:

# Forensic evidence preservation:
# 1. Do NOT turn off the infected machine
# 2. Photograph everything: screen, ransom note, file extensions
# 3. If possible, create a forensic RAM dump:
#    - Use FTK Imager or DumpIt on a clean USB
#    - Run: dumpit.exe (creates .raw memory image)
# 4. Create a forensic disk image:
#    - Boot from Linux live USB
#    - Use dd or guymager to create E01/AFF image
#    sudo dd if=/dev/sda of=/mnt/usb/evidence.dd bs=4M
# 5. Document chain of custody:
#    - Who handled the machine, when, what did they do
#    - Store original drive in anti-static bag, sealed
#    - Work only from copies/images
# 6. Save all log files:
#    - Windows Event Logs (Security, System, Application)
#    - Firewall logs
#    - Web proxy logs
#    - RDP connection logs
# 7. Note the ransom wallet address and any Bitcoin
#    transaction IDs if payment was made
