Offline Communication

Communicate without internet or cellular - Briar mesh, Meshtastic devices, LoRa radio, goTenna, and ham radio basics.

Briar (Mesh Messaging)

Briar is an open-source messaging app using Bluetooth, Wi-Fi Direct, and Tor - no internet needed for local mesh.

Use cases: Protests, disaster zones, remote areas, internet shutdowns.

# Install from briarproject.org (F-Droid)
# Create account (no phone/email)
# Add contacts by scanning QR code
# Messages sync automatically when in range

Limitation: Android only.

Meshtastic Devices

Meshtastic is open-source long-range mesh on LoRa radios. Range: 1-3 km urban, 10-30 km line-of-sight.

Features: 868/915 MHz ISM band (license-free), AES-256 encrypted, mesh relay, standalone or via Bluetooth to phone.

Hardware: LilyGO T-Beam, Heltec Wireless Tracker, RAK WisBlock. $15-60 per node.

goTenna

goTenna creates a private mobile mesh network. More plug-and-play than Meshtastic but proprietary.

Models: goTenna Mesh (~$200/pair, 1-6 km), goTenna Pro (ruggedized).

vs Meshtastic: goTenna is easier but closed-source and more expensive. Meshtastic is cheaper and open-source.

LoRa Radio

LoRa (Long Range) is the radio modulation behind Meshtastic. EU: 868 MHz, US: 915 MHz, license-free.

Spreading Factor: SF7 = fastest/shortest (~1 km urban), SF12 = slowest/longest (~10+ km urban).

Antenna is critical. Good antenna doubles range.

Briar Detailed Setup

Briar is the most practical tool for off-grid messaging over short distances. It uses Bluetooth and Wi-Fi Direct to create a mesh network of devices, with Tor for optional internet connectivity.

Installation & Account Creation

# Install Briar (Android only):
# F-Droid: Search "Briar" in F-Droid client
# Google Play: Available but F-Droid is recommended

# Creating an account:
# 1. Open Briar → Create Account
# 2. Choose a display name (can be any nickname)
# 3. Set a password (used to encrypt local data)
# 4. No email or phone number required
# 5. Account created instantly

Adding Contacts via Bluetooth/Wi-Fi Direct

To add contacts in person, both users open Briar and navigate to "Add Contact". One device shows a QR code while the other scans it. The QR code contains the cryptographic key exchange needed for end-to-end encryption. Once scanned, messages sync automatically when devices are in Bluetooth (~10m) or Wi-Fi Direct (~50m) range.

Forums & Blogs

Briar supports forums and blogs for group communication. Forums are like group chats — anyone in a forum sees all messages. Blogs allow users to broadcast messages to all their contacts. These features work entirely offline and sync when devices are in range.

# Using Briar forums:
# 1. Tap the + icon → New Forum
# 2. Name the forum → Create
# 3. Share the forum invite link with contacts (in person)
# 4. All forum posts sync when contacts are nearby

# Using Briar blogs:
# 1. Tap the + icon → New Blog Post
# 2. Write your message → Publish
# 3. The post syncs to all contacts when in range
Tip: Briar is ideal for group coordination during protests, disasters, or camping trips. Create a private forum for your group before you lose internet access, then add contacts in person as you meet.

Meshtastic Radio Configuration

Meshtastic turns inexpensive LoRa radio modules into a mesh network for text communication over kilometers. Here is a comprehensive setup guide.

Hardware Selection

The most popular Meshtastic devices range from $15 to $60 per node. The key components are an ESP32 microcontroller (for Bluetooth and processing) and a LoRa transceiver (for long-range radio).

  • LilyGO T-Beam (~$35) — Built-in GPS, 18650 battery holder, OLED display. Best all-round choice.
  • Heltec Wireless Tracker (~$30) — Built-in GPS, USB-C, small form factor.
  • RAK WisBlock (~$50) — Modular, IP67 rated, great for outdoor deployment.
  • LilyGO T-Echo (~$40) — Built-in GPS, e-ink display (low power), 18650 battery.

Channel Setup & Encryption

Meshtastic uses channels for communication. The primary channel is the default mesh channel (AES-256 encrypted). You can create additional channels for different groups or purposes.

# Configure primary channel:
meshtastic --ch-set name "private" --ch-index 0
meshtastic --ch-set psk "your-secret-passphrase" --ch-index 0

# Add a secondary channel for public announcements:
meshtastic --ch-add "announcements"
meshtastic --ch-set psk "public" --ch-index 1

# Set channel role:
# PRIMARY (index 0): The main encrypted mesh
# SECONDARY (index 1+): Additional channels
# DISABLED: Turn off unused channels

GPS Configuration

Meshtastic can share GPS positions with the mesh, enabling location tracking and mapping. Configure GPS settings carefully to balance accuracy with battery life.

# Enable GPS position broadcasting:
meshtastic --set position.gps_enabled true
meshtastic --set position.fixed_position false

# Set position broadcast interval (seconds):
# Default: 3600 (1 hour) — good for hiking
# Fast: 300 (5 minutes) — for search and rescue
meshtastic --set position.position_broadcast_secs 3600

# Set a fixed position (for stationary nodes):
meshtastic --set-position "lat=40.7128 lon=-74.0060"
meshtastic --set position.fixed_position true

Mesh Networking Fundamentals

Mesh networks are self-healing, decentralized networks where each node relays data for others. Unlike traditional star-topology Wi-Fi (where all devices connect to one router), mesh networks route traffic through neighboring nodes, extending range and resilience.

BATMAN-adv

BATMAN-adv (Better Approach To Mobile Ad-hoc Networking) is a Linux kernel module for mesh networking. It operates at layer 2 (Ethernet level), meaning all devices on the mesh appear to be on the same LAN. This makes it compatible with any application that uses standard networking.

# Install BATMAN-adv on Linux:
sudo apt install batctl

# Enable BATMAN-adv on an interface:
sudo modprobe batman-adv
sudo batctl if add wlan0
sudo ip link set up bat0
sudo ip addr add 192.168.200.1/24 dev bat0

# Check mesh neighbors:
batctl n
# Check mesh originators:
batctl o
# Check mesh gateway:
batctl gwl

OLSR (Optimized Link State Routing)

OLSR is a proactive routing protocol that maintains routing tables to all reachable nodes. It is well-suited for dense mesh networks with many nodes. The OLSRd daemon is widely used in community mesh networks.

# Install OLSRd:
sudo apt install olsrd

# Configure /etc/olsrd/olsrd.conf:
Interface "wlan0" {
  HelloInterval 2.0
  TcInterval 5.0
  LinkQualityMult "default" 1.0
}

# Start OLSRd:
sudo systemctl enable olsrd
sudo systemctl start olsrd

goTenna & BearTooth Devices

goTenna devices create private mobile mesh networks without any infrastructure. They pair with your phone via Bluetooth and use VHF radio waves for communication. BearTooth is a similar device focused on outdoor/survival use.

  • goTenna Mesh (~$200/pair) — 1-4 km urban, 4-10 km line-of-sight. VHF frequency, license-free. Proprietary protocol.
  • goTenna Pro (~$500+) — Ruggedized for professional use. Longer range, encrypted channels, higher power output.
  • BearTooth (~$150) — Similar concept, designed for hikers and emergency prep. Smaller form factor.

vs Meshtastic: goTenna is easier to set up (plug-and-play) but is closed-source and significantly more expensive. Meshtastic is cheaper ($20-50), open-source, and has a larger community. For emergency preparedness on a budget, Meshtastic is the better choice.

Amateur Radio Licensing

Amateur (ham) radio is the most resilient communication technology available to individuals. However, transmitting requires a license in most countries. Here is what you need to know.

License Classes (USA)

  • Technician — Entry level. VHF/UHF privileges (local communication). 35-question exam. Can operate on 146 MHz and 446 MHz simplex/calling frequencies. Study time: 2-4 weeks.
  • General — Mid level. HF privileges (worldwide communication). 35-question exam (adds to Technician). Can operate on 40m, 20m, 15m, 10m bands for long-distance. Study time: 4-8 weeks.
  • Amateur Extra — Full privileges. All bands and modes. 50-question exam. Study time: 3-6 months.

Exam Preparation

The Technician exam is not difficult. Free study resources include: HamStudy.org (flashcards), KB6NU's No-Nonsense Study Guide, ARRL License Manual. Exams are administered by volunteer exam coordinators (VECs) — find a session near you at arrl.org/exam.

Frequencies for Emergency Communication

# Repeater inputs/outputs (varies by region):

Packet Radio Over HAM

Packet radio is a way to send digital data over amateur radio frequencies. It is the original wireless data network — predating Wi-Fi by decades. With packet radio, you can send email, chat, and transfer files over the air.

# Basic packet radio setup:
# 1. Baofeng UV-5R or any 2m/70cm radio
# 2. Signalink USB or simple sound card interface
# 3. Dire Wolf software modem:
sudo apt install direwolf

# Configure Dire Wolf:
# /etc/direwolf.conf:
ADEVRATE 1200
CHANNEL 0
MYCALL YOUR-CALL-1
MODEM 1200
PTT RIG 2m

# Start Dire Wolf:
direwolf -c /etc/direwolf.conf

# Using xastir or similar APRS client

APRS Messaging

APRS (Automatic Packet Reporting System) is a digital communication system used by amateur radio operators for real-time tactical information, position reporting, and messaging. It operates on 144.390 MHz in the US (144.800 MHz in Europe).

APRS can send text messages between stations, display positions on maps, and relay weather station data. During emergencies, APRS is used for health-and-welfare traffic and coordination. Mobile apps like APRSdroid allow you to participate with just a phone and a radio interface.

# APRS messaging with APRSdroid (Android):
# 1. Install APRSdroid from F-Droid or Play Store
# 2. Connect to a TNC (Dire Wolf on PC, or Bluetooth KISS TNC)
# 3. Set your callsign and SSID
# 4. Send messages: tap a station, type message, send

# APRS message format:
# CALLSIGN>APRS:>Message content
# Example: KI7ABC>APRS:>Meet at shelter B, ETA 15 min

# View APRS traffic in real-time:
# https://aprs.fi — but download maps offline first

Winlink Email Over Radio

Winlink is a global radio email system that allows licensed amateur radio operators to send and receive email without internet. It uses a network of RMS (Radio Message Server) stations worldwide. Winlink supports HF (long-range, thousands of km), VHF/UHF (local, tens of km), and even satellite connections.

# Winlink setup:
# 1. Get your ham license and callsign
# 2. Download Winlink Express (Windows, free)
# 3. Connect radio to computer via sound card interface
# 4. Configure RMS expression in Winlink Express
# 5. Connect to nearest RMS station and send email

# Winlink via Telnet (when internet is available):
# Same Winlink account, just use Telnet mode instead of Radio
# Messages are stored on the Winlink server for radio pickup

CB, FRS & GMRS Radio Basics

These are license-free or low-barrier radio services that do not require a ham license. They are useful for short-range communication during emergencies.

  • CB Radio (Citizens Band, 27 MHz) — No license required. Range: 1-5 km (more with skip propagation). 40 channels. Used by truckers and off-roaders. Channel 9 is the emergency channel. Channel 19 is the trucker channel.
  • FRS (Family Radio Service, 462/467 MHz) — No license required. Range: 0.5-2 km. 22 channels. Handheld radios like Motorola Talkabout. Maximum 2W power, non-removable antenna.
  • GMRS (General Mobile Radio Service, 462/467 MHz) — License required ($35, no test). Range: 1-5 km. Up to 50W, repeater-capable, removable antennas. Better range than FRS.

Recommendation for emergency prep: Get a pair of GMRS radios (Midland GXT-series or similar) for family communication during power outages. Keep them charged and stored with your emergency kit. CB is useful for monitoring traffic and local conditions during a disaster.

Community Mesh Network Planning

Setting up a community mesh network requires planning, equipment, and coordination. Here is a step-by-step approach for building a local mesh that serves your neighborhood during internet outages.

  1. Survey your area — Map out locations of participating households. Note line-of-sight between rooftops. Use Google Earth or a physical site survey.
  2. Choose equipment — Ubiquiti LiteBeam or NanoStation for directional point-to-point links. TP-Link Omada or MikroTik for routing hardware. Keep all equipment battery-backed.
  3. Establish backbone nodes — Identify 2-3 high points (rooftops, water towers) for backbone nodes with omnidirectional antennas. These connect the mesh.
  4. Install CPE — Customer premises equipment at each home. A small directional antenna aimed at the nearest backbone node.
  5. Configure routing — Use BATMAN-adv, OLSR, or BGP for dynamic routing. Each node should auto-discover neighbors.
  6. Set up local services — Kiwix, Nextcloud, local DNS, file sharing. Make the mesh useful beyond internet access.
  7. Test regularly — Monthly mesh tests ensure everything works when needed.

Low-Tech Internet Concepts

Low-Tech Magazine's article "How to Make a Low-Tech Internet" explores the concept of a sustainable, low-bandwidth internet that works with minimal energy. Key ideas that apply to emergency scenarios:

  • Sneakernet — Physically transporting data via USB drives or SD cards. The bandwidth of a USB 3.0 drive carried 10 km is ~3 Gbps — faster than most internet connections.
  • Delay-tolerant networking — Store-and-forward messaging where data propagates through the network opportunistically. Briar and Meshtastic use this principle.
  • Low-bandwidth protocols — Use text-only communication (email, SMS over radio) instead of bandwidth-heavy web browsing. Gemini protocol is designed for this.
  • Solar-powered access points — Small Wi-Fi access points powered by solar panels can create local connectivity in areas without grid power.
  • Content distribution networks over radio — Broadcasting data via one-way radio (like RDS, FM subcarriers, or digital TV whitespace) to distribute information to many receivers.

Practical application: Set up a "community drop box" — a Raspberry Pi with Kiwix and a Samba share, accessible via local Wi-Fi. People bring their devices to the location, sync content, and leave. This is a modern version of the sneakernet concept.
