Encrypted Communication Guide

Set up end-to-end encrypted communication - Signal, Session, Keybase, PGP email, Matrix/Element, and WireGuard VPN.

Table of Contents

Signal Setup

Signal is the gold standard for encrypted messaging. E2EE by default, self-destructing messages, encrypted voice/video calls, open source.

# Download from signal.org, register with phone number
# Set PIN, enable Registration Lock
# Compare safety numbers with contacts in person

vs alternatives: Signal is most private. WhatsApp uses same protocol but owned by Meta. Telegram is NOT E2EE by default.

Session Messenger

Session is Signal without a phone number. Anonymous, uses onion routing, E2EE by default, decentralized.

When to choose: You want anonymity, are in a high-risk environment, or need service resilience.

Keybase

Keybase combines encrypted chat, file sharing (250 GB free), encrypted Git, and public key verification linked to social identities.

Note: Acquired by Zoom in 2020. Development slowed. Still works but consider Signal or Matrix for future-proof comms.

PGP Email

# GPG setup:
gpg --full-generate-key  # RSA 4096
# Encrypt: gpg -e -r recipient file
# Decrypt: gpg -d file.gpg

# Email integration: Thunderbird + Enigmail
# Or use ProtonMail (built-in PGP)

Limitations: Does not encrypt metadata. Complex for non-technical users.

Matrix/Element

Matrix is an open standard for decentralized encrypted communication. Element is the flagship client.

Features: E2EE by default, self-hostable (Synapse/Dendrite), bridges to Signal/Telegram/Slack/IRC, open protocol.

WireGuard VPN

WireGuard is a modern, fast VPN protocol built into the Linux kernel.

# Server: wg genkey, configure /etc/wireguard/wg0.conf
# Client: connect with 3 lines of config
# 10x faster than OpenVPN, modern crypto (ChaCha20)

Signal Full Guide

Beyond basic registration, Signal has several important security features that should be configured properly:

Registration & PIN Setup

When you first install Signal, you register with your phone number. You will receive an SMS or voice call to verify. After registration, Signal prompts you to set a PIN — this is not your phone lock screen PIN but a separate registration lock PIN. This PIN prevents anyone else from registering your number on Signal without it.

# Signal registration steps:
# 1. Download Signal from signal.org (Desktop) or app store (Mobile)
# 2. Open app → Accept permissions → Enter phone number
# 3. Verify via SMS code or voice call
# 4. Create a PIN (at least 4 digits, alphanumeric recommended)
# 5. Enable Registration Lock in Settings → Account
# 6. Verify safety numbers with trusted contacts

Disappearing Messages

Signal allows you to set messages to disappear after a timer on both one-on-one chats and group chats. Timer options: 5 seconds, 30 seconds, 1 minute, 5 minutes, 30 minutes, 1 hour, 4 hours, 24 hours, 1 week, 2 weeks, 4 weeks. Once enabled, messages are deleted from both sender and receiver devices after the timer expires.

Warning: Disappearing messages are not a guarantee of deletion — recipients can screenshot or photograph the screen before the timer expires. Treat disappearing messages as a courtesy, not a security feature.

Sealed Sender & Screen Security

Sealed Sender encrypts metadata about who is sending a message — even Signal's servers cannot see it. This is enabled by default for mutual contacts. For non-contacts, the first message may reveal your identity. Screen Security prevents the app content from appearing in the Android recent apps switcher. Enable it in Settings → Privacy → App Security → Screen Security.

Linked Devices

Signal Desktop can be linked to your phone for multi-device messaging. The desktop app does not work independently — it requires the phone to be connected periodically. To link: Signal Desktop → Open app → QR code appears → Phone → Settings → Linked Devices → Link New Device → Scan QR code.

Session Anonymous Setup

Session is a fork of Signal that removes the phone number requirement. It uses an onion routing network (Oxen Service Node network) for anonymity and decentralized message storage.

# Session setup:
# 1. Download Session from getsession.org (Desktop/Mobile)
# 2. Create account — no email or phone required
# 3. Session ID is generated automatically (long random string)
# 4. Share your Session ID with contacts via any channel
# 5. Session IDs are reusable — keep backups of your recovery phrase

Recovery phrase: Session generates a mnemonic seed phrase (similar to cryptocurrency wallets). Write this down and store it securely. Without it, your Session ID and all associated messages are unrecoverable if you lose your device.

Session vs Signal: Session offers better anonymity (no phone number), but the Oxen network introduces latency. Signal is faster and more mature. Use Session when anonymity is critical, Signal for everyday secure communication.

Matrix/Element Self-Hosting Guide

Self-hosting a Matrix server gives you complete control over your communication infrastructure. You can run your own Synapse or Dendrite server, federate with other Matrix servers, and bridge to other platforms.

Choosing a Server

Synapse is the reference Matrix server implementation (Python, most features, higher resource usage). Dendrite is a next-generation implementation (Go, lower resource usage, still maturing). For most self-hosters, Synapse is the recommended starting point.

# Deploy Synapse with Docker (recommended):
mkdir -p /srv/matrix/synapse
docker run -it --rm \
  -v /srv/matrix/synapse:/data \
  -e SYNAPSE_SERVER_NAME=matrix.yourdomain.com \
  -e SYNAPSE_REPORT_STATS=no \
  matrixdotorg/synapse:latest generate

# Edit homeserver.yaml:
# /srv/matrix/synapse/homeserver.yaml
# Set: enable_registration: false (disable open registration)
# Set: registration_shared_secret: "your-secret-here"

# Run Synapse:
docker run -d --name synapse \
  -v /srv/matrix/synapse:/data \
  -p 8448:8448 \
  --restart unless-stopped \
  matrixdotorg/synapse:latest

Federation Configuration

For Matrix federation to work, your server must be accessible on port 8448 (or port 443 with a reverse proxy). You need a valid TLS certificate (Let's Encrypt works). Configure your DNS with a SRV record or a delegation record.

# DNS SRV record for federation:
# _matrix._tcp.yourdomain.com. 3600 IN SRV 10 0 8448 matrix.yourdomain.com.

# Or use a .well-known delegation:
# Serve at https://yourdomain.com/.well-known/matrix/server:
# {
#   "m.server": "matrix.yourdomain.com:8448"
# }

Bridges

Matrix bridges allow you to communicate with users on other platforms directly from your Matrix client. Popular bridges: Signal (mautrix-signal), Telegram (mautrix-telegram), WhatsApp (mautrix-whatsapp), IRC (matrix-appservice-irc), Slack (matrix-appservice-slack). Bridges require separate setup but dramatically expand your reachable contacts.

# Deploy mautrix-signal bridge via Docker:
docker run -d --name mautrix-signal \
  -v /srv/matrix/bridges/signal:/data \
  --restart unless-stopped \
  dock.mau.dev/mautrix/signal:latest

# Configure the bridge in the config file,
# then start a chat with @signalbot:yourdomain.com

Tor Browser Configuration

Tor Browser is your gateway to the dark web and a critical tool for anonymous communication. Proper configuration maximizes both security and usability.

Security Levels

Tor Browser has three security levels accessible via the shield icon in the address bar:

  • Standard — All features enabled. Best for general browsing where JavaScript is needed.
  • Safer — Disables JavaScript on non-HTTPS sites, disables some fonts and math symbols. Good balance of security and usability.
  • Safest — Disables JavaScript entirely, disables all fonts, disables SVG, WebGL, and audio/video autoplay. Recommended for high-risk environments.

Bridges & Pluggable Transports

Bridges are obfuscated Tor relays that help bypass censorship. If Tor is blocked in your country, bridges provide a way to connect. Pluggable transports disguise Tor traffic as other protocols.

# Getting bridges:
# 1. Tor Browser → Settings → Tor → Request a bridge
# 2. Visit https://bridges.torproject.org/ (may be blocked)
# 3. Email bridges@torproject.org from Gmail or Riseup
# 4. Use the built-in "Request a bridge" in Tor Browser Settings

# Pluggable transport options:
# - obfs4: Most popular, mimics random traffic
# - snowflake: Decentralized, uses browser-based proxies
# - webtunnel: Disguises as HTTPS traffic
# - meek: Uses domain fronting (Azure, AWS)

.onion Services

Onion services (formerly "hidden services") are websites accessible only through Tor. They provide end-to-end encryption and anonymity for both the visitor and the site operator.

# Accessing .onion sites:
# Simply paste the .onion address into Tor Browser
# Popular .onion directories:
# - http://duckduckgogg42xjoc72x3sjasowoarfbgcmvfimaftt6twagswzczad.onion
# - http://darkfailenbsdla5mal2mxn2uz66od5vz554ajby5oq6mbeniqbqy6yd.onion

# Hosting your own .onion service:
# In /etc/tor/torrc:
# HiddenServiceDir /var/lib/tor/my_service/
# HiddenServicePort 80 127.0.0.1:80
# Then restart Tor: sudo systemctl restart tor
# Your .onion address is in /var/lib/tor/my_service/hostname
Security note: Do not resize the Tor Browser window — websites can use screen dimensions for fingerprinting. Keep the default window size. Also, avoid installing extensions in Tor Browser as they can compromise anonymity.

WireGuard VPN Full Setup

Setting up WireGuard on a VPS gives you a secure VPN tunnel for all your traffic. WireGuard is significantly faster than OpenVPN and is built into the Linux kernel since version 5.6.

Server Configuration

# Install WireGuard on server (Ubuntu/Debian):
sudo apt update && sudo apt install wireguard

# Generate server key pair:
cd /etc/wireguard/
wg genkey | tee server.key | wg pubkey > server.key.pub
chmod 600 server.key

# Create /etc/wireguard/wg0.conf:
[Interface]
Address = 10.0.0.1/24
ListenPort = 51820
PrivateKey = <server-private-key>
PostUp = iptables -A FORWARD -i wg0 -j ACCEPT
PostUp = iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
PostDown = iptables -D FORWARD -i wg0 -j ACCEPT
PostDown = iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE

# Enable IP forwarding:
echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf
sysctl -p

# Start WireGuard:
systemctl enable wg-quick@wg0
systemctl start wg-quick@wg0

Client Configuration

# Generate client key pair:
wg genkey | tee client.key | wg pubkey > client.key.pub

# Add to server's wg0.conf:
[Peer]
PublicKey = <client-public-key>
AllowedIPs = 10.0.0.2/32

# Restart WireGuard on server:
systemctl restart wg-quick@wg0

# Client config (save as client.conf):
[Interface]
PrivateKey = <client-private-key>
Address = 10.0.0.2/24
DNS = 1.1.1.1

[Peer]
PublicKey = <server-public-key>
Endpoint = your-server-ip:51820
AllowedIPs = 0.0.0.0/0
PersistentKeepalive = 25

QR Code for Mobile

WireGuard mobile apps can import configuration via QR code. Generate the QR code from the client config file:

# Install qrencode on Linux:
sudo apt install qrencode

# Generate QR code from client config:
qrencode -t ansiutf8 < client.conf

# Scan the QR code with the WireGuard mobile app
# The tunnel will be configured automatically

PGP Email Encryption Deep Dive

PGP (Pretty Good Privacy) remains the standard for end-to-end encrypted email. While complex for everyday use, it is essential for high-risk communications where metadata privacy is not a concern.

GPG Tools & Key Generation

# Install GPG:
sudo apt install gnupg  # Linux
brew install gnupg     # macOS

# Generate a key pair (RSA 4096 recommended):
gpg --full-generate-key
# Choose: RSA and RSA
# Keysize: 4096
# Expiration: 2y (then renew)
# Real name: Your Name
# Email: your@email.com

# List keys:
gpg --list-keys
gpg --list-secret-keys

# Export public key:
gpg --armor --export your@email.com > public-key.asc

# Export private key (backup securely!):
gpg --armor --export-secret-keys your@email.com > private-key.asc

Key Servers

Key servers allow others to find and download your public key. Upload your key so contacts can encrypt messages to you. Popular key servers: keys.openpgp.org, keyserver.ubuntu.com, pgp.mit.edu.

# Upload your public key to a keyserver:
gpg --keyserver keys.openpgp.org --send-key YOUR_KEY_ID

# Find someone else's key:
gpg --keyserver keys.openpgp.org --search-keys person@example.com

# Import someone's key:
gpg --keyserver keys.openpgp.org --recv-key THEIR_KEY_ID

# Encrypt a file for someone:
gpg --encrypt --recipient person@example.com secret.txt

# Decrypt a file:
gpg --decrypt secret.txt.gpg

Email Integration

For email integration, Thunderbird with the Enigmail plugin (or the built-in OpenPGP support in Thunderbird 78+) provides a clean interface. ProtonMail has built-in PGP support and can exchange encrypted emails with external PGP users. For a command-line email setup, use `mutt` with GPG integration.

Steganography Basics

Steganography is the practice of hiding the existence of a message within another medium. Unlike encryption (which hides the content but not the communication), steganography hides the communication itself.

  • Image steganography: Embedding messages in the least significant bits of pixel data. Tools: steghide, OpenStego, zsteg.
  • Audio steganography: Hiding data in audio files using low-bit encoding or spread spectrum. Tools: DeepSound, SilentEye.
  • Text steganography: Using whitespace, font variations, or word patterns to encode data.
# Steganography with steghide (images):
sudo apt install steghide

# Hide a message in an image:
steghide embed -cf image.jpg -ef secret.txt -p "passphrase"

# Extract a hidden message:
steghide extract -cf image.jpg -p "passphrase"

# Check if an image has hidden data:
steghide info image.jpg
Important: Steganography is not a replacement for encryption. Always encrypt messages before hiding them. A hidden encrypted message is far more secure than either technique alone.

Metadata Minimization

Metadata is data about your data — who you communicate with, when, how often, from where, and using what device. Even if message content is encrypted, metadata can reveal your relationships, habits, and location. Minimizing metadata is a critical part of operational security.

  • Remove EXIF data from photos before sharing. Tools: ExifTool, MAT2, or use Signal which strips EXIF automatically.
  • Use Tor to hide your IP address, which reveals your approximate location and ISP.
  • Avoid using the same username across platforms — this links your identities.
  • Disable read receipts — they reveal when you are active.
  • Use Signal's Sealed Sender to hide message metadata from Signal's servers.
# Strip EXIF data from images with ExifTool:
exiftool -all= photo.jpg

# Or use MAT2 (Metadata Anonymisation Toolkit):
pip install mat2
mat2 photo.jpg   # Removes metadata from images, PDFs, etc.
