Encrypted Communication Guide

Set up end-to-end encrypted communication - Signal, Session, Keybase, PGP email, Matrix/Element, and WireGuard VPN.

Table of Contents

  • Signal Setup
  • Session Messenger
  • Keybase
  • PGP Email

Signal Setup

Signal is the gold standard for encrypted messaging. E2EE by default, self-destructing messages, encrypted voice/video calls, open source.

# Download from signal.org, register with phone number
# Set PIN, enable Registration Lock
# Compare safety numbers with contacts in person

vs alternatives: Signal is most private. WhatsApp uses same protocol but owned by Meta. Telegram is NOT E2EE by default.

Session Messenger

Session is Signal without a phone number. Anonymous, uses onion routing, E2EE by default, decentralized.

When to choose: You want anonymity, are in a high-risk environment, or need service resilience.

Keybase

Keybase combines encrypted chat, file sharing (250 GB free), encrypted Git, and public key verification linked to social identities.

Note: Acquired by Zoom in 2020. Development slowed. Still works but consider Signal or Matrix for future-proof comms.

PGP Email

# GPG setup:
gpg --full-generate-key  # RSA 4096
# Encrypt: gpg -e -r recipient file
# Decrypt: gpg -d file.gpg

# Email integration: Thunderbird + Enigmail
# Or use ProtonMail (built-in PGP)

Limitations: Does not encrypt metadata. Complex for non-technical users.

Matrix/Element

Matrix is an open standard for decentralized encrypted communication. Element is the flagship client.

Features: E2EE by default, self-hostable (Synapse/Dendrite), bridges to Signal/Telegram/Slack/IRC, open protocol.

WireGuard VPN

WireGuard is a modern, fast VPN protocol built into the Linux kernel.

# Server: wg genkey, configure /etc/wireguard/wg0.conf
# Client: connect with 3 lines of config
# 10x faster than OpenVPN, modern crypto (ChaCha20)
