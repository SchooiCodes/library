BIOS/UEFI Recovery

Recovering a system with corrupted BIOS/UEFI firmware - CMOS reset, Flashback, CH341A programmers, and debricking motherboards.

When BIOS Fails

A corrupted BIOS can brick your motherboard - no display, no POST, no response.

Causes: Failed update, wrong BIOS file, overclocking failure, physical damage, UEFI rootkit.

Symptoms: No display, no POST beeps, fans spin but nothing happens, boot loop.

BIOS Flashback

BIOS Flashback allows updating BIOS without CPU/RAM/GPU.

# ASUS: rename BIOS file to CB****.CAP
# MSI: rename to MSI.ROM
# Gigabyte: rename to GIGABYTE.bin
# Format USB FAT32, insert in labeled port
# Press Flashback button, wait for LED to stop blinking

CH341A Programmer

The CH341A is a cheap (~$10) SPI flash programmer for direct BIOS chip read/write.

# Connect SOIC8 clip (pin 1 = red wire)
sudo apt install flashrom
sudo flashrom -p ch341a_spi -r backup.bin
sudo flashrom -p ch341a_spi -w new-bios.bin
sudo flashrom -p ch341a_spi -v new-bios.bin
Warning: Pin 1 mismatch can damage the chip. Double-check before powering on.

Recovering a Bricked Motherboard

Levels of debricking:

  1. Clear CMOS, minimal hardware, known-good PSU/RAM
  2. BIOS Flashback with correct file
  3. CH341A programmer - read, write, verify
  4. Hot flash (RISKY - identical donor board required)
  5. Buy pre-flashed chip from eBay (~$10-20)
