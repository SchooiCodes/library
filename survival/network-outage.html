Surviving Network Outages

How to survive when the internet goes down - diagnosing outages, local network access, cellular failover, offline entertainment, mesh networks.

Diagnosing the Outage

# Step 1: Try another device
# Step 2: Check modem/router lights
# Step 3: Ping router (192.168.1.1)
# Step 4: Ping 8.8.8.8 (tests ISP)
# Step 5: Check downdetector.com, call ISP

Common causes: ISP outage, router crash, loose cable, DNS failure, modem overheating.

Local Network Access

Even without internet, your LAN works. Access NAS shares, stream local media (Plex/Jellyfin), access local web apps.

Pro tip: Set up Pi-hole for local DNS resolution.

Cellular Failover

Options: Smartphone hotspot (instant but data caps), dedicated 4G/5G router (Netgear Nighthawk, GL.iNet), dual-WAN router with auto-failover, signal booster.

# Reduce data usage:
# - Disable auto-updates
# - Use text-only browsing
# - Block video autoplay
# - Set email to manual fetch

Offline Entertainment

Prepare ahead: Local media files (VLC), ebooks/audiobooks (Calibre), single-player games, ZIM archives (Kiwix), downloaded podcasts.

Emergency Communication

When cell and internet are down: Briar (mesh, 100m), Meshtastic (LoRa, 1-30 km), ham radio, CB/FRS/GMRS (1-5 km), satellite messengers (Garmin inReach, subscription).

Mesh Networks

Community meshes: NYC Mesh, Freifunk (Germany), Guifi.net (Spain), Althea. Set up a home mesh node with Ubiquiti equipment for decentralized internet.

Types of Network Outages

Understanding the type of outage you are experiencing helps determine the appropriate response. Here are the most common types and their characteristics:

  • ISP failure — Your internet service provider has a regional outage. Your modem shows no sync or intermittent connectivity. Often affects a neighborhood or city. Duration: hours to days. Response: Call ISP, use cellular failover, wait.
  • DNS attack — Your browser cannot resolve domain names (cannot load websites), but other internet services may work (ping 8.8.8.8 works). Response: Change DNS to 1.1.1.1 or 8.8.8.8. Use Pi-hole for local DNS caching.
  • Government shutdown — Intentional internet blackout imposed by authorities. Your ISP is forced to block all traffic. Response: VPN, Tor bridges, satellite internet, mesh networking.
  • Natural disaster — Physical damage to infrastructure (earthquake, flood, hurricane). Cellular towers and fiber lines may be destroyed. Response: Radio communication (ham, Meshtastic), satellite messengers, pre-staged offline resources.
  • Hardware failure — Your modem, router, or ONT (fiber terminal) has failed. Only your home is affected. Response: Power cycle, check cables, replace hardware, use backup router.

ISP Outage Troubleshooting

Before concluding your ISP is down, systematically rule out local issues. Here is a step-by-step troubleshooting procedure:

# Step-by-step ISP outage diagnosis:
# 1. Check other devices — is it just one device or the whole network?
# 2. Look at modem status lights:
#    - Power: Should be solid. If off, check power outlet.
#    - Downstream/Upstream: Flashing or off = no signal
#    - Online: Solid = connected to ISP
# 3. Power cycle: Unplug modem and router for 30 seconds,
#    plug modem in first, wait 2 minutes, then router.
# 4. Cable check: Ensure coax or fiber cable is tight.
#    Check for physical damage (pets, furniture, weather).
# 5. MAC address registration: If you added new hardware,
#    your ISP may need to register the MAC address.
#    Call them and provide the MAC on the modem label.

ISP status: Check downdetector.com (if you have cellular data) or call your ISP's automated status line. Many ISPs have an SMS-based status check that works when internet is down.

DNS Failure Mitigation

DNS failures are among the most common "outages" — your internet connection works, but you cannot browse websites because domain names cannot be resolved. Here is how to mitigate:

# Test if DNS is the problem:
ping 8.8.8.8        # Should work if internet is up
ping google.com     # Fails if DNS is down
nslookup google.com # Shows which DNS server fails

# Fix DNS on Windows:
# Settings → Network & Internet → Change adapter options
# Right-click your connection → Properties
# Select "Internet Protocol Version 4 (TCP/IPv4)" → Properties
# Use: 1.1.1.1 and 1.0.0.1 (Cloudflare)
# Or:  8.8.8.8 and 8.8.4.4 (Google)

# Fix DNS on Linux:
sudo nano /etc/resolv.conf
nameserver 1.1.1.1
nameserver 8.8.8.8

# Edit /etc/hosts for critical websites (manual DNS):
# Example: Add entries for sites you need offline
# Or use a local DNS cache like Pi-hole

Local DNS Cache

A local DNS cache stores previously resolved domain names. When DNS servers are unreachable, the cache can still resolve frequently visited sites. Pi-hole and dnsmasq both provide caching DNS services.

# Set up dnsmasq as a local DNS cache:
sudo apt install dnsmasq
# /etc/dnsmasq.conf:
cache-size=10000
server=1.1.1.1
server=8.8.8.8

# Point your devices to the machine running dnsmasq
# This caches all DNS queries for faster repeat lookups

Solar-Powered Networking Equipment

During extended outages, keeping networking equipment running on solar power maintains your ability to communicate and access local resources.

  • Router power consumption: Typical home router ~5-15W. A 100W solar panel can run a router for ~6-8 hours of direct sunlight + charge a battery for nighttime use.
  • Low-power routers: GL.iNet routers run on USB power (5V, ~2-5W). Perfect for solar + power bank setups. OpenWrt-compatible for advanced networking.
  • Solar + battery setup: 50W solar panel → charge controller → 12V 20Ah battery → 12V-to-USB converter → router. Total: ~$150. Provides 24/7 router operation.
  • PoE (Power over Ethernet): For long-distance outdoor connections, use PoE to power access points or mesh nodes. A solar-powered PoE switch can power multiple outdoor nodes.

Starlink as Backup Internet

Starlink satellite internet provides connectivity anywhere with a clear view of the sky. It is an excellent backup when terrestrial internet fails:

  • Setup: Starlink dish + router ~$599. Monthly service ~$120 (Residential) or $140 (Priority). Portable plan available for travel.
  • Power: Starlink dish consumes ~50-75W average. Requires a generator or large solar/battery system for extended use during grid outages.
  • Obstructions: Requires clear view of the northern sky (in the US). Trees, buildings, and mountains can cause intermittent drops.
  • Limitations: Subject to weather (heavy rain/snow can degrade signal). Congestion in populated areas during peak hours. Not available worldwide.

Satellite Phone Basics

Satellite phones provide voice and text communication when all terrestrial networks are down. They operate by connecting directly to satellites in orbit.

  • Providers: Iridium (global coverage, reliable), Inmarsat (voice + data, not global), Thuraya (Middle East, Africa, Asia), Globalstar (USA, limited).
  • Cost: Handset $500-1500. Service $30-100/month + $0.50-1.50/minute. Prepaid plans available.
  • Rental: For short-term emergencies, rent a satellite phone from companies like Roadpost or Outfitter Satellite. ~$50-100/week + airtime.
  • Alternative: Satellite messengers (Garmin inReach, ZOLEO). Two-way text messaging via satellite. Lower cost ($200-400), lower monthly fee ($15-30), but text-only.

Internet Via Radio (HamWAN, AREDN)

Amateur radio operators have built data networks that provide internet-like services over radio frequencies. These networks operate independently of commercial internet infrastructure.

  • HamWAN — High-speed data network on amateur radio frequencies (5.8 GHz). Provides IP connectivity between nodes at speeds up to 100 Mbps. Line-of-sight required. Equipment cost: ~$200-500 per node.
  • AREDN (Amateur Radio Emergency Data Network) — Mesh network using off-the-shelf Wi-Fi equipment (Ubiquiti, MikroTik) flashed with AREDN firmware. Provides IP connectivity, VoIP, and messaging. 2.4 GHz and 5.8 GHz bands.
  • Winlink — Radio email system over HF, VHF, or UHF. Send and receive email without any internet. Requires amateur radio license. See the Offline Communication guide for details.

# 1. Download AREDN firmware from aredn.org
# 2. Flash via the Ubiquiti web interface
# 3. Configure node name, GPS location, channel
# 4. Mount for line-of-sight to other nodes
# 5. Node automatically discovers and meshes with neighbors
# 6. Access network services via any browser on the mesh

Cellular Data Failover

When your primary internet connection fails, cellular data is the most accessible backup. Here is how to set up reliable cellular failover:

  • Smartphone hotspot — Instant, always available. Turn on Personal Hotspot in settings. Connect laptop/tablet via Wi-Fi or USB tethering. Limitations: battery drain, data caps, signal dependent.
  • Dedicated 4G/5G router — Netgear Nighthawk M6, GL.iNet GL-X300B, or TP-Link MR600. Better antennas, supports external antenna, can be always-on. Connect to your home router as a WAN backup.
  • Dual-WAN router — Router with two WAN ports (primary: cable/fiber, secondary: cellular). Auto-failover when primary goes down. Products: TP-Link ER605, Ubiquiti UDM Pro, pfSense (DIY).
  • Signal booster — WeBoost or SureCall booster improves marginal cellular signals. Adds 10-20 dB gain. Essential if you are in a fringe coverage area.
# Reduce cellular data usage during outage:
# - Use text-only browsing mode in browsers
# - Disable automatic app updates
# - Set email to manual fetch (not push)
# - Use Opera Mini or browser with data compression
# - Block video autoplay in browser settings
# - Use messaging apps (Signal, WhatsApp) instead of voice calls
# - Disable cloud backups and sync

Wi-Fi Mesh for Local Connectivity

A Wi-Fi mesh network maintains local connectivity between devices even when the internet is down. This allows file sharing, messaging, media streaming, and multiplayer gaming within your home or neighborhood.

  • Consumer mesh systems (Eero, Google Nest, TP-Link Deco) — Continue working locally when internet is out. Local traffic between nodes (file sharing, printer access) still works.
  • Ubiquiti UniFi — Professional-grade mesh. Can be configured with local DNS, DHCP, and routing that functions independently of the WAN connection. Best for advanced users.
  • OpenWrt mesh — Flash OpenWrt on compatible routers for full control. Set up 802.11s mesh or BATMAN-adv for dynamic routing.
  • Outdoor mesh nodes — Ubiquiti LiteBeam or NanoStation for point-to-point links between buildings. Create a neighborhood mesh that shares a single internet connection (if available) or provides local connectivity.

Pre-Outage Preparation

The best time to prepare for a network outage is before it happens. Here is a checklist of things to download and configure while you have internet:

  • ZIM archives — Download Wikipedia, WikiHow, Stack Overflow, and other ZIM files via Kiwix.
  • Offline maps — Download maps for your entire region plus surrounding areas in Organic Maps or OsmAnd.
  • Software repositories — Archive critical software installers, drivers, and tools on a USB drive.
  • Media content — Movies, TV shows, music, podcasts, audiobooks for entertainment.
  • Documentation — PDF manuals for your devices, medical references, survival guides.
  • Configure Pi-hole — Set up Pi-hole or AdGuard Home for local DNS resolution.
  • Back up router config — Export and save your router's configuration file. If the router needs to be reset, you can restore settings quickly.
  • Test cellular failover — Practice switching to cellular hotspot. Verify all family devices can connect and access essential services.
  • Establish meeting points — If communication is impossible, agree on physical meeting points and times with family members.
  • Print critical information — Emergency contacts, ISP phone numbers, network settings, passwords, and maps should exist on paper.

Post-Outage Communication Coordination

After an outage ends, or during an extended outage, coordinate with your community using these strategies:

  • Designated meetup times — Agree on specific times to check for messages (e.g., every hour at :00). This ensures everyone is listening at the same time.
  • Radio check-in schedule — For ham radio users: schedule regular check-in times on a specific frequency. 146.520 MHz at 9 AM and 9 PM local time.
  • Message relay — Designate relay points where messages can be passed between neighborhoods that cannot directly communicate. A person at a high point with a radio relay can bridge gaps.
  • Sneakernet — Physically transport messages via bicycle, car, or on foot between areas with no connectivity. This is slow but reliable.
