Hardware Diagnostics

Diagnose failing hardware - RAM testing with MemTest86, storage diagnostics with CrystalDiskInfo, CPU/GPU stress testing, PSU testing, and thermal diagnostics.

Table of Contents

  • Thermal Diagnostics

RAM Testing (MemTest86)

MemTest86 is the gold standard for RAM testing. Download, create bootable USB, boot, let it run for 1+ full pass (2-4 hours for 16 GB). ANY error = faulty RAM.

Symptoms: Random BSODs, program crashes, file corruption, boot failures, graphical artifacts.

Storage Testing (CrystalDiskInfo, SMART)

CrystalDiskInfo: Shows health %, temperature, power-on hours. Reallocated sectors > 0 = failing drive. Pending sectors > 0 = imminent failure.

SMART (Linux): sudo smartctl -a /dev/sda

Thresholds: Reallocated > 10: replace now. Power-on > 50,000: aging. Temp > 55C consistently: check cooling.

CPU Stress Testing

Prime95: Small FFTs for CPU heat, Blend for CPU+RAM. Run 1+ hour. Watch for worker errors, freezes, WHEA errors.

Monitor: HWiNFO - CPU temps should not exceed 95C. Throttling = check paste, cooler, fans.

GPU Testing

FurMark: Classic GPU stress test. Run 15-30 min. Watch for artifacts, driver crashes. Normal temp: 65-85C.

Unigine Heaven/Superposition: Better real 3D workload, ~20 min loop.

Signs of failure: Artifacts, driver crashes, black screens, fans not spinning.

PSU Testing

Methods: Visual inspection (bulging caps, burnt smell), paperclip test (basic on/off), multimeter (12V rail: 11.4-12.6V), PSU tester ($10-20).

Quality brands: Seasonic, Corsair (RMx/AX), EVGA (G6/P6), Be Quiet.

Thermal Diagnostics

Safe ranges: CPU idle 30-45C, load 60-85C (max 95C). GPU idle 30-45C, load 65-85C. SSD 30-50C. HDD 25-45C.

Stress test: HWiNFO + Prime95 + FurMark simultaneously, monitor for 15-30 min. Sustained 95C+ = fix urgently.

Bootable Diagnostic Tools

When a computer will not boot into Windows or Linux, bootable diagnostic environments allow you to test hardware independently of the installed OS. These are essential tools for any hardware diagnostics toolkit.

Hiren's Boot CD PE

Hiren's Boot CD PE is a comprehensive Windows PE environment with built-in diagnostic tools. It boots from USB and provides a complete Windows desktop with tools for disk testing, memory testing, file recovery, and system repair.

# Create Hiren's Boot CD PE USB:
# 1. Download ISO from hirensbootcd.org
# 2. Use Rufus (Windows) or dd (Linux):
sudo dd if=hirens.iso of=/dev/sdX bs=4M status=progress

# Tools included:
# - CrystalDiskInfo (S.M.A.R.T. monitoring)
# - MemTest86 (RAM testing)
# - HDD Regenerator (bad sector repair)
# - TestDisk (partition recovery)
# - CPU-Z (system information)
# - AIDA64 (system benchmark and sensor monitoring)

SystemRescue

SystemRescue (formerly SystemRescueCd) is a Linux-based rescue environment. It includes a comprehensive set of diagnostic and recovery tools for Linux and Windows systems.

# Create SystemRescue USB:
# Download ISO from system-rescue.org
# Use Rufus or:
dd if=systemrescue.iso of=/dev/sdX bs=4M status=progress

# Key diagnostic tools included:
# - smartctl (S.M.A.R.T. tests)
# - badblocks (sector checking)
# - memtest86+ (RAM testing)
# - ddrescue (data recovery)
# - fsck (filesystem check)
# - lshw (hardware listing)
# - htop (system monitoring)

Ultimate Boot CD (UBCD)

UBCD is a collection of bootable diagnostic tools for PC hardware. It includes dozens of tools for testing every component of a computer. While older, it remains useful for legacy hardware (IDE drives, DDR2/DDR3 RAM, older CPUs).

MemTest86+ Memory Testing Deep Dive

MemTest86+ is the industry standard for RAM testing. Running it correctly requires understanding test patterns, timing expectations, and result interpretation.

Test Patterns

MemTest86+ runs multiple test patterns to detect different types of memory errors:

  • Test 0 [Address test] — Tests address line connectivity. Detects stuck address lines.
  • Test 1 [Moving inversions, ones & zeros] — Writes and reads patterns to detect stuck bits.
  • Test 2 [Moving inversions, 8-bit pattern] — Similar but with 8-bit patterns. Detects data line shorts.
  • Test 3 [Moving inversions, random pattern] — Random patterns to detect intermittent errors.
  • Test 4 [Block move] — Stress tests the memory controller and cache.
  • Test 5 [Modulo-20, ones & zeros] — Detects errors sensitive to specific address patterns.
  • Test 6 [Bit fade test] — Writes pattern, waits, reads back. Detects leaky DRAM cells.
  • Test 7 [Random number sequence] — Stress test with random access patterns.

ECC vs Non-ECC

ECC (Error-Correcting Code) RAM can detect and correct single-bit memory errors. It is used in servers and workstations. ECC RAM will still produce errors in MemTest86+ if the errors are multi-bit or beyond the correction capability. Non-ECC RAM (consumer desktops) will show all errors — any error is a failure.

Expected Timings

A full pass of MemTest86+ on 16GB of DDR4 takes approximately 2-4 hours. Running multiple passes (overnight, 8+ passes) is recommended for comprehensive testing. Any single error — even one bit in one pass — indicates faulty RAM. Replace it immediately.

# MemTest86+ usage:
# 1. Download MemTest86+ (memtest.org)
# 2. Create bootable USB using included installer
# 3. Boot from USB
# 4. MemTest86+ starts automatically
# 5. Let it run for at least 1 full pass (overnight recommended)
# 6. Press Esc to stop early (only if testing is urgent)

# Interpreting results:
# "Pass complete, no errors" — RAM is good
# ANY error count > 0 — RAM is faulty
# Common error patterns:
#   - Single bit errors: Bad RAM chip
#   - Address line errors: Bad motherboard slot or RAM
Important: A single MemTest86+ error means the RAM is faulty, even if the computer seems stable. Memory errors cause file corruption, data loss, and mysterious crashes. Replace the faulty DIMM immediately.

CPU Stress Testing

CPU stress testing pushes the processor to its thermal and power limits to verify stability under maximum load. It detects overheating, inadequate cooling, and silicon defects.

Prime95

Prime95 is the gold standard for CPU stress testing. It runs complex mathematical calculations (Mersenne prime searches) that push the CPU to 100% utilization and maximum heat output.

# Prime95 options:
# Small FFTs: Maximum heat, minimum RAM testing
# Large FFTs: Balanced CPU + RAM testing
# Blend: Maximum RAM and CPU testing (most comprehensive)

# Run Prime95 for at least 1 hour
# Monitor temperatures with HWiNFO or Core Temp
# CPU should stay below 95C (desktop) or 100C (laptop)
# Watch for:
# - Worker thread errors in Prime95
# - System freezes or reboots
# - WHEA (Windows Hardware Error Architecture) errors
# - Thermal throttling (CPU clock speed dropping)

AIDA64 System Stability Test

AIDA64 includes a comprehensive system stability test that stresses CPU, FPU, cache, memory, and disk simultaneously. It provides real-time temperature, voltage, and clock speed graphs.

IntelBurnTest

IntelBurnTest (based on Linpack) creates the highest thermal load possible. It is more aggressive than Prime95. Run at "High" or "Very High" stress level. If a system passes IntelBurnTest, it is extremely stable.

# CPU stress testing protocol:
# 1. Run HWiNFO in sensors-only mode (log temperatures)
# 2. Run Prime95 Small FFTs for 30 minutes
# 3. Check maximum temperatures recorded
# 4. Run IntelBurnTest at High stress (10 passes)
# 5. Check for WHEA errors in Windows Event Viewer
# 6. If any errors or temperatures exceed safe limits,
#    check CPU cooler mounting, thermal paste, fan speed

GPU Testing

Graphics card testing requires dedicated tools that stress the GPU core, memory, and VRM (voltage regulator modules). Different tools test different aspects of GPU stability.

  • FurMark — The classic GPU stress test. Creates extreme heat (fur rendering). Run for 15-30 minutes. Watch for artifacts, driver crashes, or temperatures exceeding 85C (some high-end cards can reach 90-95C safely).
  • Unigine Heaven/Superposition — Realistic 3D workloads. Better for testing real-world stability. Run in loop mode for 20+ minutes. Also serves as a benchmark for performance comparison.
  • 3DMark — Industry-standard gaming benchmark. The Time Spy and Fire Strike tests stress both GPU and CPU. Compare scores with similar systems to identify underperformance.
  • OCCT — All-in-one stability tester. GPU stress test with error detection (ARTIFACT detection). Also has power supply stress test (CPU + GPU simultaneously).
# GPU diagnostic checklist:
# [ ] Check GPU driver version (clean install with DDU if issues)
# [ ] Run FurMark for 15 min — check max temperature
# [ ] Run Unigine Superposition loop mode × 3 passes
# [ ] Check for artifacts (colored dots, lines, flickering)
# [ ] Monitor fan speed — fans should ramp up under load
# [ ] Check GPU clock speed — should maintain rated boost
# [ ] If crashing: underclock GPU core by 50 MHz, retest
# [ ] If artifacts: reduce VRAM clock by 100 MHz, retest
Tips: GPU artifacts (visual glitches) indicate VRAM issues or core instability. Driver crashes (black screen, recoverable) indicate driver issues or mild instability. Full system crashes (reboot) indicate power supply or serious GPU failure. If the GPU passes at stock settings but fails when overclocked, the overclock is unstable — reduce clock speed.

Storage Testing

Storage devices fail in predictable ways. Proper testing can identify failing drives before data loss occurs.

CrystalDiskMark

CrystalDiskMark benchmarks sequential and random read/write speeds. Compare results with expected speeds for your drive model. Significantly lower speeds may indicate a failing drive, incorrect driver, or SATA mode issue (IDE vs AHCI).

HDTune

HDTune performs read/write speed tests across the entire drive surface. A healthy drive shows a smooth, gradually declining speed graph (slower at the end of the disk). Erratic drops in speed indicate bad sectors or mechanical issues.

H2testw (USB/SD Card)

H2testw is essential for verifying USB drives and SD cards. Many counterfeit drives report fake capacity — they appear as 256GB but actually have only 32GB of real storage. H2testw writes data to every sector and reads it back to verify integrity.

# H2testw usage (Windows):
# 1. Download from heise.de
# 2. Select target drive (USB or SD card)
# 3. Click "Write + Verify"
# 4. Wait for completion (1-4 hours for 128GB)
# 5. Result: "Test finished without errors" = genuine
# 6. Errors at specific addresses = counterfeit or failing

badblocks (Linux)

The Linux badblocks utility performs a destructive or non-destructive read-write test on storage devices. It is the most thorough storage testing tool available.

# Non-destructive read-write test:
sudo badblocks -svn /dev/sdX

# Destructive test (wipes all data):
sudo badblocks -svw /dev/sdX

# -s: Show progress
# -v: Verbose output
# -n: Non-destructive
# -w: Destructive (write patterns)

# Expected results:
# 0 bad blocks: Drive is healthy
# 1-10 bad blocks: Drive is failing, backup immediately
# 10+ bad blocks: Drive is critically failing, replace now

Power Supply Testing

A faulty power supply can cause crashes, component damage, and data corruption. Testing a PSU requires the right tools and knowledge of safe procedures.

PSU Tester

A dedicated PSU tester ($10-30) provides a quick pass/fail check of all voltage rails. It plugs into the 24-pin motherboard connector and displays voltages for each rail. This is the safest and easiest way to test a power supply.

Multimeter Testing

For accurate voltage measurement, use a digital multimeter. Test each voltage rail under load (while the computer is running a stress test).

# Multimeter voltage test points (ATX 24-pin connector):
# Pin 9 (Purple): +5VSB (standby) — should be 4.75-5.25V
# Pin 10 (Yellow): +12V — should be 11.4-12.6V
# Pin 4 (Red): +5V — should be 4.75-5.25V
# Pin 3 (Orange): +3.3V — should be 3.14-3.47V

# Test under idle and full load (Prime95 + FurMark)
# Voltages should remain within spec
# Drooping below minimum under load = failing PSU

POST Code Interpretation

POST (Power-On Self-Test) codes help diagnose hardware failures during boot. Different motherboard manufacturers use different code systems.

Beep Codes

Motherboards use beep codes to indicate specific failures. The pattern varies by BIOS manufacturer:

  • AMI BIOS: 1 long, 3 short = memory error; 1 long, 8 short = display error; 4 short = timer failure; 8 short = memory read/write error.
  • Award BIOS: 1 long, 2 short = video error; 1 long, 3 short = keyboard error; continuous beeps = memory or power error.
  • Phoenix BIOS: Sequence of beep groups (e.g., 1-3-1 = DRAM refresh failure, 3-1-1 = DMA failure).

Debug LED / POST Card

Modern motherboards have onboard debug LEDs that display two-digit hexadecimal POST codes. A POST card (plugs into a PCI slot or USB port on older boards) provides the same information. Common codes: 0x00 or 0xFF (no boot device), 0xD0 (CPU initialization error), 0xB0 (memory initialization), 0x7F or 0xA0 (boot device error). Consult your motherboard manual for the specific code meanings.

Network Cable Testing

Faulty network cables cause intermittent connectivity, speed drops, and packet loss. Testing cables requires a cable tester or a known-good device to verify connectivity.

  • Visual inspection — Check for bent pins in RJ45 connectors, damaged cable sheathing, or kinks near the connector.
  • Continuity tester ($10-30) — Tests each of the 8 wires for proper connection. All 8 wires should show continuity in the correct order (T568A or T568B standard).
  • Cable certifier ($500+) — Professional tool that measures signal quality, crosstalk, and impedance. Only needed for professional network installers.
  • Loopback test — Use a loopback plug on the NIC to test if the network interface itself is working. Send ping to 127.0.0.1 for a basic test.
# Basic cable testing with a multimeter:
# 1. Continuity mode
# 2. Test each pin on one end against the corresponding pin
#    on the other end
# 3. All 8 pins should show continuity (< 5 ohms)
# 4. No pins should show continuity to any other pin
#    (no shorts)

# Wire order (T568B standard):
# Pin 1: White/Orange
# Pin 2: Orange
# Pin 3: White/Green
# Pin 4: Blue
# Pin 5: White/Blue
# Pin 6: Green
# Pin 7: White/Brown
# Pin 8: Brown

Motherboard Component Inspection

Visual inspection of motherboard components can identify the root cause of hardware failures without any diagnostic tools. Here is what to look for:

  • Bulging or leaking capacitors — Electrolytic capacitors have a scored X or K on top. If the top is bulging or there is brown residue, the capacitor is failing. Replace the motherboard or the capacitors if you have soldering skills.
  • Burn marks or discoloration — Dark spots, burn marks, or discolored PCB near voltage regulators (VRMs), CPU socket, or power connectors indicate overheating or electrical failure.
  • Corrosion — Green or white corrosion on contacts, pins, or solder joints indicates liquid damage. Clean with isopropyl alcohol (99%) and a soft brush.
  • Broken traces — Scratches or cracks in the PCB that cut copper traces. These can sometimes be repaired with conductive ink or jumper wires.
  • Burnt MOSFETs — Small black squares near the CPU socket (VRMs). Cracks, burn marks, or discoloration indicate VRM failure. This can cause random shutdowns under load.

Intermittent Fault Finding

Intermittent faults are the hardest to diagnose — they occur randomly and may not reproduce consistently. Here is a systematic approach:

  1. Log everything — Start a log of when crashes occur, what you were doing, and what error messages appear. Look for patterns.
  2. Temperature correlation — Log temperatures at the time of failure. Faults that occur only when hot indicate thermal issues. Faults that occur when cold (cold boot) indicate component aging or capacitor issues.
  3. Load correlation — Does the failure happen under load (gaming, rendering) or at idle? Load failures suggest PSU, VRM, or overheating issues. Idle failures suggest driver or motherboard issues.
  4. Component isolation — Remove all non-essential components (extra RAM sticks, GPU if using integrated graphics, extra drives, expansion cards). If the fault stops, add components back one at a time until it returns.
  5. Reseat everything — Remove and reseat RAM, GPU, power cables, and data cables. Oxidation on contacts can cause intermittent faults.
  6. Replace the PSU first — Power supply issues are the most common cause of intermittent hardware faults. Test with a known-good PSU before replacing other components.
Method: When diagnosing an intermittent fault, change only one variable at a time. Test for at least 24 hours after each change. Changing multiple things at once makes it impossible to know which change fixed the problem.
