Data Recovery Guide

Step-by-step guide to recovering lost data from hard drives, SSDs, memory cards, and RAID arrays - from software recovery to professional lab services.

Table of Contents

  • Stop Using the Drive
  • Software Recovery (Recuva, TestDisk, PhotoRec)
  • Hardware Recovery
  • RAID Recovery
  • Professional Services

Stop Using the Drive

The single most important rule: stop using the drive immediately. Every second the drive stays powered on, more data can be overwritten - especially on SSDs with TRIM.

  • Power down the system - Hold power button or disconnect cord
  • Do not reinstall Windows - Writes gigabytes over your deleted files
  • Do not run CHKDSK /F - The fix switch often makes things worse
  • Remove the drive - Physically disconnect it
  • Clone first - Create sector-by-sector clone with DDRescue, work on the clone
# Clone failing drive with DDRescue
sudo ddrescue -f /dev/sda /dev/sdb rescue.log
sudo ddrescue -f -r3 /dev/sda /dev/sdb rescue.log  # retry bad sectors

SSD note: TRIM can permanently erase deleted data within minutes. Power off immediately if you deleted from an SSD.

Hardware Recovery

When the drive has physical damage - clicking, grinding, or the click of death - software will not work. Hardware recovery requires opening the drive in a cleanroom.

Common physical failures:

  • Failed read/write heads - Clicking. Needs cleanroom head replacement.
  • Stuck spindle motor - Drive does not spin up.
  • Failed PCB - Drive electrically dead. Replacing PCB with identical donor board may work, but many drives have ROM-specific encryption.
  • Bad sectors - May still be readable with DDRescue.

When NOT to attempt DIY: If data is worth more than a few hundred dollars, send to a professional lab.

RAID Recovery

RAID recovery requires reconstructing the logical volume from physical disks.

  • Identify RAID level - 0, 1, 5, 6, 10
  • Determine stripe size - Usually 64 KB or 128 KB
  • Clone drives first - Clone each physical drive before array reconstruction
  • Use RAID recovery software - R-Studio, ReclaiMe, mdadm
# Linux - assemble RAID
sudo mdadm --assemble --scan
sudo mount /dev/md0 /mnt/recovery
rsync -av /mnt/recovery/ /safe/location/

RAID 0: If one drive fails, the entire array is lost. Back up RAID 0.

Professional Services

When data is worth $300-$3000+, send to a lab:

  • Evaluation: Free diagnostic and quote
  • Cleanroom work: Class 10/100 for hardware repairs
  • Recovery rate: 80-95% of data
  • Turnaround: 1-5 business days

Labs: DriveSavers, Ontrack, Gillware, Kroll Ontrack.

Before shipping: Remove drive, pack in anti-static bag, wrap in foam, label fragile.

File Deletion Basics (FAT vs NTFS vs ext4)

Understanding how filesystems handle deletion is critical for successful recovery:

  • FAT32/exFAT — Deletion marks the directory entry as available. File data remains intact until overwritten. Highest chance of recovery. Used on USB drives and memory cards.
  • NTFS — Deletion marks the MFT entry as free. The $Bitmap is updated. File data persists but may be overwritten quickly. Windows also uses the $LogFile and $UsnJrnl for recovery clues. Used on Windows system drives.
  • ext4 — Deletion clears the inode and marks blocks as free. The file data stays until reused. Journaling may overwrite metadata. Used on Linux system drives.
  • TRIM (SSD) — Immediately wipes deleted blocks at the flash level. Recovery from SSDs after TRIM is nearly impossible. Power off immediately after accidental deletion on an SSD.
Critical: On SSDs with TRIM enabled, you have seconds to minutes before deleted data is permanently gone. Power off immediately and use hardware write-blockers for forensic imaging.

Recuva Deep Scan Steps

Recuva is the go-to tool for recovering deleted files on Windows. Here is how to perform a thorough deep scan:

# Recuva Deep Scan Procedure:
# 1. Download Recuva Portable to your emergency USB
# 2. Run recuva.exe as Administrator
# 3. Welcome screen → Next
# 4. File type: Select "All Files" for maximum coverage
# 5. Location: Choose the specific drive (not "I'm not sure")
# 6. Check "Enable Deep Scan" — takes longer but finds more
# 7. Scan starts (can take 30 min to several hours)
# 8. After scan: Sort by "Status" — look for "Excellent" first
# 9. Right-click a file → "Preview" to verify contents
# 10. Select files → "Recover..." → Save to DIFFERENT drive
#
# Deep Scan vs Quick Scan:
# - Quick Scan: Reads MFT/MBR, fast (seconds), finds recently deleted
# - Deep Scan: Scans every sector for file signatures, slow but thorough
# - Always run Deep Scan if Quick Scan misses important files
Tip: Recuva's "Preview" feature lets you see file contents before paying for the professional version. Use it to confirm recoverability before purchasing.

TestDisk Step-by-Step (Partition Recovery, MBR/GPT Repair)

TestDisk is the most powerful free tool for recovering lost partitions and repairing boot sectors:

# TestDisk — Recover a deleted partition:
# 1. Run: testdisk (or testdisk-7.2-WIP.linux)
# 2. Choose: [Create] new log file
# 3. Select the disk with the lost partition (use arrow keys)
# 4. Choose partition table type:
#    - [Intel] for MBR (most Windows PCs)
#    - [EFI GPT] for UEFI systems
#    - [None] for corrupt tables
# 5. Select [Analyse] → [Quick Search]
# 6. TestDisk scans for lost partitions
# 7. Found partitions are listed with P=Primary, L=Logical
# 8. Highlight lost partition → [P] to list files (verify it's correct)
# 9. Press [Enter] → [Write] to save partition table
# 10. Reboot — partition should reappear
#
# Repair MBR (Master Boot Record):
# testdisk → Select disk → [MBR Code]
# → Write a new MBR code to the first sector
# → Reboot
#
# Repair GPT (GUID Partition Table):
# testdisk → Select disk → [EFI GPT]
# → [Analyse] → [Backup GPT] → Repair backup GPT
# → Reboot

TestDisk cannot recover: Individual deleted files (use PhotoRec), data from physically damaged drives (use ddrescue).

PhotoRec File Carving

PhotoRec recovers files by scanning for "file headers" (magic bytes) rather than filesystem metadata. It ignores the filesystem entirely, making it ideal for formatted or corrupted drives:

# PhotoRec — Carve files from a formatted drive:
# 1. Run: photorec
# 2. Select the disk (not the partition)
# 3. Select partition table type
# 4. Select filesystem type: [Other] for FAT/NTFS/ext
# 5. Choose: [File Opt] → select which file types to recover
#    (deselect unwanted types to speed up and reduce noise)
# 6. Choose: [Whole] (extract from entire disk) or [Freed] (unallocated only)
# 7. Choose destination directory on a DIFFERENT drive
# 8. [Y] to continue → carving begins
#
# PhotoRec output:
# - Files saved as recup_dir.1/, recup_dir.2/, etc.
# - Filenames are lost — renamed as f1234567.jpg
# - Sort by type:
#   find recup_dir.1 -name "*.jpg" -exec mv {} ./jpgs/ \;
# - 480+ file types supported (JPEG, DOC, ZIP, PDF, MP4, etc.)

PhotoRec vs Recuva: PhotoRec works when Recuva fails — formatted drives, corrupted filesystems, or when the MFT/MBR is destroyed. The tradeoff is that all filenames are lost.

ddrescue for Failing Drives

When a drive has physical damage or bad sectors, ddrescue is the tool to clone it while skipping unreadable areas:

# ddrescue — Clone a failing drive:
# 1. Identify source (failing) and target (good) drives:
#    lsblk or fdisk -l
# 2. First pass: read good sectors quickly
#    sudo ddrescue -f -n /dev/sdb /dev/sdc rescue.log
# 3. Second pass: retry bad sectors (3 retries)
#    sudo ddrescue -f -r3 /dev/sdb /dev/sdc rescue.log
# 4. Third pass: direct read for remaining bad sectors
#    sudo ddrescue -f -d /dev/sdb /dev/sdc rescue.log
# 5. Check the logfile:
#    ddrescuelog rescue.log
#    Shows: number of bad sectors, total size, status
#
# Key flags:
# -f = force overwrite target
# -n = no retry (fast first pass)
# -r3 = retry bad sectors 3 times
# -d = direct disk access (bypasses kernel cache)
# -A = try to read all sectors even if marked bad
Warning: Never run ddrescue on an SSD to recover files. Use software recovery tools first. ddrescue is for HDDs only — SSD controllers remap sectors and the data you read may not be what is physically stored.

Chip-Off Recovery (When to Use Professionals)

Chip-off recovery involves desoldering the NAND flash chips from the PCB and reading them directly with specialized hardware. This is a last resort:

  • When needed: SSD with failed controller, water/fire damage, physically broken PCB, drives that click or do not spin (HDD).
  • Cost: $500-$5000+ depending on drive complexity and urgency.
  • Success rate: 60-90% for NAND chip-off, 80-95% for HDD cleanroom.
  • Risks: Chip desoldering can permanently destroy data if heat is applied incorrectly. Controller encryption may make raw NAND reads unparseable.
  • Professional labs: DriveSavers, Ontrack, Gillware, Kroll Ontrack. Always ask about "no data, no fee" policies.
Recommendation: If your data is worth more than $300, send to a professional. DIY chip-off almost always makes things worse.

RAID Recovery Basics

RAID recovery adds complexity because data is striped, mirrored, or parity-protected across multiple drives:

  • RAID 0 (Striping) — One failed drive destroys the array. All data lost unless recovered from individual drives in the correct stripe order. Very difficult.
  • RAID 1 (Mirroring) — Each drive has complete data. Recovery is simple: recover from either drive. If one failed, the other has all data.
  • RAID 5 (Striping + Parity) — One drive can fail without data loss. Replace the failed drive and rebuild. If two fail, recovery requires reconstructing parity from remaining drives.
  • RAID 6 (Dual Parity) — Two drives can fail. Rebuild with two replacements.
  • Recovery software: R-Studio (best for RAID), ReclaiMe, UFS Explorer. These tools can analyze disk order, stripe size, and parity rotation automatically.
# Linux software RAID recovery:
# 1. Identify array components:
sudo mdadm --examine /dev/sd[b-e]
# 2. Assemble the array:
sudo mdadm --assemble --scan
# 3. If auto-assembly fails, try forced assembly:
sudo mdadm --assemble --force /dev/md0 /dev/sdb /dev/sdc
# 4. Mount and recover:
sudo mount /dev/md0 /mnt/recovery
rsync -av /mnt/recovery/ /safe/location/

Data Recovery Order of Operations

Follow this sequence to maximize recovery chances without causing further damage:

  1. Stop using the drive immediately. Power off. Do not reinstall OS, run chkdsk, or fsck.
  2. Assess the situation. Is it accidental deletion, formatted drive, physical damage, or logical corruption?
  3. Clone the drive first. Use ddrescue for HDDs, or create a forensic image (.dd/.e01) with FTK Imager or DD.
  4. Work from the clone/image. Never work on the original drive.
  5. Try quick recovery. Recuva quick scan (deletion), TestDisk (partition loss).
  6. Deep scan if needed. Recuva deep scan, PhotoRec carving, Foremost.
  7. Professional help. If DIY fails and data is worth $300+, send to a lab.
Golden rule: The moment you realize you need data recovery, the best thing you can do is nothing. Power off, put the drive aside, and plan your approach.

Write-Blocker Usage

Write-blockers prevent any data from being written to the source drive during forensic imaging or recovery:

  • Hardware write-blocker — A device that sits between the drive and computer. It physically blocks write commands. Essential for forensic evidence preservation. Cost: $100-$500.
  • Software write-blocker — OS-level protection. On Linux: mount -o ro (read-only). On Windows: use Registry key or forensic software that enforces read-only.
  • Cheap alternative — Use a Linux live USB and mount the drive read-only. Not as secure as hardware blocker but better than nothing.
# Software write-block with dd on Linux:
# Mount source read-only:
sudo mount -o ro,noexec /dev/sdb1 /mnt/source
# Or use dd with iflag=noatime,conv=noerror:
sudo dd if=/dev/sdb of=/images/evidence.dd bs=4M iflag=noatime,conv=noerror,sync

Cold Storage for Frozen Drives

Freezing a failing hard drive is a controversial but occasionally effective last-ditch trick:

  • The theory: Cold temperatures cause metal components to contract slightly. A stuck spindle motor or seized read/write head may break free temporarily.
  • The risk: Condensation forms when the drive warms up, causing electrical shorts and corrosion. Moisture = permanent data loss.
  • If you must try: Seal the drive in an anti-static bag with desiccant silica gel. Freeze for 2-4 hours. Remove, keep in bag, quickly connect, try to read. You may get 5-30 minutes of operation before condensation kills it.
  • Better approach: Send to a professional lab. They have cleanroom facilities and component-level repair capabilities.
Warning: Freezing rarely works and often makes professional recovery harder. Use only as a last resort for data that is otherwise unrecoverable.

Avoiding Data Recovery Scams

The data recovery industry has many reputable labs but also scammers. Watch for these red flags:

  • Upfront charges — Reputable labs provide free evaluation and quote. Paying before diagnosis is a red flag.
  • No cleanroom — HDD head replacement requires Class 10 or Class 100 cleanroom. Ask about their cleanroom classification.
  • Flat-rate pricing — Real recovery depends on damage. Quotes should be per-case, not "everything for $99."
  • Guaranteed recovery — No ethical lab guarantees 100% recovery, especially for physically damaged drives.
  • Check reviews — Search for the lab name + "scam" or "complaint" on Reddit, Trustpilot, and BBB.
  • Insist on no-data-no-fee — You should not pay if they cannot recover anything.
  • Get a data list — Reputable labs provide a list of recoverable files before payment.

Previewing Recovery Files Before Purchase

Most recovery tools let you preview recoverable files before paying for the full version. Use this to assess whether recovery is worth it:

  • Recuva — Free version shows all recoverable files with preview. You only pay if you want to recover them.
  • DMDE — Free version recovers up to 4000 files from one folder. Preview is built in.
  • R-Studio — Demo version shows recoverable files and allows preview. Purchase unlocks bulk recovery.
  • PhotoRec — Completely free. No preview needed — it recovers everything to a destination folder.

Workflow: Scan with free tool → browse recoverable files → verify critical files exist → pay for recovery only if needed.
