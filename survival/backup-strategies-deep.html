Advanced Backup Strategies

Deep dive into backup strategies: 3-2-1 rule, local and offsite backups, cloud options, rotation schemes, restore testing, and automation.

3-2-1 Rule

3 copies of data, 2 different media types, 1 offsite. Example: laptop (primary) + external HDD (local) + Backblaze (cloud).

Protects against hardware failure, theft, fire, flood, and ransomware.

Local Backups

Fastest for restores.

Options: External USB HDD (4-8 TB, ~$80-120), External USB SSD (1-2 TB, ~$60-120), NAS (~$200-600).

# rsync (Linux/macOS):
rsync -avh --delete /home/user/Documents/ /mnt/backup/
# Robocopy (Windows):
robocopy C:\Users\Name\ D:\Backup\ /MIR

Offsite Backups

Methods: Swapped drives with trusted person (free, monthly swap), safe deposit box, cloud backup, work/office location.

Geographic distance: At least 10 miles away.

Cloud Backups

Providers: Backblaze (~$9/mo unlimited), iDrive (~$80/yr 5 TB), Proton Drive (zero-access E2EE), Rclone + any cloud.

Important: Cloud backup is NOT cloud sync. Backup keeps version history, sync propagates deletions.

Backup Rotation

GFS scheme: 7 daily + 4 weekly + 12 monthly backups. Long retention with deduplication via hardlinks.

Testing Restores

A backup you have not tested is not a backup.

Monthly: Restore 3 random files, verify checksums. Quarterly: Restore entire OS. Yearly: Full disaster recovery drill.

Automation

The best backup is automatic. Use systemd timers (Linux), Task Scheduler (Windows), or cron. Monitor with healthcheck.io.
