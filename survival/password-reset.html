Password Reset & Recovery

Reset or recover passwords on Windows, Linux, and macOS - plus Bitwarden emergency access and 2FA recovery codes.

Table of Contents

Windows Password Reset (chntpw)

chntpw is the most reliable tool for resetting local Windows passwords.

# Boot from SystemRescue or Ubuntu Live
sudo mount /dev/sda2 /mnt/windows
cd /mnt/windows/Windows/System32/config
sudo chntpw SAM
# Select user, choose clear or change password
# Reboot and log in

Microsoft account: Reset at account.live.com. BitLocker requires recovery key first.

Linux Password Reset

# Single-user mode via GRUB:
# Press e to edit kernel, add init=/bin/bash
mount -o remount,rw /
passwd username
reboot -f

# From Live CD:
# Mount root, chroot, passwd, exit

LUKS: Need original passphrase for encrypted home.

macOS Password Reset

# Apple ID: wrong password 3x - reset via Apple ID
# Recovery: Hold power/Cmd+R - Terminal - resetpassword
# Single-user: Cmd+S - mount -uw / - passwd username

FileVault: Need 64-char recovery key first.

chntpw Detailed Commands

chntpw is the definitive tool for resetting local Windows passwords from rescue media. Here are the full commands:

# chntpw — Full command reference:
# Boot from SystemRescue or Ubuntu Live USB

# Mount Windows system partition:
sudo mount /dev/sda2 /mnt/windows
# Navigate to SAM file:
cd /mnt/windows/Windows/System32/config

# List all users with details:
sudo chntpw -l SAM
# Output shows: RID, Username, Admin?, Locked?

# Clear password (interactive mode — recommended):
sudo chntpw SAM
# Menu options:
# 1 - Clear password (blank)
# 2 - Edit user data and flags
# 3 - Promote to administrator
# 4 - Unlock account
# q - Quit

# Clear a specific user's password (non-interactive):
sudo chntpw -u Administrator SAM
sudo chntpw -u UserName SAM

# Promote user to admin:
sudo chntpw -u UserName SAM  # then choose option 3
# Or edit the SAM file directly:
# The 'F' flag = disabled, 'L' = locked
# Change 'F' to 'A' for admin in the user flags

# Unlock all accounts:
sudo chntpw -u UserName SAM  # then choose option 4

# Batch clear all user passwords:
for user in $(sudo chntpw -l SAM | grep -oP '^\d+'); do
    echo -e "1\ny\ny\nq" | sudo chntpw -u $user SAM
done

# Reboot and log in with blank password
# Set a new password once in Windows:
# Ctrl+Alt+Del → Change a password
Important: chntpw only works for local Windows accounts. Microsoft account passwords must be reset online. BitLocker encrypted drives require the 48-digit recovery key before you can mount the partition.

Linux Live USB Password Reset Walkthrough

If you forget your Linux password, here is the complete recovery procedure:

# Method 1: GRUB init=/bin/bash (easiest):
# 1. Reboot → hold Shift (BIOS) or press Esc (UEFI) for GRUB menu
# 2. Highlight kernel line → press 'e' to edit
# 3. Find "ro quiet splash" → replace with:
#    rw init=/bin/bash
# 4. Ctrl+X or F10 to boot
# 5. At the bash prompt:
passwd username
# 6. If you don't know the username:
cat /etc/passwd | grep /home
# 7. Reboot: exec /sbin/init

# Method 2: From Live USB (for systems with encrypted root):
# 1. Boot from Ubuntu Live USB
# 2. Open terminal:
sudo fdisk -l  # identify root partition
sudo mount /dev/sda2 /mnt
# If root is encrypted (LUKS):
sudo cryptsetup luksOpen /dev/sda2 cryptroot
sudo mount /dev/mapper/cryptroot /mnt
# 3. Chroot and reset:
sudo chroot /mnt
passwd username
exit
sudo umount /mnt
sudo reboot

# Method 3: Emergency mode:
# 1. Add systemd.unit=emergency.target to GRUB boot params
# 2. At the shell prompt:
mount -o remount,rw /
passwd username
reboot -f

Windows RE Password Reset via utilman.exe Trick

The utilman.exe trick replaces the Ease of Access utility with Command Prompt on the login screen:

# utilman.exe password reset trick:
# Requires: Windows installation USB or access to WinRE

# Step 1: Boot from Windows installation USB
# Step 2: On the install screen press Shift+F10
# Step 3: Identify the Windows drive:
diskpart
list volume
exit
# Usually C: is Windows, but in WinRE it may be D: or E:

# Step 4: Backup and replace utilman.exe:
ren D:\Windows\System32\utilman.exe utilman.exe.bak
copy D:\Windows\System32\cmd.exe D:\Windows\System32\utilman.exe

# Step 5: Reboot normally (remove USB)
# Step 6: On the login screen, click the Ease of Access icon
# Step 7: A command prompt opens (with SYSTEM privileges):
net user Administrator newpassword
# Or for a specific user:
net user UserName newpassword
# To create a new admin user:
net user backupadmin Password123! /add
net localgroup Administrators backupadmin /add

# Step 8: Log in with the new password
# Step 9: Restore utilman.exe:
ren C:\Windows\System32\utilman.exe utilman.exe.cmd
ren C:\Windows\System32\utilman.exe.bak utilman.exe
Note: This trick works on all Windows versions from 7 to 11. It requires physical access to the machine. The Ease of Access icon must be present on the login screen.

macOS Password Reset (Recovery Mode, Terminal, FileVault)

Complete macOS password recovery options:

# macOS Recovery Mode Password Reset:
# 1. Shut down Mac
# 2. Press and hold Cmd+R (Apple Silicon: hold power button)
# 3. Continue holding until you see Apple logo or startup options
# 4. From Recovery menu: Utilities → Terminal
# 5. Type:
resetpassword
# 6. GUI opens → Select user → Enter new password
# 7. Restart and log in with new password

# If FileVault is enabled:
# You MUST have the 64-character recovery key to unlock
# the drive before resetting. Find it:
# - iCloud account (if stored)
# - Printed copy from initial setup
# - FileVault master password (institutional)
# Without the recovery key, data is unrecoverable

# Apple ID password reset (for iCloud/Mac login):
# 1. Enter wrong password 3 times
# 2. Click "Reset using Apple ID"
# 3. Enter Apple ID credentials
# 4. Set new Mac login password
# Note: This does NOT work if FileVault is enabled

# Single-user mode (Intel Macs only):
# 1. Cmd+S at boot
# 2. At the shell:
/sbin/mount -uw /
passwd username
reboot

# Apple Silicon recovery:
# Hold power button → Options → Continue
# → Terminal → resetpassword
# Apple Silicon Macs require recovery key or Apple ID reset

BIOS Password Removal Methods

When a BIOS/UEFI password blocks access to a computer:

  • CMOS clear (most common) — Remove the CR2032 battery, short the CLR_CMOS jumper, or press the clear CMOS button. This clears BIOS settings including passwords stored in CMOS RAM.
  • System discharge — Remove battery AND power cord. Press power button 30 seconds to drain capacitors. Wait 10 minutes. Reconnect and boot.
  • Manufacturer backdoor passwords — Many laptop brands have master passwords. For Dell, HP, Lenovo — search "[brand] BIOS master password" with your service tag. Some generate codes based on a "system disabled" message.
  • SPI chip reflash — If the password is stored in the firmware flash (common on modern laptops), the only way to clear it is by reflashing the BIOS chip with CH341A. Read the chip, flash a known clean BIOS, verify.
  • Soldered BIOS chip — On ultrabooks, the chip is often soldered to the motherboard. This requires professional desoldering/reballing or a precision clip for WSON-8 packages.
  • Apple EFI/ Firmware Password — Cannot be removed by CMOS clear. Requires SMC reset (Shift+Ctrl+Opt+Power) or PRAM reset (Cmd+Opt+P+R). If that fails, the logic board needs professional EFI reprogramming.

Bitwarden Emergency Access Setup

Bitwarden Emergency Access grants trusted contacts vault access after a waiting period:

# Bitwarden Emergency Access Setup:
# 1. Log into Bitwarden web vault
# 2. Settings → Emergency Access → Add Contact
# 3. Enter contact's email
# 4. Choose access level:
#    - View: Can see all vault items (read-only)
#    - Takeover: Can take ownership of vault (can use passwords)
# 5. Set waiting period:
#    - 7 days (recommended for personal)
#    - 14 days (for business/critical)
#    - 30 days (maximum delay)
# 6. Contact receives invitation email → accepts

# How emergency access works:
# 1. Contact requests access from their Bitwarden account
# 2. You receive email notification
# 3. Waiting period starts (7-30 days)
# 4. You can DENY access at any time during waiting period
# 5. If you do not respond, access is granted after the period
# 6. Contact receives your vault decrypted data

# Offline backup:
# Even with Emergency Access, keep a paper backup:
# Print master password on paper, seal in envelope
# Store in home safe + safe deposit box
# Include: master password, 2FA backup codes, phone PIN
# Update whenever master password changes

2FA Recovery Code Storage (Offline Physical Copy)

2FA recovery codes are your last resort when you lose access to authenticator apps. Store them carefully:

  • Print and store in safe — Print recovery codes on paper, seal in a fireproof safe. Update when you change services.
  • Encrypted USB — Store recovery codes in a VeraCrypt-encrypted container on your emergency USB.
  • Password manager secure note — Add recovery codes as a secure note in Bitwarden/KeePassXC. Tag them as "Recovery."
  • Steel backup — For crypto exchange or email recovery codes, stamp onto steel washers or use a Cryptosteel. Survives fire and flood.
  • What to store: Service name, account email, recovery codes (full list), date generated, and a note about what each code is for (e.g., "backup code #3 of 10").

Domain-Joined Computer Password Reset

For computers joined to a corporate domain, local password reset is limited:

  • Local admin password — If you know the local admin password (not domain), you can use it to log in and reset domain user passwords via Ctrl+Alt+Del.
  • Domain admin intervention — A domain administrator can reset your password from Active Directory Users and Computers. Contact IT.
  • Offline reset (chntpw) — Works on domain-joined machines for LOCAL accounts. Does not reset domain passwords. You can create a new local admin to access files.
  • Cached credentials — Windows caches the last 10 domain logins. If you can log in with cached credentials before they expire, you can change your password.

Android FRP Bypass Options

Factory Reset Protection (FRP) locks Android devices to the last Google account after a factory reset:

  • Legitimate recovery — Enter the previous Google account email/password. If you forgot it, use Google account recovery.
  • Account recovery first — Go to accounts.google.com/signin/recovery. Verify your identity and recover the account. Then enter it on the FRP screen.
  • OEM tools — Samsung has "Samsung Find My Mobile" remote unlock. Other OEMs have similar tools — check their account dashboards.
  • Professional tools — Services like BypassFRP or Octoplus can remove FRP for a fee ($20-50). These are reputable if you own the device.
  • Scam warning — Do NOT pay random YouTube channels or Telegram groups claiming to bypass FRP for free. Most are scams or malware.
Note: FRP is a theft deterrent. If you legitimately own the device, contact the OEM or use official account recovery. Third-party bypass tools may not work on the latest Android versions.

iOS/iCloud Password Recovery

If you are locked out of your iPhone, iPad, or iCloud account:

  • iPhone passcode forgotten — Connect to computer with iTunes/Finder. Put iPhone in recovery mode (volume up, volume down, hold power). Choose "Restore" (wipes device) or "Update" (preserves data). Requires the computer to be trusted.
  • Apple ID password — Reset at iforgot.apple.com. Uses email verification or trusted phone number. Can also use account recovery.
  • iCloud Activation Lock — Requires original proof of purchase (receipt, box) to send to Apple for removal. This is an anti-theft feature.
  • Screen Time passcode — If locked out of Screen Time restrictions, contact Apple Support with proof of purchase.
  • Backup password (encrypted iTunes backup) — There is no recovery. You must try all possible passwords or lose the backup data.

Password Manager Master Password Recovery

If you forget your password manager master password, recovery options are limited by design:

  • Bitwarden — No master password recovery (zero-knowledge). If you have Emergency Access set up, a trusted contact can grant access after the waiting period. If not, the vault is permanently lost.
  • KeePassXC — No recovery option. The database file (.kdbx) is encrypted with your master password + key file (if used). Without both, the database is unrecoverable.
  • 1Password — Has a "Secret Key" (part of the 2-factor setup). If you have the Secret Key + account password, you can recover. Emergency Kit PDF has both.
  • Dashlane — Offers master password reset via email if you set up the recovery option while logged in. Without it, no recovery possible.
  • LastPass — Master password cannot be recovered. Account recovery options exist but are limited.
  • Prevention: Write master password on paper, seal in envelope, store in safe. Update when you change it. This is the only guaranteed recovery method for zero-knowledge managers.
