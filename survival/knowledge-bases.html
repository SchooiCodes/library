DevDocs — Multi-API Documentation

DevDocs combines dozens of API documentation sets into a single, fast, searchable interface. It works in the browser and can be run entirely offline.

  • Online: devdocs.io — Aggregates docs for 200+ APIs and languages including JavaScript, Python, React, Docker, Git, HTML/CSS, Node.js, Rust, Go, and more.
  • Offline via DevDocs Desktop: The official desktop app caches entire documentation sets locally. Search without internet.
  • Self-hosted: DevDocs is open-source. You can run your own instance locally via Docker or Ruby.
# DevDocs Desktop (cross-platform)
# Download from: https://github.com/egoist/devdocs-desktop
# Install via package manager:
winget install devdocs-desktop  # Windows
brew install --cask devdocs   # macOS
sudo snap install devdocs    # Linux

# Self-host DevDocs via Docker:
docker run --name devdocs -d -p 8080:8080 \
 -e DOCS_ENABLED="javascript python react" \
 ghcr.io/egoist/devdocs

# Open http://localhost:8080
# Pre-cache the docs you need:
# In the app, click "Preferences" → select documentation sets → "Install"
Tip: DevDocs Desktop automatically updates cached docs when you have internet. In an emergency, the cached docs remain fully functional offline. Pre-load as many docsets as your disk allows.

Zeal — Offline Docset Browser

Zeal is a cross-platform (Windows, Linux) offline documentation browser based on the Dash docset format. Choose from 200+ documentation sets and use them instantly without internet.

  • Platform: Windows, Linux (macOS users use Dash — native version)
  • Docsets: Zeal uses the same docset format as Dash. 200+ docsets available. Includes Python, JavaScript, React, Vue, Angular, Docker, Kubernetes, Git, Linux man pages, PostgreSQL, Redis, and many more.
  • Features: Fuzzy search, snippet integration, offline-first, global keyboard shortcut for quick lookup.
# Install Zeal
# Linux:
sudo apt install zeal    # Debian/Ubuntu
sudo dnf install zeal    # Fedora
sudo pacman -S zeal     # Arch

# Windows: Download from https://zealdocs.org/download.html
# or: winget install zeal

# Download docsets in-app:
# Zeal → Edit → Preferences → Docsets → Download
# Or download .docset files manually from:
# https://raw.github.com/zealdocs/zeal/master/docsets.md

# Install docset from command line:
# Download the .tgz file, then:
tar xzf python_3.docset.tgz -d ~/.local/share/Zeal/docsets/

Zeal is indispensable for any developer preparing for a hackathon, coding in a remote location, or just wanting fast, offline API reference. It's lightweight (~20MB) and docsets average 5-50MB each.

Dash for macOS

Dash is the original docset browser, exclusive to macOS. It inspired both Zeal and DevDocs. It remains the most polished offline documentation experience on Apple platforms.

  • Features: Instant API search, docset management, code snippet manager, integration with Xcode, VS Code, Sublime Text, and 30+ other IDEs and editors.
  • Docsets: 200+ including Apple frameworks, web technologies, languages, databases, and tools. Download all of them if you have the disk space (~5 GB total).
  • Integration: Press the Dash shortcut from within any editor to search docs instantly. Supports custom docset creation from any HTML documentation.
# Install Dash (macOS):
# Download from https://kapeli.com/dash
# Or: brew install --cask dash

# Download all docsets (offline complete set):
# Dash → Preferences → Downloads → Download All
# This gives you ~5 GB of comprehensive offline documentation

# Integrate with VS Code:
# Install "Dash" extension
# Cmd+Shift+H opens selected text in Dash

# Create custom docset from HTML docs:
# Use docsetutil or the "Generate" feature in Dash Preferences
Cross-platform tip: Use Zeal on Windows/Linux, Dash on macOS, and DevDocs Desktop anywhere. They all use the same underlying docset format, so you can share docsets between them.

MDN Web Docs Offline

MDN Web Docs is the definitive reference for HTML, CSS, JavaScript, and Web APIs. The complete offline version is available through multiple methods:

  • MDN via DevDocs/Zeal — Both DevDocs and Zeal include MDN docsets. This is the easiest way to get MDN offline.
  • MDN ZIM archive — The Kiwix project provides MDN Web Docs as a ZIM file. ~2 GB, searchable via Kiwix.
  • MDN Web Docs dump — The raw HTML/JSON export from Mozilla's developer site. Available via the MDN archive project on GitHub.
  • MDN PWA — The MDN website can be installed as a Progressive Web App and caches recently viewed pages for offline use.
# MDN via Kiwix (ZIM format)
wget https://download.kiwix.org/zim/other/mdn_all_all.zim
kiwix-serve --port=8080 mdn_all_all.zim
# Browse at http://localhost:8080

# MDN as Zeal docset
# Zeal → Preferences → Docsets → Search "MDN"
# Download: ~500 MB, covers all web documentation

Given that web development changes rapidly, having an offline snapshot of MDN is invaluable when you're without internet. The ZIM/ZIM format means full-text search across all MDN articles.

tldr-pages

tldr stands for "Too Long; Didn't Read" — it's a community-maintained collection of simplified man pages. Each page shows the most common use cases for a command with practical examples.

  • What it is: Practical examples for 1,000+ commands. Each command gets a page with 5-10 common usage examples.
  • How it helps: Instead of wading through a 100-line man page, tldr shows you exactly what you need: "tar -xzf file.tar.gz" and "tar -czf archive.tar.gz dir/".
  • Installation: Available as a Node.js package, Python package, Go binary, or standalone app. Works offline after initial cache.
# Install tldr
npm install -g tldr        # Node.js
pip install tldr          # Python
brew install tldr         # macOS
sudo apt install tldr       # Debian/Ubuntu
sudo pacman -S tldr        # Arch

# Usage examples:
tldr tar      # Show tar command examples
tldr find     # Show find command examples
tldr ffmpeg    # Show ffmpeg examples
tldr --list    # List all available commands

# Update local cache (do this while you have internet):
tldr --update

# Offline mode: tldr works fully offline after cache is populated
# Cache location: ~/.tldr/ or ~/.config/tldr/

Combine tldr with Zeal/DevDocs for a comprehensive offline reference: tldr for quick command syntax, Zeal for deep API documentation.

man Pages — The Original Unix Docs

Every Unix and Linux system ships with man pages — the original offline documentation system. They're always available, always relevant, and work without any setup.

  • Access: man <command> — e.g., man grep, man bash, man 5 crontab (section 5 for config files)
  • Sections: 1=User commands, 2=System calls, 3=C library functions, 4=Devices, 5=File formats, 6=Games, 7=Misc, 8=System admin commands
  • Search: man -k keyword (apropos) or apropos keyword
  • Printable: man -t command | lp — print a man page as PostScript
# Essential man page navigation
man ls        # Manual for ls
man 3 printf     # Section 3 (C library) man page
man -k database   # Search for "database" in all man pages
man -f ls      # Same as "whatis ls" — short description

# View man page as PDF (for offline reading on any device)
man -t ls | ps2pdf - ls.pdf

# Export to plain text
man ls | col -b > ls.txt

# Offline man pages for embedded systems:
# Install man-db and man-pages on any Linux system
sudo apt install man-db man-pages  # Debian/Ubuntu
sudo pacman -S man-db man-pages   # Arch

When learning or troubleshooting, use this order: tldr for quick examples, man for full documentation, info for GNU-style manuals.

Cheat Sheets and Cheat.sh

Cheat sheets are condensed, single-page references for tools, languages, and concepts. Multiple offline and online sources exist:

  • cheat.sh — Online/offline cheat sheet aggregator. Supports curl-based queries: curl cheat.sh/tar. Can be installed as a CLI tool.
  • navi — Interactive cheat sheet browser. Community repositories cover 200+ tools. Fully offline.
  • eglot/cheat — Python-based cheat sheets for Linux commands and programming languages.
  • Printed cheat sheets: Keep a binder of printed cheat sheets for critical tools (Vim, Git, Bash, SQL, Python, regex).
# cheat.sh — query from any device with curl
curl cheat.sh/tar       # Examples for tar
curl cheat.sh/python/regex   # Python regex quick reference
curl cheat.sh/git~merge    # Git merge cheat sheet

# Install cheat.sh as CLI (requires Ruby or Python)
# Or use the curl method — it's always available

# navi — interactive cheat sheet browser
brew install navi        # macOS
sudo apt install navi      # Debian/Ubuntu (if available)
# Or: cargo install navi
# Then download cheat sheets:
navi repo add https://github.com/denisidoro/navi

# cheat — Python-based offline cheat sheets
pip install cheat
cheat tar            # Show cheat sheet for tar
cheat -l            # List all available cheats
cheat -e vim          # Edit/create cheat sheet for vim

Build your own cheat sheet collection by curating the most-used commands from each tool you use. Store them in a Git repo for versioning and syncing.

Local Wiki and Note Systems

For personal knowledge management that works entirely offline, these tools let you build and search your own local wiki:

  • Obsidian — Plain-text markdown notes with bidirectional linking, graph view, full-text search. Works completely offline. Extendable with plugins.
  • Wiki.js — Self-hosted wiki that runs locally. Supports Markdown, HTML, diagrams, authentication, and full-text search. Docker deployment.
  • Dokuwiki — PHP-based wiki that runs on any web server (Apache, Nginx, even IIS). No database required. All content is plain text files.
  • TiddlyWiki — A single HTML file that is a complete wiki. Edit in the browser, save locally. Extremely portable — lives on a USB stick.
  • Zim Desktop Wiki — Desktop wiki with WYSIWYG editing, page linking, attachments, and versioning. Stores content in plain text files.

# Download from https://obsidian.md/
# Or on Linux via AppImage:
chmod +x Obsidian*.AppImage
./Obsidian*.AppImage

# Create a vault in any folder
# Install "Local REST API" plugin for programmatic access
# All data is plain Markdown — future-proof and portable

# Download empty.html from https://tiddlywiki.com/
# Open in browser, start typing
# Save changes to the same file (works in Firefox, Chrome)
# Entire wiki is a single self-contained HTML file

For developers, Obsidian + a community plugin that indexes your codebase creates an offline, searchable development knowledge base.

Keeping Your Docs Updated

Documentation changes over time. Here's how to keep your offline library current:

  • DevDocs: Open the app periodically with internet — it auto-updates docsets.
  • Zeal/Dash: Manually trigger "Update All" in Preferences when you have internet.
  • tldr: Run tldr --update weekly as part of your routine.
  • Kiwix ZIMs: Subscribe to the Kiwix RSS feed for update notifications. Redownload major ZIM files quarterly.
  • Obsidian vault: Sync via Git or Syncthing for local-only versioning.
# Automated docset update script
#!/bin/bash
# Run weekly via cron to keep offline docs fresh

# Update tldr cache
tldr --update 2>/dev/null || true

# Update Zeal docsets (if installed)
if command -v zeal &> /dev/null; then
 # Zeal has no CLI update — open it manually
 echo "Remember to update Zeal docsets manually"
fi

# Update cheat sheets
if command -v cheat &> /dev/null; then
 cheat -u 2>/dev/null || true
fi

# Notify to update Kiwix ZIMs
echo "Check https://download.kiwix.org/zim/ for new ZIM versions"

# Save as ~/bin/update-offline-docs.sh
# chmod +x ~/bin/update-offline-docs.sh
# Add to crontab: 0 10 * * 1 ~/bin/update-offline-docs.sh
Reminder: Set a recurring calendar reminder (monthly) to update your offline documentation. When SHTF and the internet is gone, you'll be glad you have current docs — not a snapshot from 2 years ago.

Kiwix Installation & Configuration

Kiwix is the primary reader for ZIM files. It is available as a desktop application, a mobile app, a command-line tool, and a server. Here is a comprehensive installation guide.

Desktop Installation

# Windows:
# Download MSI installer from kiwix.org
# Or via winget: winget install kiwix

# macOS:
# Download DMG from kiwix.org
# Or via brew: brew install --cask kiwix

# Linux (AppImage - works on any distro):
wget https://download.kiwix.org/release/kiwix-desktop/kiwix-desktop_x86_64.appimage
chmod +x kiwix-desktop*.AppImage
./kiwix-desktop*.AppImage

# Linux (package manager):
sudo snap install kiwix-desktop  # Snap
flatpak install flathub org.kiwix.desktop  # Flatpak

Kiwix Serve (Headless Server)

Kiwix Serve allows you to host ZIM files on a network and access them via any browser. Ideal for sharing ZIM archives across your LAN or a community mesh network.

# Install kiwix-tools:
wget https://download.kiwix.org/release/kiwix-tools/kiwix-tools_linux-x86_64.tar.gz
tar xzf kiwix-tools*.tar.gz
sudo cp kiwix-serve /usr/local/bin/

# Serve a single ZIM file:
kiwix-serve --port=8080 wikipedia_en_all_maxi.zim

# Serve a library of ZIMs:
kiwix-serve --port=8080 --library=library.xml

# Create a library file:
kiwix-manage library.xml add wikipedia_en_all_maxi.zim
kiwix-manage library.xml add wikihow_en.zim
kiwix-manage library.xml add gutenberg_en_all.zim

Kiwix Hotspot

Kiwix Hotspot turns a Raspberry Pi into a Wi-Fi access point that serves ZIM content. No network infrastructure needed — devices connect directly to the Pi's Wi-Fi and access the content via a captive portal.

# Kiwix Hotspot on Raspberry Pi:
# 1. Flash Raspberry Pi OS Lite to SD card
# 2. Install kiwix-hotspot:
wget https://download.kiwix.org/release/kiwix-hotspot/kiwix-hotspot-linux-armhf.tar.gz
tar xzf kiwix-hotspot*.tar.gz
cd kiwix-hotspot
sudo ./kiwix-hotspot

# 3. Place ZIM files in /data/zim/
# 4. The Pi creates a Wi-Fi network called "Kiwix Hotspot"
# 5. Connect any device to this Wi-Fi
# 6. Open browser → captive portal shows ZIM library

ZIM File Sources

ZIM files are available from multiple sources. The official Kiwix library is the primary repository, but there are mirrors and alternative sources as well.

  • Kiwix Library — The official in-app download catalog. Browse, search, and download ZIM files directly within Kiwix Desktop or mobile app.
  • download.kiwix.org/zim/ — Direct ZIM file repository. Organized by category (wikipedia, wiktionary, wikihow, stack_exchange, etc.) and then by language.
  • Internet Archive — Many ZIM files are mirrored on the Internet Archive. Search for "ZIM" or "Kiwix" to find community-created archives.
  • FAU Mirror (Germany) — Fast European mirror of the Kiwix ZIM repository.
  • MIT Mirror (US) — Fast US-based mirror of the Kiwix repository.

Zeal/Dash for Developer Documentation

Zeal (Windows/Linux) and Dash (macOS) are specialized offline documentation browsers for developers. They use the docset format — a collection of HTML documentation files bundled with a search index. Both support 200+ documentation sets covering virtually every programming language, framework, and tool.

Recommended Docset Collections

  • Python 3 — Complete Python standard library reference
  • JavaScript/Node.js — JS core, Node.js API, npm packages
  • React/Vue/Angular — Frontend framework documentation

Offline Programming References

Beyond docsets and ZIM archives, there are several other essential offline programming references to prepare:

  • MDN Web Docs ZIM (~2 GB) — Complete HTML, CSS, JavaScript, and Web API reference via Kiwix. This is the single most useful offline resource for web developers.
  • Python docs ZIM — Complete Python documentation including tutorials, library reference, and language reference.
  • Docker docs — Available as a docset for Zeal/Dash. Includes Dockerfile reference, compose file spec, and CLI commands.
  • Linux man pages — Pre-installed on every Linux system. man bash, man syscalls, man hier for filesystem hierarchy.
  • Stack Overflow ZIM (~80 GB) — All programming Q&A. Huge but invaluable. Download only if you have significant storage.

PDF Library Management

Managing a large collection of PDF documents (manuals, textbooks, papers) requires proper tools. Here are the best options for offline PDF management:

  • Calibre — Full-featured ebook manager. Converts between formats, manages metadata, serves books over the network via content server. Supports PDF, EPUB, MOBI, and 20+ other formats. Runs on Linux, Windows, macOS.
  • SumatraPDF — Lightweight Windows PDF reader. Opens PDF, EPUB, MOBI, CHM, XPS, DjVu, and CBZ/CBR. Portable version available for USB drive.
  • Zathura — Minimalist document viewer for Linux. Vim-like keybindings. Supports PDF, DjVu, and EPUB via plugins.

Scientific Paper Archives

For researchers and students, offline access to scientific literature is critical. Here are the best sources and methods:

  • ArXiv — Download paper PDFs in bulk via the ArXiv bulk API. Over 2 million papers in physics, mathematics, computer science, and related fields. Use arxiv-sanity or similar tools to curate your collection.
  • PubMed Offline — The PubMed/MEDLINE database is available as a ZIM archive (~2 GB). Medical references, abstracts, and links to full text where available.
  • Wikipedia ZIM subsets — Thematic Wikipedia subsets for medicine, science, and technology are available from the Kiwix library at much smaller sizes than the full Wikipedia.
  • PDF scraping — For specific papers, use tools like `scidownl` to bulk download papers from known DOIs while you have internet access.

Offline Maps with Notes

Combining offline maps with personal notes creates a powerful situational awareness tool. You can annotate maps with locations, routes, and notes for emergency scenarios.

  • OsmAnd — Supports audio notes, photo notes, and POI creation directly on the map. Add notes about water sources, shelters, or hazards while offline.
  • Organic Maps — Bookmark and categorize locations. Export bookmarks as GPX/KML for sharing.
  • QGIS — Professional-grade map creation with annotation layers. Create custom maps with waypoints, routes, and notes for emergency planning.
  • Gaia GPS — Powerful offline topographic maps with waypoints and route planning. Popular with hikers and overlanders.

Building Custom ZIM Archives

You can create your own ZIM files from web content using Zimwriterfs. This is useful for preserving specific websites, documentation, or personal wikis in a portable format.

# Install Zimwriterfs:
sudo apt install zimwriterfs  # Debian/Ubuntu
# Or build from source: https://github.com/openzim/zim-tools

# Create a ZIM file from a local directory of HTML files:
zimwriterfs --welcome=index.html \
  --language=eng \
  --title="My Custom Archive" \
  --description="Personal documentation collection" \
  --creator="Your Name" \
  --publisher="Offline Library" \
  ./html-content/ \
  my-custom-archive.zim

# Create a ZIM from a web crawl (using wget first):
wget --recursive --level=5 --convert-links \
  --html-extension --restrict-file-names=windows \
  --page-requisites --no-parent \
  https://docs.example.com/
zimwriterfs docs.example.com/ my-docs.zim

Sharing Knowledge Bases Over Local Network

Once you have built your offline knowledge library, share it with others on your local network. Multiple methods are available depending on your setup:

  • Kiwix Serve — The simplest solution for sharing ZIM archives. Start the server on any machine, and all devices on the LAN can access the content via browser.
  • Samba share — Export your ZIM and docset directories via Samba. Other devices can browse the files and open them locally. Works well for laptops with Kiwix installed.
  • Calibre content server — For ebook sharing. All devices on the LAN can browse and download books from the Calibre library.
  • Raspberry Pi hotspot — Create a self-contained portable knowledge server. The Pi creates its own Wi-Fi network and serves ZIM content via Kiwix Hotspot. No existing network required.
# Quick LAN sharing setup (one command):
# Python HTTP server (serves current directory):
python3 -m http.server 8000

# Access from other devices:
# http://your-ip:8000

# For a permanent solution, use nginx:
sudo apt install nginx
sudo ln -s /usr/share/nginx/www /path/to/your/zims
