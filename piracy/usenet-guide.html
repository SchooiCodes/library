What is Usenet?

Usenet is a decentralized discussion and file distribution system that predates the World Wide Web. Unlike torrenting’s P2P model, Usenet uses a client-server architecture: you download files directly from news servers that store the data.

  • How it works: Files are split into posts, encoded into NZB files (like .torrent files), and uploaded to news servers. Your NZB client downloads all the pieces from the server and reassembles them.
  • No seeding required: Unlike torrents, you don’t need to upload anything back. Download speeds are typically maxed out on your connection.
  • Retention: Good providers keep data for 10-15 years. Older content may be taken down via DMCA.
  • Cost: Usenet is not free. You pay for a provider (server access) and optional indexers (search engines).

Indexers

Indexers are the search engines for Usenet. They crawl newsgroups and build a searchable database of NZB files.

  • NZBGeek — Popular, affordable, good API. ~$12/year or lifetime option.
  • Drunkenslug — Excellent API, reasonable price. ~$15/year.
  • NZBPlanet — Large index, decent API. ~$10/year.
  • AltHub — Free tier available, paid for more API hits.

NZBGet / SABnzbd Setup

  • SABnzbd — Popular, modern web interface, extensive settings, great for beginners.
  • NZBGet — Extremely lightweight (C++), very fast, low memory usage. Ideal for Raspberry Pi or NAS.

Automation with Radarr/Sonarr

The Arr stack supports Usenet natively via SABnzbd/NZBGet. Prowlarr handles indexer integration across all your apps.

# Sonarr + SABnzbd + Prowlarr workflow:
# 1. Prowlarr connects to your indexers
# 2. Sonarr connects to Prowlarr for indexer access
# 3. Sonarr connects to SABnzbd as download client
# 4. Add a show in Sonarr, set quality profile
# 5. Sonarr searches indexers, finds NZB files
# 6. Sends to SABnzbd which downloads
# 7. Sonarr renames and moves to media folder
# 8. Plex/Jellyfin scans and makes available
