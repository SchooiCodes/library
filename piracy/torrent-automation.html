Sonarr Setup

Sonarr automates TV show downloads. Add shows, set quality profiles, and it handles the rest. Install via Docker or native package. Configure indexers through Prowlarr. Connect to your download client (qBittorrent or SABnzbd).

# Sonarr Docker:
docker run -d --name=sonarr   -p 8989:8989   -v /path/to/config:/config   -v /path/to/tv:/tv   -v /path/to/downloads:/downloads   linuxserver/sonarr

Radarr Setup

Radarr is the movie equivalent of Sonarr. Same setup pattern: add movies, set quality profiles, connect indexers and download client. Supports custom formats for advanced release preferences (e.g., prefer certain release groups).

Prowlarr

Prowlarr is your single indexer management point. Add your torrent and Usenet indexers once, then sync to all your Arr apps. Supports hundreds of indexers, automatic updates, and search history.

# Prowlarr setup:
# 1. Install Prowlarr (docker: linuxserver/prowlarr)
# 2. Go to http://localhost:9696
# 3. Settings → Indexers → Add your indexers
# 4. Settings → Apps → Add Sonarr, Radarr, Lidarr
# 5. Enter API keys from each app
# 6. Indexers are automatically synced to all connected apps

qBittorrent Integration

qBittorrent is the recommended torrent client for the Arr stack. Enable the Web UI: Tools → Options → Web UI. Set a username and password. Sonarr/Radarr connect via the Web UI API.

# qBittorrent Web UI setup:
# Tools → Options → Web UI
# Enable: Web User Interface
# IP: 0.0.0.0 (all interfaces)
# Port: 8080
# Authentication: Set username/password
#
# In Sonarr: Settings → Download Clients → Add qBittorrent
# Enter host, port, username, password

Overseerr

Overseerr provides a Netflix-like interface for requesting media. Users browse trending and popular content, request movies and shows, and track their requests. Integrates with Sonarr and Radarr for automatic processing. Supports Plex authentication.

Usenet Automation

The Arr stack also supports Usenet through SABnzbd or NZBGet. Add your Usenet indexers to Prowlarr alongside torrent indexers. Configure both a torrent client and a Usenet client in Sonarr/Radarr. Content automatically routes through the best available source.

 Sonarr Deep Dive

    
Sonarr is the heart of your TV automation setup. Beyond basic add-and-download, these advanced features are essential:


    
Series Types

    
When adding a show, set the correct series type to ensure proper episode matching:

    
    
     
  • Standard: Regular TV series. Episodes follow a standard season/episode numbering (S01E01, S01E02, etc.).
    • 
     
  • Daily: Shows that air daily (talk shows, news, soap operas). Sonarr matches by airdate instead of episode number.
    • 
     
  • Anime: Uses absolute episode numbering (001, 002, etc.) instead of season/episode. Handles Japanese numbering conventions.
    • 
    


    
Quality Profiles

    
Quality profiles control what quality Sonarr downloads and how it upgrades:

    
# Recommended quality profiles (from trash-guides.info):
#
# HD-1080p:  1080p BluRay + WEB-DL (sweet spot for most users)
# HD-720p:   720p encodes (for limited bandwidth/storage)
# 4K:        2160p BluRay + WEB-DL (requires storage space)
# SD:        480p (for older shows only available in SD)
#
# Each profile ranks qualities in order of preference.
# Sonarr will upgrade until the highest-ranked quality is met.


    
Release Profiles

    
Release profiles let you prefer or reject specific release groups, terms, or categories:

    
    
     
  • Preferred terms: Boost releases containing certain keywords. Set scores for terms like "NTb" or "FLUX" (quality release groups).
    • 
     
  • Must contain / Must not contain: Require or reject specific terms. For example, reject "x264" if you only want x265.
    • 
     
  • Tags: Apply release profiles to specific shows or groups of shows via tags.
    • 
    


    
Delay Profiles

    
Delay profiles control how long Sonarr waits before grabbing a release:

    
    
     
  • Purpose: Early releases are often poor quality (CAM, HDTV). Waiting a few hours or days lets better releases (WEB-DL, BluRay) appear.
    • 
     
  • Protocol-based delays: Set different delays for Usenet vs Torrents. Usenet is fast, so a short delay suffices. Torrents may need longer delay for quality releases.
    • 
     
  • Seed ratio/goal: Set minimum seed ratio/ time before Sonarr considers a release "grabbed" and stops searching.
    • 
    


    
Custom Formats

    
Custom formats are the most powerful feature in modern Sonarr/Radarr. They replace the old "Release Profiles" system with a scoring-based approach:

    
# Example custom format: "HDR10+"
# Conditions:
# - Video dynamic range: HDR10+
# - OR Video dynamic range: HDR10
# Score: +50
#
# Example custom format: "x265 (No HDR)"
# Conditions:
# - Video codec: x265
# - AND NOT Video dynamic range: HDR
# Score: -50 (prefer x264 for non-HDR content)
#
# Best custom format packs:
# - trash-guides.info has the definitive collection
# - Includes: HDR formats, audio codecs, release groups
# - Import via JSON or GUI


    
 Radarr Deep Dive

    
Radarr mirrors Sonarr’s functionality for movies. Key differences and features:

    
    
     
  • Minimum availability: Set when Radarr considers a movie available:
      
      
       
    • Announced: Starts monitoring as soon as a movie is announced (future releases).
      • 
       
    • In Cinemas: Starts monitoring when the movie hits theaters.
      • 
       
    • Physical/Web: Starts monitoring when the BluRay or WEB-DL releases.
      • 
       
    • Released: Starts monitoring when widely available.
      • 
      
    
     
    • 
     
  • Collections: Add entire movie franchises at once (MCU, Fast & Furious, James Bond). Radarr monitors all movies in the collection.
    • 
     
  • Lists: Sync watchlists from Trakt, IMDb, Plex, RSS feeds, or custom lists. Radarr automatically adds movies from these sources.
    • 
     
  • Custom Formats: Same as Sonarr. Prefer certain release groups, audio codecs (DTS-HD, TrueHD), or HDR types. The trash-guides.info site has the definitive custom format collection.
    • 
    


    
 Prowlarr Indexer Management

    
Prowlarr is your single point of truth for indexer management:

    
    
     
  • Supported indexer types: Torrent (public + private trackers), Usenet (NZB indexers), and generic (Newznab, Torznab).
    • 
     
  • Sync to all apps: Add an indexer once in Prowlarr, and it automatically pushes to Sonarr, Radarr, Lidarr, Readarr, and other connected apps.
    • 
     
  • Indexer proxies: Route indexer traffic through Prowlarr’s built-in proxy (FlareSolverr) for Cloudflare-protected sites.
    • 
     
  • Search history: Prowlarr logs all searches. Useful for debugging why a specific movie or show isn’t being found.
    • 
     
  • Indexer health checks: Prowlarr periodically tests your indexers and reports failures (login issues, API errors, rate limiting).
    • 
    

    
# Prowlarr Docker environment for FlareSolverr:
  prowlarr:
    image: linuxserver/prowlarr:latest
    container_name: prowlarr
    ports: ["9696:9696"]
    volumes: ["./prowlarr:/config"]
    environment:
      - PROWLARR__AUTH__APIKEY=your_api_key
    restart: unless-stopped

  flaresolverr:
    image: ghcr.io/flaresolverr/flaresolverr:latest
    container_name: flaresolverr
    ports: ["8191:8191"]
    restart: unless-stopped


    
 qBittorrent Integration

    
qBittorrent is the recommended torrent client for the Arr stack. Beyond basic setup:

    
    
     
  • Web UI setup: Enable Web UI in Tools → Options → Web UI. Set a strong username/password. Bind to your VPN interface for leak protection.
    • 
     
  • Category management: Set up categories (tv, movies, music) in qBittorrent. Sonarr/Radarr automatically apply the correct category, which determines save paths.
    • 
     
  • Seeding limits: Set per-category seeding limits:
      
    # In qBittorrent options:
# Category: tv     -> Seed until ratio 1.0 OR 7 days
# Category: movies -> Seed until ratio 1.0 OR 14 days
# Category: music  -> Seed until ratio 2.0 OR 30 days
    
     
    • 
     
  • Label management: qBittorrent labels created by Sonarr/Radarr help identify which app downloaded what.
    • 
    


    
 Overseerr Setup

    
Overseerr provides a Netflix-like interface for requesting media:

    
    
     
  • Plex authentication: Overseerr uses Plex for user authentication. Authorize your Plex server and users can log in with their Plex credentials.
    • 
     
  • Integration setup: Add your Sonarr and Radarr API keys in Settings → Services. Overseerr auto-discovers quality profiles and root folders.
    • 
     
  • Request management: Users search for movies/shows, request them, and Overseerr automatically sends to Sonarr/Radarr. Admin reviews and approves/denies requests.
    • 
     
  • Notifications: Configure Discord, Slack, Telegram, Pushover, or email notifications for new requests, approvals, and availability.
    • 
    


    
 Notification Setup

    
Stay informed about your automation stack with notifications:

    
    
     
  • Sonarr/Radarr notifications: Configurable in Settings → Connect. Events:
      
      
       
    • On Grab: Release was grabbed by download client
      • 
       
    • On Download: Release was imported to media folder
      • 
       
    • On Upgrade: Existing file was replaced with a better quality
      • 
       
    • On Failure: Download failed or import failed
      • 
      
    
     
    • 
     
  • Platforms: Discord (webhook), Pushover (direct push notification), Telegram (bot), Slack, email, custom webhooks.
    • 
     
  • Recommended setup:
      
      
       
    • Discord channel for grab/import notifications (shows what was downloaded)
      • 
       
    • Pushover for failure notifications (immediate attention needed)
      • 
      
    
     
    •

 Hardlinks vs Copy

    
Understanding hardlinks is essential for efficient media management:

    
    
     
  • Copy: Sonarr/Radarr copies the file from the download folder to the media folder. You have two copies of the same file — double the storage usage.
    • 
     
  • Hardlink: Sonarr/Radarr creates a hardlink (directory entry pointing to the same data on disk). You have one file with two names. No extra storage used.
    • 
     
  • Why hardlinks matter: You can continue seeding the download while the file appears in your media library. No storage waste. Crucial for private trackers where you must seed long-term.
    • 
     
  • Requirements:
      
      
       
    • The download folder and media folder must be on the same filesystem (same Docker volume, same partition).
      • 
       
    • Sonarr/Radarr must use the same root folder path for both download and media.
      • 
       
    • Docker volumes must be mounted at the same path inside the container.
      • 
      
    
     
    • 
    

    

     Recommended setup: Mount a single volume (/media) that contains both /media/downloads (where torrent client saves) and /media/tv, /media/movies (media folders). This enables hardlinks. Never use separate Docker volumes for downloads vs media.
    


    
 trash-guides.info for Optimal Setup

    
trash-guides.info is the definitive resource for configuring the Arr stack:

    
    
     
  • Quality profiles: Pre-built quality profiles optimized for different use cases (HD, 4K, Remux). Import via JSON.
    • 
     
  • Custom formats: The largest collection of custom formats for Radarr and Sonarr. Covers release groups, HDR types, audio codecs, and more. Importable via text files or JSON.
    • 
     
  • Recommended settings: Comprehensive guides for Sonarr, Radarr, Bazarr, Prowlarr, qBittorrent, SABnzbd, and other tools.
    • 
     
  • Hardlinks guide: Detailed explanation of how to configure Docker volume mounts for proper hardlink support.
    • 
     
  • How to use: Go to the site, select your app (e.g., Radarr), copy the JSON for the custom formats you want, paste into Radarr"s Custom Formats settings. Then apply the recommended quality profile.
