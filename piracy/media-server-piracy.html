Media Servers for Piracy

Plex Setup

Plex organizes your media library and streams it to any device. Install Plex Media Server on a dedicated machine or NAS. Organize media in proper folder structures (Movies/, TV Shows/). Access from phones, TVs, and browsers. Plexamp for music.

Jellyfin Setup

Jellyfin is the open-source alternative to Plex. No paid features, no telemetry, full customization. Supports all media types, hardware transcoding, live TV, and plugin system. Docker deployment is recommended.

Sonarr/Radarr

Sonarr (TV) and Radarr (Movies) automate media acquisition. Add content, set quality profiles, and they automatically search indexers, download, rename, and organize your media. Connect to Prowlarr for indexer management.

Prowlarr

Prowlarr is a centralized indexer manager. Add your torrent and Usenet indexers once, and sync them to Sonarr, Radarr, Lidarr, Readarr, and other Arr apps. Supports hundreds of indexers and automatic updates.

Overseerr

Overseerr provides a user-friendly web interface for requesting media. Users browse, search, and request movies and TV shows. Integrates with Sonarr/Radarr for automatic processing. Plex authentication for user management.

Usenet/Torrent Integration

Sonarr/Radarr support both Usenet and Torrent clients simultaneously. Configure SABnzbd for Usenet and qBittorrent for torrents. Content is automatically routed through the best available source.

Plex Setup Guide

Setting up Plex Media Server properly is crucial for a smooth streaming experience. Follow this comprehensive guide:

Library Organization

Plex relies on proper folder structures and file naming to match content correctly:

# Recommended folder structure:
/media/
  movies/
    Movie.Name.2024.1080p.BluRay.x264-GROUP.mkv
    Movie.Name.2025.2160p.WEB-DL.DDP5.1.x265-GROUP.mkv
  tv/
    Show Name/
      Season 01/
        Show.Name.S01E01.1080p.WEB-DL.x264-GROUP.mkv
        Show.Name.S01E02.1080p.WEB-DL.x264-GROUP.mkv
  music/
    Artist/
      Album (Year)/
        01 - Song Title.flac
  photos/
  • Movies: Each file in a flat folder or individual movie folders. Naming: Movie.Name.Year.Quality.Source-GROUP.ext
  • TV Shows: Folder per show, subfolder per season. Naming: Show.Name.S01E01.episode-title.ext
  • Music: Artist → Album → Track files. Plex matches by embedded metadata (ID3 tags).

Remote Access Setup

To stream outside your home network, you must configure remote access:

  1. In Plex Settings → Remote Access, enable "Show Advanced."
  2. Plex will attempt automatic port mapping (UPnP). If it fails, manually forward port 32400 on your router.
  3. Set a static IP for your Plex server to keep the port forward working.
  4. Test access by disabling Wi-Fi on your phone and streaming over cellular.
Remote access without port forwarding: Use a reverse proxy (Nginx/Caddy) combined with Cloudflare Tunnel or Tailscale. This is more secure and works on restrictive networks (CGNAT).

Hardware Transcoding

Hardware transcoding offloads video encoding/decoding to your GPU, reducing CPU usage. You need Plex Pass ($4.99/month or $119.99 lifetime):

  • Intel QuickSync (QSV): Best option. Intel CPUs with iGPU (7th gen+). In Plex Settings → Transcoder, set "Use hardware acceleration when available."
  • NVIDIA NVENC: Works with GTX 1050+ and all RTX cards. Requires an NVIDIA driver that supports NVENC. Consumer cards are artificially limited to 2 simultaneous transcodes (use a driver patch for unlimited).
  • AMD AMF: Supports VCE hardware encoding. Less tested than Intel/NVIDIA but works well on modern AMD APUs.
  • Apple Video Toolbox: Available on macOS servers with M-series or Intel Macs.

Complete Arr Stack Setup

The "Arr stack" automates everything from acquisition to organization. Here is the full setup:

  • Prowlarr: Central indexer management. Add your torrent and Usenet indexers once, sync to all other apps.
  • Sonarr: TV show management. Monitors shows, searches indexers, sends to download clients, renames and organizes.
  • Radarr: Movie management. Same as Sonarr but for movies. Supports custom formats for quality (release groups, HDR, audio codecs).
  • Lidarr: Music management. Monitors artists, downloads albums, organizes your music library.
  • Readarr: Ebook and audiobook management. Same concept for books.
  • Bazarr: Subtitle management. Automatically downloads subtitles for your media library from multiple providers.
  • Overseerr: Request management. Web interface for family/friends to request content. Integrates with Sonarr/Radarr.
  • Tautulli: Plex/Jellyfin monitoring. Tracks plays, user activity, bandwidth usage. Alerts for issues.

Reverse Proxy Setup

A reverse proxy provides secure remote access (HTTPS), subdomain routing, and protects your services from direct exposure:

Option 1: Nginx Reverse Proxy

# Sample Nginx config for media services:
server {
    listen 443 ssl;
    server_name sonarr.yourdomain.com;

    ssl_certificate /etc/ssl/certs/yourdomain.pem;
    ssl_certificate_key /etc/ssl/private/yourdomain.key;

    location / {
        proxy_pass http://localhost:8989;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# Repeat for radarr (7878), prowlarr (9696), overseerr (5055), etc.

Option 2: Caddy (Automatic HTTPS)

# Caddyfile for automatic Let's Encrypt:
sonarr.yourdomain.com {
    reverse_proxy localhost:8989
}

radarr.yourdomain.com {
    reverse_proxy localhost:7878
}

overseerr.yourdomain.com {
    reverse_proxy localhost:5055
}

Option 3: Cloudflare Tunnel (No open ports)

# Install cloudflared:
# docker run -d --name=cloudflared cloudflare/cloudflared tunnel --no-autoupdate run --token YOUR_TOKEN

# Configure tunnel to point at your services:
# In Cloudflare Zero Trust Dashboard:
# - Public hostname: sonarr.yourdomain.com -> localhost:8989
# - Public hostname: radarr.yourdomain.com -> localhost:7878
# Cloudflare handles SSL/TLS, DDoS protection, and hides your IP.

Tautulli Monitoring

Tautulli (formerly PlexPy) is a monitoring and tracking tool for Plex media servers:

  • Watch history: See who watched what, for how long, and from which device.
  • Bandwidth monitoring: Track transcoding quality, bandwidth usage, concurrent streams.
  • Alerts: Get notifications for new streams, transcoding, user sign-ins, server issues via Discord, Telegram, Pushover, email.
  • Library stats: Total media counts, sizes, bitrates, codec distributions.
  • Graphs: Daily/weekly/monthly play counts, concurrent streams, bandwidth usage.
  • Newsletter: Weekly/monthly email reports to users showing recently added content.

Seedbox Integration

Integrate a seedbox with your local media server for the best of both worlds:

  1. Choose a seedbox: Providers like WhatBox, Seedhost, UltraSeedbox. Ensure they support public/private trackers and have FTP/HTTP access.
  2. Set up automation on the seedbox: Many seedboxes offer one-click install of Sonarr/Radarr/Prowlarr. Or run them locally and point the download client to your seedbox's torrent client.
  3. Sync files home: Use rclone to mount the seedbox filesystem locally, Syncthing for bidirectional sync, or LFTP for efficient transfer. Automate sync with cron jobs.
  4. Media server watches the sync folder: Plex/Jellyfin monitors the media directories. As files sync, they appear in your library automatically.
# rclone mount example (seedbox to local):
rclone mount seedbox:media /media/seedbox --daemon \
  --vfs-cache-mode writes \
  --transfers 4 \
  --buffer-size 64M

# Then configure Sonarr/Radarr to use /media/seedbox as an
# additional root folder or a remote path mapping.

Cloud Storage with rclone

Mount cloud storage as local media drives using rclone. This allows unlimited storage for your media server at minimal cost:

  • Google Drive: Use a shared drive (unlimited storage). Mount with rclone and direct Plex/Jellyfin to scan it. $20/year for Google Workspace.
  • Dropbox + Dropbox Decryption: For smaller libraries. rclone supports encrypted remotes.
  • OneDrive: 1TB with Office 365. Works with rclone for media streaming.
  • rclone cache: rclone --vfs-cache-mode full to cache frequently accessed files locally and reduce cloud API calls.

Backup Strategies

Your media server configuration is valuable. Protect it:

  • Config backups: Back up Docker volume configs (Sonarr, Radarr, Plex, tautulli). These contain your library state, watch history, quality profiles, and custom settings.
  • Database backups: Each Arr app has a SQLite database. Back up these files daily. Script: rsync -av /docker/configs/ /backup/media-config/ --link-dest=/backup/media-config-yesterday
  • Media backups: Media files themselves are replaceable (re-download). Prioritize backing up irreplaceable personal media (home videos, photos, rare content).
  • Off-site backup: Copy configs to Backblaze B2 ($5/TB/month), a second cloud drive, or a friend's server.
  • Restore test: Periodically test restoring from backup to a test server to ensure your backup works.
Pro tip: Automate nightly backups with a simple script: docker stop sonarr radarr && tar -czf backup-$(date +%Y%m%d).tar.gz /configs && docker start sonarr radarr
