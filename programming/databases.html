Databases

Databases Guide

Relational, NoSQL, and NewSQL databases — understanding their trade-offs and choosing the right one for your project.

Quick recommendation: Use PostgreSQL for most projects. Pick MongoDB for flexible schemas, Redis for caching, and SQLite for embedded/local storage.

Relational Databases

Relational databases store data in structured tables with rows and columns. They use SQL for queries and enforce ACID transactions. Best for applications where data integrity and relationships matter.

PostgreSQL

Advanced open-source relational database with ACID compliance, JSONB support, full-text search, and extensive extensions. Widely considered the most capable open-source database.

  • ACID compliant with MVCC
  • JSONB for hybrid relational/NoSQL
  • Full-text search engine
  • Extensions: PostGIS, TimescaleDB, pgvector
  • Advanced indexing (B-tree, GiST, GIN, BRIN)
  • Excellent for complex queries and analytics

MySQL

Widely used relational database, popular for web applications (especially with LAMP stack). Known for ease of use and strong community support.

  • Mature and battle-tested
  • Strong replication support
  • InnoDB storage engine
  • Great performance for reads
  • Widely hosted (RDS, Aurora)
  • Excellent tooling (phpMyAdmin, Workbench)

SQLite

Embedded, serverless, zero-configuration relational database engine. The most deployed database in the world (every smartphone uses it).

  • No server setup required
  • Single file storage
  • Ideal for mobile & desktop apps
  • In-memory mode available
  • Fully ACID compliant
  • Surprisingly capable (CTEs, window functions)

MariaDB

Open-source fork of MySQL with additional storage engines, performance improvements, and enhanced security features. Drop-in replacement for MySQL.

  • Faster than MySQL in many benchmarks
  • Aria, ColumnStore, Spider engines
  • Improved query optimizer
  • Built-in JSON functions
  • Galera clustering support
  • 100% MySQL compatible

NoSQL Databases

NoSQL databases offer flexible schemas, horizontal scaling, and specialized data models (document, key-value, graph, column-family). They sacrifice ACID guarantees for performance and scalability.

MongoDB

Document-oriented NoSQL database storing JSON-like documents. Flexible schema, rich query language, and powerful aggregation pipeline.

  • Dynamic schemas
  • Aggregation pipeline
  • Sharding for horizontal scaling
  • Atlas managed service
  • Change streams
  • Geospatial queries

Redis

In-memory key-value data structure store. Used for caching, real-time data, session management, message queues, and rate limiting.

  • Sub-millisecond latency
  • Data structures: strings, hashes, lists, sets, sorted sets
  • Pub/Sub messaging
  • Redis Stack: JSON, Search, TimeSeries
  • Built-in replication & persistence
  • Perfect for caching layer

Firebase

Google's mobile and web application development platform with a realtime NoSQL database. Syncs data across clients in real time.

  • Real-time synchronization
  • Automatic offline support
  • Built-in authentication
  • Cloud Firestore & Realtime DB
  • Serverless scaling
  • Integrates with Google Cloud

Cassandra

Apache Cassandra is a wide-column NoSQL database designed for massive scalability with no single point of failure. Used by Netflix, Apple, and Uber.

  • Linear horizontal scaling
  • No single point of failure
  • Tunable consistency
  • CQL (Cassandra Query Language)
  • Multi-datacenter replication
  • Best for time-series IoT data

Neo4j

Leading graph database. Stores data as nodes and relationships, making it ideal for connected data like social networks, recommendations, and fraud detection.

  • Property graph model
  • Cypher query language
  • ACID compliant
  • Native graph storage
  • Excellent for relationship queries
  • Used in recommendation engines

NewSQL Databases

NewSQL databases combine the ACID guarantees of relational databases with the horizontal scalability of NoSQL. They are designed for cloud-native, distributed environments.

CockroachDB

Distributed SQL database with PostgreSQL wire protocol compatibility. Automatic scaling, fault tolerance, and strong consistency across regions.

  • PostgreSQL-compatible
  • Automatic sharding & rebalancing
  • Survives node failures automatically
  • Multi-region deployment
  • Strong consistency (serializable)
  • Cloud-native design

PlanetScale

MySQL-compatible serverless database platform powered by Vitess. Branch-based workflow for schema changes with zero-downtime migrations.

  • MySQL compatible
  • Branching for dev databases
  • Zero-downtime schema migrations
  • Automatic sharding
  • Serverless scaling
  • Built-in connection pooling
